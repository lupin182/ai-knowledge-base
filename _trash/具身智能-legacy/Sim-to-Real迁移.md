# Sim-to-Real 迁移

## 核心问题

在仿真中训练的策略部署到真实机器人时，性能会显著下降。这种差距称为 **Sim-to-Real Gap**（也叫 Reality Gap）。

差距来源：

| 类型 | 仿真 vs 真实的差异 | 示例 |
|------|-------------------|------|
| **物理差异** | 接触动力学、摩擦、形变 | 仿真中抓稳的物体真实中滑落 |
| **视觉差异** | 渲染 vs 真实图像 | 光照、纹理、反射差异 |
| **传感器差异** | 理想传感器 vs 真实噪声 | 深度相机噪声、IMU 漂移 |
| **执行差异** | 理想电机 vs 真实延迟 | 电机响应延迟、齿轮间隙 |
| **时间差异** | 离散时间步 vs 连续物理 | 控制频率、通信延迟 |

## 主要方法

### 1. 域随机化 (Domain Randomization)

**核心思想**：训练时大幅随机化仿真参数，使策略对参数变化鲁棒，只要真实参数落在随机范围内即可迁移。

#### 物理参数随机化

```python
# Isaac Lab 中的物理域随机化示例
randomization_params = {
    "friction": {"range": [0.3, 1.5]},        # 摩擦系数
    "object_mass": {"range": [0.1, 2.0]},      # 物体质量
    "joint_damping": {"range": [0.5, 3.0]},    # 关节阻尼
    "motor_strength": {"range": [0.8, 1.2]},   # 电机力矩系数
    "gravity": {"range": [9.5, 10.1]},         # 重力
    "action_delay": {"range": [0, 3]},         # 动作延迟 (帧数)
    "observation_noise": {"std": 0.02},        # 传感器噪声
    "external_force": {"range": [0, 10]},      # 随机外力扰动
}
```

#### 视觉域随机化

```python
visual_randomization = {
    "lighting": {
        "intensity": [0.3, 3.0],
        "color": random_RGB,
        "direction": random_direction,
    },
    "texture": random_textures,          # 随机纹理
    "camera": {
        "position_noise": 0.02,          # 相机位置扰动
        "fov": [55, 75],                 # 视场角
    },
    "distractors": random_background,    # 随机背景物体
}
```

**Tobin et al. (2017)**：开山之作，视觉域随机化用于目标检测 Sim-to-Real。

**OpenAI Rubik's Cube (2019)**：域随机化的标杆，数百个参数同时随机化，零样本迁移 Shadow Hand 解魔方。

#### 自适应域随机化 (ADR)

自动调整随机化范围：
- 策略在当前范围内成功率高 → 扩大范围
- 策略在当前范围内成功率低 → 缩小范围
- 逐步扩展到策略能承受的极限

### 2. 域适应 (Domain Adaptation)

不靠"暴力随机化"，而是学习**对齐仿真和真实的特征表示**。

#### 视觉域适应

```
仿真图像 ──→ 编码器 ──→ 共享特征空间 ←── 编码器 ←── 真实图像
                         │
                     策略网络
                         │
                       动作
```

方法：
- **对抗训练**：判别器区分仿真/真实特征，编码器对抗 → 特征对齐
- **图像翻译**：CycleGAN 将仿真图像转换为真实风格
- **风格迁移**：保留内容（物体位置）但改变风格（纹理、光照）

#### RL-CycleGAN

```
仿真图像 ──→ G_sim2real ──→ "仿真真实图像" → 策略
真实图像 ──→ G_real2sim ──→ "真实仿真图像"
           对抗 + 循环一致性损失
```

### 3. 系统辨识 (System Identification)

精确测量/估计真实环境参数，在仿真中匹配：

- **物理参数辨识**：通过真机实验拟合摩擦、惯量等
- **学习式辨识**：训练网络从传感器数据推断仿真参数

**Online System Identification**：
```
真实传感器数据 → 参数估计网络 → 仿真参数
     ↓
在估计参数的仿真环境中训练/微调策略
```

### 4. 仿真增强与高保真仿真

提高仿真本身的逼真度：

- **GPU 加速物理**：更精细的物理仿真（更小时间步、更准确接触模型）
- **光线追踪渲染**：NVIDIA RTX 渲染接近照片级真实
- **可微分仿真**：通过真实数据梯度反传优化仿真参数
- **数据驱动物理**：用神经网络学习接触/形变模型

### 5. Real-to-Sim-to-Real

先从真实环境重建仿真场景，再在仿真中训练：

```
真实场景扫描 (NeRF/3DGS) → 仿真场景重建 → RL/IL 训练 → 部署到真实
```

- **NeRF/3D Gaussian Splatting** 重建视觉场景
- 加上物理属性 → 可交互仿真
- 缩小了视觉 gap，物理 gap 仍需处理

## Teacher-Student 框架详解

这是 locomotion Sim-to-Real 的标准范式：

```
┌─────────────────────────────────────────────────────┐
│ 阶段 1: 仿真中训练 Teacher                            │
│                                                     │
│ 观测 = [关节角, 角速度, IMU,                          │
│         地形高度图, 接触力, 真实速度]   ← 特权信息     │
│         │                                           │
│    ┌────▼────┐                                      │
│    │ Teacher │  ← PPO 训练                           │
│    │ Policy  │                                      │
│    └────┬────┘                                      │
│         │                                           │
│    目标关节角 → PD 控制器 → 力矩                       │
└─────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────┐
│ 阶段 2: 蒸馏 Student                                 │
│                                                     │
│ 观测 = [关节角, 角速度, IMU]   ← 仅真机可得           │
│         │                                           │
│    ┌────▼────┐     ┌─────────┐                      │
│    │ Student │ ──→ │ 隐式估计 │  ← 学会从有限传感器   │
│    │ Policy  │     │ 地形/接触│     推断隐藏信息       │
│    └────┬────┘     └─────────┘                      │
│         │                                           │
│    模仿 Teacher 的输出动作 (行为蒸馏)                  │
└─────────────────────────────────────────────────────┘

阶段 3: Student 部署到真实机器人
```

**为什么 Teacher-Student 有效**：
- Teacher 有完美信息，容易训练出强策略
- Student 通过模仿 Teacher，隐式学会从噪声传感器中估计隐藏状态
- 等价于一个端到端的状态估计器 + 控制器

## 典型 Sim-to-Real 流程

以四足机器人为例：

```
1. 搭建仿真环境
   - 导入机器人 URDF/MJCF 模型
   - 设置地形（平地 → 台阶 → 随机地形）
   - 标定基本物理参数

2. 训练 Teacher (PPO, ~1-2小时)
   - 特权观测 (地形扫描、真实速度、接触力)
   - 课程学习：平地 → 复杂地形
   - 域随机化：摩擦、质量、电机延迟等

3. 蒸馏 Student (~30分钟)
   - 仅用传感器可得的观测
   - 行为克隆损失模仿 Teacher

4. 真机测试与调试
   - 先在简单场景测试
   - 检查: 关节跟踪误差、步态稳定性
   - 常见问题: 电机过热、振荡、滑倒

5. 迭代优化
   - 根据真机表现调整奖励/随机化范围
   - 可能需要 3-5 轮迭代
```

## 方法对比

| 方法 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| 域随机化 | 简单、通用、零样本迁移 | 过度随机化可能降低性能 | Locomotion、简单操控 |
| 域适应 | 利用真实数据对齐 | 需要真实数据、训练复杂 | 视觉策略 |
| 系统辨识 | 精确、可解释 | 测量困难、泛化差 | 精密操控 |
| Real-to-Sim-to-Real | 缩小视觉gap | 重建质量有限 | 特定场景操控 |
| Teacher-Student | 标准、有效 | 两阶段训练 | Locomotion 标配 |

## 前沿趋势

1. **Foundation Model 降低迁移难度**：VLA 的视觉理解能力天然对视觉变化更鲁棒
2. **可微分仿真**：端到端优化仿真参数，使仿真更匹配真实
3. **混合数据训练**：仿真 + 真实数据联合训练（如 π₀）
4. **数字孪生**：高精度重建真实环境 → 仿真中测试 → 直接部署
5. **持续学习**：部署后从真实交互中持续改进
