# 仿真环境详解

## 为什么需要仿真

具身智能的训练需要大量交互数据，真实机器人采集存在三大瓶颈：

1. **效率低**：真实世界 1 秒 = 1 秒，GPU 仿真可达 1000x 实时
2. **成本高**：机器人磨损、场景搭建、人工监督
3. **安全风险**：探索阶段策略可能造成碰撞损坏

仿真的核心价值：**用算力换数据**，大规模并行训练后通过 Sim-to-Real 迁移到真实环境。

## 主流仿真器对比

| 仿真器 | 物理引擎 | GPU 加速 | 渲染质量 | 适用场景 | 维护方 |
|--------|----------|----------|----------|----------|--------|
| **Isaac Sim / Isaac Lab** | PhysX 5 | ✅ 数千并行 | 光线追踪 (RTX) | 操控、locomotion | NVIDIA |
| **MuJoCo** | MuJoCo 引擎 | ✅ (MJX on JAX) | 中等 | 连续控制、locomotion | Google DeepMind |
| **Habitat** | Bullet (可选) | ✅ | 高 (场景重建) | 室内导航 | Meta |
| **SAPIEN** | PhysX 5 | ✅ | 高 (Ray tracing) | 操控、灵巧手 | UCSD |
| **PyBullet** | Bullet | ❌ | 低 | 教学、快速原型 | 社区 |
| **Genesis** | 多后端 | ✅ | 高 | 通用、可微分 | 开源社区 |
| **RoboCasa** | MuJoCo | ✅ | 中 | 家庭环境操控 | UT Austin |

## NVIDIA Isaac 生态

### Isaac Sim

基于 Omniverse 平台的高保真仿真器：

- **物理引擎**：PhysX 5，支持刚体、柔体、流体、布料
- **渲染**：RTX 光线追踪，PBR 材质，逼真视觉
- **域随机化 (Domain Randomization)**：材质、光照、相机参数、物理参数自动随机化
- **传感器模拟**：RGB-D、LiDAR、IMU、力/力矩传感器

### Isaac Lab (原 Orbit)

Isaac Sim 之上的 RL/IL 训练框架：

```python
# Isaac Lab 典型训练流程
import isaaclab

env = isaaclab.make("Isaac-Franka-Reach-v0", num_envs=4096)  # 4096 并行环境
obs = env.reset()

for step in range(10000):
    actions = policy(obs)       # 策略网络推理
    obs, reward, done, info = env.step(actions)  # GPU 并行 step
    # 所有 4096 个环境同时执行，吞吐量极高
```

核心优势：
- **大规模并行**：单卡可运行数千个环境
- **端到端 GPU**：观测、物理、渲染全部在 GPU 上，避免 CPU-GPU 数据搬运
- **丰富的任务库**：Reach、Pick-and-Place、Locomotion、Dexterous Hand 等

### Isaac GR00T

面向人形机器人的基础模型框架：
- 提供 **GR00T N1** 基础模型（双系统架构：VLM 高层规划 + 扩散策略底层控制）
- 从仿真数据 + 真实遥操作数据联合训练

## MuJoCo

**Multi-Joint dynamics with Contact**，最经典的机器人仿真器：

### 核心特点

- **快速精确的接触动力学**：凸优化求解接触力，物理精度高
- **可微分物理 (MJX)**：基于 JAX，支持自动微分，可用于可微分 RL
- **轻量级**：C 语言实现，单核即可达到数千 FPS
- **XML 模型定义 (MJCF)**：声明式场景描述

```xml
<!-- MuJoCo MJCF 模型示例 -->
<mujoco>
  <worldbody>
    <body name="robot" pos="0 0 1">
      <joint name="hip" type="hinge" axis="0 1 0"/>
      <geom type="capsule" size="0.05 0.3"/>
      <body name="shin" pos="0 0 -0.6">
        <joint name="knee" type="hinge" axis="0 1 0" range="0 150"/>
        <geom type="capsule" size="0.04 0.3"/>
      </body>
    </body>
  </worldbody>
  <actuator>
    <motor joint="hip" ctrlrange="-1 1"/>
    <motor joint="knee" ctrlrange="-1 1"/>
  </actuator>
</mujoco>
```

### 经典 Benchmark 环境

| 环境 | 维度 | 任务 |
|------|------|------|
| HalfCheetah | obs=17, act=6 | 跑步 |
| Ant | obs=111, act=8 | 四足行走 |
| Humanoid | obs=376, act=17 | 双足行走 |
| Shadow Hand | obs=211, act=20 | 灵巧操作 |
| Adroit (DAPG) | obs~40, act=24-30 | 工具使用 |

### MJX: GPU 加速的 MuJoCo

```python
import mujoco
from mujoco import mjx
import jax

# 加载模型
model = mujoco.MjModel.from_xml_path("humanoid.xml")
mjx_model = mjx.put_model(model)

# JAX 批量仿真
@jax.vmap
def step(state, action):
    state = mjx.step(mjx_model, state.replace(ctrl=action))
    return state

# 数千环境并行
batch_states = jax.vmap(lambda key: mjx.make_data(mjx_model))(keys)
```

## Meta Habitat

专注于**视觉导航**的仿真平台：

### Habitat-Sim

- **真实 3D 扫描场景**：基于 Matterport3D、HM3D、Gibson 等数据集
- **高速渲染**：单 GPU 达数千 FPS
- **语义标注**：每个物体有语义类别标签

### Habitat-Lab

任务定义与训练框架，内置标准任务：

- **PointNav**：给定坐标，导航到目标点
- **ObjectNav**：给定物体类别（如"找到沙发"），导航到目标
- **ImageNav**：给定目标位置照片，导航过去
- **Rearrangement**：移动物体到指定位置（导航 + 操控）

### Habitat Challenge

每年举办的导航竞赛，推动了具身导航研究：
- 从 PointNav → ObjectNav → 多物体重排 → 社交导航
- 逐步接近真实世界复杂度

## SAPIEN

来自 UCSD 的操控仿真平台：

- **丰富的可交互物体**：基于 PartNet-Mobility 数据集，包含铰接物体（抽屉、门、水龙头）
- **GPU 并行**：基于 PhysX 5 / Warp
- **高质量渲染**：支持 Ray tracing，适合视觉策略训练

### ManiSkill

ManiSkill（Manipulation Skill）不是独立仿真器，而是基于 SAPIEN 之上的**操控任务 Benchmark + 训练框架**。类比关系：ManiSkill 之于 SAPIEN ≈ Isaac Lab 之于 Isaac Sim ≈ Habitat-Lab 之于 Habitat-Sim。

**提供什么**：
- **标准化任务集**：刚体抓取、软体操作、铰接物体交互（开抽屉、拧水龙头）等
- **统一 API**：兼容 Gymnasium 接口，方便接入各种 RL/IL 算法
- **多种观测模式**：RGB-D、点云、状态向量
- **丰富的可交互物体**：依托 PartNet-Mobility 数据集，物体都有铰接结构标注

**版本演进**：

| 版本 | 特点 |
|------|------|
| ManiSkill1 | 基础操控 benchmark |
| ManiSkill2 | 增加软体、更多任务 |
| **ManiSkill3** (2024) | GPU 并行加速（基于 PhysX 5 / Warp），速度大幅提升，支持视觉 RL 大规模训练 |

## Genesis

2024 年新出的开源通用仿真器：

- **多物理引擎后端**：刚体、MPM（软体）、SPH（流体）、FEM、PBD
- **生成式仿真**：可以用自然语言描述生成仿真场景
- **可微分**：支持自动微分用于梯度优化
- **极高速度**：号称比 Isaac Lab 快 10-80x（特定任务）

## 仿真器底层原理：从 action 到关节运动

做具身智能研究**一般不需要深入物理引擎内部**，但理解 `env.step(action)` 背后发生了什么有助于调参和 sim-to-real 迁移。

### action 到力矩的转换链路

```
policy 输出 actions (归一化的 [-1, 1])
        │
        ▼
┌─────────────────────────┐
│  Action Scaling          │  actions × scale + offset → 目标值
│  (归一化 → 物理量)       │  例: [-1,1] → [-30°, 30°] 关节角度
└─────────┬───────────────┘
          ▼
┌─────────────────────────┐
│  Actuator Model          │  模拟真实电机的响应特性
│  (执行器模型)            │  目标值 → 实际施加的力/力矩
└─────────┬───────────────┘
          ▼
┌─────────────────────────┐
│  Physics Engine          │  根据力/力矩 + 接触 → 求解下一时刻状态
│  (PhysX / MuJoCo)       │  牛顿-欧拉方程 + 约束求解
└─────────┬───────────────┘
          ▼
    新的 obs, reward
```

### 三种执行器模型

**1. 理想力矩控制**（最简单，MuJoCo 经典 benchmark 常用）

```python
τ_applied = action * torque_limit  # 直接施加力矩，无电机动态
```

**2. PD 控制器**（最常用，Isaac Lab 默认）

策略输出**目标关节角度**，PD 控制器计算需要的力矩：

$$\tau = K_p (q_{target} - q_{current}) + K_d (\dot{q}_{target} - \dot{q}_{current})$$

- $K_p$：刚度（弹簧系数），$K_d$：阻尼（阻力系数）
- 真实机器人也用 PD 控制器，所以 **sim-to-real 接口一致**

**3. 隐式执行器模型**（高保真）

在 PD 基础上增加电机带宽限制（低通滤波）、摩擦模型、力矩延迟，更接近真实电机响应。

### 控制频率分层

一个关键细节：**物理仿真频率远高于策略频率**。

```
策略 50Hz 输出一个目标角度
    → PD 控制器以 500Hz 持续跟踪（每个 policy step 跑 10 个物理 step）
        → 物理引擎每步求解接触、碰撞、约束
```

### 观测反馈（传感器建模）

| 观测量 | 物理引擎来源 | 对应真实传感器 |
|--------|-------------|---------------|
| 关节角度 $q$ | 关节状态 | 编码器 (encoder) |
| 关节角速度 $\dot{q}$ | 状态微分 | 编码器差分 |
| 接触力 | 约束求解器 | 力/力矩传感器 |
| RGB-D 图像 | 光线追踪渲染 | 相机 |
| IMU | 刚体角速度 + 重力投影 | IMU |

通常还会加噪声模拟真实传感器：`obs_noisy = obs_clean + noise`。

## 主流具身智能工作的仿真选型

| 工作 | 仿真器 | 方向 |
|------|--------|------|
| **RoboTwin** (2024, 上交) | SAPIEN / ManiSkill | 双臂操控数据生成（LLM 生成脚本 → 仿真执行 → 收集轨迹） |
| **RoboCasa** (2024, UT Austin) | MuJoCo (robosuite) | 家庭场景操控 |
| **GR00T / Isaac Lab** | Isaac Sim (PhysX) | 人形机器人 locomotion + 操控 |
| **Legged Gym** | Isaac Gym (PhysX) | 四足/双足运动 |
| **SIMPLER** (2024) | SAPIEN / ManiSkill | RT-2 等 VLA 的仿真评估 |
| **Habitat Rearrangement** | Habitat-Sim | 导航 + 操控 |

总结：**操控类基本是 SAPIEN/MuJoCo 二选一，locomotion 基本是 Isaac 生态**。

```
你要做什么？

├── 连续控制/locomotion RL 研究

│
├── 大规模操控训练 (工业级)

│
├── 室内视觉导航

│
├── 铰接物体交互 / benchmark
│   └── → SAPIEN + ManiSkill
│
├── 需要软体/流体物理
│   └── → Genesis / Flex
│
└── 教学 / 快速原型
    └── → PyBullet / MuJoCo (原生)
```
