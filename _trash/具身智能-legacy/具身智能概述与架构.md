# 具身智能概述与架构

## 什么是具身智能

**具身智能 (Embodied AI)** 指的是将智能体 (Agent) 嵌入到物理或模拟环境中，通过感知 (Perception)、决策 (Decision-making) 和执行 (Action) 的闭环交互来完成任务的人工智能范式。

与传统"纯数字"AI（如 NLP、CV）的核心区别：

| 维度 | 传统 AI | 具身智能 |
|------|---------|----------|
| 输入 | 静态数据集 | 实时传感器流（RGB、深度、点云、触觉、力/力矩） |
| 输出 | 标签/文本 | 连续控制信号（关节角度、末端位姿、轮速） |
| 评价 | 离线指标（accuracy, BLEU） | 任务成功率、泛化能力、安全性 |
| 交互 | 无/单轮 | 长时序闭环，动作影响后续观测 |

关键思想：**智能不仅来自大脑（模型），也来自身体与环境的交互**。这一观点源于认知科学的 "Embodied Cognition" 理论。

## 核心架构：感知-决策-执行

一个典型的具身智能系统可以抽象为三层架构：

```
┌─────────────────────────────────────────────┐
│              高层任务规划 (Task Planning)       │
│   自然语言指令 → 子任务分解 → 技能调度          │
│   (LLM/VLM, 如 SayCan, Code as Policies)     │
├─────────────────────────────────────────────┤
│            中层技能策略 (Skill Policy)          │
│   子任务 → 连续动作序列                        │
│   (RL Policy / Diffusion Policy / VLA)        │
├─────────────────────────────────────────────┤
│            底层运动控制 (Motor Control)         │
│   目标位姿/力 → 关节力矩                       │
│   (PID / 阻抗控制 / 运动规划器)                │
└─────────────────────────────────────────────┘
        ↑ 感知反馈              ↓ 执行指令
   ┌──────────┐          ┌──────────┐
   │  传感器   │          │  执行器   │
   │ RGB-D    │          │ 电机/气动 │
   │ LiDAR    │          │ 夹爪/灵巧手│
   │ IMU/触觉  │          │ 轮/足    │
   └──────────┘          └──────────┘
```

### 1. 感知层 (Perception)

负责从原始传感器数据中提取结构化信息：

- **视觉感知**：物体检测、6DoF 位姿估计、语义分割、深度估计
- **触觉感知**：接触检测、滑动检测、物体属性估计（硬度、纹理）
- **多模态融合**：RGB + 深度 + 点云 + 力/力矩信号的统一表示
- **场景理解**：3D 场景图（Scene Graph）、可供性 (Affordance) 预测

常见传感器：
- **RGB-D 相机**：Intel RealSense, Azure Kinect
- **LiDAR**：Velodyne, Livox
- **触觉传感器**：GelSight (视触觉)、BioTac
- **力/力矩传感器**：六维力传感器，安装在腕部

### 2. 决策层 (Decision-making)

将感知信息映射为动作或子目标：

**经典方法**：
- 状态机 (FSM)：手工设计状态转移，简单但不灵活
- 行为树 (Behavior Tree)：模块化、可组合，游戏和机器人广泛使用
- 任务与运动规划 (TAMP)：符号规划 + 运动规划结合

**学习方法**：
- **强化学习 (RL)**：通过试错学习最优策略，适合连续控制
- **模仿学习 (IL)**：从专家演示中学习，数据效率高
- **大模型方法**：LLM/VLM 作为高层规划器，生成代码或 API 调用

### 3. 执行层 (Execution)

将高层动作指令转化为电机级控制信号：

- **关节空间控制**：直接控制各关节角度/角速度
- **笛卡尔空间控制**：控制末端执行器的 6DoF 位姿
- **阻抗控制**：同时控制位置和力，适合接触任务
- **运动规划**：在关节空间中搜索无碰撞路径（RRT、RRT*、CHOMP）

## 主要研究方向

```
具身智能
├── 机器人操控 (Manipulation)
│   ├── 抓取 (Grasping)
│   ├── 灵巧操作 (Dexterous Manipulation)
│   └── 长horizon 任务 (Long-horizon Tasks)
├── 导航 (Navigation)
│   ├── 点目标导航 (PointNav)
│   ├── 目标导航 (ObjectNav)
│   └── 视觉-语言导航 (VLN)
├── 移动操控 (Mobile Manipulation)
│   └── 导航 + 操控联合
├── 人形机器人 (Humanoid)
│   ├── 双足运动 (Locomotion)
│   └── 全身控制 (Whole-body Control)
└── 多智能体协作
```

## 发展脉络

```
传统机器人 (1960s-2000s)     学习驱动 (2010s)          大模型驱动 (2023-)
─────────────────────────────────────────────────────────────────────
手工编程/PID控制     →    Deep RL / IL           →   LLM/VLM 规划
结构化环境           →    仿真训练               →   Foundation Model
单任务               →    多任务泛化             →   通用具身智能
```

**关键里程碑**：
- **2016**: OpenAI Gym 发布，统一 RL 接口
- **2018**: HABITAT 平台发布，大规模视觉导航
- **2020**: DexMV、DexYCB 灵巧操作 benchmark
- **2022**: SayCan (Google)，LLM 作为机器人规划器
- **2023**: RT-2 (Google DeepMind)，VLA 模型开山之作
- **2024**: π₀ (Physical Intelligence)，通用操控基础模型
- **2025.02**: Figure Helix 端到端 VLA，双频 S1/S2 架构
- **2025.03**: NVIDIA GR00T N1 首个开源 VLA，Gemini Robotics 发布
- **2025.04**: π₀.5 开放世界家庭泛化
- **2025**: 人形机器人量产爆发，工业场景百台级部署（Digit@GXO、Figure@BMW）
- **2026**: 数据飞轮 + 端侧 VLA 芯片（Jetson Thor）+ 世界模型耦合
