# 数据与训练范式演进（2026）

> "具身智能的瓶颈是数据，不是模型。"—— 这是 2025-2026 行业的主流判断。本文梳理数据来源、数据工厂、训练范式与飞轮设计。

## 一、为什么数据是瓶颈

```
对比 LLM vs 具身智能的数据规模:

LLM (GPT-4):
  预训练: ~10 万亿 tokens (整个互联网)
  RLHF: 数十万条人类偏好

具身智能 (π₀ 训练集):
  真机数据: ~1 万小时 (~数百万条轨迹)
  ≈ 2-3 年全人类机器人数据总量

差距: 5-6 个数量级
```

### 数据采集的四个难点

1. **需要物理接触**：不能从网页爬，必须真机采集
2. **跨本体不通用**：Franka 数据不能直接用于 UR5（夹爪、关节结构差异）
3. **任务特异性**：叠衣服的数据对做饭意义不大
4. **质量要求高**：失败轨迹需要筛选，专家水平遥操作稀缺

### 2.3 仿真数据

```

```

**仿真数据的质量衰减问题**：
- **Sim-to-Real gap** 在 VLA 时代依然存在
- 纯仿真训练的 VLA 真机性能通常比"仿真+少量真机混合"差 20-40%
- 最佳实践：仿真预训练 + 真机微调

  少量真实轨迹 → Cosmos 生成变体 → 大规模训练

优势:
  - 基于真实视觉分布（vs 仿真）
  - 可控增广（光照/物体/视角）
  
局限:
  - 动作维度难以"生成"（物理正确性）

### 2.5 人类视频学习

人类每天产生海量第一视角视频（Ego4D, YouTube 操作教程），如何利用？

**1. 视觉 encoder 预训练**
```
R3M (2022):      视频 + 语言对比学习 → 视觉特征
VIP (2023):      视频的 value function 学习
MVP:             Masked Video Pretraining
→ 作为 VLA 视觉 encoder 初始化
```

**2. 动作 retargeting**（最难）
```
人手视频 → 3D 手部位姿估计 → 映射到机器人夹爪/灵巧手动作
代表: H2O, AnyTeleop, DexCap
Embodiment gap 仍是最大挑战
```

**3. 视觉 → 动作的隐式预测**
```
UniPi, Vid2Robot:
  视频 predictor 生成"目标视频" → inverse dynamics 提取动作
  思路: 用视频生成做"规划"，动作由 ID 模型提取
```

**4. 互联网规模预训练**
```
π₀.5 声明: 训练时用了"大量多模态互联网数据"
Cosmos Reason: 视频推理 VLM, 作为世界知识
```

## 四、跨本体学习（Cross-Embodiment）

### 4.1 问题定义

> 不同机器人（Franka vs UR5 vs 人形）有不同关节结构、夹爪、相机位置，如何让一个模型在多种本体上工作？

### 4.2 主流方法

**方法 1：动作空间归一化**
```
每种机器人定义本体专属 action head:
  Franka head → 7D 目标末端位姿 + 夹爪
  UR5 head → 6D 末端位姿 + 夹爪
  Humanoid head → 35D 关节角
主干 (VLM) 共享
```

**方法 2：统一末端位姿**
```
所有本体都用末端执行器 6DoF pose (世界坐标) 作为动作
问题: 忽略关节冗余/约束
优势: π₀ 采用此方案，跨本体泛化好
```

**方法 3：潜在动作空间 (Latent Action)**
```
训练: 学习潜在动作 z
推理: z → 每种本体有独立 decoder 解码为具体关节/末端动作
代表: Latent Action Models, UniAct
优势: 真正本体无关
挑战: 训练复杂
```

**方法 4：本体参数作为条件**
```
输入: (obs, instruction, robot_description)
robot_description = URDF 嵌入 / 类别 ID
一个模型自适应不同本体
代表: Octo, GR00T N1
```

### 4.3 实践经验

- **Open X-Embodiment 实验**：跨本体训练的模型，在每个本体单独性能略降 10-20%，但获得零样本迁移新本体的能力
- **π₀** 在 8+ 种本体验证，微调 ~100 条数据即可适配新本体
- **Humanoid 与机械臂**：全身协调差异巨大，当前跨本体通常不混训

## 五、训练范式演进

### 5.1 五个阶段

```
阶段 1 (2015-2019): 单任务 RL
  ── DDPG, SAC, PPO
  ── 每个任务从头训练，数据 10-100 万 step

阶段 2 (2019-2022): 仿真 RL + Sim-to-Real
  ── 域随机化、Teacher-Student
  ── OpenAI Rubik's Cube, ANYmal Parkour

阶段 3 (2022-2023): Foundation Model 初现
  ── RT-1 (13 万真实数据)
  ── 大规模并行采集 + Transformer

阶段 4 (2023-2024): VLA 兴起
  ── RT-2, OpenVLA, Octo, π₀
  ── 利用 VLM 常识 + 动作头
  ── Open X-Embodiment 跨本体

阶段 5 (2025-2026): 多源融合 + 持续学习
  ── 真实 + 仿真 + 世界模型 + 人类视频
  ── 在线数据飞轮
  ── VLA + RL 后微调
```

### 5.2 当前主流训练流程（π₀ / GR00T 范式）

```
Step 1: VLM 预训练 (或用现成的)
  基础: PaliGemma, Qwen2-VL, Eagle 等

Step 2: 机器人数据联合训练 (Co-training)
  数据源混合:
    - 真实机器人数据 (OXE + 自有)
    - 仿真数据
    - 世界模型增广 (2025 新增)
    - (可选) 多模态互联网数据
  损失: Flow Matching / Diffusion 动作损失
  
Step 3: 任务特化微调
  目标场景 100-1000 条数据
  LoRA 或全参微调
  
Step 4 (2026 新): RL 后微调
  在仿真或真机 rollout
  Reward: 稀疏任务成功 + 专家偏好
  方法: DPPO, ReinFlow, IQL
  
Step 5: 持续学习 / 数据飞轮
  部署后收集 → 筛选 → 下一版
```

### 5.3 Flow Matching 动作头训练细节

```python
# 伪代码: π₀ 风格动作头训练
def train_step(obs, expert_action):
    # obs: 视觉 + 语言 + 本体状态
    # expert_action: shape (T, action_dim), T=50 future steps
    
    # 1. VLM encoding
    cond = vlm_encoder(obs)  # latent condition
    
    # 2. Flow Matching
    t = uniform(0, 1)
    noise = randn_like(expert_action)
    a_t = (1 - t) * noise + t * expert_action
    target_velocity = expert_action - noise
    
    # 3. 预测速度场
    pred_velocity = action_head(a_t, t, cond)
    loss = MSE(pred_velocity, target_velocity)
    
    return loss

# 推理: 从噪声积分
def inference(obs):
    cond = vlm_encoder(obs)
    a = randn(50, action_dim)  # 初始噪声
    dt = 0.1
    for step in range(10):  # 10步即可
        t = step * dt
        v = action_head(a, t, cond)
        a = a + v * dt
    return a  # shape (50, action_dim)
```

### 5.4 VLA + RL 后微调（2025-2026 新趋势）

VLA 用 imitation 训练，行为上限 = 专家。**RL 可以突破专家水平**，但传统 RL 需要大量数据。

```
DPPO (Diffusion Policy PPO, 2024):
  将 Diffusion Policy 视为策略
  用 PPO 优化去噪链
  在仿真中 finetune, 性能显著提升

ReinFlow (Flow Matching + RL, 2025):
  对 π₀ 类 Flow Matching 的 RL 微调
  用仿真 rollout 改善难任务

RLDG (RL + Diffusion-guided, 2024):
  RL 引导 Diffusion 采样
  数据效率高于纯 RL
```

**实践要点**：
- 真机 RL 代价巨大，优先仿真
- Reward shaping 仍然棘手
- 稳定性问题（KL 约束 / reference policy）

## 六、数据飞轮设计

### 6.1 飞轮循环

```
     ┌──────────────────────────────────┐
     ▼                                  │
  数据生产       ←───────────   部署反馈   │
     │                                  │
     │ 筛选/标注                          │
     ▼                                  │
  训练           ───────────→  策略部署   │
     │                                  │
     └──────────────────────────────────┘
```

**关键环节**：
1. **部署 → 反馈**：机器人出错/人工纠正时回传数据
2. **筛选**：过滤低质量/重复数据
3. **标注**：任务描述、失败原因、是否成功
4. **训练**：增量训练或全量重训

### 6.2 成功案例

**Tesla FSD 的启示**：
- 2M+ 车队持续回传数据
- "corner case" 自动识别并回收
- 每月迭代一版模型
- 这是**具身飞轮的目标状态**

**当前具身玩家的飞轮阶段**：

| 公司 | 阶段 | 数据回传规模 |
|------|-----|------------|
| Figure | Beta | BMW 工厂 + 内部测试 |
| Physical Intelligence | Alpha | 合作伙伴试点 |
| Tesla Optimus | Alpha | 工厂内部 |
| 智元 AgiBot | Alpha | 合作场景 |
| 1X | Beta | Home Beta 用户 |

**尚未真正跑通"小时级"数据飞轮**。

### 6.3 RLHF 类似的机器人反馈

- 人类偏好数据：两段相似轨迹选"更好"的
- 用于 reward model 或 DPO 式微调
- 2025 开始在具身领域试水（e.g., RoboArena）

  - 联合训练 (多源混合) 1-4 周
  - 场景特化微调 3-7 天

## 九、2026 数据主题预测
