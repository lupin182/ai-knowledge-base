# 模仿学习与强化学习 (具身智能视角)

## 模仿学习 (Imitation Learning)

模仿学习从专家演示中学习策略，无需设计奖励函数，是目前具身智能最主流的训练范式。

### 为什么模仿学习在机器人中如此重要

- **奖励设计困难**：操控任务的奖励函数极难手工定义（如"叠衣服"的奖励是什么？）
- **数据效率高**：几十到几百条演示即可训练有效策略
- **行为可控**：策略的行为风格直接来自演示，安全可预测

### 数据采集方式

| 方式 | 设备 | 优点 | 缺点 |
|------|------|------|------|
| **遥操作 (Teleoperation)** | VR 控制器、SpaceMouse、外骨骼 | 高质量、动作对齐 | 效率低、需硬件 |
| **运动学教示** | 手把手拖动机器人 | 直觉、无需额外设备 | 精度受限 |
| **VR 遥操作** | Quest 3、Vision Pro | 沉浸式、适合双臂 | 延迟、习惯差异 |
| **人体视频** | 普通摄像头 | 海量数据 | 形态差异 (embodiment gap) |
| **仿真自动生成** | 脚本化/RL 策略 | 无限量 | Sim-to-Real gap |

#### ALOHA 系统

斯坦福 Tony Zhao 团队的双臂遥操作系统：
- **硬件**：2 个 ViperX 机械臂 (leader) + 2 个 ViperX (follower)
- **操作者**控制 leader 臂，follower 臂实时跟随
- 采集一对一映射的动作数据
- **Mobile ALOHA**：加了底盘，可移动
- 低成本 (~$20K)，大量实验室复制

### 行为克隆 (Behavioral Cloning, BC)

最简单的模仿学习：监督学习直接拟合 $\pi_\theta(a|o) \approx \pi^*(a|o)$。

$$\mathcal{L} = \mathbb{E}_{(o,a) \sim \mathcal{D}} \left[ \| \pi_\theta(o) - a \|^2 \right]$$

#### 问题：Covariate Shift

训练时看到的是专家的状态分布，部署时策略会犯小错误，导致偏离专家分布 → 错误累积 → 完全失控。

**缓解方法**：
- **DAgger**：部署策略 → 专家标注新状态 → 加入训练集 → 迭代
- **Action Chunking**：预测多步动作而不是单步，减少累计误差
- **扩散策略**：多模态分布建模

#### 现代 BC 方法

| 方法 | 策略表示 | 特点 |
|------|----------|------|
| **BC-MLP** | 简单 MLP | 基线，容易训练 |
| **BC-RNN** | LSTM | 处理部分可观测 |
| **ACT** | CVAE + Transformer | Action chunking，ALOHA 配套 |
| **Diffusion Policy** | 扩散模型 | 多模态，SOTA |
| **VQ-BeT** | VQ-VAE + Transformer | 离散化动作空间 |

### 逆强化学习 (Inverse RL)

从演示中推断奖励函数，再用 RL 优化该奖励：

$$r^* = \arg\max_r \mathbb{E}_{\pi^*}[r(s,a)] - \mathbb{E}_{\pi_r}[r(s,a)]$$

使专家策略在推断奖励下的回报高于其他策略。

**GAIL (Generative Adversarial Imitation Learning)**：
- 判别器区分专家轨迹和策略轨迹
- 策略优化：骗过判别器
- 类似 GAN，但生成的是轨迹

## 强化学习在具身智能中的应用

### 与纯 RL 研究的区别

具身智能 RL 的特殊挑战：
- **仿真到真实的差距 (Sim-to-Real Gap)**：仿真中学到的策略需要能迁移
- **安全约束**：不能像 Atari 那样随意探索（机器人可能摔倒/碰撞）
- **高维连续动作**：关节空间通常 6-30 维连续
- **部分可观测**：传感器噪声、遮挡

### 常用 RL 算法

| 算法 | 类型 | 特点 | 具身智能适用场景 |
|------|------|------|-----------------|
| **PPO** | On-policy | 稳定、易调参 | Locomotion、大规模并行训练 |
| **SAC** | Off-policy | 样本高效、自动温度 | 操控、真机训练 |
| **TD3** | Off-policy | SAC 的确定性版 | 连续控制 |
| **DrQ-v2** | Off-policy + 数据增强 | 直接从像素学习 | 视觉 RL |

### 奖励工程

具身智能中的奖励设计是个核心问题。以四足行走为例：

```python
# 典型四足运动奖励
reward = (
    1.0 * velocity_tracking_reward    # 跟踪目标速度
    - 0.1 * torque_penalty            # 减小力矩（省能量）
    - 0.5 * action_rate_penalty       # 动作平滑
    - 2.0 * fall_penalty              # 不摔倒
    - 0.1 * joint_limit_penalty       # 不超关节限位
    + 0.5 * feet_air_time_reward      # 鼓励抬脚（trot步态）
    - 0.1 * base_height_penalty       # 保持躯干高度
)
```

**自动奖励设计**：
- **Eureka (2023, NVIDIA)**：用 LLM 自动生成和迭代奖励函数代码
- **Text2Reward**：从自然语言描述生成奖励

### 课程学习 (Curriculum Learning)

逐步增加任务难度，帮助策略从简单到复杂：

```
Level 1: 平地行走
Level 2: 小斜坡
Level 3: 台阶
Level 4: 随机地形
Level 5: 极端地形 (碎石、悬崖边)
```

实现方式：
- **自动课程**：根据策略当前成功率自动调整难度
- **地形课程**：Isaac Lab 内置，按成功率分配不同难度地形

### 域随机化 (Domain Randomization)

训练时随机化仿真参数，使策略对参数变化鲁棒，从而迁移到真实：

| 随机化类型 | 参数示例 | 随机范围 |
|-----------|----------|----------|
| **物理参数** | 摩擦系数、质量、惯量 | ±30-50% |
| **视觉参数** | 光照、纹理、相机位置 | 大范围 |
| **动力学参数** | 电机延迟、PD 增益 | ±20-30% |
| **干扰** | 外力推动、负载变化 | 随机脉冲 |

**关键 insight**：不需要精确仿真，只需要真实参数落在随机范围内。

### Teacher-Student 训练

两阶段训练范式，解决观测差异问题：

```
阶段 1: Teacher 策略 (在仿真中)
    - 输入: 特权信息 (ground truth 状态、接触力、地形高度图)
    - 用 RL 训练到很强

阶段 2: Student 策略 (可部署)
    - 输入: 仅传感器可获取的信息 (关节编码器、IMU)
    - 通过蒸馏模仿 Teacher 的行为
    - 学会从有限传感器中隐式估计特权信息
```

这在 locomotion 中几乎是标准做法。
