# 导航与运动规划

## 导航 (Navigation)

### 经典导航框架

传统机器人导航基于 **SLAM + 路径规划 + 控制** 的模块化架构：

```
传感器 (LiDAR/RGB-D)
    │
    ▼
┌────────────┐
│   SLAM     │  ← 同步定位与地图构建
│ (建图+定位) │     输出：地图 + 机器人位姿
└─────┬──────┘
      │
      ▼
┌────────────┐
│  路径规划   │  ← 全局: A*/Dijkstra/RRT
│            │     局部: DWA/TEB
└─────┬──────┘
      │
      ▼
┌────────────┐
│  运动控制   │  ← 跟踪规划路径
│ (PID/MPC)  │     输出：线速度/角速度
└────────────┘
```

### SLAM 方法

| 方法 | 传感器 | 特点 |
|------|--------|------|
| **GMapping** | 2D LiDAR | 粒子滤波，经典 2D SLAM |
| **Cartographer** | 2D/3D LiDAR | Google 开源，回环检测优秀 |
| **ORB-SLAM3** | 单目/双目/RGB-D | 特征点法视觉 SLAM，支持 IMU 融合 |
| **RTAB-Map** | RGB-D/LiDAR | 实时、可大规模建图 |
| **LOAM/LIO-SAM** | 3D LiDAR + IMU | 激光惯性里程计 |

### 学习式导航

#### PointNav (点目标导航)

给定目标坐标 $(x, y)$，导航过去。

**DD-PPO (Wijmans et al., 2019)**：
- 在 Habitat 中大规模分布式训练 PPO
- 2.5 亿帧训练，PointNav 接近完美 (99%+ SPL)
- 关键 insight：**足够多数据 + 简单 RL = 非常强的导航策略**

#### ObjectNav (目标导航)

给定物体类别（如"找到电视"），导航到该物体前。

比 PointNav 难得多：
- 需要语义理解（什么是电视、电视通常在哪）
- 需要探索策略（先去客厅找而不是浴室）
- 当前最好方法约 30-50% 成功率

**方法**：
- **模块化**：物体检测 → 语义地图 → 前沿探索 → 规划
- **端到端 RL**：直接从 RGB-D 学习导航策略
- **LLM 增强**：利用常识推理物体可能在哪

#### Vision-Language Navigation (VLN)

根据自然语言指令导航："走出卧室，穿过走廊，在第二个门右转，你会看到厨房里的冰箱。"

**R2R (Room-to-Room) Benchmark**：
- 基于 Matterport3D 真实室内扫描
- 自然语言路径描述 → 导航轨迹

**代表方法**：
- **VLN-BERT**：预训练视觉-语言-动作 Transformer
- **DUET**：双尺度图 Transformer（拓扑图 + 细粒度视图）
- **NavGPT**：利用 GPT-4 进行零样本导航推理

### 语义地图与场景理解

现代导航越来越依赖丰富的场景表示：

**传统地图**：
- 栅格地图 (Occupancy Grid)：2D/3D，标记可通行/不可通行
- 拓扑地图：节点(地点) + 边(连通性)

**语义地图**：
- 每个栅格不仅有占据信息，还有语义类别
- 如 "这是厨房台面"、"这是沙发"

**3D 场景图 (Scene Graph)**：
```
房间 (卧室)
├── 物体: 床 (位置, 属性)
├── 物体: 台灯 (位置, on/off)
├── 关系: 台灯 在 床头柜 上面
└── 连接: 门 → 走廊
```

**VLMap / ConceptFusion (2023)**：
- 将 VLM 特征（CLIP/DINO）嵌入到 3D 地图中
- 支持开放词汇查询："找到红色的杯子"

## 运动规划 (Motion Planning)

运动规划解决的问题：在高维关节空间中找到一条无碰撞路径，将机器人从当前构型移动到目标构型。

### 构型空间 (Configuration Space)

$$\mathcal{C} = \{q \in \mathbb{R}^n : q \text{ 是关节角度向量}\}$$

- **自由空间 $\mathcal{C}_{free}$**：无碰撞的构型集合
- **障碍空间 $\mathcal{C}_{obs}$**：导致碰撞的构型集合
- 目标：在 $\mathcal{C}_{free}$ 中找到从 $q_{start}$ 到 $q_{goal}$ 的路径

### 采样式规划器

#### RRT (Rapidly-exploring Random Tree)

```
算法 RRT(q_start, q_goal):
    T.init(q_start)
    for i = 1 to N:
        q_rand = RandomSample(C)           # 随机采样
        q_near = NearestNode(T, q_rand)     # 找最近节点
        q_new = Steer(q_near, q_rand, δ)    # 向随机点扩展 δ 距离
        if CollisionFree(q_near, q_new):     # 碰撞检测
            T.add(q_new, parent=q_near)
            if Distance(q_new, q_goal) < ε:
                return ExtractPath(T)
    return FAILURE
```

**RRT\*** 改进：
- 在附近节点中选择路径代价最小的父节点
- 对邻域节点重新布线 (Rewire)
- **渐近最优**：采样数趋向无穷时收敛到最优路径

#### PRM (Probabilistic Roadmap)

- **预处理阶段**：在 $\mathcal{C}_{free}$ 中均匀采样节点，连接邻近无碰撞节点
- **查询阶段**：将起终点连入路线图，用图搜索（A*）找路径
- 适合多次查询同一环境

### 优化式规划器

#### CHOMP (Covariant Hamiltonian Optimization)

将轨迹优化为：

$$\xi^* = \arg\min_\xi \underbrace{f_{smooth}(\xi)}_{\text{平滑性}} + \lambda \underbrace{f_{obs}(\xi)}_{\text{避障}}$$

- 用梯度下降优化整条轨迹
- 需要障碍物距离场 (SDF) 的梯度

#### TrajOpt

- 类似 CHOMP，使用**顺序凸优化 (SCP)**
- 将碰撞约束线性化，迭代求解凸问题
- 通常比 CHOMP 更快收敛

### MoveIt (ROS 运动规划框架)

ROS 生态中最主流的运动规划框架：

- **MoveIt 2**：支持 ROS 2
- 集成 OMPL (Open Motion Planning Library)：包含 RRT、RRT*、PRM 等
- 支持笛卡尔路径规划、约束规划
- 提供碰撞检测、场景管理、抓取规划

```python
# MoveIt Python 接口示例
import moveit_commander

arm = moveit_commander.MoveGroupCommander("arm")
arm.set_pose_target(target_pose)  # 目标末端位姿
plan = arm.plan()                  # 规划
arm.execute(plan)                  # 执行
```

## Locomotion (足式运动)

### 四足运动

四足机器人（如 Unitree Go2、Spot）的运动控制：

**经典步态**：
- Walk: 每次一条腿移动
- Trot: 对角线两腿同时移动
- Gallop: 类似马的奔跑

**RL 训练四足运动** 是近年最成功的应用之一：

1. 在仿真中 (Isaac Gym) 训练 RL 策略
2. 观测：关节角/角速度、IMU、命令速度
3. 动作：目标关节角度 (PD 控制器跟踪)
4. 奖励：速度跟踪 + 平稳 + 能耗低 + 不摔倒
5. 域随机化 → Sim-to-Real 零样本迁移

**代表工作**：
- **Legged Gym / Isaac Lab**: NVIDIA 的四足/人形 RL 训练框架
- **Walk These Ways (2023)**: 单一策略覆盖多种步态和速度
- **Parkour (2023, ETH Zurich)**: 四足机器人翻越障碍、跳跃

### 人形运动

人形机器人双足行走更难（欠驱动、平衡困难）：

**ZMP (Zero Moment Point)**：
- 经典方法：确保 ZMP 在支撑多边形内
- 用于 ASIMO、NAO 等

**RL 训练人形运动**：
- **2024-2025 爆发**：Unitree H1 后空翻、各种敏捷运动
- 关键：大规模仿真并行 + 课程学习 (Curriculum) + 域随机化

**从人体动作捕捉学习**：
- 用 CMU MoCap、AMASS 数据集提供参考运动
- RL 策略模仿参考运动同时适应物理约束
- **AMP (Adversarial Motion Priors)**：用判别器代替手工奖励
- **PHC (Perpetual Humanoid Control)**：持续人形控制
