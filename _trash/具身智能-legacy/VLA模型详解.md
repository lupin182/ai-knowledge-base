# VLA 模型详解 (Vision-Language-Action)

## 什么是 VLA

**VLA (Vision-Language-Action)** 模型是将视觉语言模型 (VLM) 扩展到机器人动作生成的基础模型。核心思想：像 LLM 预测下一个 token 一样，VLA 预测下一个动作。

```
传统流程:  图像 → 检测/分割 → 规划 → 控制
VLA:      图像 + 语言指令 → 直接输出动作
```

VLA 代表了具身智能从模块化走向端到端的趋势。

## RT 系列 (Google DeepMind)

### RT-1 (Robotics Transformer, 2022)

不是 VLA，但奠定了基础：

- **架构**：EfficientNet (视觉) + TokenLearner + Transformer
- **输入**：RGB 图像 + 语言指令
- **输出**：离散化的动作 token (末端位姿增量 + 夹爪)
- **数据**：13 万条真实机器人演示（17 个月采集）
- **结果**：97% 成功率（见过的任务），76%（未见任务组合）

关键 insight：**数据规模和多样性**比模型架构更重要。

### RT-2 (2023) — 第一个 VLA

**核心创新**：直接复用预训练的 VLM（PaLM-E / PaLI-X），将动作表示为文本 token。

```
输入:  [图像] "pick up the blue cup"
输出:  "1 128 91 241 1 0 0 1"   ← 这是动作！
       (x  y   z  rx ry rz grip terminate)
```

**动作离散化**：
- 将连续动作空间均匀分桶为 256 个离散值
- 每个维度映射到一个整数 token
- VLM 自回归生成这些 token

**为什么这么做**：
- 复用 VLM 的预训练知识（视觉理解 + 语言推理）
- 不需要新架构，只需要微调
- 涌现出 VLM 未训练过的推理能力（如理解"移到左边的物体旁"）

**局限**：
- 大模型推理慢（5B/55B 参数），控制频率仅 1-3 Hz
- 动作离散化损失精度
- 依赖大量真实数据

### RT-H (2024)

引入语言化的中间动作表示，提高人类可读性和动作复用。

## Octo (2024, Berkeley)

开源的通用操控模型：

- **架构**：Transformer，支持多种观测（单/双腕相机、外部相机）和动作空间
- **预训练**：Open X-Embodiment 数据集（80 万轨迹、多种机器人）
- **微调**：少量新任务数据即可适配新机器人
- **特点**：模块化 tokenizer 设计，适配不同机器人配置

```
┌──────────────────────────────────────────────┐
│                  Octo 架构                    │
│                                              │
│  任务 Token: [语言指令 / 目标图像]             │
│  观测 Token: [外部相机] [腕部相机] [本体感觉]   │
│         │                                    │
│    ┌────▼────┐                               │
│    │Transformer│  ← 注意力机制融合多模态        │
│    │ Backbone │                               │
│    └────┬────┘                               │
│         │                                    │
│    ┌────▼────┐                               │
│    │ Diffusion │  ← 扩散头生成连续动作          │
│    │  Head    │                               │
│    └─────────┘                               │
│         │                                    │
│    动作: [Δx, Δy, Δz, Δrx, Δry, Δrz, grip]  │
└──────────────────────────────────────────────┘
```

## OpenVLA (2024, Stanford)

开源 VLA，基于 Prismatic VLM 微调：

- **架构**：SigLIP (视觉) + DinoV2 (视觉) + Llama 2 7B (语言/动作)
- **输入**：单张 RGB 图像 + 语言指令
- **输出**：离散化的 7DoF 动作 token（类似 RT-2）
- **训练数据**：Open X-Embodiment 子集 (97 万轨迹)
- **开源**：模型权重、代码、训练流程完全开源

**特点**：
- 证明了 7B 参数的 VLA 可以有效泛化
- 微调简单：LoRA 即可适配新任务
- 控制频率 ~5-10 Hz（得益于较小模型）

**OpenVLA 2 (2025)**：
- 改用扩散动作头替代离散化 → 精度更高
- 支持多图像输入（多视角）
- 性能大幅提升

## π₀ (Physical Intelligence, 2024)

通用操控基础模型，由 Physical Intelligence 公司开发（团队包含 Chelsea Finn、Sergey Levine 等）。

### 架构：VLM + Flow Matching

```
┌─────────────────────────────────────────┐
│              π₀ 架构                     │
│                                         │
│  ┌─────────┐   ┌───────────┐            │
│  │ 视觉编码 │   │ 语言编码   │            │
│  │ (SigLIP) │   │ (Gemma)   │            │
│  └────┬────┘   └─────┬─────┘            │
│       │              │                   │
│       └──────┬───────┘                   │
│              │                           │
│         ┌────▼────┐                      │
│         │  VLM    │  ← 预训练的 VLM 主干  │
│         │ Backbone│    理解图像和指令      │
│         └────┬────┘                      │
│              │                           │
│         ┌────▼────┐                      │
│         │  Flow   │  ← 动作生成头        │
│         │ Matching│    连续动作去噪       │
│         │  Head   │                      │
│         └─────────┘                      │
│              │                           │
│    动作 chunk (未来 ~1s 的动作序列)        │
└─────────────────────────────────────────┘
```

### Flow Matching 动作头

比 DDPM 更简单高效的生成方法：
- 学习从噪声到目标动作的**直线流** (Rectified Flow)
- 推理只需 5-10 步去噪（DDPM 需 50-100 步）
- 数学更简洁：$v_\theta(x_t, t) = x_1 - x_0$（速度场近似）

### 关键特点

- **跨任务泛化**：折叠衣物、收拾桌子、装箱等多种任务
- **跨机器人**：单臂、双臂、不同夹爪
- **精细操控**：能处理布料、食物等形变物体
- **数据规模**：万级真实轨迹 + 仿真数据

### π₀.5 (2025)

π₀ 的升级版，加入高层语言推理：
- VLM 不仅理解指令，还能**自主推理**下一步该做什么
- 实现更长 horizon 的自主任务完成
- 在真实家庭环境中长时间自主运行

## VLA 核心技术对比

| 维度 | RT-2 | Octo | OpenVLA | π₀ |
|------|------|------|---------|-----|
| 动作表示 | 离散 token | 扩散 | 离散 token | Flow Matching |
| VLM 基座 | PaLM-E (55B) | 无 (from scratch) | Llama 2 (7B) | Gemma 系列 |
| 控制频率 | 1-3 Hz | 5-10 Hz | 5-10 Hz | 10-50 Hz |
| 开源 | ❌ | ✅ | ✅ | ❌ |
| 训练数据 | 130K 真实 | 800K OXE | 970K OXE | 万级真实+仿真 |

## VLA 面临的挑战

1. **推理速度**：大模型推理慢，难以高频控制 (需要 > 20Hz)
2. **动作精度**：离散化 token 损失精度，连续动作头（扩散/Flow Matching）正在解决
3. **数据瓶颈**：真实机器人数据采集仍然昂贵
4. **泛化边界**：对全新物体/场景的泛化仍然有限
5. **安全性**：端到端模型的行为难以保证安全

## 未来方向

- **规模化**：更大数据、更大模型 → 涌现更多能力
- **仿真预训练 + 真实微调**：降低真实数据需求
- **多模态输入**：加入触觉、力反馈、点云
- **分层 VLA**：高层 VLA 规划 + 底层专家策略执行
- **世界模型 + VLA**：学习环境动力学，用于规划和想象
