# 前沿 VLA 与世界模型（2026）

> 本文聚焦 2025-2026 年 VLA（Vision-Language-Action）与世界模型（World Model）两条主线的最新工作。VLA 基础与早期模型（RT-2 / Octo / OpenVLA / π₀ 初版）见 [VLA 模型详解](VLA模型详解.md)。

## 一、2025-2026 VLA 架构共识

经过 2023-2024 年的探索，2025 年起行业形成如下共识：

```
┌────────────────────────────────────────────────────────┐
│              2026 主流 VLA 双系统架构                     │
├────────────────────────────────────────────────────────┤
│                                                        │
│  System 2 (慢思考, 7-9 Hz)                              │
│  ┌─────────────────────────┐                           │
│  │  VLM backbone           │                           │
│  │  (7-9B, e.g. PaliGemma, │                           │
│  │   Eagle, Qwen2-VL)      │                           │
│  └──────────┬──────────────┘                           │
│             │ latent / token                           │
│             ▼                                          │
│  System 1 (快执行, 100-200 Hz)                          │
│  ┌─────────────────────────┐                           │
│  │  Action Expert          │                           │
│  │  (80-300M, Flow-Match / │                           │
│  │   Diffusion / Transformer)│                          │
│  └──────────┬──────────────┘                           │
│             │                                          │
│             ▼  连续动作 chunk (1-2秒, 50步)              │
│                                                        │
└────────────────────────────────────────────────────────┘
```

**关键特点**：
1. **异步双频**：VLM 低频理解"做什么"，Action Expert 高频生成"怎么做"
2. **VLM 参数冻结 or 半冻结**：保留预训练的视觉/语言能力，不破坏常识
3. **动作专家独立设计**：Flow Matching 主流，少数用 Diffusion 或 Transformer 自回归
4. **Chunk Prediction**：预测 1-2 秒未来动作，减少推理频次
5. **多本体通用**：一个模型训多种机器人

## 二、Physical Intelligence: π₀ 系列

### π₀.5 (2025.04)

π₀ 的重要升级，**面向家庭开放世界的泛化**。

**关键创新**：
- **Co-Training**：在"全网多模态数据 + 真实机器人数据 + 仿真数据"上联合训练
- **高层推理能力**：模型自主分解长任务（如"清理厨房"→ 多步子任务）
- **新家新物体零样本**：在未见过的真实住宅中执行叠衣、整理玩具、擦厨房

**架构变化**：
- 基于 PaliGemma (3B VLM)
- Flow Matching 动作头
- 加入"discrete language action"：模型可以输出中间自然语言规划步骤

**定量结果（π₀.5 论文）**：
- 未见过的家庭场景，20-60 分钟长时任务成功率 40-90%
- 物体泛化：训练未见过的 100+ 物体

### π-fast (2025H2)

面向低延迟部署的轻量版：
- 参数减半，推理速度 3-5 倍
- 保留 π₀.5 主体能力
- 目标 30Hz+ 闭环

### π₀ 技术细节

```
输入:
  - 多视角 RGB (通常 2-3 路)
  - 机器人本体状态 (proprioception)
  - 语言指令

主干: PaliGemma-3B (SigLIP-So400m + Gemma-2B)
动作头: Flow Matching Transformer, 50 步 chunk
训练数据:
  - ~10,000 小时真实遥操作数据
  - Open X-Embodiment 子集
  - 仿真增强数据
训练策略:
  1. 预训练 (VLM on multimodal) 
  2. 机器人联合训练 (cross-embodiment)
  3. 任务特化微调

Flow Matching Loss:
  L_FM = E_{t, a_0, a_1} || v_θ(a_t, t, obs) - (a_1 - a_0) ||²
  其中 a_t = (1-t)·a_0 + t·a_1
  推理只需 5-10 步 Euler integration
```

## 三、NVIDIA GR00T 系列

### GR00T N1 (2025.03, 首个开源)

面向人形机器人的基础模型，也是**首个 Apache 2.0 开源的 VLA**。

```
架构: System 2 + System 1
  System 2: Eagle-2 VLM (2B)
  System 1: Diffusion Transformer (DiT, 1B)
            输入: VLM 的 token + 本体状态
            输出: 动作 chunk (16步)

训练数据:
  - Open X-Embodiment
  - AgiBot-World (100万条)
  - 仿真数据 (Isaac Sim / GR00T Dreams)
  - Fourier GR-1 / Unitree H1 真实数据

控制频率: VLM 7 Hz / DiT 30+ Hz
开源: 模型权重、训练代码、数据 pipeline
支持本体: Fourier GR-1, Unitree H1, Neura 4NE-1, etc.
```

**训练 pipeline（3 阶段）**：

1. **Stage 1**：VLM 预训练（保留 Eagle-2 的多模态能力）
2. **Stage 2**：Action Expert 在大规模跨本体数据上训练
3. **Stage 3**：特定本体微调

### GR00T N1.5 / N2 (2025H2 - 2026)

预期改进：
- 更大的动作专家（支持更长 horizon）
- 触觉输入通道
- 与 Cosmos 世界模型联合训练
- 开源持续扩展

### GR00T Dreams

NVIDIA 基于 Cosmos 的**合成数据生成 pipeline**：
- 从少量真实轨迹 → 生成大量变体轨迹（新视角、新物体、新场景）
- 用于扩展训练数据规模
- 2025 年 GTC 宣布对公众开放

## 四、Figure Helix (2025.02)

Figure AI 的首个端到端 VLA，部署在 Figure 02 上。

### 架构细节

```
System 2 (S2):
  - 7B VLM (internet-scale 预训练)
  - 9 Hz
  - 输入: 多相机 RGB + 语言
  - 输出: continuous latent vector (256维)

System 1 (S1):
  - 80M Transformer
  - 200 Hz
  - 输入: S2 latent + 本体状态 (关节角、速度、末端力)
  - 输出: 连续动作 (全身控制, 35-DOF 上半身)

训练:
  - ~500 小时高质量 Figure 02 遥操作数据
  - 单一策略覆盖: 抓取、放置、递送、开柜门、家居任务
  - 无需任务特定微调
```

**技术亮点**：

1. **单一神经网络做全身协调**：双臂 + 躯干 + 手 + 头，一个模型全部输出
2. **双机器人协作**：Helix 版本支持两台 Figure 02 协同工作（一个递物体另一个放置）
3. **快慢系统解耦的端到端**：不是"VLM 规划 + 技能库"，而是端到端训练的双频模型
4. **商业闭环数据**：BMW 工厂部署持续生成真实数据

**Helix v2 (2025H2)**：
- 更长 horizon 推理
- 人形家庭场景（洗碗、叠衣）
- 在 Figure 03 上部署

## 五、Google DeepMind Gemini Robotics (2025.03)

### Gemini Robotics

基于 Gemini 2.0 的 VLA，DeepMind 回归机器人领域的旗舰工作。

```
主干: Gemini 2.0 Vision (经过机器人数据继续训练)
动作头: Diffusion Transformer
场景: 桌面双臂操控 (ALOHA 2 类硬件)
控制频率: 10-30 Hz
```

**关键能力**：
- **Generality**：大量跨物体 / 跨任务泛化
- **Interactivity**：响应自然语言指令（包括中途修正）
- **Dexterity**：折纸船、拉拉链、装配乐高

### Gemini Robotics-ER (Embodied Reasoning)

**ER 版本专注"具身推理"**，不直接输出动作，而是：
- 空间理解：物体位置、可供性（affordance）预测
- 长 horizon 规划：分解为子步骤
- 代码生成：生成控制代码（类似 Code as Policies）

**思路**：VLM 做"大脑"，接到任何动作后端（策略网络 / 经典控制 / 第三方 VLA）都可用。

### DeepMind-Apptronik 合作

2025 年 DeepMind 与 Apptronik 深度合作：
- Gemini Robotics 在 Apollo 人形上部署
- 共享研究成果
- 对标 Figure + OpenAI 的组合

## 六、其他重要 VLA（2025）

### RDT-2 (2025, 清华/星动)

Robotics Diffusion Transformer 的第二代：
- 基于 SigLIP + T5
- **1B 参数**，双臂/双机协作
- **MoE 动作头**：不同任务激活不同专家
- 开源，国内最强 VLA 之一

### OpenVLA-OFT (2024.10)

Stanford 对 OpenVLA 的重要改进：
- **Optimized Fine-Tuning**：更高效微调
- 并行解码（vs 自回归）→ 控制频率 26Hz → 109Hz
- L1 回归代替 cross-entropy
- **关键发现**：OpenVLA 原版受限于离散 token 和自回归，而非模型能力

### CogACT (2024.11)

微软亚研：VLA 中解耦认知（cognition）和动作（action）：
- VLM 编码器 + 独立的 diffusion action decoder
- 30Hz，SOTA 级成功率

### TinyVLA / SmolVLA (2025, HuggingFace LeRobot)

**轻量化 VLA**：
- 1B 参数以内
- 端侧部署（Jetson / Mac Mini 可跑）
- 降低研究门槛，HF LeRobot 生态
- 2025.05 SmolVLA 发布，~500M 参数

### Dexbotic / iDP3 / SpiRit / GO-1

- **Dexbotic (2024)**：灵巧操作 VLA
- **Improved 3D Diffusion Policy**：3D 输入版 Diffusion Policy
- **SpiRit-LM**：多模态生成 + 动作
- **GO-1 (智元, 2025)**：开源大规模 VLA

### Helix 类型的"全身"VLA

2025 趋势：从"桌面双臂"到"全身协调"：
- Helix (Figure)
- GR00T N1（Fourier GR-1 全身）
- RT-H, Mobile ALOHA VLA
- UniAct (智元)

## 七、VLA 训练范式演进

### 2023 离散 token 范式
```
VLM 自回归输出: "15 128 91 241 0 1"
→ 直接解码为动作维度
问题: 控制频率低、精度差、无多模态建模
```

### 2024 扩散 / Flow Matching 范式
```
VLM 作为条件编码器 → 扩散/FM 动作头 → 连续动作 chunk
改进: 高频输出、多模态分布、精细控制
代表: π₀, Octo, Diffusion Policy, RDT
```

### 2025 异步双系统范式
```
VLM (低频) → latent/token → Action Expert (高频)
改进: 实时性、保留 VLM 能力、可扩展
代表: Helix, GR00T N1, π₀.5
```

### 2026 趋势：世界模型耦合
```
VLM + World Model (做 rollout / 想象) → Action Expert
改进: 长 horizon 规划、反事实推理、sample efficiency
代表: π₀.5 + Cosmos, Gemini Robotics + Genie
```

## 九、世界模型与具身智能

### 9.1 为什么世界模型重要

```
传统 RL/IL:       策略 π(a|s) 直接映射
世界模型加持:     策略可以在"想象"中 rollout 
                 P(s'|s,a) + R(s,a) → 规划/搜索
                 P(o'|o,a) (视频世界模型)

三大价值:
  1. 数据增广: 合成逼真轨迹
  2. 规划: 在模型中想象未来
  3. 无监督学习: 从海量视频学习物理/因果
```

### 9.2 NVIDIA Cosmos (2025.01)

NVIDIA 在 CES 2025 发布的**"世界基础模型"(World Foundation Model)** 平台：

```
Cosmos 组件:
├── Cosmos Predict (视频生成，多模态条件)
│   ├── Cosmos-1.0-Diffusion (5B / 14B)
│   └── Cosmos-1.0-Autoregressive (12B)
├── Cosmos Transfer (真实到仿真风格迁移)
├── Cosmos Reason (视频推理 VLM)
└── Cosmos Nemotron (合成训练数据生成)
```

**用于机器人**：
- **数据增广**：真实一条轨迹 → 视频 WM 生成 N 条变体（不同光照/物体/角度）
- **Sim2Real 桥梁**：仿真画面 → Cosmos Transfer → 真实风格画面
- **策略评估**：在世界模型中 rollout 策略
- **开源商用许可**：Apache 2.0-like，推动生态

### 9.3 Google DeepMind Genie 系列

```
Genie 1 (2024.02): 从无标签视频学习"可交互世界"
Genie 2 (2024.12): 可控 3D 场景，支持人物/物体交互
Genie 3 (2025.08): 更长一致性 + 物理约束 (推测)
```

**特点**：从 YouTube 级无标签视频学习世界动力学，生成可用方向键/动作控制的"游戏"。
用于机器人：作为**无限生成式仿真环境**。

### 9.4 Meta V-JEPA 2 (2025.06)

Meta FAIR 的**非生成式世界模型**：
- 在**隐空间**预测未来（而非像素空间）
- 更高抽象、更少算力
- **zero-shot planning**：给定目标图像，在隐空间搜索动作序列

```
V-JEPA 2 pipeline:
  video → encoder → z_t (隐表示)
  (z_t, a_t) → predictor → ẑ_{t+1}
  
具身应用:
  encode current obs → z_current
  encode goal image → z_goal
  plan actions to minimize ||ẑ_T - z_goal||
```

### 9.5 1X World Model

1X Technologies 专门为 Neo 机器人训练的世界模型：
- 基于 Neo 本体第一视角视频
- 用于在家庭场景中预测动作后果
- **"超越仿真"**：真实数据驱动的"软仿真器"

### 9.6 UniSim / Universe (Pieter Abbeel)

学术界早期提出"通用仿真器"概念：
- 从多源视频学习统一世界模型
- 任何动作 → 预测画面
- 2024 由 Pieter Abbeel 商业化（Generalist AI）

### 9.7 世界模型对比

| 模型 | 方向 | 隐 vs 像素 | 主要用途 | 开源 |
|------|-----|----------|---------|------|
| Cosmos | 视频 WM | 像素 | 数据增广、Sim2Real | ✅ |
| Genie 2/3 | 可交互 WM | 像素 | 仿真环境 | ❌ |
| V-JEPA 2 | 抽象 WM | 隐空间 | 规划、表示学习 | ✅ 模型 |
| 1X WM | 家庭专用 | 像素 | 策略训练 | ❌ |
| UniSim | 通用 | 像素 | 通用仿真 | ❌ |

## 十、VLA + 世界模型融合趋势

### 10.1 Dreamer 式耦合

经典 Dreamer 路线：在 WM 中学策略。具身版：
```
真机数据 → 训练 WM
WM 中 RL rollout → 策略
少量真机微调
```
代表：Dreamer V3, DayDreamer (2022), IRIS。

### 10.2 VLA + Cosmos 增广

```
π₀ + Cosmos:
  小规模真实数据 → Cosmos 增广 → 大规模训练 → π₀
实际案例:
  NVIDIA GR00T Dreams pipeline
```

### 10.3 VLA 内嵌世界模型

```
多任务 VLA 中加入 future prediction head:
  训练损失 = 动作回归 + 未来观测预测
代表:
  某些 2025 论文 (e.g., DreamGen, DynaMo)
```

### 10.4 规划 + 执行分离

```
WM 在抽象 / 隐空间规划 → 产出"语言子目标" 
→ VLA 执行每个子目标
代表:
  V-JEPA 2 做规划 + 下层 VLA 执行
  π₀.5 的 language-in-the-loop 路线
```

## 十二、开放问题（2026）

1. **Scaling Law 是否成立**：数据 2x 性能提升多少？目前不清晰。
2. **触觉如何整合进 VLA**：视觉主导 → 视+触多模态，仍处早期。
3. **RL 微调的价值**：VLA + RL (DPPO, ReinFlow) 在真机数据少场景可能关键。
4. **安全对齐**：类似 RLHF，如何"对齐"机器人行为边界。
5. **长 horizon 与记忆**：当前 context 只有几秒，分钟级任务需要记忆机制。
6. **灾难性遗忘**：持续学习新本体/任务时不忘旧能力。
