# 核心概念与三层架构

> 本文系统讲解具身智能的定义、与传统 AI 的核心差异、感知-决策-执行三层架构、主要研究方向与主流硬件平台。是理解后续 VLA、世界模型、策略学习等专题的基础。

## 一、什么是具身智能

**具身智能（Embodied AI）** 指将智能体（Agent）嵌入到物理或模拟环境中，通过**感知（Perception）→ 决策（Decision-making）→ 执行（Action）** 的闭环交互来完成任务的人工智能范式。

其核心理念源于认知科学的 "Embodied Cognition" 理论：**智能不仅来自大脑（模型），也来自身体与环境的交互**。一个没有身体、只会处理静态数据的模型（例如纯文本 LLM）不具备具身智能属性；而一个能根据传感器反馈实时调整动作的策略（即便模型很小）属于具身智能范畴。

### 与传统 AI 的核心差异

| 维度 | 传统 AI（NLP / CV） | 具身智能 |
|------|---------------------|----------|
| **输入** | 静态数据集（文本、图像） | 实时传感器流（RGB、深度、点云、触觉、力/力矩、IMU） |
| **输出** | 标签、文本、边界框 | 连续控制信号（关节角度、末端位姿、力矩、轮速） |
| **时间维度** | 单步或离线批量 | 长时序闭环，当前动作影响未来观测 |
| **评价指标** | Accuracy / BLEU / mIoU 等离线指标 | 任务成功率、泛化能力、鲁棒性、安全性 |
| **数据成本** | 爬取、标注 | 真机遥操作 / 仿真 / 世界模型增广，单位成本高 |
| **错误代价** | 一次误识别 | 可能损坏设备、伤人、破坏环境 |
| **部署环境** | 云端 GPU | 端侧机器人 + 实时约束 |

### 为什么现在是具身智能爆发期

三个条件在 2023-2026 年同时成熟：

1. **基础模型（Foundation Model）**：VLM 预训练提供了通用的视觉语言理解能力，迁移到机器人只需微调。
2. **数据基础设施**：Open X-Embodiment（80 万轨迹，22 种本体）、AgiBot World（100 万条）、DROID 等共享数据集让跨本体训练成为可能。
3. **硬件成本下降**：Unitree G1（不到 10 万元）、LeRobot 教育平台、Jetson Thor 端侧算力（2000 TOPS）让研究门槛和部署门槛同步下降。

## 二、核心架构：感知-决策-执行（经典三层）

一个典型的具身智能系统可以抽象为三层架构：

```
┌─────────────────────────────────────────────┐
│              高层任务规划 (Task Planning)       │
│   自然语言指令 → 子任务分解 → 技能调度          │
│   (LLM/VLM, 如 SayCan, Code as Policies)     │
├─────────────────────────────────────────────┤
│            中层技能策略 (Skill Policy)          │
│   子任务 → 连续动作序列                        │
│   (RL Policy / Diffusion Policy / VLA)        │
├─────────────────────────────────────────────┤
│            底层运动控制 (Motor Control)         │
│   目标位姿/力 → 关节力矩                       │
│   (PID / 阻抗控制 / 运动规划器)                │
└─────────────────────────────────────────────┘
        ↑ 感知反馈              ↓ 执行指令
   ┌──────────┐          ┌──────────┐
   │  传感器   │          │  执行器   │
   │ RGB-D    │          │ 电机/气动 │
   │ LiDAR    │          │ 夹爪/灵巧手│
   │ IMU/触觉  │          │ 轮/足    │
   └──────────┘          └──────────┘
```

> 注：2024-2026 年 VLA 范式兴起后，中层技能策略与高层任务规划逐步合流为"**大脑-小脑**"双系统架构，详见 [README.md § 2026 共识](README.md#发展现状2026-共识) 与 [../01-VLA模型/双系统架构.md](../01-VLA模型/双系统架构.md)。

### 1. 感知层（Perception）

负责从原始传感器数据中提取结构化信息：

- **视觉感知**：物体检测、6DoF 位姿估计、语义分割、深度估计、光流
- **触觉感知**：接触检测、滑动检测、物体属性估计（硬度、纹理、摩擦）
- **多模态融合**：RGB + 深度 + 点云 + 力/力矩信号的统一表示
- **场景理解**：3D 场景图（Scene Graph）、可供性（Affordance）预测、开放词汇识别

**常见传感器**：

| 类别 | 代表产品 | 典型作用 |
|------|----------|----------|
| RGB-D 相机 | Intel RealSense D435/D455、Azure Kinect、Orbbec Femto | 桌面操控的主力视觉源 |
| LiDAR | Velodyne、Livox Mid-360、Ouster | 导航、建图、SLAM |
| 触觉传感器 | GelSight（视触觉）、DIGIT、BioTac、Fanuc/ATI 力传感器 | 灵巧操作、接触闭环 |
| 力/力矩传感器 | ATI Axia、Bota SensONE（六维力） | 装配、插拔、擦拭等接触任务 |
| IMU | Xsens、ADIS 系列 | 运动估计、行走平衡 |
| 事件相机 | Prophesee、iniVation | 高速运动、低延迟场景 |

### 2. 决策层（Decision-making）

将感知信息映射为动作或子目标。

**经典方法**：

- **有限状态机（FSM）**：手工设计状态转移，简单但不灵活，适合结构化任务
- **行为树（Behavior Tree）**：模块化、可组合，游戏 AI 与工业机器人广泛使用
- **任务与运动规划（TAMP）**：符号规划（PDDL）+ 运动规划结合，代表 PDDLStream

**学习方法**：

- **强化学习（RL）**：通过试错学习策略，适合连续控制与接触任务；代表 PPO、SAC、DreamerV3
- **模仿学习（IL）**：从专家演示学习，数据效率高；代表 Behavior Cloning、DAgger、Diffusion Policy
- **大模型方法**：LLM/VLM 作为高层规划器或端到端 VLA；代表 SayCan、Code as Policies、RT-2、π₀

详见 [../03-策略学习方法/](../03-策略学习方法/)。

### 3. 执行层（Execution）

将高层动作指令转化为电机级控制信号：

- **关节空间控制**：直接控制各关节角度/角速度（适合位置明确的任务）
- **笛卡尔空间控制**：控制末端执行器的 6DoF 位姿（适合任务空间描述的任务）
- **阻抗控制 / 导纳控制**：同时控制位置和力，适合擦拭、装配、人机协作
- **运动规划**：在关节空间中搜索无碰撞路径（RRT、RRT*、CHOMP、TrajOpt）
- **力/力矩控制**：直接闭环力信号，适合插拔、磨抛

2026 年主流架构中，VLA 输出 1-2 秒的"末端位姿 chunk"，由底层低级控制器以 200-1000 Hz 跟踪，而非直接写电机寄存器。

## 三、主要研究方向

```
具身智能
├── 机器人操控 (Manipulation)
│   ├── 抓取 (Grasping)              — GraspNet、AnyGrasp
│   ├── 灵巧操作 (Dexterous Manipulation) — Allegro/LEAP/Shadow Hand
│   ├── 双臂协同 (Bimanual)           — ALOHA 类
│   └── 长 horizon 任务               — 家庭场景复合任务
├── 导航 (Navigation)
│   ├── 点目标导航 (PointNav)
│   ├── 目标导航 (ObjectNav)
│   ├── 视觉-语言导航 (VLN)
│   └── 社交导航 (Social Nav)
├── 移动操控 (Mobile Manipulation)
│   └── 导航 + 操控联合（Mobile ALOHA、Stretch）
├── 人形机器人 (Humanoid)
│   ├── 双足运动 (Locomotion)         — Unitree H1/G1、Figure 02/03
│   ├── 全身控制 (Whole-body Control) — Helix 类端到端
│   └── 人形操控                      — GR00T N1 / Apollo
├── 足式 / 轮式机器人
│   ├── 四足 (Quadruped)              — Spot、Unitree Go2、ANYmal
│   └── 轮式平台                      — Stretch、Hello Robot
└── 多智能体协作                       — Helix 双机、Menagerie
```

## 五、感知-决策-执行的端到端演化

2023 年以前，三层架构的每一层通常由独立团队/模块开发，接口是显式的物体位姿、抓取点、路径点等。2023 年起 VLA 范式将"感知 + 决策"压缩到一个端到端模型里：

```
经典模块化:
  RGB-D → 检测 → 位姿估计 → 抓取规划 → 运动规划 → 控制
   每个模块独立训练/调试，泛化依赖每一层都正确

VLA 端到端（2023-）:
  RGB + 语言 → VLA 模型 → 末端动作 chunk → 低级控制
   一个模型端到端训练，泛化依赖数据覆盖面
```

但"完全端到端"并非最终形态。2025-2026 年主流方案是**"双系统 + 保留底层控制"** 的混合架构：

- **System 2 + System 1 端到端**（VLA 内部）
- **底层仍保留 PID/阻抗控制、安全监控、碰撞检测**（非学习）

这种折衷保证了安全性和可部署性，详见 [../01-VLA模型/双系统架构.md](../01-VLA模型/双系统架构.md)。
