## 发展现状（2026 共识）

### 时间线：三次范式跃迁

```
┌─────────────────────────────────────────────────────────────────────┐
│   传统机器人 (1960s-2000s)                                            │
│   手工编程 / PID / 运动规划 / 结构化环境 / 单任务                       │
│                                ↓                                      │
│   学习驱动 (2010s-2022)                                              │
│   Deep RL / 模仿学习 / 仿真训练 / 多任务泛化                           │
│                                ↓                                      │
│   大模型驱动 (2023-至今)                                              │
│   LLM/VLM 规划 → VLA 端到端 → 双系统双频 → 世界模型耦合                 │
└─────────────────────────────────────────────────────────────────────┘
```

### 2026 主流架构共识：三层 → 双系统重划分

2023 年以前学术界普遍沿用"高层任务规划 / 中层技能策略 / 底层运动控制"的三层架构；2024-2025 年随着 VLA（Vision-Language-Action）基础模型的成熟，行业在**大脑（System 2）与小脑（System 1）的界面**上形成了新的共识：

```
┌─────────────────────────────────────────────────────────────────┐
│  大脑 System 2（7-9 Hz，慢思考）                                  │
│    任务理解、长 horizon 规划、常识推理                             │
│    代表：PaliGemma / Eagle / Qwen2-VL / Gemini Robotics           │
├─────────────────────────────────────────────────────────────────┤
│                  ↓ latent token / 语言子目标                      │
├─────────────────────────────────────────────────────────────────┤
│  小脑 System 1（100-200 Hz，快执行）                              │
│    连续动作生成、反应式控制、力/触闭环                              │
│    代表：Flow-Match / Diffusion / Transformer Action Expert        │
├─────────────────────────────────────────────────────────────────┤
│                  ↓ 关节/末端指令                                   │
├─────────────────────────────────────────────────────────────────┤
│  执行器层：电机控制、阻抗控制、安全监控（延续传统控制栈）             │
└─────────────────────────────────────────────────────────────────┘
```

**与旧三层架构的关键变化**：

1. **中层策略被 VLA 吸收**：从前"子任务 → 连续动作"的技能策略，现在由 System 1 动作专家直接端到端生成。
2. **高层规划融合进 VLM 主干**：SayCan 式"LLM 当规划器再调用技能库"的分阶段范式，被 π₀.5 / Gemini Robotics 这种"VLM 内部自主分解"的端到端范式取代。
3. **底层控制层保留**：PID、阻抗控制、安全停机仍在底层保留，VLA 输出的目标不直接写电机寄存器。

### 2026 当前共识（六条）

1. **双系统双频成行业默认**：π₀ / GR00T N1 / Helix / Gemini Robotics 无一例外采用 S2 慢思考 + S1 快执行。
2. **Flow Matching 成动作头主流**：相比 Diffusion 少一个数量级的推理步，相比离散 token 精度高，是 2025-2026 年 π 系列、SmolVLA、Helix v2 的默认选择。
3. **Chunk Prediction + 时序集成**：一次预测 1-2 秒（~50 步）动作 chunk，用时序集成平滑接缝，已成为标配。
4. **跨本体（Cross-Embodiment）训练是基础设施**：Open X-Embodiment、AgiBot World、DROID 等共享数据集让"一个模型多个机器人"可行。
5. **端侧部署开始落地**：Jetson Thor（2000 TOPS）、华为/地平线 SoC 让 7B 级 VLA 可以直接跑在机器人上。
6. **世界模型成为数据飞轮**：Cosmos、Genie、V-JEPA 2 从视频学物理，再反哺 VLA 训练数据，详见 [../02-世界模型/](../02-世界模型/)。

### 2026 开放问题

- **Scaling Law 是否成立**：数据 2 倍带来的性能提升尚无定论。
- **触觉/力反馈如何进 VLA**：当前视觉主导，触觉 token 化仍处早期。
- **长 horizon 记忆**：当前 context 窗口仅数秒，分钟/小时级任务需要显式记忆机制。
- **安全对齐**：类似 RLHF 的机器人行为边界对齐尚无成熟方案。
- **持续学习**：新本体/新任务加入时避免灾难性遗忘。
- **Sim2Real gap**：仿真数据占比越来越大，但与真实数据的对齐仍是瓶颈。
