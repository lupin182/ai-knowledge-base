# 具身智能术语速查表

> 一份面向入门者的术语字典，按类别分组。每个词条包含英文原词、中文释义和一句话语境说明。
> 最后更新：2026-04

---

## 一、机器人本体与硬件

| 术语 | 中文 | 一句话解释 |
|------|------|-----------|
| **Embodiment** | 本体 / 具身形态 | 机器人的物理形态（臂、四足、人形…），不同 embodiment 意味着不同的传感器和动作空间 |
| **Articulation** | 铰接体 / 关节体 | 由多个刚体通过关节连接而成的系统，就是仿真里"一个机器人"的完整模型 |
| **Joint** | 关节 | 连接两段刚体的活动部件，常见有 revolute（旋转）和 prismatic（滑动）两种 |
| **Link** | 连杆 / 刚体段 | 铰接体中由关节连接的一截刚体，如大腿、小腿、前臂 |
| **DoF (Degree of Freedom)** | 自由度 | 机器人能独立运动的维度数。四足 12 DoF，人形 20-30+，灵巧手 16-24 |
| **End-Effector (EE)** | 末端执行器 | 机器人与环境接触的末端，如夹爪（gripper）、灵巧手、吸盘 |
| **Gripper** | 夹爪 | 最常见的末端执行器，平行指夹或多指手 |
| **Dexterous Hand** | 灵巧手 | 多指多关节手（16-24 DoF），能做手内旋转、捏取等精细操作。代表：Shadow Hand、Allegro、LEAP |
| **Actuator** | 执行器 / 电机 | 驱动关节运动的动力元件，主流为无刷电机 + 谐波/行星减速器 |
| **IMU (Inertial Measurement Unit)** | 惯性测量单元 | 测量加速度 + 角速度的传感器，是机器人"感知自己身体姿态"的核心硬件 |
| **F/T Sensor** | 力/力矩传感器 | 装在腕部或脚底，测量接触力和力矩，用于力控和碰撞检测 |
| **Tactile Sensor** | 触觉传感器 | 装在指尖/皮肤上，感知接触位置、压力、滑动。代表：GelSight、Digit 360 |
| **URDF / MJCF** | 机器人描述文件 | 定义机器人结构（link、joint、惯量…）的 XML 格式，URDF 用于 ROS，MJCF 用于 MuJoCo |
| **BOM (Bill of Materials)** | 物料清单 / 硬件成本 | 一台机器人的零件总成本。2026 人形 BOM 降至 $20-35K |
| **Humanoid** | 人形机器人 | 双足双臂类人形态。代表：Atlas、Unitree H1/H2、Figure 02、Tesla Optimus |
| **Quadruped** | 四足机器人 | 四条腿的机器人。代表：ANYmal、Spot、Unitree Go2、MIT Mini Cheetah |
| **Mobile Manipulator** | 移动操作平台 | 底盘 + 机械臂的组合，能边移动边操作。代表：Mobile ALOHA、Fetch |

## 二、感知与观测

| 术语 | 中文 | 一句话解释 |
|------|------|-----------|
| **RGB-D** | 彩色-深度相机 | 同时输出彩色图和深度图的传感器，如 RealSense D435 |
| **Point Cloud** | 点云 | 三维空间中的点集合，由深度相机或 LiDAR 生成，是 3D 感知的基本表示 |
| **Ego View / Egocentric** | 第一人称视角 | 从机器人自身视角（如头部/腕部相机）看到的画面，相对于 exocentric（第三人称/外部相机） |
| **Exocentric / Third-Person** | 第三人称视角 | 外部固定相机拍摄的画面。实验室常用，但真实部署通常没有 |
| **Wrist Camera** | 腕部相机 | 装在手腕处的相机，看到的是手附近的近场画面，操控任务常用 |
| **Proprioception** | 本体感知 | 机器人"感知自身状态"的能力：关节角度、角速度、力矩、IMU 读数等 |
| **Observation Space** | 观测空间 | RL/IL 中策略能看到的所有信息的总和 |
| **Privileged Information** | 特权信息 | 仿真中可获取但真机上拿不到的信息（如精确接触力、地形高度图），Teacher-Student 训练中 Teacher 使用 |
| **POMDP** | 部分可观马尔可夫决策过程 | 智能体看不到完整环境状态——这是真实机器人的常态 |
| **Grounding** | 接地 / 语义落地 | 把语言/概念映射到物理世界中的具体物体或位置。例如理解"拿红色杯子"→ 定位到桌上哪个物体 |
| **Visual Grounding** | 视觉接地 | 给一段文字，在图像中定位对应的物体或区域 |
| **Spatial Reasoning** | 空间推理 | 理解物体之间的空间关系（左边、里面、上方…），对操控任务至关重要 |
| **Pose Estimation** | 位姿估计 | 估计物体在 3D 空间中的位置 + 朝向（6D pose: x,y,z + roll,pitch,yaw） |
| **Grasp Pose** | 抓取位姿 | 夹爪/手应该以什么位置和角度去抓取物体 |

## 三、动作与控制

| 术语 | 中文 | 一句话解释 |
|------|------|-----------|
| **Action Space** | 动作空间 | 策略输出的维度和范围。可以是关节角度、末端位姿、速度指令等 |
| **Joint Position Control** | 关节位置控制 | 策略输出目标关节角度，底层 PD 控制器追踪。最常见的 RL 动作类型 |
| **Cartesian Control** | 笛卡尔空间控制 | 策略输出末端执行器的目标位姿（xyz + 旋转），用逆运动学转换为关节指令 |
| **Impedance Control** | 阻抗控制 | 让机器人表现得像一个弹簧-阻尼系统，能柔顺地与环境接触，不硬撞 |
| **PD Controller** | PD 控制器 | 底层关节控制器：τ = Kp·(q_target - q) + Kd·(0 - q̇)，几乎所有 RL 部署都用 |
| **Torque Control** | 力矩控制 | 策略直接输出关节力矩，最灵活但最难控——真机很少直接用 |
| **IK (Inverse Kinematics)** | 逆运动学 | 已知末端目标位姿，求各关节角度。传统控制的基础工具 |
| **Motion Planning** | 运动规划 | 规划从 A 到 B 的无碰撞轨迹。经典方法：RRT、RRT*、CHOMP |
| **Trajectory** | 轨迹 | 一系列带时间戳的状态或动作序列，是模仿学习的基本数据单元 |
| **Action Chunk** | 动作块 | 策略一次预测一段连续动作（如 50 步），而非逐步预测。2025-2026 VLA 标配 |
| **WBC (Whole-Body Control)** | 全身控制 | 同时协调机器人全身关节（四肢 + 躯干）来完成任务的控制框架 |
| **Locomotion** | 运动 / 行走 | 机器人的移动能力：四足步态、双足行走、跑、跳 |
| **Manipulation** | 操控 / 操作 | 机器人用手/臂与物体交互：抓取、放置、插入、旋转、装配等 |
| **Loco-Manipulation** | 移动操控 | 边走边操作，需要同时协调运动和操控 |
| **Gait** | 步态 | 行走时腿的运动模式。如 trot（对角步态）、gallop（奔跑）、bound（跳跃） |
| **Contact** | 接触 | 机器人与环境的物理接触（脚触地、手抓物体），是具身 AI 的核心物理事件 |

## 四、学习范式

| 术语 | 中文 | 一句话解释 |
|------|------|-----------|
| **IL (Imitation Learning)** | 模仿学习 | 从专家演示中学习策略，不需要奖励函数。子类有 BC 和 IRL |
| **BC (Behavior Cloning)** | 行为克隆 | 最简单的 IL：把（观测, 动作）当监督学习来训练 |
| **RL (Reinforcement Learning)** | 强化学习 | 通过试错 + 奖励信号学习最优策略 |
| **On-policy / Off-policy** | 在线策略 / 离线策略 | On-policy（如 PPO）：只用当前策略产生的数据；Off-policy（如 SAC）：可重用旧数据，样本效率高 |
| **PPO** | 近端策略优化 | 2026 具身 RL 的事实标准，稳定易调 |
| **SAC** | 软演员-评论家 | Off-policy 算法，样本效率高，适合真机或灵巧手 |
| **DAgger** | 数据聚合 | IL 改进：用学生策略采集数据 → 让老师标注 → 混入训练集，解决分布偏移 |
| **Diffusion Policy** | 扩散策略 | 用扩散模型生成动作序列，能表示多模态动作分布。2024 爆火 |
| **Flow Matching** | 流匹配 | 扩散的替代方案，推理更快（~10 步 vs 扩散 ~100 步），2025-2026 VLA 动作头主流 |
| **VLA (Vision-Language-Action)** | 视觉-语言-动作模型 | 输入图像+语言，输出机器人动作的端到端大模型。代表：RT-2、π₀、GR00T N1 |
| **VLM (Vision-Language Model)** | 视觉-语言模型 | 理解图像和语言的多模态模型。VLA 的"大脑"部分 |
| **Teacher-Student** | 教师-学生蒸馏 | 仿真中用特权信息训 Teacher → 蒸馏出只用传感器的 Student → 部署。Locomotion 标配 |
| **Reward Shaping** | 奖励塑形 | 手工设计中间密集奖励，引导 RL 学习。具身 RL 的核心工程问题 |
| **Curriculum Learning** | 课程学习 | 从易到难逐步增加训练难度，如地形从平地到台阶到碎石 |
| **HER (Hindsight Experience Replay)** | 后见经验回放 | 把失败轨迹的终点当作"另一个目标"来重用，缓解稀疏奖励 |
| **Goal-Conditioned** | 目标条件化 | 策略以目标为额外输入，一个策略处理多种目标 |
| **System 1 / System 2** | 快系统 / 慢系统 | 具身双系统架构：S2（VLM，7-9Hz）负责理解和规划，S1（动作专家，100-200Hz）负责快速执行 |

## 五、仿真与 Sim2Real

| 术语 | 中文 | 一句话解释 |
|------|------|-----------|
| **Sim-to-Real (Sim2Real)** | 仿真到真实迁移 | 在仿真中训练策略，直接部署到真实机器人 |
| **Domain Randomization (DR)** | 域随机化 | 训练时随机化仿真参数（摩擦、质量、光照…），让策略对参数变化鲁棒 |
| **ADR (Automatic DR)** | 自动域随机化 | 根据策略表现自动扩大/缩小随机范围 |
| **Sim-to-Real Gap** | 仿真-真实差距 | 仿真和真实世界之间的差异，是具身 AI 的核心挑战之一 |
| **Real-to-Sim-to-Real** | 真实→仿真→真实 | 先从真实世界重建仿真场景，再在仿真中训练，再迁回真实 |
| **Isaac Sim / Isaac Lab** | NVIDIA 仿真平台 | GPU 加速的机器人仿真器，支持 4096+ 环境并行 RL 训练 |
| **MuJoCo / MJX** | 物理仿真器 | DeepMind 的高精度物理仿真引擎，MJX 是 JAX 加速版 |
| **Genesis** | 生成式仿真器 | 2025 新一代仿真器，试图统一各种物理求解器 |
| **Parallel Environments** | 并行环境 | 同时运行数千个仿真实例收集 RL 数据，Isaac Gym 的杀手特性 |
| **Zero-Shot Transfer** | 零样本迁移 | 仿真训练后不做任何真机微调，直接部署——DR 做得好就能实现 |

## 六、数据与遥操作

| 术语 | 中文 | 一句话解释 |
|------|------|-----------|
| **Teleoperation (Teleop)** | 遥操作 | 人远程操控机器人执行任务，同时录制数据用于训练 |
| **UMI (Universal Manipulation Interface)** | 通用操控接口 | 一种低成本遥操作方案，用手持夹爪 + GoPro 采集操控演示，无需真机 |
| **ALOHA** | 低成本遥操作系统 | Stanford 的双臂遥操作平台（~$20K），广泛用于 IL 数据采集 |
| **Demonstration / Demo** | 演示 / 示范 | 专家完成任务的一条轨迹录制，是模仿学习的训练数据 |
| **Trajectory Dataset** | 轨迹数据集 | 大量 demo 的集合。代表：Open X-Embodiment（80 万条）、DROID、AgiBot World |
| **Cross-Embodiment** | 跨本体 | 让一个模型同时在多种机器人上工作。OXE 数据集覆盖 22 种本体 |
| **Data Flywheel** | 数据飞轮 | 部署 → 收集数据 → 改进模型 → 更好部署 → 更多数据 的正循环 |
| **MoCap (Motion Capture)** | 动作捕捉 | 用标记点/无标记视觉追踪人体运动，用于人形机器人的模仿学习（HumanPlus） |

## 七、任务与评测

| 术语 | 中文 | 一句话解释 |
|------|------|-----------|
| **Pick-and-Place** | 抓取放置 | 最基础的操控任务：抓起物体放到目标位置 |
| **In-Hand Manipulation** | 手内操控 | 在手中旋转、翻转物体（不放下来），非常难。代表：OpenAI 解魔方 |
| **Long-Horizon Task** | 长时序任务 | 需要多步骤完成的复杂任务，如"做一杯咖啡"需要十几个子步骤 |
| **Generalization** | 泛化 | 策略能否在没见过的物体、场景、指令上工作 |
| **Robustness** | 鲁棒性 | 面对干扰（推搡、遮挡、传感器噪声）时策略还能不能工作 |
| **Success Rate** | 成功率 | 最核心的评测指标：任务完成的比例 |
| **Benchmark** | 评测基准 | 标准化的测试环境和任务集。代表：SIMPLER、LIBERO、CALVIN、Meta-World |
| **Parkour** | 跑酷 | 机器人在复杂地形（台阶、斜坡、间隙）上做 agile 动作。代表：ANYmal Parkour、Extreme Parkour |

## 八、架构与系统

| 术语 | 中文 | 一句话解释 |
|------|------|-----------|
| **End-to-End (E2E)** | 端到端 | 从原始输入（像素 + 语言）直接到动作输出，不做人工分阶段 |
| **Modular Pipeline** | 模块化流水线 | 感知 → 规划 → 控制分模块设计，每个模块独立开发。与端到端相对 |
| **Foundation Model** | 基础模型 | 大规模预训练模型，通过微调适配下游任务。VLA 是机器人领域的 foundation model |
| **Action Head** | 动作头 | VLA 中负责生成动作的输出模块（可以是 MLP、Diffusion、Flow Matching） |
| **Backbone** | 骨干网络 | 提取特征的主干模型（如 ViT、ResNet），VLA 中通常是 VLM |
| **Tokenization** | Token 化 | 把连续信号离散化为 token 序列，RT-2 就是把动作 token 化后用 LLM 预测 |
| **Latent Space** | 隐空间 | 模型内部的压缩表示空间，动作/观测在这里被编码为向量 |
| **World Model** | 世界模型 | 能预测"如果我做动作 a，环境会变成什么样"的模型。代表：Dreamer、Cosmos、Genie |
| **ROS (Robot Operating System)** | 机器人操作系统 | 机器人软件的标准中间件框架，管理传感器、驱动、通信 |
