# 动作生成方法

> 本文系统对比 VLA 中四种主流的动作生成范式：**离散 token / 扩散 / Flow Matching / 自回归 Transformer**。给出各自的数学直觉、训练目标、推理流程、优劣与代表模型，重点展开 Flow Matching 的推导。

## 一、四范式总览

| 范式 | 训练目标 | 推理 | 连续/离散 | 多模态分布 | 代表模型 | 典型频率 |
|------|----------|------|-----------|------------|----------|----------|
| **离散 token** | Cross-entropy | 自回归 1 步/维度 | 离散（分桶） | 隐式（softmax） | RT-2, OpenVLA | 1-10 Hz |
| **扩散（Diffusion）** | 噪声预测（ε-pred / v-pred） | 50-100 步去噪 | 连续 | **显式多模态** | Octo, Diffusion Policy, GR00T DiT, Gemini Robotics | 5-30 Hz |
| **Flow Matching** | 速度场回归 `v_θ = x_1 - x_0` | 5-10 步 Euler | 连续 | 显式多模态 | π₀ / π₀.5 / π₀.7, SmolVLA | 10-50 Hz |
| **自回归 Transformer**（Helix S1 类） | MSE / L1 回归 | 并行一次前向 | 连续 | 弱（条件依赖） | Figure Helix S1, OpenVLA-OFT | 100-200 Hz |

> "多模态分布"指对于同一观测，策略能表达多个可行动作模式（例如"从左/右绕过障碍"都是有效的）。

---

## 二、离散 token（RT-2 范式）

### 2.1 思想

把连续动作 `a ∈ R^D` 每个维度均匀分桶为 N 个离散值（典型 N=256），映射到 VLM 词表中的某些 token。VLA 像语言模型一样自回归预测。

```
a = [Δx, Δy, Δz, Δrx, Δry, Δrz, grip]
  → "1 128 91 241 0 1 0"   (长度 D 的 token 序列)
```

### 2.2 训练

标准 cross-entropy：

```
L = -Σ_{d=1}^{D} log p_θ(token_d | image, lang, token_{<d})
```

### 2.3 推理

自回归：`token_1 → token_2 → ... → token_D`。每个维度一次前向。

### 2.4 优劣

| 优 | 劣 |
|----|----|
| 复用 VLM 预训练能力 | 离散化精度损失（末端姿态尤其敏感） |
| 不需新架构 | 自回归 → 慢（D 次前向） |
| 工程简单 | 多模态表达弱（隐式 softmax） |

### 2.5 代表与命运

- **RT-2（2023）**：开创路线
- **OpenVLA（2024.06）**：开源 7B 基线
- **OpenVLA-OFT（2024.10）**：用 L1 回归 + 并行解码替代，频率从 26 → 109 Hz

**命运**：2024 H2 起基本被 Flow Matching / 扩散替代，仅在极端端侧场景保留。

## 三、扩散（Diffusion Policy 范式）

### 3.1 思想

把动作生成建模为**从噪声到目标动作的去噪过程**。VLM / Transformer 作为条件编码器，Diffusion Head 学习去噪。

```
前向过程（训练数据生成）:
  a_0 ~ data
  a_t = √(α_t) · a_0 + √(1 - α_t) · ε,   ε ~ N(0, I)
  
反向过程（推理）:
  a_T ~ N(0, I)
  for t = T, T-1, ..., 1:
      predict ε̂_θ(a_t, t, obs)
      a_{t-1} = denoise(a_t, ε̂_θ)
  return a_0
```

### 3.2 训练目标（ε-prediction）

```
L_DDPM = E_{t, a_0, ε} || ε - ε̂_θ(a_t, t, obs) ||²
```

也可用 v-prediction、score-matching 等变体。

### 3.3 推理步数

- DDPM：50-100 步
- DDIM：10-50 步（非马尔可夫采样加速）
- DPM-Solver：更激进的求解器

### 3.4 优劣

| 优 | 劣 |
|----|----|
| **显式多模态分布** | 推理步多 → 频率受限 |
| 连续动作，精度高 | 训练目标相对复杂（VLB） |
| 成熟稳定（图像生成领域已验证） | 少步求解器需要经验调参 |

### 3.5 代表

- **Diffusion Policy**（Chi et al. 2022.12）：开创操控扩散范式
- **Octo**（2024.05）：跨本体 Diffusion 头
- **GR00T N1** 的 DiT 动作专家（2025.03）
- **Gemini Robotics** 的 Diffusion Transformer（2025.03）
- **RDT-1B / RDT-2**（清华）：双臂扩散 MoE
- **CogACT**（2024.11）

## 四、Flow Matching（π₀ 范式）

### 4.1 核心思想

**直接学习从噪声到目标动作的"速度场"**，避免 DDPM 的曲线路径与多步求解。

```
选定一条路径（通常是直线）:
  a_t = (1 - t) · a_0 + t · a_1
  其中 a_0 ~ N(0, I)  (噪声)
        a_1 ~ data     (目标动作)
        t ~ Uniform[0, 1]

定义速度场（target）:
  v*(a_t, t) = d a_t / d t = a_1 - a_0

神经网络拟合:
  v_θ(a_t, t, obs) ≈ a_1 - a_0
```

### 4.2 训练目标（conditional flow matching）

```
L_FM = E_{t, a_0, a_1} || v_θ(a_t, t, obs) - (a_1 - a_0) ||²
```

这是一个**简洁的 MSE**，比 DDPM 的 VLB 目标容易训练。

### 4.3 Rectified Flow（直线流）

当路径选择为线性插值 `a_t = (1-t)·a_0 + t·a_1` 时，得到 **Rectified Flow**：

- 路径是直线，理论上最优传输的近似
- 可以 "re-flow"：用学到的 v_θ 生成数据对 `(a_0, a_1)`，再重新训练，路径进一步变直
- 极限情况下 1 步 Euler 积分就足够

### 4.4 推理：Euler 积分

```
a_0 ~ N(0, I)
K = 5 (或 10)
for k = 0 to K-1:
    t = k / K
    a_{t + 1/K} = a_t + (1/K) · v_θ(a_t, t, obs)
return a_1
```

**K = 5-10 步** 即可达到 DDPM 50 步的质量。

### 4.5 与扩散的关系

| 维度 | Diffusion (DDPM) | Flow Matching |
|------|------------------|---------------|
| 路径 | 曲线（高斯过程） | **直线**（Rectified Flow） |
| 训练目标 | ε-pred / v-pred VLB | **v_θ = a_1 - a_0 MSE** |
| 推理 | 50-100 步 | **5-10 步 Euler** |
| 理论 | 随机微分方程 | 常微分方程 |
| 多模态 | 显式 | 显式（来自不同 `a_1`） |

理论上 Flow Matching 是 Diffusion 的一种推广/简化（参考 Lipman et al. 2022, Liu et al. 2022 Rectified Flow）。

### 4.6 代表

- **π₀**（2024.10）：首个大规模 Flow Matching VLA
- **π₀.5 / π-fast / π₀.7**：PI 系列全线 Flow Matching
- **SmolVLA**（2025.05）：端侧 FM
- 越来越多 2025-2026 开源项目默认 FM

### 4.7 实现要点

```python
# 训练（伪代码）
def train_step(obs, action_gt):
    B, T, D = action_gt.shape
    t = torch.rand(B, 1, 1)               # 采样 t
    noise = torch.randn_like(action_gt)   # a_0
    a_t = (1 - t) * noise + t * action_gt # 插值
    target = action_gt - noise            # v* = a_1 - a_0
    v_pred = net(a_t, t, obs)
    loss = F.mse_loss(v_pred, target)
    return loss

# 推理
def sample(obs, K=5):
    a = torch.randn(B, T, D)
    for k in range(K):
        t = torch.full((B, 1, 1), k / K)
        v = net(a, t, obs)
        a = a + (1/K) * v
    return a
```

## 五、自回归 Transformer 动作专家（Helix S1 范式）

### 5.1 思想

在**双系统架构**下，S1（小脑）直接回归连续动作，不做扩散也不做 Flow Matching。由 S2（大脑）提供的 latent 承担条件编码的重任，S1 只做"翻译 latent → 电机"。

```
S2 latent (256-d, 9 Hz)  ──┐
                            │
本体状态 (关节角/速度/力) ───┼──→ S1 Transformer (80M) ──→ 动作 chunk
                            │
历史动作 (可选) ─────────────┘
                                     200 Hz 闭环
```

### 5.2 训练

- **L1 或 MSE 回归** 到真机数据的动作
- 可加上 **Action Chunking**（一次预测多步）
- 可加上 **时序集成**（Temporal Ensembling）

### 5.3 推理

**一次前向**得到整个 chunk，无需多步去噪。这是能达到 200 Hz 的关键。

### 5.4 优劣

| 优 | 劣 |
|----|----|
| **最快**（一次前向） | 多模态表达弱 |
| 实现简单 | 依赖 S2 latent 的信息量 |
| 易于端侧部署 | 单靠 S1 无法独立泛化 |

### 5.5 代表

- **Figure Helix S1**（80M Transformer, 200 Hz）
- **OpenVLA-OFT** 的 L1 并行解码（介于这一类与离散 token 之间）
- 一些端侧实时控制器

## 六、四范式对比

### 6.1 精度 vs 频率 vs 多模态

```
                 ▲ 精度
                 │
         π₀ / π₀.5 ●    Helix S1 ●
         (FM)         (Transformer)
                 │
 RDT-2 / GR00T ● Diffusion
   Octo       
                 │
        RT-2 ●     ● OpenVLA
      (离散 token)
                 │
                 └──────────────────────▶ 频率
               1Hz    10Hz   100Hz  200Hz

多模态分布表达能力: Diffusion ≈ FM > 离散 token > 纯回归 Transformer
```

### 6.2 选型矩阵

| 场景 | 推荐范式 |
|------|----------|
| 需要多模态分布（多种可行解） | **Diffusion / FM** |
| 极高频率（>100 Hz） | **纯回归 Transformer（S1）** |
| 精度要求高 + 可接受 10-30 Hz | **Flow Matching**（推理步 5-10） |
| 端侧 + 极小模型 | **Flow Matching**（SmolVLA） |
| 论文基线 | Diffusion（Diffusion Policy / Octo） |
| 复用 VLM 最小改动 | 离散 token（已逐渐淘汰） |

## 七、辅助技巧

### 7.1 Action Chunking

**一次预测未来 N 步动作**（典型 N=16 或 N=50，即 1-2 秒）。

- 减少推理频次（每 N 步调用一次模型）
- 缓解模型抖动（同一 chunk 内的连贯性）
- 改善长 horizon（覆盖更大时间窗）

### 7.2 Temporal Ensembling（时序集成）

当同一时刻 `t` 被多个重叠 chunk 预测时，取加权平均：

```
a_t = Σ_{k} w_k · a_{t}^{(k)},   w_k ∝ exp(-k/τ)
```

平滑接缝、提升稳定性。ALOHA 首次系统化使用。

### 7.3 Classifier-Free Guidance（CFG）

扩散/FM 可以用 CFG 加强条件跟随：

```
v_cfg = v_θ(a_t, t, obs, lang) + s · [v_θ(a_t, t, obs, lang) - v_θ(a_t, t, obs, ∅)]
```

代价是每步两次前向，实践中 s = 1-3。

### 7.4 Flow Matching 的 Euler vs Midpoint

- Euler：一阶，最常用，K=5-10
- Midpoint / Heun：二阶，K=3-5 即可，但每步两次前向
- 实践中 Euler K=5 已足够，不常用高阶

## 八、前沿：四范式的混合

2025-2026 出现的趋势是**混合**：

- **VLM + Flow Matching 主动作头 + 轻量 Transformer S1 二次精修**：
  - FM 给 chunk，S1 在 chunk 上做高频细化（例如 Helix 类结构 + PI 类 FM）
- **Diffusion Policy + RL 微调**（DPPO、ReinFlow）：
  - 用 Diffusion 预训练，RL 微调提升成功率

未来可能出现**一种统一框架**：不同频率层用不同范式，共享 VLM 主干。
