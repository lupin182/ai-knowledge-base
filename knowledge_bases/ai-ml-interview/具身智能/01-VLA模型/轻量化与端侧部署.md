# 轻量化与端侧部署

> 本文聚焦 2025-2026 年 VLA 从云端走向端侧的工程实践：SmolVLA / π-fast / OpenVLA-OFT 的轻量化技术、蒸馏 / INT4 量化 / MoE VLA 等压缩手段，以及 Jetson Thor / 华为 SoC / 地平线 J6 等端侧芯片。包含延迟-精度权衡表。

## 一、为什么要端侧部署

### 1.1 云端部署的痛点

| 痛点 | 影响 |
|------|------|
| **网络延迟** | 100-300 ms 不可接受（反射性动作需 <10 ms） |
| **网络不稳** | 工厂 / 家庭 Wi-Fi 丢包直接导致停机 |
| **隐私** | 家庭机器人摄像头数据上传不可接受 |
| **成本** | 每台机器人长期云推理成本 > 硬件成本 |
| **算力分配** | 一个云端 A100 服务不了多台高频机器人 |

### 1.2 端侧 VLA 的可行性（2025-2026）

三个条件同时成熟：

1. **模型压缩**：SmolVLA 证明 500M 参数可达 7B 模型 80% 性能
2. **并行解码**：OpenVLA-OFT 把 7B 模型从 26 Hz 拉到 109 Hz
3. **端侧芯片**：Jetson Thor（2000 TOPS FP8）让 7B 级 VLA 单卡跑 30 Hz

## 二、轻量化模型

### 2.1 SmolVLA（2025.05，HuggingFace LeRobot）

- **~500M 参数**
- Flow Matching 动作头
- 面向 LeRobot 教育生态（SO-100、Koch、Moss 等低成本机械臂）
- 可在 **Jetson Orin / Mac Mini / 普通笔记本 GPU** 上运行
- 开源：权重、代码、训练数据 pipeline

**技术点**：

- 小型 VLM 主干（SmolLM2 基础）
- 共享视觉编码器（节省参数）
- Flow Matching 仅 5 步 Euler，推理极快

### 2.2 π-fast（2025 H2，Physical Intelligence）

- 基于 π₀.5 蒸馏而来，参数减半（~1.5B）
- 推理速度 3-5×
- 保留 π₀.5 主体能力
- 目标 30 Hz+ 闭环

### 2.3 OpenVLA-OFT（2024.10）：并行解码突破

**关键发现**：7B OpenVLA 的瓶颈不是模型能力，而是**离散 token + 自回归解码**。

改进：

1. **并行解码**：一次前向同时输出所有动作维度
2. **L1 回归代替 cross-entropy**：连续回归头替代离散 token
3. **Action chunking**：一次输出多步

**结果**：控制频率从 **26 Hz → 109 Hz**（提升 4.2 倍），成功率还略有提升。

### 2.4 TinyVLA（2025）

- <1B 参数
- Diffusion 动作头
- 桌面场景
- 面向研究与小型部署

## 三、压缩技术

### 3.1 知识蒸馏（Distillation）

```
Teacher: π₀.5 (3B, 云端)
Student: π-fast (~1.5B, 端侧)

蒸馏损失:
  L = α · L_task  +  β · L_feature_match  +  γ · L_output_match
      任务损失         中间特征对齐           输出分布对齐
```

实践要点：

- **Flow Matching 友好蒸馏**：让 student 学 teacher 的速度场，而非最终动作
- **分层蒸馏**：视觉特征、语言特征、动作特征各自对齐
- **数据增广**：在 teacher 生成的轨迹上训 student，数据可近乎无限

### 3.2 INT4 / INT8 量化

VLM 主干的量化：

| 精度 | 模型大小（7B） | 性能损失 | 推理加速 |
|------|----------------|----------|----------|
| FP16 | 14 GB | 0 | 1× |
| INT8 | 7 GB | ~1% | 1.5-2× |
| **INT4** | **3.5 GB** | **2-5%** | **3-4×** |

动作专家因参数本身小，通常保留 FP16（否则精度损失明显）。

主流工具：

- **GPTQ**：权重量化，机器人领域主流
- **AWQ**：激活感知量化
- **SmoothQuant**：权重 + 激活联合量化
- **TensorRT-LLM INT4**：NVIDIA 端到端部署栈

### 3.3 MoE VLA（Mixture of Experts）

**核心思想**：每个任务 / 子技能激活不同的专家子网络，总参数大但每次推理只用一小部分。

```
输入 → Router → Expert_k (k ∈ {1, 2, ..., K})
       (软门控)

总参数 = K × N_expert + N_shared
激活参数 = 1-2 × N_expert + N_shared (稀疏)
```

代表：

- **RDT-2**（清华 / 星动）：1B 参数，MoE 动作头，不同任务激活不同专家
- **未来方向**：MoE 在 S1 上做"技能专家"，S2 做"规划通才"

**端侧优势**：总参数可以很大（学到多技能），但激活参数少（推理快）。

### 3.4 结构化剪枝

- **注意力头剪枝**：去掉冗余 head（典型可剪 20-40%）
- **层剪枝**：跳过某些 Transformer 层（典型可剪 10-20%）
- **通道剪枝**：MLP 隐层通道裁减

机器人场景的特殊性：**动作空间低维**（<50），部分参数是冗余的，剪枝潜力比 LLM 更大。

### 3.5 动作专家的极致小型化

- 动作专家（S1）通常只有 80M-1B
- 可以全 FP16 或 INT8 部署
- 带宽主要花在 S2 latent 传输（256-2048 维，低频）

## 四、端侧芯片

### 4.1 NVIDIA Jetson Thor（2025-2026）

- **2000 TOPS FP8** / **1000 TOPS FP16**
- 128 GB LPDDR5X 统一内存
- 对标服务器级 A100 的单机版
- 支持 TensorRT-LLM、Isaac ROS、GR00T 全栈
- **机器人端侧旗舰芯片**

典型部署：

- 7B VLM (S2) INT4 + 1B 动作专家 FP16 → 30 Hz 闭环

### 4.2 NVIDIA Jetson Orin（2024-）

- 275 TOPS (Orin AGX)
- 上一代旗舰，2025 年主流
- SmolVLA、OpenVLA-OFT、GR00T N1 蒸馏版可跑
- 典型：500M-2B VLA，30-100 Hz

### 4.3 华为昇腾 SoC（国内）

- Atlas 系列边缘推理模块
- 支持 MindSpore + CANN 栈
- 国内机器人厂商如宇树、智元部分产品采用

### 4.4 地平线 J6（2026）

- 汽车级 AI SoC（~200+ TOPS INT8）
- 人形机器人潜在选项
- 低成本量产方向

### 4.5 寒武纪、黑芝麻、芯原等

国内多家 AI 芯片公司在 2025-2026 年切入机器人端侧市场，主打"国产替代 Jetson"。

### 4.6 Apple Silicon

- Mac Studio M3 Ultra / M4 Max
- 统一内存 + Metal 后端
- LeRobot / SmolVLA 研究友好
- 家庭机器人潜在选项（1X Neo 早期原型）

| OpenVLA (原版) | 7B | A100 | 26 Hz | ~75% | 离散 token 瓶颈 |

## 七、端侧部署实战清单

### 7.1 模型侧

- [ ] VLM 主干 INT4 量化（AWQ / GPTQ）
- [ ] 动作专家保留 FP16
- [ ] 视觉编码器 INT8 + kernel 融合
- [ ] Flow Matching 或回归 Transformer（避免 Diffusion 多步）
- [ ] Action chunking（减少推理频次）

### 7.2 系统侧

- [ ] S2 / S1 异步双 stream
- [ ] 本体状态 zero-copy 传输
- [ ] 相机 GPU Direct（避免 CPU 中转）
- [ ] 低级控制器独立 RT 线程
- [ ] Watch-dog 安全兜底

### 7.3 数据侧

- [ ] 端侧日志采集（真机数据飞轮起点）
- [ ] 失败案例回传云端
- [ ] 持续学习管道

## 八、趋势：2026 之后

### 8.1 更大的 MoE + 端侧稀疏激活

- 总参数 10-30B 但激活仅 1-3B
- Jetson Thor 级硬件可负担
- 代表方向：RDT-3（推测）、智元 MoE

### 8.2 神经形态硬件

- IBM TrueNorth、Intel Loihi 2 等
- 超低功耗（1W 级）
- 2026 仍处早期，机器人应用尚未落地

### 8.3 世界模型辅助的端侧规划

- S3（世界模型）在云端或更强算力上做"分钟级规划"
- S2 / S1 在端侧做实时执行
- 代表：V-JEPA 2 + 下层 VLA 架构（参见 [../02-世界模型/](../02-世界模型/)）

### 8.4 芯片定制化

- Figure / Tesla / 1X 可能自研 SoC（ASIC 级定制）
- 对标 Tesla FSD 芯片路径
- 单位算力成本再降一个数量级
