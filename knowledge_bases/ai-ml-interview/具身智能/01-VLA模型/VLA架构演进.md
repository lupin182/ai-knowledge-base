# VLA 架构演进

> 本文梳理 Vision-Language-Action 模型从 2022 年 RT-1 到 2026 年双系统 VLA 的完整演进，重点刻画动作表示方式的四次跃迁：**离散 token → Diffusion → Flow Matching → 双频双系统**。

## 二、RT-1（2022.12）：VLA 前身

RT-1 严格说不是 VLA（它不是基于 VLM 预训练的），但奠定了基础。

| 维度 | RT-1 |
|------|------|
| 架构 | EfficientNet（视觉） + TokenLearner + Transformer（35M） |
| 输入 | RGB 图像序列 + 语言指令 |
| 输出 | 离散化 7DoF 动作 token（末端位姿增量 + 夹爪 + 终止） |
| 数据 | 13 万条真实机器人演示（17 个月采集） |
| 结果 | 见过任务 97% 成功率；未见组合 76% |
| 频率 | ~3 Hz |

**关键 insight**：**数据规模与多样性比模型架构更重要**。这条经验直接指导了后来的 RT-2、Open X-Embodiment、π₀。

---

## 三、RT-2（2023.07）：第一个 VLA

**核心创新**：直接复用预训练的 VLM（PaLM-E 12B / PaLI-X 55B），将连续动作离散化为文本 token。

```
输入:  [图像] "pick up the blue cup"
输出:  "1 128 91 241 1 0 0 1"   ← 这是动作！
       (x  y   z  rx ry rz grip terminate)
```

### 动作离散化

- 将连续动作空间均匀分桶为 256 个离散值
- 每个动作维度映射到一个整数 token（占用 VLM 词表中某些 token）
- VLM 自回归生成这些 token，解码回连续动作

### 为什么这么做

1. **复用 VLM 预训练**：视觉理解 + 语言推理能力"白嫖"
2. **零新架构**：不引入新模块，只微调
3. **涌现能力**：能理解"移到左边的物体旁"这种训练中未见过的空间指令

### 局限

- 大模型推理慢（12B / 55B），控制频率仅 1-3 Hz
- 动作离散化损失精度（特别是末端姿态）
- 依赖大量真实数据且闭源

---

## 四、Octo（2024.05）：扩散动作头进场

Octo 由 Berkeley 开源，是第一个在**多种观测空间 + 多种动作空间**上通用的 VLA 式 Transformer Policy。

```
┌──────────────────────────────────────────────┐
│                  Octo 架构                    │
│                                              │
│  任务 Token: [语言指令 / 目标图像]             │
│  观测 Token: [外部相机] [腕部相机] [本体感觉]   │
│         │                                    │
│    ┌────▼────┐                               │
│    │Transformer│  ← 注意力机制融合多模态        │
│    │ Backbone │                               │
│    └────┬────┘                               │
│         │                                    │
│    ┌────▼────┐                               │
│    │ Diffusion │  ← 扩散头生成连续动作          │
│    │  Head    │                               │
│    └─────────┘                               │
│         │                                    │
│    动作: [Δx, Δy, Δz, Δrx, Δry, Δrz, grip]  │
└──────────────────────────────────────────────┘
```

### 关键进步

1. **扩散动作头（Diffusion Head）**：用 DDPM 对动作 chunk 去噪，避免离散化精度损失
2. **Open X-Embodiment 预训练**：80 万轨迹，跨 22 种机器人
3. **模块化 tokenizer**：同一模型可适配不同相机配置 / 动作空间
4. **开源**：代码 + 权重完整公开

### 局限

- 从零训练（非基于 VLM 预训练），缺少常识知识
- 扩散头需要 50-100 步去噪，频率受限 5-10 Hz

---

## 五、OpenVLA（2024.06）：开源 VLA 基线

Stanford 开源，基于 Prismatic VLM 微调而成。

| 维度 | OpenVLA |
|------|---------|
| 视觉编码 | SigLIP + DinoV2（双视觉塔） |
| 语言/动作主干 | Llama 2 7B |
| 动作表示 | 离散 token（类似 RT-2） |
| 训练数据 | Open X-Embodiment 子集（97 万轨迹） |
| 频率 | 5-10 Hz |
| 微调成本 | LoRA 可在单卡 4090 适配新任务 |

**意义**：OpenVLA 让学术界和创业公司**首次可以基于 7B 级 VLA 做研究**，成为后续各种 OpenVLA-* 变种的起点。

### OpenVLA-OFT（2024.10）：并行解码突破

**核心洞察**：OpenVLA 的瓶颈不是模型能力，而是**离散 token + 自回归解码**：每个动作维度都要一次前向。

改进：

1. **并行解码**：一次前向同时输出所有维度，频率从 26 Hz 跃升到 **109 Hz**
2. **L1 回归代替 cross-entropy**：用连续回归头替代离散 token，精度更高
3. **Action chunking**：一次输出多步动作

这一工作标志着：**离散 token 路线基本被 OpenVLA-OFT 这类改进终结**，扩散/Flow Matching 成为下一阶段主流。

## 六、π₀（2024.10）：Flow Matching 范式确立

Physical Intelligence 发布，是第一个大规模使用 **Flow Matching** 的 VLA。

```
┌─────────────────────────────────────────┐
│              π₀ 架构                     │
│                                         │
│  ┌─────────┐   ┌───────────┐            │
│  │ SigLIP  │   │  Gemma    │            │
│  │ (视觉)  │   │  (语言)    │            │
│  └────┬────┘   └─────┬─────┘            │
│       └──────┬───────┘                   │
│              │                           │
│         ┌────▼────┐                      │
│         │ PaliGemma│  ← 3B VLM 主干      │
│         │ Backbone │                      │
│         └────┬────┘                      │
│              │                           │
│         ┌────▼────┐                      │
│         │  Flow   │  ← 动作生成头        │
│         │ Matching│    5-10 步 Euler    │
│         └────┬────┘                      │
│              │                           │
│    动作 chunk (未来 ~1s, 50 步)          │
└─────────────────────────────────────────┘
```

### 为什么 Flow Matching 胜出

| 维度 | DDPM | Flow Matching |
|------|------|---------------|
| 推理步数 | 50-100 | **5-10** |
| 训练目标 | 复杂 VLB | **简洁 MSE**：`v_θ(x_t, t) = x_1 - x_0` |
| 路径 | 曲线 | **直线（Rectified Flow）** |
| 数值稳定性 | 一般 | 更好 |
| 控制频率 | 5-10 Hz | **10-50 Hz** |

详见 [动作生成方法.md § Flow Matching](动作生成方法.md)。

### π₀.5（2025.04）：加入高层语言推理

- **discrete language action**：模型在动作 token 之间穿插输出自然语言规划步骤
- **Co-Training**：同时在"网络多模态数据 + 真机数据 + 仿真数据"上训练
- **家庭泛化**：在完全未见过的真实住宅中长时任务（20-60 分钟）成功率 40-90%

### π-fast（2025 H2）

低延迟版本：参数减半、推理 3-5×、目标 30 Hz+ 闭环。

### π₀.6 / π₀.7（2026 H1，据披露）

PI 的最新迭代。据披露/预期：

- **多模态推理增强**：更强的空间推理与物理常识
- **更长 horizon**：从 π₀.5 的几十分钟向小时级任务扩展
- **家庭泛化深度**：从"会做"升级到"稳健做"，跨家跨物体 zero-shot
- **世界模型耦合**：与 Cosmos 类世界模型深度集成，支持反事实规划

具体架构细节官方尚未全部披露，以 PI 后续论文/博客为准。

## 七、双系统 VLA（2025-2026）：S2 + S1

2025 年 2 月 Figure Helix 发布后，行业迅速围绕"**大脑 S2 + 小脑 S1**"形成共识。详细架构见 [双系统架构.md](双系统架构.md)，此处给出演进视角。

### 单系统 VS 双系统对比

| 维度 | 单系统 VLA（π₀、OpenVLA） | 双系统 VLA（Helix、GR00T N1、π₀.5+） |
|------|---------------------------|--------------------------------------|
| 参数规模 | 3-7B 一体 | S2 7-9B + S1 80-300M |
| 频率 | 10-50 Hz 全局 | **S2 7-9 Hz + S1 100-200 Hz 异步** |
| VLM 更新 | 每步前向 | 仅 S2 低频前向 |
| 反应性 | 受限于 VLM 前向延迟 | S1 可高频反应，达反射级 |
| 实现复杂度 | 简单 | 异步流水线、token 缓存 |

### 代表作

- **Figure Helix**（2025.02）：7B VLM (9 Hz) + 80M Transformer (200 Hz)
- **NVIDIA GR00T N1**（2025.03）：Eagle-2 2B VLM + 1B DiT 扩散动作专家
- **π₀.5**（2025.04）：PaliGemma 3B + Flow Matching 动作专家（内部也是快慢解耦）
- **Gemini Robotics**（2025.03）：Gemini 2 驱动的双臂版

## 八、动作表示方式的四次跃迁

```
第一代 (2022-2023)  离散 token           RT-1, RT-2, OpenVLA
                   优: 复用 VLM
                   劣: 精度/频率差
                            │
                            ▼
第二代 (2024)       Diffusion 动作头      Octo, Diffusion Policy, RDT
                   优: 多模态分布
                   劣: 50-100 步推理
                            │
                            ▼
第三代 (2024-2025)  Flow Matching        π₀ / π₀.5 / π₀.7, SmolVLA
                   优: 5-10 步推理
                   保留多模态分布
                            │
                            ▼
第四代 (2025-2026)  双频双系统            Helix, GR00T N1, Gemini Robotics
                   VLM 慢 + 动作专家快
                   S1 可用 FM/Diffusion/Transformer
```

四种动作表示的详细对比（数学、训练、推理、代表模型）见 [动作生成方法.md](动作生成方法.md)。

## 九、演进驱动力

| 演进 | 核心驱动力 |
|------|------------|
| RT-1 → RT-2 | VLM 预训练知识的可用性（PaLM-E / PaLI-X 成熟） |
| RT-2 → Octo | 离散化精度瓶颈 + Open X-Embodiment 数据集发布 |
| Octo → π₀ | Flow Matching 取代 Diffusion 的训练效率 |
| π₀ → Helix / GR00T N1 | 实时性要求（100 Hz+）+ VLM 复用成本 |
| 单系统 → 双系统 | 大脑-小脑解耦便于独立扩展与部署 |
| 云端 → 端侧 | Jetson Thor、地平线 J6 等 SoC 算力成熟；部署经济性 |

---

## 十、小结

VLA 的四年演进可以概括为：**"如何让一个大模型输出高频、高精度、多模态、可部署的连续动作"**。四代动作表示各有其历史位置，但到 2026 年：

- **Flow Matching + 双系统双频** 是主流工程默认
- **离散 token** 仅在端侧极轻量场景保留
- **纯 Diffusion** 被 Flow Matching 基本替代
- **自回归 Transformer 动作专家**（Helix S1）是双系统下的一种新选择

下一步的主战场是：**长 horizon 记忆、世界模型耦合、触觉融入、RL 微调**。
