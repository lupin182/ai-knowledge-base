# 双臂与全身操控 (Bimanual & Whole-Body Manipulation)

从单臂到双臂，再到全身控制（Whole-Body Control with Manipulation），是 2023–2026 年具身智能硬件与算法共同推动的大趋势。本篇聚焦算法层面的双臂协作、全身操控与动作空间设计；详细的 locomotion 见 [../05-运动与导航/](../05-运动与导航/)。

## 1. 为什么需要双臂 / 全身

- **真实世界很多任务单臂做不了**：开瓶盖（一手扶瓶、一手拧）、折叠衣物、抱箱子、剪纸。
- **人形机器人天生双臂**：不用双臂等于浪费硬件。
- **全身才能 Loco-Manipulation**：拿着箱子走路、蹲下捡东西、爬楼梯送货。
- **家庭 / 工厂场景的大部分任务需要移动底座 + 双臂协同**。

## 2. 双臂协作

### 2.1 ALOHA (Stanford, 2023)

- **硬件**：2× ViperX 主臂 + 2× ViperX 从臂（约 2 万美元）
- **采集方式**：人握持主臂，从臂 1:1 跟随，录制动作
- **学习方法**：ACT（Action Chunking with Transformers）+ CVAE
- **关键任务**：电池穿线、打开小袋、挤牙膏、系鞋带
- **核心洞察**：**简单模仿学习 + 好的数据采集系统即可完成复杂双臂任务**，不一定要 RL

### 2.2 Mobile ALOHA (Stanford, 2024)

- ALOHA + 移动底盘（两轮差速）
- 家务任务：炒虾、擦桌子、开柜子取东西、洗盘子
- 核心发现：**少量移动数据 + 大量静态 ALOHA 数据 co-training** 效果最好
- 全身（底盘 + 双臂）一起学习，但动作空间仍然分开：底盘速度 + 双臂位姿

### 2.3 RDT-2 (清华 + 智元, 2024)

Robotics Diffusion Transformer 的第二代：

- **扩散式 Transformer 作为 VLA 动作头**
- **MoE 动作专家**：不同子任务激活不同专家
- **预训练 + fine-tune**：在 46 个数据集预训练，少量数据 fine-tune
- **原生支持双臂**：动作空间为两臂关节拼接
- **特点**：在双臂灵巧任务上 SOTA，开源

## 3. 全身 VLA：Figure Helix 系列

### 3.1 Helix v1 (Figure, 2025)

- **单一神经网络输出 35-DOF 上半身全身动作**
  - 左右手臂 + 5 指双手 + 腰 + 头，统一动作空间
- **双系统架构（System 1 / System 2）**
  - System 2 (VLM, 7B)：低频（1–7 Hz）做语义理解与子任务分解
  - System 1 (Transformer, 80M)：高频（200 Hz）出连续动作
- **核心突破**：**没有上下身解耦**——腰、头、手臂、手指被同一网络联合决策，避免人为切分导致的动作不协调
- **在两台 Figure 02 上跑同一套权重**：天然支持多机协同

### 3.2 Helix v2 / 多机协作 (2025-2026)

- **双机 Figure 02 合作**：一台递物给另一台，共享视觉 context
- **物流场景量产**：BMW、GXO 仓储部署
- **关键**：不同机器人实时共享任务状态与意图（类似人类团队协作）

### 3.3 为什么 Helix 架构重要

传统路线：
```
高层 planner → 手臂 policy + 底盘 policy + 手策略  (三个网络)
```
Helix 路线：
```
语言 + 视觉 → 单一 VLA → 所有关节同时出动作
```

**优势**：
- 全身动作自然协调（如搬重物时腰自动配合）
- 数据效率更高：一次学习，全身受益
- 减少模块接口 bug

**挑战**：
- 需要大规模全身遥操作数据（Figure 有专门的全身动捕采集）
- 推理延迟敏感：35 DoF 输出在 200 Hz 下需要专用优化

## 4. 全身控制的动作空间设计

### 4.1 三类动作空间对比

| 动作空间 | 描述 | 优势 | 劣势 | 代表 |
|----------|------|------|------|------|
| **关节空间 (Joint Space)** | 直接输出关节角度/力矩 | 最直接、硬件友好 | 维度高、语义弱 | Helix, 多数人形 RL |
| **末端空间 (Task Space)** | 输出末端位姿 + IK 求解 | 语义清晰、跨 embodiment | IK 可能无解、奇异点 | ACT, Diffusion Policy |
| **Latent Action** | 学习低维潜在动作空间 | 压缩高 DoF、跨机器人 | 解释性差、需预训练 | LAPA, QueST |

### 4.2 关节空间 vs 末端空间的选择

| 维度 | 关节空间更好 | 末端空间更好 |
|------|--------------|--------------|
| 高速 / 敏捷运动（走跑跳） | ✓ | |
| 精细抓取 / 装配 | | ✓ |
| 跨具身学习 | | ✓ |
| 全身协调 | ✓ | |
| 触觉反馈闭环 | ✓ | |
| 小数据模仿学习 | | ✓ |

**实际趋势（2026）**：
- **人形 locomotion**：关节空间（RL + PPO 的默认选择）
- **操控**：末端空间或关节空间 + 双手混合
- **全身 VLA**：Helix 走关节空间；部分采用"末端 + 关节"混合

### 4.3 Latent Action：跨具身统一

**LAPA** (Latent Action Pretraining from videos, 2024)：
- 从人类视频学习 latent action
- 不同机器人学一个 decoder 把 latent 映射到各自关节
- 实现"一次预训练，多机器人部署"

**QueST** (2024)：
- 向量量化动作空间（类似 VQ-VAE）
- 离散 token + Transformer 生成

**Vector-Quantized Behavior Transformer (VQ-BeT)** (2024)：
- k-means 聚类动作，变成离散 token
- Transformer 自回归生成

2026 趋势：latent action 正在成为跨机器人迁移的基础设施。

## 6. UniAct 与中国全身 VLA

**UniAct (智元, 2025)**：
- 面向智元**远征 A2**、**灵犀 G1** 等自研人形
- 全身统一动作空间：腰、颈、双臂、双手、底盘
- 预训练 + post-training 路线，跨不同形态机器人
- 与 Figure Helix 技术路径类似，但更强调开放生态和数据飞轮

国内其他方向：
- **银河通用**：桌面操控为主
- **宇树科技**：H1/G1 硬件 + 部分 RL 全身运动
- **小鹏 Iron**：自动驾驶 VLA 经验迁移到人形
- **智元**：UniAct + 灵犀 X2

## 7. 2026 开放问题

1. **全身控制的数据瓶颈**：全身遥操作难度远高于单臂；数据量相比 ALOHA 少一个数量级。
2. **Contact-rich 双臂任务**：两手同时受力时，力闭环 + VLA 的最佳结合方式未定。
3. **双机协作的通信协议**：是共享观测、共享 action、还是共享隐藏状态？
4. **异构双机**：一个人形 + 一个轮式机器人如何协作？
5. **评测标准缺失**：目前没有被广泛认可的双臂 / 全身 benchmark。
