## 二、真机遥操作

### 2.1 主流硬件对比

| 方式 | 代表设备 | 采集速率 | 成本 | 精度 | 特点 |
|------|---------|---------|------|------|-----|
| **Leader-Follower 机械臂** | ALOHA (ViperX), GELLO | 50-100 条/小时 | $10K-25K | 高 | 力觉自然，学术标配 |
| **VR 手柄** | Meta Quest 3, Apple Vision Pro | 60-120 条/小时 | $500-3500 | 中 | 成本低，易部署 |
| **外骨骼** | Tesla Optimus 套装, Shadow 外骨骼 | 20-60 条/小时 | $50K+ | 高 | 全身动作、人形主力 |
| **动作捕捉 (MoCap)** | OptiTrack, Vicon | 批量 | $100K+ | 极高 | 专业级、场地固定 |
| **手部视觉追踪** | UMI, AnyTeleop | 30-80 条/小时 | $100-2K | 中 | 极低成本 |
| **触觉手套** | HaptX, SenseGlove | 20-50 条/小时 | $5K-50K | 中-高 | 力反馈，灵巧手专用 |

### 2.2 ALOHA 生态（学术事实标准）

**ALOHA**（Stanford, 2023）：
- 双臂 ViperX 机器人 + 两个 leader arm
- 一对一"影子"模式：人手操作 leader，robot follower 复现动作
- 成为学术界遥操作"事实标准"
- 开源硬件 BOM ~$30K

**Mobile ALOHA**（Stanford, 2024）：
- 在 ALOHA 基础上加移动底盘
- 一天可采集 50 条长任务（煮菜、擦桌、叠衣）
- 演示家庭场景全流程

**国内复刻与扩展**：
- **AIRBOTs**（国仪量子）：国产双臂平替
- **UMI**（Stanford）：手持夹爪，低成本采集
- **AgilSense、星海图**：国内多家厂商复刻

### 2.3 GELLO（简化版 leader）

- Stanford 2024 发布
- 把 leader arm 简化为 3D 打印 + 便宜电机
- 单臂成本 $500 以下
- 适合低成本学术研究

### 2.4 Apple Vision Pro 遥操作（2024-2025 热点）

Vision Pro 提供高精度手部追踪 + 空间定位，成为灵巧手遥操作的新平台。

#### Bunny-VisionPro

- 零成本接入灵巧手
- Vision Pro 手部关键点直接映射到灵巧手关节
- 特点：入门门槛低，适合快速实验

#### Open-TeleVision（MIT, 2024）

视觉 + 手部追踪的完整方案：

```
Apple Vision Pro:
  ├─ 双目相机画面 → 操作者看到机器人视角 (Stereo VR)
  ├─ 手部追踪 → 映射到灵巧手 (26 DoF)
  ├─ 头部姿态 → 控制机器人"颈部" 相机云台
  └─ 手腕姿态 → 控制机械臂末端

优势:
  - 第一人称沉浸感，操作更自然
  - 腕部姿态精度高
  - 完全无穿戴（vs 外骨骼）
  - 单人单设备，低成本规模化
```

意义：让**每个研究组都能用 $3500 设备做灵巧手 + 全身遥操作**。

### 2.5 Quest 3 遥操作

- Meta Quest 3 ~$500
- 手柄按键 + 控制器追踪
- 精度略逊于 Vision Pro，但便宜 7 倍
- 国内机器人公司（自变量、银河通用等）广泛使用

### 2.6 外骨骼与 Tesla Optimus 训练套装

- Tesla 员工穿戴套装：全身 IMU + 手部传感器
- 动作直接映射到 Optimus
- 适合大规模工厂数据采集（>1000 人规模）
- 缺点：穿戴成本、人力成本高

### 2.7 手部视觉追踪：UMI 与 AnyTeleop

**UMI (Universal Manipulation Interface, Stanford 2024)**：
- 手持夹爪 + GoPro 相机
- 人拿着夹爪做任务，视频 + 夹爪开合度 = 轨迹
- 不需要机器人参与采集
- 极低成本，但 embodiment gap 较大

**AnyTeleop**：
- 视觉追踪 + 重定向算法
- 支持多种夹爪 / 灵巧手
- 开源，MIT license

## 三、自主策略采集 (Autonomous Rollout)

### 3.1 流程

```
初始策略 π₀ (弱)
  │
  ▼
真机 rollout
  │
  ▼
人工筛选成功 (或自动)
  │
  ▼
加入训练集
  │
  ▼
训练 π₁ (更强)
  │
  ▼
重复...
```

### 3.2 DAgger（经典算法，2011）

**Dataset Aggregation**：
- 专家演示初始策略
- 策略执行时，专家实时纠正错误动作
- 专家纠正数据加入训练集
- 迭代收敛

优势：缓解 covariate shift（策略自己走出来的分布偏差）。

### 3.3 RoboCat（DeepMind, 2023）

- 自我改进的多任务机器人策略
- 流程：训练 → 部署 → 自动采集轨迹 → 下一版
- 展示了**大规模机器人自主数据积累**的可行性

### 3.4 π₀ 的自采集循环

- 初版 π₀ 用遥操作 + OXE 训练
- 部署后持续采集自主轨迹
- 成功轨迹加入训练，模型迭代

### 3.5 Figure 02 @ BMW 工厂

- 工厂连续运行 + 数据回传
- 2025 年"第一个进入真实工厂的 VLA"案例
- 数据飞轮雏形

### 3.6 关键挑战

- **初始策略分布偏差**：弱策略只走自己"熟悉"的动作，覆盖有限
- **筛选成本**：自动判定成功率低，人工筛选昂贵
- **失败数据的价值**：失败轨迹 + 改进注释可能比成功轨迹更值钱（RLHF 思路）

## 四、仿真数据采集

详见 [../06-仿真与Sim2Real/](../06-仿真与Sim2Real/)。

```
2024-2026 主流:
  Isaac Lab / Sim: locomotion, 灵巧手 RL
  ManiSkill 3: 操控 benchmark
  Genesis: 大规模并行
  MuJoCo MJX: 可微分

典型 scale:
  单 GPU 并行 1000-4000 环境
  一天可生成 ~百万条仿真轨迹
```

### 仿真数据的质量衰减

- **Sim-to-Real gap** 在 VLA 时代依然存在
- 纯仿真训练的 VLA 真机性能通常比"仿真+少量真机混合"差 20-40%
- 最佳实践：**仿真预训练 + 真机微调**

**RoboTwin (上交, 2024)**：LLM 生成双臂任务脚本 → 仿真执行 → 合成数据。代表了"LLM + 仿真"的新路径。

## 六、人类视频学习

人类每天产生海量第一视角视频，如何利用？

### 6.1 视觉 encoder 预训练

```
R3M (2022):      视频 + 语言对比学习 → 视觉特征
VIP (2023):      视频的 value function 学习
MVP:             Masked Video Pretraining
→ 作为 VLA 视觉 encoder 初始化
```

### 6.2 动作 retargeting（最难）

```
人手视频 → 3D 手部位姿估计 → 映射到机器人夹爪/灵巧手动作
代表: H2O, AnyTeleop, DexCap
Embodiment gap 仍是最大挑战
```

### 6.3 视觉 → 动作的隐式预测

```
UniPi, Vid2Robot:
  视频 predictor 生成"目标视频" → inverse dynamics 提取动作
  思路: 用视频生成做"规划"，动作由 ID 模型提取
```

### 6.4 互联网规模预训练

π₀.5 声明: 训练时用了"大量多模态互联网数据"

## 七、采集工程实践

### 7.1 数据采集流水线

```
遥操作工位
  ├─ 任务描述卡 (LLM 辅助生成)
  ├─ 场景布置
  ├─ 机器人 + 遥操作设备
  └─ 数据记录器 (相机, 状态, action @ 10-30 Hz)

↓

数据预处理
  ├─ 时间同步 (camera vs state)
  ├─ 标注 (成功/失败, 任务描述)
  ├─ 去重与筛选
  └─ 格式转换 (HDF5 / LeRobot format)

↓

数据版本管理
  ├─ git-lfs / DVC
  ├─ 数据集 hub (HuggingFace)
  └─ 增量发布
```

### 7.2 常见格式

- **LeRobot format**（HuggingFace 主推，2024+）
- **RLDS (Reinforcement Learning Datasets Format)**：OXE 使用
- **HDF5**：传统
- **自定义 pickle / zarr**：各实验室常见

### 7.3 质量标注

- **任务描述**：自然语言（"pick up the red cube and place it in the bowl"）
- **成功/失败**：二分类或分段
- **关键帧**：子任务切分（抓取 / 运输 / 放置）
- **物体 bbox / 掩码**：视觉增强用
