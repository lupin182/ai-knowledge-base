# 跨本体学习 (Cross-Embodiment Learning)

> 不同机器人（Franka 机械臂 / UR5 协作臂 / 双臂 ALOHA / 人形全身）在关节结构、夹爪类型、相机位置上差异巨大。如何让**一个模型在多种本体上训练、迁移到新本体只需少量微调**，是 2023 年 Open X-Embodiment 开启、2025-2026 走向主流的研究方向。

## 一、问题定义

### 1.1 本体差异的四个维度

```
┌────────────────────────────────────────────┐
│              本体差异维度                     │
├────────────────────────────────────────────┤
│  1. 运动学结构                               │
│     - 关节数: 6/7 DoF 机械臂, 35 DoF 人形    │
│     - 链路长度、关节极限、运动雅可比           │
│                                            │
│  2. 末端执行器                               │
│     - 平行夹爪（1 DoF）                     │
│     - 吸盘（0 DoF）                         │
│     - 灵巧手（16-22 DoF）                   │
│                                            │
│  3. 传感器配置                               │
│     - 相机数量/位置：外部 / 腕部 / 头戴      │
│     - 深度 / 触觉 / 力矩的有无               │
│                                            │
│  4. 控制接口                                 │
│     - 位置控制 vs 力矩控制 vs 阻抗控制        │
│     - 控制频率 30Hz vs 200Hz vs 1kHz        │
└────────────────────────────────────────────┘
```

### 1.2 为什么重要

- **数据复用**：Franka 上 10 万条数据不能白白废弃
- **新本体 zero-shot**：买一台新机器人不必从零采集
- **加速泛化**：跨本体训练的模型在单本体上也更稳定（类似数据增强）
- **生态效应**：foundation model 的竞争力取决于覆盖多少本体

## 二、四种主流方法

### 2.1 方法 1：动作空间归一化（每本体独立 head）

```
共享主干:   VLM  ──┐
                  │
动作头分支:        ├── Franka head     → 7D 末端 + 夹爪
                  ├── UR5 head         → 6D 末端 + 夹爪
                  ├── ALOHA head       → 14D 双臂
                  └── Humanoid head    → 35D 关节角
```

**代表**：RT-2（部分）、早期 Octo

**优点**：每个本体的动作语义清晰，训练稳定
**缺点**：加新本体需要加 head + 独立数据

### 2.2 方法 2：统一末端位姿（π₀ 方案）

所有本体都用**末端执行器 6DoF pose（世界坐标）+ 夹爪宽度**作为动作。

```
动作 = (Δx, Δy, Δz, Δroll, Δpitch, Δyaw, gripper)
       -------- 世界坐标下末端增量 --------
```

控制器（底层 IK / MPC）负责把末端位姿映射到关节命令。

**代表**：π₀, π₀.5, 部分 OpenVLA

**优点**：
- 真正的跨本体通用
- 数据可以直接跨机器人混合训练
- VLA 模型无需理解具体关节结构

**缺点**：
- 丢失关节冗余信息
- 灵巧手 / 人形全身动作无法直接用末端表达
- 依赖底层 IK 质量

### 2.3 方法 3：潜在动作空间（Latent Action）

```
共享 VLA → 潜在动作 z (例如 64D 连续向量)
           │
           ├── Decoder_Franka   → 7D 关节 / 末端
           ├── Decoder_UR5      → 6D
           ├── Decoder_ALOHA    → 14D
           └── Decoder_Humanoid → 35D
```

**代表**：Latent Action Models (LAM)、UniAct（智元 2024）、部分 RDT-2 模式

**训练方式**：
1. 每种本体有自己的 decoder（以该本体数据训练）
2. 跨本体数据对齐到共享的 z 空间（用对比学习 / 重建损失）
3. VLA 只需输出 z

**优点**：
- 完全本体无关
- 新本体只需训练 decoder，不改 VLA
- 潜在空间可做语义插值

**缺点**：
- 训练复杂（需要对齐 loss）
- z 空间语义不直观，调试难

### 2.4 方法 4：本体描述作为条件（Conditioning）

把本体信息作为**额外输入**喂给 VLA：

```
输入:
  - 观测 obs
  - 语言指令 instr
  - 本体描述 robot_desc
    ├─ URDF 嵌入
    ├─ 机器人类别 ID
    ├─ 关节数 / DoF
    └─ 自然语言描述 ("this is a 7-DoF Franka arm with parallel gripper")

输出:
  - 动作（所有本体共享同一个 head，按条件输出不同维度）
```

**代表**：Octo（用 tokenizer 区分本体）、GR00T N1、Gemini Robotics（据披露）

**优点**：
- 单一模型、单一 head
- 加新本体只需提供描述
- 零样本迁移可能（未见过的 URDF）

**缺点**：
- 本体描述设计空间大，效果依赖实现
- 新本体 zero-shot 仍不如少量微调

## 三、Open X-Embodiment 的启示

### 3.1 数据集概况

```
Open X-Embodiment (OXE, 2023.10)
  来源:     21 家机构联合（Google, Stanford, Berkeley, 清华, 上交, ...）
  规模:     160 万条真机轨迹
  本体:     22 种机器人
  任务:     527 种任务
  总时长:   >10000 小时演示
```

### 3.2 OXE 论文的核心结论

- **RT-X 系列**：在 OXE 上训练的 RT-1-X / RT-2-X，比单本体训练平均提升 **50%**
- **正向迁移确认**：Franka 数据能帮助 UR5 的任务成功率
- **涌现能力**：训练时未见过的"本体 + 任务"组合也能部分成功
- **当前上限**：跨本体能提升 40-60%，但"通用机器人 foundation model" 仍需 10x 数据

### 3.3 OXE v2（2025-2026）

- 更多人形数据（GR-1, H1, Optimus）
- 更长 horizon 轨迹
- 引入触觉 / 力反馈
- 统一的任务描述 schema

## 四、代表工作详解

### 4.1 π₀ / π₀.5（Physical Intelligence）

**已验证本体**：Franka, UR5, ARX 双臂, ALOHA, YAM 等 **8+ 种**

**跨本体策略**：
- 统一的末端位姿动作空间
- Flow Matching 动作头（连续动作 chunk）
- 微调新本体仅需 **~100 条演示**即可达可用水平

**实验结果**（据 π₀ 论文）：
- 跨本体联合训练相比单本体：单本体性能微降 5-10%，新本体迁移成功率 +30-50%
- π₀.5 在家庭新场景零样本成功率 40-90%

### 4.2 Octo（Berkeley, 2024）

- Transformer 主干，多模态 tokenizer
- 支持**任意视角组合**（0-3 个外部相机 + 0-2 个腕部相机）
- 本体信息编码为特殊 token
- 开源：`octo-1.5b` / `octo-small`

**关键发现**：
- 即使新本体的 observation 与训练时完全不同，微调 50 条数据即可恢复性能
- 多视角适应性是跨本体的隐藏关键

### 4.3 GR00T N1（NVIDIA, 2025.03）

- 专为人形机器人设计
- 支持 Fourier GR-1, Unitree H1, Neura 4NE-1, 等
- 本体信息通过 URDF 嵌入 + 训练时本体 ID
- 跨本体训练显著提升新本体零样本性能

### 4.4 UniAct（智元，2024）

- Latent Action Model 路线
- 训练数据：AgiBot World 100 万 + OXE
- 潜在动作空间 64D，覆盖双臂 + 人形
- 新本体适配只需训练 decoder（~1000 步迭代）

### 4.5 Gemini Robotics-ER（Google DeepMind, 2025.03）

- 不直接输出动作，而是输出"语言子目标 + 代码"
- 天然本体无关（代码由本体专属控制器执行）
- 号称 **zero-shot 迁移到新本体**
- 本质是把跨本体问题推给下层控制器

## 五、实践经验总结

### 5.1 "有效跨本体"的五个要点

1. **动作空间要抽象**：末端位姿 > 关节角；latent > 末端位姿（理论上）
2. **视觉 encoder 要够强**：冻结的预训练 VLM（PaliGemma, Qwen2-VL）>> from-scratch
3. **数据混合策略**：每种本体 10-30% 混合比例，避免某本体数据支配
4. **本体标识要明确**：通过 token / prompt / URDF 让模型知道"我是谁"
5. **微调数据量**：新本体 50-500 条通常足够

### 5.2 常见失败模式

- **夹爪不匹配**：平行夹爪数据训练后部署到吸盘机器人 → 抓取策略失效
- **相机位置偏移**：训练时腕部相机 + 部署时外部相机 → 观测分布失配
- **控制频率不一致**：30Hz 数据部署到 200Hz 机器人 → 动作 chunking 断层
- **坐标系错配**：不同机器人 base frame 定义差异导致末端位姿语义改变

### 5.3 微调配方（2026 常用）

```
新本体适配配方:
  Step 1: 采集 100-300 条遥操作数据覆盖目标任务
  Step 2: 加载 π₀ / GR00T N1 预训练权重
  Step 3: 冻结 VLM, 微调 action head + adapter
  Step 4: 5-20 epoch 全参或 LoRA 微调
  Step 5: 真机评测 + DAgger 式数据扩充

时间成本:  ~1-3 天（单卡 H100）
数据成本: ~50 人-小时遥操作
```

## 六、开放问题

### 6.1 humanoid vs 机械臂：混训还是分训？

- **混训**：增加数据量，但两类本体动作语义差异大（桌面操控 vs 全身协调）
- **分训**：稳定但数据量受限
- 当前共识：**分别预训练，共享 VLM backbone**

### 6.2 触觉 / 力反馈的跨本体

- 不同触觉传感器输出格式不同（GelSight 图像 vs BioTac 信号）
- 目前 VLA 基本忽略触觉 → 2026 是整合窗口

### 6.3 跨本体 RL 的可行性

- IL 跨本体已被验证（OXE）
- RL 跨本体几乎没有成功案例（reward 本体相关）
- 潜在方向：学习本体无关的 value function

### 6.4 真正的 "foundation model"何时到来

- 当前最通用的 π₀ / GR00T N1 仍然在 "跨本体微调" 阶段
- "一个模型一键部署到任何机器人" 仍是 2027+ 目标
- 瓶颈：数据多样性 > 模型能力
