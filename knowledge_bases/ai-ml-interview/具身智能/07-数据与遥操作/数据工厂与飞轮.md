# 数据工厂与飞轮

> "谁能跑通小时级数据飞轮，谁就赢得具身智能的下半场。" 本文梳理数据工厂定义、代表玩家、产能估算、飞轮四环节与 Tesla FSD 的启示。

## 一、数据工厂定义

> **数据工厂（Data Factory）**：以工业化流水线方式规模化采集、标注、生产机器人数据的专业组织。

### 工厂标配

```
硬件层：
  - 数十至数百个遥操作工位
  - 机器人阵列（Franka / ALOHA / 自研本体）
  - 中央视觉监控、传感器校准

软件层：
  - 任务脚本库（LLM 辅助生成）
  - 数据采集平台（同步记录 + 回放）
  - 数据版本管理（DVC / git-lfs）
  - 自动 QA 流程

人力层：
  - 24 小时三班制
  - 遥操作员 + 场景布置员 + 质检员
  - 数据标注团队

产出：
  单工位：50-200 条轨迹 / 小时
  100 工位工厂：~1 万轨迹 / 天
  年产能：~300 万轨迹（理想）
```

### 经济学

- 单条轨迹成本：$1-10（取决于任务复杂度）
- 100 万条数据集：$100 万-1000 万
- 数据工厂年运营：$5M-50M（100-1000 工位规模）

## 三、产能估算

### 单工位产能

```
采集速率: 50-200 条/小时
  取决于:
    - 任务复杂度（抓取 vs 叠衣服）
    - 遥操作设备（ALOHA 快 / 外骨骼慢）
    - 操作员熟练度

每日有效工时: 6-7 小时（含休息、场景重置）
单工位日产: 300-1400 条
单工位月产: 9000-42000 条
```

### 工厂产能

```
100 工位:
  日产: 3 万-14 万条
  月产: 90 万-420 万条
  年产: 1000 万-5000 万条

500 工位（Tesla 水平）:
  日产: 15 万-70 万条
  年产: 5000 万-25000 万条
```

### 对比：LLM vs 具身数据

```
GPT-4 预训练: ~10 万亿 tokens
  一条轨迹 ~5000 tokens（50 步 × 100 tokens）
  等价: 20 亿条轨迹 ≈ 100 万工位 × 1 年
```

所以具身智能的数据总量与 LLM 还差 **5-6 个数量级**——这也是"数据是瓶颈"的核心依据。

## 四、数据飞轮四环节

```
┌─────────────────────────────────────────────┐
│                                              │
│   ① 数据生产   ←───────────  ④ 部署反馈       │
│       │                         ▲            │
│       ▼                         │            │
│   ② 筛选 / 标注                  │            │
│       │                         │            │
│       ▼                         │            │
│   ③ 训练                ───→   策略部署        │
│                                              │
└─────────────────────────────────────────────┘
```

### 环节 1：数据生产

- 遥操作 + 自主策略 + 仿真 + 世界模型 + 人类视频
- 五源融合
- 工业化流水线

### 环节 2：筛选与标注

```
输入: 原始轨迹 (带成功/失败，带视频)
过滤:
  - 自动: 任务完成检测、异常检测
  - 人工: 专家审核
增强:
  - 自然语言描述 (LLM 辅助生成)
  - 关键帧切分
  - 失败原因标注
输出: 训练就绪数据
```

**关键挑战**：失败数据的价值！一条失败轨迹 + 改进说明可能比成功轨迹更值钱（RLHF 思路）。

### 环节 3：训练

- 增量训练：新数据加入，LoRA 微调
- 全量重训：周期性（月度/季度）
- 版本管理：策略 v1 / v2 / v3 ...

### 环节 4：部署反馈

- 机器人出错时自动回传
- 人工纠正数据（DAgger 式）
- 家庭 Beta 用户偏好选择
- Corner case 自动识别

**目标状态**：**小时级闭环**——机器人早上出错，下午训练好新模型。

## 五、Tesla FSD 的启示

Tesla FSD 被公认为"数据飞轮的黄金标准"：

```
车队规模: 2M+ (持续增长)
数据类型: 视频 + 传感器 + 驾驶员接管
回传机制: 后台自动上传 "有价值" 片段

关键技术:
  - Shadow mode: 影子模式对比 FSD 决策 vs 人类
  - 自动 corner case 识别
  - 标注流水线（部分自动 + 人工）
  - 每月一版模型

产出:
  每月数百万公里新数据
  模型持续改进
  "长尾问题" 逐步被覆盖
```

### 具身智能能借鉴什么

1. **规模即护城河**：车队/机器人部署越多，数据越多
2. **Corner case 自动识别**：异常检测 + 不确定性估计
3. **影子模式**：让模型与人类操作对比
4. **闭环迭代**：每月/每周一版
5. **专用标注流水线**：大规模 + 半自动

### 为什么具身还没跑通

- 机器人部署规模 < Tesla 车队（Figure/1X 等刚开始）
- 任务多样性远高于驾驶（驾驶只有有限任务）
- 失败代价可能高（家庭场景安全要求）
- 数据回传的法律合规更复杂（家庭隐私）

| 公司 | 阶段 | 数据回传规模 | 特点 |
|------|-----|------------|-----|
| **Figure AI** | Beta | BMW 工厂 + 内部测试 | 工厂场景真实部署 |
| **Physical Intelligence** | Alpha | 合作伙伴试点 | 多机器人类型 |
| **Tesla Optimus** | Alpha | 工厂内部 | 员工穿戴数据为主 |
| **智元 AgiBot** | Alpha | 合作场景 | 开源驱动社区 |
| **1X** | Beta | Home Beta 用户 | 家庭场景 |
| **银河通用** | Alpha（合成） | 仿真为主 | 合成数据飞轮 |
| **自变量** | Alpha | 合成为主 | 合成优先 |
| **Agility Digit** | Beta | 仓库客户 | GXO 等物流场景 |

### 阶段定义

- **概念阶段**：只有单次采集，无回传
- **Alpha**：少量部署，手动回传，月级迭代
- **Beta**：中等规模部署，自动回传，周级迭代
- **成熟飞轮**：大规模部署，自动闭环，日级迭代（目前无）

**2026 年现实**：**尚未真正跑通"小时级"数据飞轮**。最接近的是 Figure @ BMW 与 1X Home Beta。

## 七、RLHF 类反馈：偏好数据与 DPO

灵感来自 LLM 的 RLHF / DPO，2025 年开始在具身领域试水：

### 7.1 偏好数据采集

```
两段相似轨迹 → 人类选择"更好"的那条

更好 = 更快 / 更稳 / 更省电 / 更安全 / 更拟人

→ 偏好数据
→ 训练 reward model 或直接 DPO 微调
```

### 7.2 DPO / IPO 在具身领域的应用

- **Robot-DPO / AI-DPO**（2024-2025）：偏好对 → 直接优化策略
- 无需显式 reward model
- 适合大模型（VLA）后微调

### 7.3 RoboArena

- 2025 发布的真机跨本体评测平台
- 参考 LLM Arena，用户对比两个策略
- 产出偏好数据供研究使用
- 早期探索阶段

### 7.4 挑战

- 偏好数据采集成本仍高
- "什么是更好"主观（LLM 有明确文本质量，机器人任务多样）
- 部署后 reward 稀疏，难以自动评估

## 八、合成数据 vs 真实数据：两条路线

### 合成派

**代表**：银河通用、自变量、Genesis 生态

**论点**：
- 真机采集 cost 太高
- 仿真 + 世界模型无限数据
- 只要 Sim-to-Real 够好，合成足够

**挑战**：Sim-to-Real gap 仍大，长 horizon 任务难

### 真实派

**代表**：智元、Figure、Tesla Optimus

**论点**：
- 真实数据才有真实分布
- 合成永远有 gap
- 规模化采集是可行的

**挑战**：成本、效率、扩展性

### 折中派（主流）

**代表**：Physical Intelligence (π₀)、NVIDIA GR00T

**做法**：真实 + 仿真 + 世界模型混合，各取所长。

## 九、2026 预测

1. **第一代真实飞轮落地**：Figure/Tesla/Optimus 工厂数据开始真正闭环
2. **10 万小时级开源数据**：2027 年前可能出现（AgiBot + 合作伙伴）
3. **世界模型作为"无限数据"**：Cosmos 类模型部分取代仿真
4. **人类视频 VLA 预训练标准化**：类似 CLIP 之于 VLM
5. **RL 微调 VLA 成为标配**：小规模但显著提升复杂任务
6. **持续学习**从论文走向产品
7. **数据安全与隐私**成为家庭机器人部署的法律焦点
