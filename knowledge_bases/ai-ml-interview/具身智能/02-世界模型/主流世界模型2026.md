# 主流世界模型 2026

> 本文聚焦 2025-2026 年最具代表性的世界基础模型（WFM）：NVIDIA Cosmos、DeepMind Genie 1/2/3、Meta V-JEPA 2、1X World Model、UniSim / Generalist AI，覆盖技术路线、开源状态、具身应用与 2026 新趋势。

## 一、NVIDIA Cosmos（WFM 平台开创者）

### 1.1 Cosmos 1.0（2025.01 CES 发布）

NVIDIA 在 CES 2025 发布的**"世界基础模型"（World Foundation Model, WFM）平台**，开创了"WFM"这一行业概念。

```
Cosmos 1.0 四大组件:
├── Cosmos Predict (视频生成，多模态条件)
│   ├── Cosmos-1.0-Diffusion (5B / 14B DiT)
│   └── Cosmos-1.0-Autoregressive (12B, VQ-VAE + Transformer)
├── Cosmos Transfer (真实 ↔ 仿真 风格迁移)
├── Cosmos Reason (视频推理 VLM)
└── Cosmos Nemotron (合成训练数据 pipeline)
```

**开源条款**：Apache 2.0-like 商用友好许可，权重、推理代码、部分训练代码开源，显著区别于 Genie 的闭源。

### 1.2 Cosmos 2.0（2026.01，据披露）

2026 年初 NVIDIA 在 CES 2026 更新 Cosmos 到 2.0（以公开信息为准）：

- **更长 horizon**：单次生成从 1.0 的 ~30 秒延长至分钟级
- **多视角一致**：支持多相机同步生成（对双目/环视机器人关键）
- **动作条件加强**：更精细的控制信号输入（关节轨迹、末端位姿）
- **与 GR00T 深度集成**：GR00T Dreams 2.0 作为配套 pipeline

### 1.3 具身应用：GR00T Dreams Pipeline

NVIDIA 为 GR00T VLA 配套的合成数据 pipeline：

```
输入: 少量真实机器人轨迹 (视频 + 动作 + 语言)
      ↓
Step 1: Cosmos Transfer
        真实画面 ↔ 仿真画面 相互迁移
      ↓
Step 2: Cosmos Predict
        以原始动作为条件，生成多种变体视频
        (不同光照 / 物体外观 / 干扰物 / 视角)
      ↓
Step 3: Cosmos Reason
        过滤生成质量差的样本
      ↓
输出: 10-100 倍扩增的训练数据
      ↓
用于: GR00T N1/N2 的视觉 encoder 与下游 VLA 训练
```

**产业意义**：2025 年 GTC 上 NVIDIA 宣布 GR00T Dreams 对公众开放，成为人形机器人公司标配工具链。

## 二、Google DeepMind Genie 系列

### 2.1 Genie 1（2024.02）

> DeepMind, "Genie: Generative Interactive Environments", 2024.

**突破**：从**无标签**游戏视频学习"可交互世界"。

```
输入训练数据: 20 万小时互联网 2D 游戏视频 (纯视频，无动作标注)
关键组件:
  - Latent Action Model (LAM): 从连续帧推断潜在动作
  - Video Tokenizer (ST-ViViT): 时空 tokenize
  - Dynamics Model: (token, latent_action) → 下一 token

推理阶段: 用户按"方向键"(对应 latent action) → 交互式生成下一帧
```

**意义**：证明可以**从观察中学出"动作概念"**，无需动作标签。

### 2.2 Genie 2（2024.12）

> DeepMind Blog, "Genie 2: A large-scale foundation world model", 2024.12.

**升级**：
- 从 2D 像素游戏升级到 **3D 场景**
- 支持 **第一/第三人称视角切换**
- 物体交互、NPC 行为
- 单张图像 → 生成完整可控 3D 世界
- 一致性约 1 分钟

### 2.3 Genie 3（2025.08，据披露）

**关键改进**：
- **长一致性**：物体持久性推至 **2-3 分钟**
- **物理约束加强**：碰撞、重力更符合真实物理
- **多模态条件**：支持文本描述、目标图像、动作序列作为条件
- 用于 **Gemini Robotics** 的合成环境训练（推测）

### 2.4 Genie 系列的具身价值

```
用途: 作为"无限生成式仿真环境"
  - 学术：研究策略泛化（相比 Isaac Lab 等少限制）
  - 工业：生成罕见场景（corner case）训练数据

局限:
  - 闭源，仅 DeepMind 内部使用
  - 物理正确性未在真机验证
  - 与 Gemini Robotics 耦合度高，难以外部复现
```

## 三、Meta V-JEPA 2（2025.06）

> Meta FAIR, "V-JEPA 2: Self-Supervised Video Models Enable Understanding, Prediction and Planning", 2025.06.

### 3.1 核心创新：非生成式世界模型

V-JEPA（Joint-Embedding Predictive Architecture）与 Cosmos/Genie 路线**完全不同**：**在隐空间预测，不生成像素**。

```
V-JEPA 2 pipeline:

  视频 x_1:T
    ↓ Encoder (ViT-H, 参数冻结)
  z_1:T (隐表示)
    ↓
  给定观测 z_1:t 和动作 a_1:t
    ↓ Predictor
  ẑ_{t+1:T}

  训练损失: || ẑ_{t+1} - z_{t+1}.detach() ||²
                             ↑
                       目标从 target encoder 来 (EMA 平均)
```

**为何不生成像素？**
- Yann LeCun 观点：像素生成浪费算力，智能只需要学"有用的抽象"
- 隐空间预测更稳定，不会陷入"生成细节 vs 长期一致性"的 trade-off

### 3.2 关键技术

- **两阶段训练**：
  - Stage 1：在 Ego4D 等大规模视频上自监督预训练（无动作）
  - Stage 2：在机器人数据上加入动作条件微调
- **规模**：主力模型 ~1B 参数，已开源权重

### 3.3 具身应用：Zero-Shot Planning

V-JEPA 2 最激动人心的应用是**给定目标图像的 zero-shot 规划**：

```
输入:
  当前观测图像 o_current
  目标观测图像 o_goal

规划:
  z_current = encoder(o_current)
  z_goal    = encoder(o_goal)
  
  搜索动作序列 a_1:T 使得:
    ẑ_T = predictor(z_current, a_1:T) ≈ z_goal
  
  用 MPC (CEM / Gradient) 在隐空间搜索

执行:
  执行 a_1 → 观测 o_1 → 重新规划 (receding horizon)
```

**验证任务**：Franka 抓取、推物体等；无需任务特定微调，**对任意目标图像都能 plan**。

### 3.4 优势与局限

| 优势 | 局限 |
|------|------|
| 无任务 reward 即可规划 | 精细操作精度有限（隐空间没有接触力） |
| 推理算力小 | 目标必须能表示为图像（语言目标要借助 CLIP 映射） |
| 模型开源 | 长 horizon 仍受 predictor 误差累积影响 |
| 与 LeCun 学术路线绑定 | 产业界采纳速度慢于 Cosmos |

## 四、1X World Model

### 4.1 背景

1X Technologies（挪威/美国，2023 获 OpenAI 投资）为 Neo / EVE 机器人训练的**家庭场景专用世界模型**。

### 4.2 特点

```
训练数据: Neo 本体第一视角视频（家庭环境大规模采集）
  - 非公开估计 1000+ 小时家庭场景
  - 涵盖起居室、厨房、卧室等真实家庭

用途:
  - 在 WM 中预测策略动作的后果
  - 作为"软仿真器"评估策略
  - 替代部分真机 rollout

对外定位: "超越仿真"(Beyond Simulation)
  传统仿真 = 脚本化物理引擎
  1X WM = 真实视觉分布 + 学习的动力学
```

### 4.3 公开信息有限

1X 将 WM 作为内部核心资产，不开源权重。2024.06 和 2025.03 发布过两篇 demo 视频，展示：
- 给定机器人动作，生成未来 5-10 秒家庭场景画面
- 视觉真实性接近真实视频

### 4.4 具身价值

- **Home Beta 用户数据回传 → WM 持续训练**：据披露这是 1X 数据飞轮核心
- **验证"真实视频 > 仿真渲染"**：视觉分布对齐自然优于渲染

## 五、UniSim / Generalist AI（Pieter Abbeel 商业化）

### 5.1 UniSim 学术渊源

> Yang et al. (Pieter Abbeel 组), "Learning Interactive Real-World Simulators", 2023.

**思想**：从多源视频（人类 demo、机器人数据、游戏、驾驶）统一学习世界模型。

```
任意动作/语言/控制输入 → UniSim → 预测未来画面

多源数据融合:
  Ego4D (人类视角)
  Kinetics (动作分类)
  Open X-Embodiment (机器人)
  游戏视频 / YouTube 操作教程
  ...
```

**贡献**：提出了"通用仿真器"的技术路径，早于 Cosmos 但影响力不及。

### 5.2 Generalist AI（2024 成立）

Pieter Abbeel 于 2024 年成立 **Generalist AI**（据披露估值 10 亿+），将 UniSim 商业化：

- **目标**：为机器人公司提供"世界模型即服务"
- **技术路线**：大规模视频 WM + 机器人策略联合训练
- **融资方**：Sequoia、Lux Capital 等
- 与 Physical Intelligence、Skild AI 等同属"具身基础模型创业公司"阵营

## 六、2026 新趋势

### 6.1 长 horizon 一致性

```
2024:  秒级一致 (Genie 1)
2025:  1-2 分钟 (Genie 2, Cosmos 1.0)
2026:  3-5 分钟 (Genie 3, Cosmos 2.0 据披露)
目标:  分钟→小时级 (研究前沿)
```

技术手段：分层世界模型（高层抽象轨迹 + 低层细节）、显式记忆模块、retrieval-augmented WM。

### 6.2 多模态条件

早期 WM 仅以动作为条件；2026 年标配多模态条件：

```
条件输入:
  ├── 动作 (离散 token / 连续关节角)
  ├── 语言指令 ("拿起杯子放到桌子右边")
  ├── 目标图像 (目标状态)
  ├── 轨迹草图 (末端路径 sketch)
  └── 物体类别 / 场景描述

代表: Cosmos 1.0 Predict, Genie 3
```

### 6.3 VLA 端到端耦合

从"先训 WM，再训策略"到"WM 与策略联合训练"：

```
2025 分阶段:
  WM 独立训练 → 数据增广 → VLA 训练

2026 端到端:
  VLA 的 future prediction head 即是 WM
  联合损失 = 动作回归 + 未来观测预测
  代表: DynaMo, DreamGen, π₀.5 + Cosmos 的内部集成
```

详见 [VLA 与世界模型融合](VLA与世界模型融合.md)。

## 八、选型建议

| 场景 | 推荐 | 理由 |
|------|------|------|
| 工业级数据增广、VLA 训练 | **Cosmos** | 开源商用、pipeline 成熟 |
| 学术研究、zero-shot 规划 | **V-JEPA 2** | 开源、算力友好、规划 API 清晰 |
| 生成式仿真 demo | Genie 3 (若开放) | 交互性最好，但闭源 |
| 真机 RL 样本效率 | **Dreamer V3** | 成熟、跨任务通用 |
| 家庭人形专用 | 自训 WM（参考 1X） | 家庭第一视角数据难共享 |
