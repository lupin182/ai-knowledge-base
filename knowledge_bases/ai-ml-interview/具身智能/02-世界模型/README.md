## 2026 世界模型发展现状

### 时间线：从 RL 辅助到世界基础模型

```
┌───────────────────────────────────────────────────────────────────────┐
│  阶段 1 (2018-2022): 隐空间世界模型 + RL                               │
│    Dreamer V1/V2/V3、PlaNet、DayDreamer                               │
│    用途：仿真中 rollout、数据效率高的 RL                                │
│                              ↓                                         │
│  阶段 2 (2023-2024): 视频生成世界模型                                   │
│    Genie 1、UniSim、Sora（非具身导向）、IRIS                            │
│    用途：从视频学动力学、生成式仿真器雏形                                │
│                              ↓                                         │
│  阶段 3 (2025): 世界基础模型（WFM）时代                                 │
│    Cosmos（NVIDIA, 2025.01）、Genie 2（2024.12）→ Genie 3（2025.08）    │
│    V-JEPA 2（Meta, 2025.06）、1X World Model                           │
│    用途：数据增广、Sim2Real 桥梁、zero-shot planning                    │
│                              ↓                                         │
│  阶段 4 (2026-): 与 VLA 端到端耦合                                      │
│    GR00T Dreams pipeline、π₀.5 + Cosmos、V-JEPA 2 + VLA 规划层分离     │
│    长 horizon、多模态条件、"想象中学策略"                               │
└───────────────────────────────────────────────────────────────────────┘
```

### 2026 世界模型的三大用途

具身智能视角下，世界模型不是目的，而是**服务于策略学习**的基础设施：

```
┌──────────────────────────────────────────────────────────────────┐
│  用途 1：数据增广（Data Augmentation）                             │
│    真实 1 条轨迹 → WM 生成 N 条变体（新光照/新物体/新视角）         │
│    代表：NVIDIA GR00T Dreams（Cosmos 为底座）                      │
│    价值：真机数据稀缺的硬约束被"视觉条件合成"缓解                    │
├──────────────────────────────────────────────────────────────────┤
│  用途 2：Sim2Real 桥梁（Domain Transfer）                          │
│    仿真画面 → Cosmos Transfer → 真实风格画面                       │
│    代表：Cosmos Transfer、UniSim                                   │
│    价值：闭合仿真渲染与真实分布的 visual gap                        │
├──────────────────────────────────────────────────────────────────┤
│  用途 3：Zero-Shot Planning（想象中规划）                          │
│    给定目标图像/隐表示 → 在 WM 中搜索动作序列                       │
│    代表：V-JEPA 2、Dreamer V3                                      │
│    价值：无需任务奖励即可完成新任务                                  │
└──────────────────────────────────────────────────────────────────┘
```

### 2026 当前共识（五条）

1. **世界模型不取代仿真，而是"软仿真器"**：传统刚体仿真（Isaac Sim / MuJoCo）在物理正确性上仍不可替代；WM 强在视觉多样性与真实分布对齐。
2. **像素与隐空间两条路线并行**：Cosmos / Genie 走像素路线，强在"可视化、生成数据"；V-JEPA 走隐空间路线，强在"高效规划"。两者目标不同并非竞争。
3. **开源化是 2025 的分水岭**：Cosmos（Apache 2.0-like）、V-JEPA 2（模型开源）、Genie（闭源）形成明显阵营；开源路线推动产业快速扩散。
4. **"世界基础模型"概念被行业接受**：NVIDIA 主推的 WFM（World Foundation Model）概念——类比 LLM 之于语言——已被 Meta / DeepMind / 1X 采纳。
5. **VLA + WM 端到端耦合是 2026 关键词**：不再是两个独立模块，而是联合训练的统一架构（参见 [VLA与世界模型融合](VLA与世界模型融合.md)）。

### 2026 开放问题

- **动作维度生成的物理正确性**：视频 WM 可以生成"好看的视频"，但生成的动作是否满足刚体/摩擦/接触约束？目前仅做视觉增广，动作仍靠真实或仿真提供。
- **长 horizon 一致性**：Genie 3 将一致性推到 2-3 分钟，但分钟级以上的物体持久性、记忆仍是难题。
- **条件可控性**：如何精确控制"这只手从左边抓这个杯子"而非"随便生成一个抓取视频"。
- **与 VLA 的计算瓶颈**：WM rollout 的算力开销巨大，实时部署需要模型蒸馏或分层规划。
- **评测标准缺失**：如何量化 WM 生成数据对下游策略的真实提升，尚无统一 benchmark。
