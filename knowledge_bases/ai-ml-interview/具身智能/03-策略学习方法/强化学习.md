# 强化学习

> 本文聚焦具身智能中的强化学习：经典算法（PPO / SAC / DDPG / TD3）、机器人 RL 的特色问题（稀疏奖励、reward shaping、课程学习）、Sim-to-Real 的 Teacher-Student 范式，以及 ANYmal Parkour、OpenAI Rubik's Cube、Extreme Parkour 等代表案例。

## 一、具身智能 RL 的特殊挑战

与 Atari / 围棋等纯 RL 研究相比，具身 RL 面临：

- **Sim-to-Real Gap**：仿真中学到的策略需要迁移到真机
- **安全约束**：不能随意探索（机器人可能摔倒/碰撞/损坏硬件）
- **高维连续动作**：关节空间通常 6-30 维连续，灵巧手可达 20+
- **部分可观测**：传感器噪声、遮挡、本体感知受限
- **稀疏奖励**：复杂操控任务难以设计密集奖励
- **真机代价高**：真机 RL 训练 1 小时可能比仿真 1 天还少

## 二、经典 RL 算法

### 2.1 PPO（Proximal Policy Optimization）

> Schulman et al. (OpenAI), 2017.

**On-policy** 算法，2026 仍是具身 RL 的**事实标准**。

```
核心思想: 限制每步策略更新幅度 (trust region)

目标函数:
  L^CLIP(θ) = E_t [ min( r_t(θ) · Â_t,  clip(r_t(θ), 1-ε, 1+ε) · Â_t ) ]
  
  其中 r_t(θ) = π_θ(a_t|s_t) / π_θ_old(a_t|s_t)  (重要性采样比)
       Â_t = GAE 优势估计

典型 ε = 0.2
```

**优势**：
- 实现简单、超参鲁棒
- 支持大规模并行（Isaac Gym 4096 环境同步 rollout）
- 是 ANYmal Parkour、Figure Helix locomotion 等工作的主力

**适用场景**：
- **Locomotion**：并行仿真爆炸式数据效率
- 大规模仿真训练
- 初学者入门

### 2.2 SAC（Soft Actor-Critic）

> Haarnoja et al. (Berkeley), 2018.

**Off-policy** 算法，样本效率高，适合真机训练或仿真效率有限的场景。

```
核心思想: 最大化 reward + 策略熵 (鼓励探索)

目标:
  J(π) = Σ_t E[ r(s_t, a_t) + α · H(π(·|s_t)) ]

关键组件:
  - 两个 Q 网络 (double Q)
  - 策略网络 (Gaussian / Tanh-squashed)
  - 自动温度 α 调节
  - Replay buffer (off-policy)
```

**优势**：
- 样本效率高（off-policy）
- 自动探索-利用平衡
- 对超参鲁棒

**适用场景**：
- **真机操控训练**（样本宝贵）
- DayDreamer 这类真机 RL
- 灵巧手任务

### 2.3 DDPG / TD3

```
DDPG (Lillicrap, 2015):
  Deterministic policy gradient + Actor-Critic
  早期连续控制基线, 目前被 SAC / TD3 取代

TD3 (Fujimoto, 2018):
  DDPG 的改进:
    - Twin critics (减少过估计)
    - Delayed policy updates
    - Target policy smoothing
  SAC 的"确定性表亲"
```

当前实用程度：SAC > TD3 > DDPG，新项目通常直接选 SAC 或 PPO。

### 2.4 DrQ-v2（像素 RL）

> Yarats et al., 2021.

**直接从像素学习**的 off-policy 算法，关键贡献：
- 数据增强（random shift）
- SAC 架构
- 在 DMControl Suite 上显著优于之前方法

机器人领域用 DrQ-v2 + 域随机化是像素 RL 的常见方案。

### 2.5 算法选型速查

| 算法 | 类型 | 特点 | 具身智能适用场景 |
|------|------|------|-----------------|
| **PPO** | On-policy | 稳定、易调参 | Locomotion、大规模并行训练 |
| **SAC** | Off-policy | 样本高效、自动温度 | 操控、真机训练、灵巧手 |
| **TD3** | Off-policy | SAC 的确定性版 | 连续控制 |
| **DrQ-v2** | Off-policy + 数据增强 | 直接从像素学习 | 视觉 RL |
| **Dreamer V3** | Model-based | 样本效率极高 | 真机 RL、跨任务（见 [../02-世界模型/世界模型范式.md](../02-世界模型/世界模型范式.md)） |

## 三、奖励工程

具身智能中奖励设计是**核心工程问题**，远比算法选择重要。

### 3.1 手工 reward shaping：四足行走案例

```python
# 典型四足运动奖励
reward = (
    1.0 * velocity_tracking_reward    # 跟踪目标速度
    - 0.1 * torque_penalty            # 减小力矩（省能量）
    - 0.5 * action_rate_penalty       # 动作平滑
    - 2.0 * fall_penalty              # 不摔倒
    - 0.1 * joint_limit_penalty       # 不超关节限位
    + 0.5 * feet_air_time_reward      # 鼓励抬脚（trot 步态）
    - 0.1 * base_height_penalty       # 保持躯干高度
    - 0.05 * orientation_penalty      # 保持躯干水平
)
```

**设计原则**：
- 主目标用正奖励（velocity tracking）
- 约束用负惩罚（fall, joint limit）
- 平滑项鼓励自然行为（action rate, smoothness）
- 每一项权重需手动调参，通常需要数天实验

### 3.2 自动奖励设计

**1. Eureka**（NVIDIA, 2023）：

```
LLM 自动生成奖励函数 Python 代码
→ 在仿真中评估策略性能
→ 根据性能反馈迭代改进奖励
→ 结果: 在 29/29 个 Isaac Gym 任务上超过人类专家设计的奖励
```

**2. Text2Reward**：

```
输入: 自然语言任务描述 ("让机器人绕过障碍物跑到目标点")
输出: 结构化奖励函数代码
```

**3. RL-VLM-F**（使用 VLM 作为 reward model）：

```
预训练 VLM 对状态打分
→ 作为密集 reward 信号
→ 省去手工设计, 也不需要任务成功标注
```

### 3.3 稀疏奖励与 curriculum

复杂任务常面临**稀疏奖励**（只在任务结束有奖励），收敛极慢。缓解：

- **Hindsight Experience Replay (HER)**：把失败轨迹的终点当作"另一个目标"来重用
- **Curriculum learning**：从易到难（下节详述）
- **Reward shaping**：添加中间密集奖励
- **Goal-conditioned RL**：把目标作为输入，一次训练多目标

## 四、课程学习（Curriculum Learning）

逐步增加任务难度，帮助策略从简单到复杂：

### 4.1 地形课程（Locomotion）

```
Level 1: 平地行走
Level 2: 小斜坡 (<10°)
Level 3: 台阶 (5-10cm)
Level 4: 随机地形 (碎石)
Level 5: 极端地形 (大台阶、悬崖边)

实现 (Isaac Lab 内置):
  按策略当前成功率自动升/降级
  每个并行环境独立评估
  成功率 > 0.8 → 升级; < 0.3 → 降级
```

### 4.2 速度课程

```
Stage 1: v_target ∈ [0, 0.5] m/s  (慢速稳行)
Stage 2: v_target ∈ [0, 1.5] m/s  (中速)
Stage 3: v_target ∈ [0, 3.0] m/s  (高速)
```

### 4.3 扰动课程

```
Stage 1: 无外部干扰
Stage 2: 小力推动 (5N 脉冲)
Stage 3: 大力推动 (20N) + 负载变化
Stage 4: 极端干扰 (连续推挤)
```

**关键 insight**：ANYmal Parkour、Extreme Parkour 等工作的成功，70% 来自精心设计的课程，30% 来自算法本身。

## 五、域随机化（Domain Randomization）

训练时随机化仿真参数，使策略对参数变化鲁棒，从而迁移到真实。

### 5.1 随机化维度

| 随机化类型 | 参数示例 | 随机范围 |
|-----------|---------|----------|
| **物理参数** | 摩擦系数、质量、惯量 | ±30-50% |
| **视觉参数** | 光照、纹理、相机位置 | 大范围 |
| **动力学参数** | 电机延迟、PD 增益 | ±20-30% |
| **干扰** | 外力推动、负载变化 | 随机脉冲 |
| **传感器噪声** | IMU 漂移、编码器噪声 | 真实噪声模型 |
| **本体参数** | 肢长、关节间距 (跨本体时) | 真实分布 |

### 5.2 关键 insight

**不需要精确仿真，只需要真实参数落在随机范围内。**

```
类比:
  如果策略在 μ - 2σ 到 μ + 2σ 的所有参数组合下都成功
  那么真实系统（某个 μ*）只要在该范围内，就能零样本迁移
```

### 5.3 代表案例：OpenAI Rubik's Cube（2019）

> OpenAI, "Solving Rubik's Cube with a Robot Hand", 2019.

**成就**：仿真训练 + 大规模域随机化 → 真实 Shadow Hand 零样本解魔方。

```
随机化维度（50+ 项）:
  - 方块大小、摩擦、质量
  - 关节限位、电机响应
  - 相机位置、光照、纹理
  - 重力方向扰动
  - 执行器噪声

训练量:
  13000 GPU·年等效 (超大规模分布式)
  PPO + ADR (Automatic Domain Randomization)
  
结果:
  真实 Shadow Hand 0-shot 部署
  证明了 "仿真 + DR" 的强大潜力
```

### 5.4 ADR（Automatic Domain Randomization）

手动设计随机范围需要调参，ADR 自动扩展：

```
初始化: 小的随机范围 (接近 nominal)
若当前范围下策略成功率 > 阈值:
  扩大范围
若失败率过高:
  缩小范围

效果: 范围自动匹配策略能力, 持续 push boundary
```

## 六、Teacher-Student 训练

**两阶段训练范式**，解决观测差异问题，是 locomotion 的事实标准。

### 6.1 基本思路

```
阶段 1: Teacher 策略 (在仿真中 RL 训练)
    - 输入: 特权信息 (ground truth 状态、接触力、地形高度图)
    - 用 RL (如 PPO) 训练到很强
    - 动作: 关节目标角

阶段 2: Student 策略 (可部署到真机)
    - 输入: 仅传感器可获取的信息 (关节编码器、IMU、深度相机)
    - 通过蒸馏模仿 Teacher 的行为 (监督学习)
    - 学会从有限传感器中隐式估计特权信息
```

### 6.2 为什么这样做

```
若直接用 noisy obs 训 RL:
  - 探索难 (POMDP)
  - 收敛慢
  - 容易陷入局部最优

两阶段分离:
  - 阶段 1 只关心"最优策略长什么样"(不管可部署性)
  - 阶段 2 只关心"模仿专家"(监督学习简单高效)
```

### 6.3 代表案例

**ANYmal Parkour**（ETH Zurich, 2022-2024）：

> Hoeller et al., "ANYmal Parkour: Learning Agile Navigation for Quadrupedal Robots", 2024.

```
Teacher:
  观测: 地形高度图 (privileged) + 本体状态
  输出: 关节目标角
  RL: PPO, Isaac Gym, 4096 并行

Student:
  观测: 深度相机原始图像 + 本体状态 (真实可获取)
  通过 DAgger 式蒸馏学习模仿 Teacher
  
真机部署: ANYmal C 四足机器人可以跳跃、攀爬、钻洞等 agile 动作
```

**Extreme Parkour**（UCSD / Stanford, 2024）：

> Cheng et al., 2024.

```
目标: 小型四足 (MIT Mini Cheetah / Unitree Go1) 做跑酷
方法: Teacher-Student + 深度视觉
成就: 跳 2 倍身高的台阶、高速奔跑下楼梯
开源: https://extreme-parkour.github.io/
```

**人形 Parkour / Humanoid Locomotion**（2024-2026）：
- **Humanoid Parkour Learning** (CMU, 2024)
- **OmniH2O** (CMU, 2024)：全身遥操作
- **HumanPlus** (Stanford, 2024)：人类动捕 → 人形蒸馏
- **Expressive Whole-Body Control** (Stanford, 2024)

共同范式：仿真 PPO Teacher + 真机可部署 Student + 大量域随机化。

## 八、2025-2026 新趋势：VLA 预训练 + 专家 RL 微调

### 8.1 新共识

```
2019-2023 范式:  从零 RL（每任务独立训练）
2024-2025 范式:  IL 大规模预训练（ALOHA, Diffusion Policy, π₀）
2025-2026 范式:  IL 预训练 + RL 后微调  (见 [VLA 后微调](VLA后微调.md))
```

**动机**：
- 纯 RL 对真机样本需求过大
- 纯 IL 上限 = 专家
- 组合：**IL 提供好起点（专家水平），RL 突破上限（超专家）**

### 8.2 代表方法

详见 [VLA 后微调](VLA后微调.md)。简要预览：

- **DPPO** (2024)：把 Diffusion Policy 视为策略，PPO 优化去噪链
- **ReinFlow** (2025)：Flow Matching 动作头的 RL 微调
- **RLDG** (2024)：RL 引导 Diffusion 采样
- **IQL / CQL**：离线 RL 方法，适合用已有数据做微调

### 8.3 真机 RL 的持久挑战

```
真机 RL 的三大硬约束:
  1. 硬件损耗 (机器人一天跑坏多个是常事)
  2. 安全 (学习过程中可能伤人/自毁)
  3. 数据效率 (真机 1000 步 ≠ 仿真 1000 步, 时间成本天差地别)

2026 工程共识:
  - 仿真训练 80-95%, 真机微调 5-20%
  - 真机 RL 仅用于"最后 2%"的精细调优
  - 主战场仍在仿真 (Isaac Lab, Genesis, Mujoco MJX)
```
