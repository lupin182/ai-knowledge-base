# 扩散策略（Diffusion Policy）

> 将扩散模型引入机器人控制，是 2023 年以来操控领域最重要的方法创新。本文覆盖 Diffusion Policy (Chi et al. 2023) 的核心思想、多模态动作分布、架构（观测编码器 + U-Net / Transformer 去噪网络）、3D Diffusion Policy (DP3) 点云输入、以及 Action Chunking / 归一化 / DDIM 加速等实践细节。

## 一、核心思想：动作生成 = 去噪

> Chi et al. (Columbia / Toyota Research), "Diffusion Policy: Visuomotor Policy Learning via Action Diffusion", 2023.

把动作生成视为**去噪过程**：从随机噪声出发，逐步去噪得到动作序列。

$$a_{0:T} = \text{Denoise}(a^K_{0:T}, \text{obs}), \quad a^K \sim \mathcal{N}(0, I)$$

其中 $a_{0:T}$ 是预测的未来 $T$ 步动作序列，$K$ 是去噪步数。

### 1.1 DDPM 训练目标

每个训练步：

```
1. 从数据采样: (obs, a*_{0:T})    [专家动作 chunk]
2. 采样时间步 k ~ U(1, K)
3. 采样噪声 ε ~ N(0, I)
4. 前向加噪: a^k = √(α_k) a*_{0:T} + √(1-α_k) ε
5. 模型预测: ε̂ = ε_θ(a^k, k, obs)
6. 损失: L = MSE(ε̂, ε)
```

推理时从 $a^K \sim N(0, I)$ 开始，迭代去噪 $K$ 步到 $a^0$。

### 1.2 关键论点

原论文的核心实验观察：

1. **在 robomimic / push-T / ALOHA 等 benchmark 上，Diffusion Policy 显著优于 BC-MLP、BC-RNN、BET、LSTM-GMM、IBC 等基线**
2. **对噪声观测鲁棒**
3. **可以建模复杂的多模态动作分布**（下一节详解）
4. **训练稳定**（不像 GAN 不收敛，不像 VAE 后验塌陷）

## 二、为什么扩散模型适合机器人动作

### 2.1 多模态动作分布

同一个观测可能对应多种合理动作（如绕过障碍物左边或右边都行）。传统方法的问题：

```
MSE 回归 (BC-MLP):
  把多模态"平均掉" → 输出中间错误动作

分类式 (BET, VQ-BeT):
  离散化动作空间 → 表达力受限

CVAE (ACT):
  隐变量 z 建模多模态, 但推理时 z 采样不总是稳定
  β (KL 权重) 需要仔细调
  
扩散模型:
  天然表达任意复杂分布
  不需要建模先验 z
  推理确定性（给定噪声和观测，输出确定）
```

### 2.2 时间一致性

Diffusion Policy 预测**整个动作 chunk** $a_{0:T}$（通常 T = 16 步）而不是单步。因为扩散过程在整个 chunk 维度联合去噪，**chunk 内部的动作天然连贯**。

### 2.3 训练稳定

```
与 GAN 比: 扩散不需要对抗训练, 不会 mode collapse
与 VAE 比: 扩散不需要编码器, 不会后验塌陷
与 MLE 回归比: 扩散不把多模态平均
```

这些性质让 Diffusion Policy 在工程上**极易训练**——一个脚本、几百条数据、一张 GPU 训 1-3 天就能得到好策略。

## 三、架构

```
观测序列 [o_{t-k}, ..., o_t]    (观测历史, k ≈ 2-3)
         │
    ┌────▼────┐
    │ 观测编码器│  (ResNet18 / ViT / 点云 PointNet)
    └────┬────┘
         │ 条件向量 c (条件化每一步去噪)
         │
    ┌────▼─────────────────────────┐
    │ 条件去噪网络                   │
    │  输入: 噪声动作 a^k, 时间步 k  │
    │  条件: c (观测特征)            │
    │  输出: 预测噪声 ε 或动作 â     │
    │                               │
    │  架构选择:                     │
    │    - U-Net 1D (原论文主推)      │
    │    - Transformer (2024 主流)   │
    └──────────────┬────────────────┘
                   │
           K 步 DDPM / DDIM 去噪
                   │
       动作 chunk [a_t, a_{t+1}, ..., a_{t+T-1}]
```

### 3.1 U-Net 1D 去噪器

原论文主推 U-Net 1D：

```
输入: a^k ∈ R^{T × action_dim}  (序列, 如 16×7)
条件: c (FiLM 调制每层)
     k (正弦位置编码 → FiLM)

架构:
  Down-sampling (1D conv + FiLM)
  Bottleneck
  Up-sampling + skip connection

输出: ε̂ ∈ R^{T × action_dim}
```

**优势**：卷积感受野天然处理序列平滑性；参数少；训练快。

### 3.2 Transformer 去噪器

2024 年起主流改为 Transformer（更易与 VLM 对接）：

```
将 a^k 的每个时间步作为一个 token
加入时间步 k 的 embedding
条件 c 通过 cross-attention 注入

架构:
  Self-attention on a^k tokens
  Cross-attention with obs tokens
  FFN

代表: 
  Diffusion Policy (Transformer version)
  GR00T N1 的 DiT action head
  RDT-1B / RDT-2
```

**优势**：与预训练 VLM 对接更自然；更容易 scale；长 chunk 支持更好。

### 3.3 DiT（Diffusion Transformer）

GR00T N1 采用 DiT 风格去噪器（源自 Sora 的技术路线）：

```
输入: noisy action tokens
条件: VLM latent + proprioception via adaLN-Zero

关键设计:
  - adaLN-Zero: 将条件信号通过 AdaLayerNorm 注入
  - 可 scale 到 1B+ 参数
  - 初始化为"identity mapping"（零初始化），训练更稳定
```

## 四、3D Diffusion Policy (DP3)

> Ze et al. (Shanghai Jiao Tong), "3D Diffusion Policy: Generalizable Visuomotor Policy Learning via Simple 3D Representations", 2024.

### 4.1 动机

原 Diffusion Policy 用 2D 图像作为观测。问题：
- 跨视角泛化差
- 对光照/背景敏感
- 难以精确定位 3D 物体

**DP3 的贡献**：用 **3D 点云**替代 2D 图像作为观测。

### 4.2 架构

```
输入: 场景点云 P ∈ R^{N × 3} (典型 N = 512-1024)
      可选: RGB 附加到每个点 → R^{N × 6}

点云编码器: 简化 PointNet (MLP + max pooling)
  → 紧凑特征 (e.g., 64-256 维)

其余部分与 Diffusion Policy 一致:
  条件 → U-Net / Transformer 去噪 → 动作 chunk
```

### 4.3 核心发现

- **10 条演示就够**：用 10 条演示训练 DP3 超过 100 条演示训练的 2D Diffusion Policy
- **仿真到真实迁移好**：3D 表示对视觉域差异不敏感
- **泛化能力强**：不同光照、背景、相机位姿下表现稳定

### 4.4 后续工作

- **Improved 3D Diffusion Policy (iDP3, 2024)**：改进点云编码
- **RDT 系列**：在 3D 输入上加入双臂与灵巧手适配
- **与 VLA 融合**：GR00T N1 等部分模型探索加入 3D 输入通道

## 五、实践细节

### 5.1 Action Chunking 与 Receding Horizon

```
训练: 预测 T = 16 步未来动作
推理:
  执行前 T_a = 8 步 (action horizon)
  然后重新规划下一个 chunk
  (类似 MPC 的 receding horizon)

参数建议:
  obs history: 2 帧
  prediction horizon T: 16 步
  action horizon T_a: 8 步
  推理频率: 每 T_a 步调用一次模型
```

### 5.2 归一化

**非常重要，常见踩坑点**：

```
动作归一化:
  每个动作维度计算训练集 min / max
  归一化到 [-1, 1]
  (扩散模型的高斯噪声 ~N(0,1), 动作需在同尺度)

观测归一化:
  图像: ImageNet mean/std
  关节状态: 训练集 mean/std
  点云: 中心化 + scale 到单位球

推理:
  模型输出归一化动作 → 反归一化 → 真实动作
```

遗漏归一化通常导致训练损失不收敛或推理动作失控。

### 5.3 去噪步数与 DDIM 加速

```
训练: 1000 个 diffusion 步 (DDPM schedule)
推理: DDIM 加速到 10-100 步

典型选择:
  离线任务 (精细操作): 100 步 DDIM
  实时任务 (10-30Hz): 10-20 步 DDIM
  极速 (100Hz+): Consistency Model / Flow Matching 替代

近期趋势:
  Flow Matching (π₀, 2024) 5-10 步即可
  逐渐取代 DDPM / DDIM
```

### 5.4 训练超参

```
# 典型 Diffusion Policy 训练配置
optimizer: AdamW
lr: 1e-4
batch size: 256
epochs: 500-3000
gradient clip: 1.0
ema decay: 0.995  # Exponential Moving Average on model weights

# 扩散过程
num_train_timesteps: 1000
num_inference_timesteps: 16 (DDIM)
beta_schedule: squaredcos_cap_v2
prediction_type: epsilon  # 预测噪声而非动作
```

### 5.5 EMA 权重

Diffusion Policy 训练时维护一份 **EMA 平均的模型权重**，推理时使用 EMA 权重而非最新权重。这显著提升推理稳定性，是几乎所有扩散模型的标准实践。

### 5.6 计算开销

```
参数量:
  U-Net 1D: 66M (小)
  Transformer: 100-500M
  DiT action head (GR00T N1): 1B

训练:
  单 A100, 500 条演示: ~1 天
  8× A100, 10K 演示: ~3-7 天

推理延迟 (单次):
  U-Net + 16 DDIM 步: ~30ms (消费级 GPU)
  + action chunking (执行 8 步): 有效决策频率 30-60Hz
```

## 六、ACT vs Diffusion Policy vs Flow Matching

| 维度 | ACT (2023) | Diffusion Policy (2023) | Flow Matching (π₀, 2024) |
|------|-----------|-------------------------|-------------------------|
| 多模态建模 | CVAE 隐变量 | 扩散过程 | ODE 流 |
| 推理步数 | 1 次前向 | 10-100 步 | 5-10 步 |
| 训练稳定性 | 需调 β | 非常稳定 | 非常稳定 |
| 动作质量 | 中 | 高 | 高 |
| 推理速度 | 最快 | 慢 | 中 |
| 与 VLM 对接 | Transformer 天然 | Transformer 可对接 | Transformer 天然 |
| 2026 地位 | 仍有应用 | 仍主流 | 主流、逐渐取代 Diffusion |

**行业演进**：Diffusion Policy (2023) → Flow Matching (π₀, 2024) → 未来可能 Consistency Model / One-step 生成。Flow Matching 本质是 Diffusion 的等价替代，推理步数更少。

## 八、常见失败模式

| 症状 | 可能原因 | 修复 |
|------|---------|------|
| 推理动作抖动 | 无 EMA / 归一化错误 | 启用 EMA、检查归一化 |
| 动作幅度饱和 | 归一化范围错误 | 用训练集 min/max 归一化 |
| 训练损失不降 | 学习率过高 / schedule 错 | 1e-4 + cosine / warmup |
| 推理太慢 | DDPM 1000 步 | DDIM 10-20 步 or Flow Matching |
| 策略死板不反应 | action horizon 过长 | 减小 T_a (e.g., 8 → 4) |
| 陷入 OOD | 观测归一化漂移 | 部署时验证归一化统计 |
