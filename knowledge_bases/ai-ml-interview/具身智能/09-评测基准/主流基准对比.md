# 主流基准对比（2023-2026）

> 系统梳理具身智能评测基准的两代演进：2023-2024 经典（仿真为主）与 2025-2026 新生（真机、跨本体、人类评分）。文末给出大表对比与开放问题。

## 一、2023-2024 经典基准

### 1.1 SIMPLER（2024）

**全称**：Simulated Manipulation Policy Evaluation for Real Robots
**机构**：UC Berkeley / Google / Stanford 合作（Xuanlin Li, Kyle Hsu et al.）
**类型**：仿真，但做过"真实性校准"

```
核心思路:
  - 在 SAPIEN / ManiSkill 仿真中复现 RT-1 / RT-2 / OpenVLA 训练任务
  - 系统地对比仿真结果 vs 真实机器人结果，做校准
  - 提供"校准后的仿真成功率"作为真实成功率的 proxy

支持模型: RT-1, RT-2, OpenVLA, Octo, π₀ 等
任务集: Google Robot / Bridge V2 风格的抓放任务
价值: 快速评测 VLA 模型，不必每次真机测试
局限: 仅覆盖 RT-1/Bridge 风格任务，泛化有限
```

**行业地位**：2024-2025 年 VLA 论文最常报告的基准之一；局限是"校准系数"只对训练分布内有效。

### 1.2 LIBERO（2023）

**全称**：LIfelong Robot learning BEnchmark
**机构**：UT Austin (Yifeng Zhu et al.)
**类型**：仿真（基于 RoboSuite）

```
任务规模: 130 个操控任务，分 4 个 suite
  - LIBERO-Spatial (空间关系)
  - LIBERO-Object (物体泛化)
  - LIBERO-Goal (目标泛化)
  - LIBERO-Long (长 horizon, 10 任务)

每任务: 50 条遥操作示范（固定）
平台: Franka Panda
语言指令: 每任务配置自然语言
评价: 任务成功率
```

**行业地位**：2024-2025 年 VLA 论文的主力基准之一，尤其用于测试**持续学习**和**多任务**能力。

### 1.3 CALVIN（2022）

**全称**：Composing Actions from Language and Vision
**机构**：Freiburg / ETH
**类型**：仿真（PyBullet）

```
任务: 34 个桌面操控子任务的长 horizon 组合
场景: 4 个环境（A/B/C/D）用于测试环境泛化
      -> A→D：在 A 训练，D 测试
特色: 长序列语言指令（5 个步骤连续完成）
评价: 单步成功率 + 连续成功率（完成前 N 步的概率）
```

**行业地位**：CALVIN ABCD→D 是长 horizon 语言条件操控的最常用基准。2024 年 OpenVLA / RDT / CogACT 均报告。

### 1.4 Meta-World（2019）

**机构**：UC Berkeley (Tianhe Yu et al.)
**类型**：仿真（MuJoCo）

```
任务规模: 50 个操控任务（MT50）或 10 任务子集（MT10）
平台: Sawyer 机械臂
评价: 多任务成功率（ML1/MT10/MT50 不同 split）
用途: 多任务 RL / 模仿学习算法的标准测试
```

**行业地位**：RL 时代的经典基准，VLA 时代使用减少但仍作为多任务对照。

### 1.5 RoboSuite（2020+）

**机构**：UT Austin + Stanford（Yuke Zhu et al.）
**类型**：仿真框架（MuJoCo 上的上层）

```
定位: 不是单一基准，而是仿真任务框架
内置任务: Lift, Stack, NutAssembly, Door, Wipe, PickPlace, ...
可扩展: LIBERO 等基于 RoboSuite 构建
特色: 多机器人支持（Panda / Sawyer / Jaco / IIWA），双臂
```

**行业地位**：仿真框架，不是"分数板"，但是学术社区操控基线的重要基础。

### 1.6 其他 2023-2024 基准

| 基准 | 定位 | 备注 |
|-----|------|------|
| **RoboCasa**（2024） | 厨房场景大规模仿真 | 25 个场景 × 100+ 物体 |
| **BEHAVIOR-1K**（Stanford） | 1000 个家庭任务 | 偏长 horizon，采用 OmniGibson |
| **Habitat 2.0 / 3.0**（Meta） | 导航 + 移动操控 | 多智能体社交导航 |
| **RLBench**（ICL） | 100 个任务仿真（Coppelia） | 偏 RL |
| **Franka-Kitchen / D4RL Kitchen** | 厨房多任务 | 早期经典 |
| **Bridge V2**（UC Berkeley） | 真机数据集 + 基准 | 60k+ 轨迹，WidowX 机械臂 |

## 二、2025-2026 新生基准

### 2.1 PARTNR（Meta, 2024.11）

**全称**：Planning And Reasoning Tasks in humaN-Robot collaboration
**机构**：Meta FAIR

```
核心定位: 人-机/机-机 协作与规划评测
任务: 100,000 个家庭协作任务（LLM 生成 + 人工验证）
场景: 基于 Habitat 3.0（家庭环境）
特色:
  - 多智能体（1+ 机器人 + 1+ 模拟人类）
  - 自然语言协同指令
  - 长 horizon（数十步）
  - 包含约束 / 依赖 / 同步逻辑
评价:
  - 任务完成率
  - 协调效率（人机等待时间）
  - 对话轮数
```

**行业地位**：2025 年家庭协作方向最常引用的基准；NeurIPS D&B Track 发表。

### 2.2 RoboArena（2024）

**机构**：多机构联合（含 UC Berkeley, Stanford, Deepmind 等）
**定位**：跨本体 + 人类评分

```
核心思路:
  - 不固定任务脚本，而是开放场景下的"自由测试"
  - 评测员观察机器人执行，打分（像 Chatbot Arena）
  - 多机器人参与：Franka, UR5, ARX, ALOHA, GR-1, H1 ...
  - 多模型参与：π₀, OpenVLA, GR00T N1, RDT-2, ...

评价:
  - Elo 排名（两两对比，众包打分）
  - 维度: 成功率、流畅度、安全性、响应速度
价值:
  - 接近"机器人 Chatbot Arena"
  - 避免过拟合单一固定任务集
  - 引入人类偏好维度
局限:
  - 评测成本高（真机 + 人工）
  - 主观性强，需大样本
```

**行业地位**：2025-2026 年最受关注的跨本体真机基准；对标 LM Arena。

### 2.3 RoboSpatial-VLM（2025）

**机构**：多所（含 Meta、UCSD）
**定位**：空间理解与几何推理评测

```
核心思路:
  - 评测 VLM / VLA 的空间理解能力
  - 任务形式: "把盘子放在杯子左边 10cm"、"从哪个方向抓取不会倾倒"
  - 不要求执行成功，重点是空间理解的正确性

评测维度:
  - 物体相对位置判断
  - 可供性（affordance）预测
  - 抓取点推理
  - 物理约束理解（稳定性、重量）
  
数据: 真实 + 合成混合；开放 leaderboard
```

**行业地位**：VLM/VLA 空间理解能力的早期标准化评测。Gemini Robotics-ER 等"推理型 VLA"的主要展示基准。

### 2.4 GR-1 / H1 Benchmark（NVIDIA，2025）

**定位**：人形机器人专用基准

```
平台: Fourier GR-1、Unitree H1
来源: 配套 GR00T N1 / N1.5 的训练与评测

任务:
  - 人形全身操控：上肢抓放 + 躯干稳定
  - Locomotion + Manipulation 结合
  - 双手协作

评价:
  - 任务成功率
  - Loco-Mani 同时评估（不能只看抓取忽略站立稳定性）

特色:
  - 与 Isaac Lab 仿真联动
  - 真机 + 仿真混合评测
  - NVIDIA 作为"平台方"推动统一基准
```

**行业地位**：人形基础模型（GR00T N1、Helix、π₀ 人形版）对标的实际基准之一。

### 2.5 AgiBot World Eval（智元，2025）

**机构**：智元机器人（AgiBot）
**定位**：大规模真机场景评测集

```
数据 / 场景:
  - 基于 AgiBot World 100 万+轨迹数据集
  - 覆盖工业、家庭、医疗、商业 4 大场景
  - 5000+ 物体、1000+ 任务类型

评价:
  - 真机场景回放 + 新场景泛化
  - 标准化评分流程
  - 对开放 VLA（GO-1, RDT-2, π₀, GR00T N1）都做过对照

特色:
  - 中文指令评测（国际基准普遍英文）
  - 中国家庭/工厂场景（不同于美国数据集）
  - 持续扩充
```

**行业地位**：国内最重要的真机基准；与 AgiBot World 数据集配套。

### 2.6 其他 2025-2026 新生基准

| 基准 | 机构 | 特色 |
|-----|------|------|
| **SimplerEnv-v2**（2025） | SIMPLER 团队 | SIMPLER 升级，扩展到更多 VLA |
| **LIBERO-Plus**（2025） | UT Austin | LIBERO 扩展到更多任务 + 人形 |
| **HOMEROBOT**（Meta 2022+延续） | Meta | 家庭长 horizon |
| **RoboTHOR / ProcTHOR** | AI2 | 程序化生成场景导航 |
| **BEHAVIOR Challenge 2024/25** | Stanford | 大规模家庭任务挑战赛 |
| **OVMM / HomeRobot OVMM** | Meta | 开放词汇移动操控 |

## 三、评测指标的演进

### 3.1 从单一到多维

```
Gen 1 (2019-2022):  任务成功率（二元）
  → Meta-World MT50 成功率, RoboSuite Lift 成功率

Gen 2 (2022-2024):  多任务平均 + 长 horizon
  → CALVIN 连续 5 步成功率, LIBERO 130 任务平均

Gen 3 (2024-2025):  跨本体 + 开放场景
  → RoboArena Elo, π₀ 在 8+ 本体上的综合表现

Gen 4 (2025-2026):  人类偏好 + 长尾鲁棒性
  → 真人打分（流畅度/安全/自然度）
  → OOD 物体/场景成功率
```

### 3.2 新兴指标

| 指标 | 定义 | 代表基准 |
|------|-----|---------|
| **连续成功率（Sequential Success Rate）** | 连续完成 N 步子任务的概率 | CALVIN, LIBERO-Long |
| **跨本体迁移成功率** | 一个模型在新本体上 zero-shot / few-shot | Open X-Embodiment, RoboArena |
| **人类偏好（Elo / Preference）** | 众包/专家两两对比打分 | RoboArena, PARTNR |
| **长尾鲁棒性（Tail Robustness）** | 训练分布外物体/场景/光照下的成功率下降 | π₀.5 paper, 内部真机测试 |
| **安全指标** | 碰撞次数、误伤风险、紧急停机次数 | 真机部署 |
| **响应延迟** | 从指令到首次动作的时间 | Helix 报告，RoboArena |
| **协作效率** | 多智能体任务中的等待/冲突时间 | PARTNR |

### 3.3 Sim2Real 校准系数

SIMPLER 引入了一个有用的概念：**仿真-真实校准系数 α**。

```
  SR_real ≈ α · SR_sim   （α 通常 0.4-0.9，视任务而定）
  
  α 由少量真机 + 对应仿真任务标定
  新模型只需报告仿真结果 + 预测 α·SR_sim 作为真实结果估计
```

2025-2026 年这一思路被扩展到跨仿真平台（Isaac Sim vs SAPIEN vs PyBullet）的校准。

## 六、开放问题

### 6.1 仿真-真实差距

- 光照、纹理、接触力学、关节摩擦仍存在系统性偏差
- 柔软物（布料、液体）仿真精度低
- **世界模型**（Cosmos、V-JEPA、Genie）有望成为"高保真第三方仿真"，但尚未标准化

### 6.2 评测可重复性

- 真机评测：硬件批次、环境光、操作员差异带来结果波动
- 不同团队的 LIBERO 报告成功率方差可达 10-20%
- 需要**标准化协议**：录像、日志、硬件 ID 追踪

### 6.3 评测与产品价值的错位

- 学术基准偏"成功率"，但产品关心"可靠性 + 用户满意度 + 长尾"
- **RoboArena**、**AgiBot World Eval** 是走向产品评测的尝试，但仍有主观性挑战

### 6.4 跨本体评测的公平性

- 不同本体的任务可行集不同（一个 6 DoF 臂与 7 DoF 臂不能直接比）
- 如何设计"本体无关"的任务？潜在动作空间（Latent Action Space）是研究方向之一

### 6.5 长尾与安全

- 当前基准覆盖 < 1% 的真实长尾情景
- 安全事件（碰撞、跌倒、误伤）的评测框架仍不成熟
- 监管需求（ISO 13482 等）将推动安全评测标准化

### 6.6 "机器人的 MMLU"是什么？

- LLM 有 MMLU、GPQA、ARC-AGI 等作为能力基准
- 具身智能尚无等价的"通用能力测试"
- RoboArena 最接近但仍是开放问题

## 七、给研究者的建议

1. **不要只报告一个基准**：LIBERO + SIMPLER + 至少一个真机是基本组合。
2. **关注分布外（OOD）**：训练集外物体/场景/光照的成功率比平均分更有说服力。
3. **报告方差 / 标准差**：不要只给单次运行的成功率。
4. **与多个 baseline 对比**：π₀、OpenVLA、GR00T N1、Diffusion Policy 至少对比 2-3 个。
5. **关注可复现性**：开源模型权重 + 评测脚本 + 硬件配置。
