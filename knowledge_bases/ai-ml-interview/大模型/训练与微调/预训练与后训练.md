# 预训练与后训练

大模型的训练流程分为两大阶段：**预训练 (Pre-training)** 和 **后训练 (Post-training)** 。

## 一、预训练 (Pre-training)

### 目标

从海量无标注文本中学习语言的通用表示，获得世界知识和语言能力。

### 核心方法

- **自回归语言建模 (Causal LM)**：给定前文，预测下一个 token
  - 损失函数：交叉熵，$L = -\sum_t \log P(x_t | x_{\lt t})$
  - 代表：GPT 系列、LLaMA、Qwen、DeepSeek
- **掩码语言建模 (Masked LM)**：随机遮盖部分 token，预测被遮盖的内容
  - 代表：BERT（但主流 LLM 已不使用此范式）

### 数据

- 规模：数万亿 token（如 LLaMA 3 用了 15T token）
- 来源：互联网网页、书籍、代码、学术论文、百科等
- 质量控制：去重、过滤低质量/有害内容、语言识别

### 训练规模

- 参数量：数十亿到数千亿
- 算力：数千到数万张 GPU，训练数周到数月
- 遵循 **Scaling Laws**：模型性能随参数量、数据量、算力的增长而可预测地提升

### 预训练产出

预训练完成后得到的是一个 **Base Model（基座模型）**：
- 具备强大的语言理解和生成能力
- 拥有广泛的世界知识
- **但不擅长遵循指令**，行为不可控，可能输出有害内容
- 倾向于"续写"而非"回答问题"

## 二、后训练 (Post-training)

### 目标

将基座模型转变为一个**有用、安全、符合人类偏好**的助手模型。

### 核心阶段

后训练通常包含以下阶段（不同团队的流程有差异）：

**Base Model → SFT → RLHF/DPO/PPO/GRPO → Chat Model**

1. **SFT** (有监督微调): 学会遵循指令、格式化输出
2. **RLHF/DPO/GRPO** (强化学习对齐): 学会符合人类偏好的回答（多种技术路线可选）

> 详细流程图见 [RLHF与PPO详解](RLHF与PPO详解.md) 中的 InstructGPT 三阶段图。

### 各阶段对比

| 维度 | 预训练 | SFT | RLHF/DPO/GRPO |
|------|--------|-----|----------------|
| 数据量 | 万亿 token | 数万~数十万条 | 数万~数十万条偏好对 |
| 数据类型 | 无标注文本 | (指令, 回答) 对 | (指令, 好回答, 差回答) 三元组 |
| 学习目标 | 预测下一个 token | 模仿高质量回答 | 优化人类偏好 |
| 算力消耗 | 极高 | 中等 | 中等~高 |
| 影响 | 知识和能力的上限 | 激活能力、学会格式 | 提升质量、安全对齐 |

### 各阶段分别获得什么能力？

| 能力 | 主要训练阶段 | 说明 |
|------|-------------|------|
| **对话/指令遵循** | SFT | 大量 (指令, 回答) 对，学会问答格式 |
| **工具调用 (Tool Use)** | SFT + RL | SFT 学输出 JSON 调用格式；RL 学**判断**——该不该调、调哪个 |
| **RAG（基于上下文回答）** | 不需要专门训练 | RAG 是工程侧把检索结果塞进 prompt，模型只需"会读上下文"——SFT 自然就会 |
| **安全/拒绝** | SFT + RLHF | SFT 给拒绝样本，RLHF 强化安全边界 |
| **推理/CoT** | RL（如 GRPO） | DeepSeek-R1 用 RL 让模型学会长链推理 |
| **代码生成** | SFT（+ 预训练） | 预训练语料含代码，SFT 阶段用代码任务数据激活 |

> **注意**：SFT 阶段通常**只训一次**，不是分能力多次训练。所有类型的训练数据（对话、工具调用、代码、数学、安全等）**混合在一起**一次训练完成。RL 阶段则在 SFT 之后进行，进一步优化判断力和输出质量。

### 关键认知

1. **预训练决定能力上限**：后训练无法教会模型它在预训练中没有学到的知识
2. **SFT 激活能力**：基座模型其实"什么都会"，SFT 只是教它用正确的方式表达
3. **对齐税 (Alignment Tax)**：过度的后训练可能导致模型能力下降，需要平衡
4. **后训练越来越重要**：OpenAI o1/o3、DeepSeek-R1 等模型表明，精心设计的后训练（尤其是 RL）可以显著提升推理能力

## 三、完整训练流水线示例

以 LLaMA 3 / Qwen 2.5 类模型为例：

```
1. 预训练 (Pre-training)
   └── 海量文本 → Base Model

2. 有监督微调 (SFT)
   └── 高质量指令数据 → SFT Model

3. 奖励模型训练 (Reward Model Training)  [仅 RLHF/PPO 路线需要]
   └── 人类偏好标注数据 → Reward Model

4. 强化学习 / 偏好优化
   ├── PPO 路线：用 RM 打分 + PPO 优化
   ├── DPO 路线：直接用偏好数据优化，跳过 RM
   └── GRPO 路线：组采样 + 相对奖励，无需 Critic

5. 后续迭代
   └── 继续收集数据 → 迭代优化
```
