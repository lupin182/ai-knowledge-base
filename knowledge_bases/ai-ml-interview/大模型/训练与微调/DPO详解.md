# DPO 详解 (Direct Preference Optimization)

DPO 由 Stanford 团队于 2023 年提出，核心思想是**跳过奖励模型，直接用偏好数据优化语言模型**。

## 一、动机：简化 RLHF

RLHF (PPO) 的痛点：
1. 需要单独训练奖励模型
2. PPO 训练不稳定，超参数敏感
3. 同时加载 4 个模型，显存巨大

DPO 的核心发现：**RLHF 的优化目标存在解析解**，可以直接推导出一个监督学习的损失函数，完全绕开 RL。

## 二、数学推导（关键洞察）

### RLHF 的优化目标

$$\max_\pi \; \mathbb{E}_{x, y \sim \pi} [r(x, y)] - \beta \cdot \text{KL}(\pi \| \pi_{ref})$$

### 该目标的最优解

经过推导，最优策略 $\pi^*$ 满足：

$$\pi^*(y|x) = \frac{1}{Z(x)} \pi_{ref}(y|x) \cdot \exp\left(\frac{r(x,y)}{\beta}\right)$$

### 反解出隐式奖励

从上式可以反解出奖励函数：

$$r(x, y) = \beta \log \frac{\pi^*(y|x)}{\pi_{ref}(y|x)} + \beta \log Z(x)$$

> 关键洞察：**奖励函数可以用策略模型和参考模型的对数概率比来表示**，不需要单独的奖励模型。

### 代入 Bradley-Terry 偏好模型

将隐式奖励代入偏好概率：

$$P(y_w \succ y_l | x) = \sigma\left(\beta \log \frac{\pi_\theta(y_w|x)}{\pi_{ref}(y_w|x)} - \beta \log \frac{\pi_\theta(y_l|x)}{\pi_{ref}(y_l|x)}\right)$$

> $Z(x)$ 项在做差时互相抵消了。

### 最终 DPO 损失函数

$$L_{DPO} = -\mathbb{E}_{(x, y_w, y_l)} \left[ \log \sigma \left( \beta \log \frac{\pi_\theta(y_w|x)}{\pi_{ref}(y_w|x)} - \beta \log \frac{\pi_\theta(y_l|x)}{\pi_{ref}(y_l|x)} \right) \right]$$

## 三、直观理解

DPO 做的事情：
- **增大**模型对 chosen 回答的概率（相对于参考模型）
- **减小**模型对 rejected 回答的概率（相对于参考模型）
- 通过 $\beta$ 控制偏离参考模型的幅度

```
                  chosen 回答
                  概率 ↑ 提升
                 ╱
    模型 ──────
                 ╲
                  rejected 回答
                  概率 ↓ 降低
```

## 四、训练流程

```
输入：偏好数据集 D = {(x, y_w, y_l)}, 参考模型 π_ref (冻结)

1. 对每条数据，计算四个对数概率：
   - log π_θ(y_w | x)    ← 当前模型对 chosen 的打分
   - log π_θ(y_l | x)    ← 当前模型对 rejected 的打分
   - log π_ref(y_w | x)  ← 参考模型对 chosen 的打分（预计算，不参与梯度）
   - log π_ref(y_l | x)  ← 参考模型对 rejected 的打分（预计算）

2. 计算 DPO loss，反向传播

3. 只更新 π_θ 的参数
```

### 只需 2 个模型

```
┌──────────────────┐    ┌──────────────────┐
│ Policy Model     │    │ Reference Model  │
│ π_θ（正在训练）   │    │ π_ref（冻结）     │
└──────────────────┘    └──────────────────┘
```

对比 PPO 的 4 个模型，显存减半。

## 五、超参数

| 参数 | 典型值 | 说明 |
|------|--------|------|
| $\beta$ | 0.1 ~ 0.5 | KL 约束强度。越大越保守，越小偏好优化越激进 |
| 学习率 | 1e-6 ~ 5e-6 | 比 SFT 更低，避免过拟合偏好数据 |
| Epoch | 1 ~ 3 | 偏好数据量小，多轮容易过拟合 |

## 六、DPO 的优缺点

### 优点

| 优点 | 说明 |
|------|------|
| **简单** | 不需要 RL，就是监督学习，代码量极少 |
| **稳定** | 没有 PPO 的训练不稳定问题 |
| **省显存** | 只需要 2 个模型，不需要 Critic 和 RM |
| **易复现** | 超参数少，调参成本低 |

### 缺点

| 缺点 | 说明 |
|------|------|
| **离线学习** | 只能用预先收集的偏好数据，不能在线探索 |
| **分布偏移** | 训练过程中模型在变，但偏好数据是固定的 |
| **难以超越数据** | 不像 RL 那样可以通过探索发现更好的策略 |
| **对数据质量敏感** | 偏好标注错误会直接误导优化方向 |

## 七、DPO 的变体

| 变体 | 改进 |
|------|------|
| **IPO** | 添加正则项，减少过拟合 |
| **KTO** | 不需要偏好对，只需要 "好/坏" 的二元标签 |
| **ORPO** | 将 SFT 和偏好优化合并为一步 |
| **SimPO** | 用平均对数概率代替求和，不需要参考模型 |
| **Iterative DPO** | 用当前模型生成新数据，迭代训练，缓解分布偏移 |
