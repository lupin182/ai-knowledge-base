# 知识蒸馏详解

> 更新日期: 2026-04-08

知识蒸馏（Knowledge Distillation）是模型压缩和知识迁移的核心技术，贯穿 CV、NLP、多模态等所有深度学习领域。

数学基础参见: [KL 散度](../../机器学习基础/KL散度.md)

---

## 一、核心思想

### 1.1 为什么需要蒸馏？

```
大模型 (如 GPT-4, ViT-Giant):
  ✓ 性能好
  ✗ 推理慢、显存大、部署成本高

直接训练小模型:
  ✓ 推理快
  ✗ 直接用 hard label 训练，性能显著差于大模型

知识蒸馏:
  用大模型（教师）指导小模型（学生）训练
  学生模型: 推理效率接近小模型，性能接近大模型
  → "站在巨人的肩膀上学习"
```

### 1.2 Hinton 的经典论文 (2015)

Hinton et al., "Distilling the Knowledge in a Neural Network" 奠定了整个领域。

**核心 Insight：模型的"暗知识"藏在 softmax 的输出分布中，而非 hard label。**

```
分类任务举例 (图片是一只猫):

Hard label:    [1, 0, 0, 0, 0]  (猫, 狗, 虎, 豹, 兔)
               → 只告诉模型"是猫"，没有其他信息

Teacher softmax (T=1):  [0.92, 0.03, 0.04, 0.008, 0.002]
               → 告诉模型"大概率是猫，跟虎和狗有点像，跟兔子完全不像"
               → 类间相似性关系 = 暗知识 (dark knowledge)

Teacher softmax (T=5):  [0.56, 0.12, 0.18, 0.09, 0.05]
               → 温度升高，分布更平滑
               → 暗知识被"放大"：虎(0.18) > 狗(0.12) > 豹(0.09) > 兔(0.05)
               → 类间的相对关系更清晰
```

### 1.3 标准蒸馏框架

```
教师模型 (frozen)
    │
    ├── logits_T → softmax(logits_T / T) → 软标签 P
    │                                         │
    │                                    KL(P || Q)  ← 蒸馏损失
    │                                         │
学生模型 (trainable)                          │
    │                                         │
    ├── logits_S → softmax(logits_S / T) → 学生分布 Q
    │
    └── logits_S → softmax(logits_S / 1) → 预测 → CE(y, pred) ← 标准损失

总损失:
  L = α · T² · KL(P || Q) + (1-α) · CE(y, pred)
```

**关键超参数**:

| 参数 | 含义 | 典型值 | 选择依据 |
|------|------|--------|---------|
| T (温度) | 控制 softmax 平滑程度 | 2~20 | 类别多/教师置信度高 → T 大 |
| α (平衡系数) | 蒸馏损失的权重 | 0.5~0.9 | 教师越强 → α 越大 |
| T² 补偿 | 抵消温度缩放导致的梯度缩小 | 自动计算 | 数学推导所得 |

**T² 补偿的推导**（面试常问）：

```
softmax 在高温下的梯度:
  ∂ softmax(z/T) / ∂z = softmax(z/T) · (1 - softmax(z/T)) / T

→ 梯度被缩小了 T 倍
→ KL loss 中两个 softmax(·/T) 各贡献一个 1/T
→ 总梯度缩小 T² 倍
→ 乘 T² 补偿回来，使蒸馏梯度和标准 CE 梯度在同一量级
```

## 二、蒸馏方法分类

按"教师传递给学生什么知识"分类：

```
知识蒸馏
├── Response-based (输出层蒸馏) ← 最经典
│   └── 对齐教师和学生的最终输出分布
│
├── Feature-based (特征层蒸馏)
│   └── 对齐中间层的特征表示
│
├── Relation-based (关系蒸馏)
│   └── 对齐样本之间的关系结构
│
└── 特殊形式
    ├── Self-distillation (自蒸馏)
    ├── Online distillation (在线互蒸馏)
    └── Multi-teacher distillation (多教师蒸馏)
```

### 2.1 Response-based Distillation（输出层蒸馏）

就是 Hinton 的经典方法，对齐教师和学生的**最终输出分布**：

```
教师 logits → soft label P
学生 logits → soft prediction Q
Loss = KL(P || Q)

适用: 分类、生成（对齐 token 概率分布）
优势: 实现最简单，效果稳定
局限: 只用了教师的最终输出，丢弃了中间层信息
```

### 2.2 Feature-based Distillation（特征层蒸馏）

不只看最终输出，还对齐教师和学生的**中间层特征**：

```
教师网络:  input → [Layer1] → F_T1 → [Layer2] → F_T2 → ... → output
                      ↓                 ↓
                   对齐损失           对齐损失
                      ↑                 ↑
学生网络:  input → [Layer1] → F_S1 → [Layer2] → F_S2 → ... → output
```

**核心问题：教师和学生的中间层维度通常不同，怎么对齐？**

```
解决方案: 加投影层 (Projection Layer)

学生特征 F_S ∈ R^(d_s)  →  线性投影 W ∈ R^(d_t × d_s)  →  F̂_S ∈ R^(d_t)

L_feature = MSE(F̂_S, F_T)  或  L1(F̂_S, F_T)

投影层 W 和学生网络一起训练
```

**代表工作**：

| 方法 | 对齐什么 | 特点 |
|------|---------|------|
| FitNets (2015) | 中间层激活值 | 开山之作，用"hint layer"指导 |
| Attention Transfer (2017) | 注意力图 | 对齐空间注意力分布，更轻量 |
| PKT (2018) | 特征的概率分布 | 用核密度估计对齐特征分布 |
| CRD (2020) | 对比学习目标 | 用 contrastive loss 对齐特征 |

**FitNets 的详细机制**：

```
教师: 深而宽 (如 ResNet-152)
学生: 浅而窄 (如 ResNet-18)

选择教师的第 k 层为 "hint layer"
选择学生的第 j 层为 "guided layer"

训练分两阶段:
  阶段 1 (Hint Training):
    冻结学生的 layer j 之后的层
    只用 hint loss 训练 layer 1~j + 投影层
    目标: 让学生中间层学到教师中间层的表示

  阶段 2 (Standard KD):
    用标准蒸馏 loss 端到端训练整个学生网络
```

### 2.3 Relation-based Distillation（关系蒸馏）

不对齐个体特征，而是对齐**样本之间的关系结构**：

```
核心思想:
  一个 mini-batch 有 N 个样本
  教师对这 N 个样本的特征形成某种"关系结构"
  让学生也学到相同的关系结构

具体做法 (RKD, Relational Knowledge Distillation):

  Distance-wise:
    教师: d_T(i,j) = ||F_T(x_i) - F_T(x_j)||  (样本 i,j 在教师特征空间的距离)
    学生: d_S(i,j) = ||F_S(x_i) - F_S(x_j)||
    Loss: Huber(d_S - d_T)  → 让学生保持样本间的距离关系

  Angle-wise:
    教师: 三元组 (i,j,k) 在教师特征空间形成的角度
    学生: 同样的三元组在学生特征空间形成的角度
    Loss: 对齐角度 → 让学生保持样本间的角度关系

优势:
  - 不要求特征维度匹配（只比较关系，不比较绝对值）
  - 更关注"结构化知识"而非个体知识
  - 对教师和学生架构差异的鲁棒性更好

局限:
  - 计算量随 batch size 增大（需要计算样本对/三元组）
  - 关系是二阶信息，可能不如直接对齐特征信息丰富
```

## 三、自蒸馏 (Self-Distillation)

### 3.1 概念

没有独立的教师模型——**模型自己教自己**。

```
传统蒸馏:  教师 (大模型) → 学生 (小模型)
自蒸馏:    模型自身的某个版本/部分 → 模型自身

几种形式:
1. 时间维度: 上一个 epoch 的模型 → 当前 epoch 的模型
2. 深度维度: 深层特征 → 浅层特征 (同一网络内)
3. 动量维度: EMA 版本 → 当前版本 (DINO 的做法)
```

### 3.2 DINO 的自蒸馏（视觉基础模型核心方法）⭐

DINO (Self-**DI**stillation with **NO** labels) 是自蒸馏在视觉领域最成功的应用。

```
架构:
  Student 网络: ViT, 参数 θ_s
  Teacher 网络: ViT, 参数 θ_t (Student 的 EMA 版本)

  θ_t ← m · θ_t + (1-m) · θ_s   (每步更新，m=0.996~0.999)

训练流程:
  输入图像 x
    │
    ├── 随机增强 1 (全局 crop) → view_1
    ├── 随机增强 2 (全局 crop) → view_2
    ├── 随机增强 3 (局部 crop) → view_3
    └── ... (多个局部 crop)
    
  Teacher 只看全局 crop → P_t = softmax(teacher(view_1) / τ_t)
  Student 看所有 crop  → P_s = softmax(student(view_k) / τ_s)
  
  Loss = Σ CE(P_t, P_s)   (交叉熵，不是 KL)
  
  只更新 Student 参数 (梯度反传)
  Teacher 通过 EMA 被动更新

为什么 EMA Teacher 有效？
  - EMA 是 Student 历史参数的加权平均 → 更稳定、更平滑
  - 提供了比 Student 当前状态更"可靠"的监督信号
  - 类似于 temporal ensemble → 隐式地集成了多个 checkpoint

防止模式坍塌 (所有输出变成同一个向量):
  - Teacher 使用 centering: 减去所有样本的均值
  - Teacher 使用更低的温度 τ_t (更 sharp 的分布)
  - 这两个技巧是 DINO 成功的关键
```

### 3.3 Born-Again Networks

```
思路: 训练完一个模型后，用它当教师，训练一个同样大小的学生

Round 1: 随机初始化 → 训练 → Model_1
Round 2: Model_1 当教师 → 训练 → Model_2 (同样架构)
Round 3: Model_2 当教师 → 训练 → Model_3

反直觉的发现: Model_2 > Model_1, Model_3 > Model_2
即使学生和教师架构完全一样，蒸馏仍然有效！

解释:
  软标签提供了比 hard label 更丰富的训练信号
  → 相当于"更好的标签" → 训练出更好的模型
  → 这进一步验证了暗知识的价值
```

## 四、在线蒸馏与多教师蒸馏

### 4.1 在线互蒸馏 (Online / Mutual Distillation)

```
传统蒸馏: 先训好教师 → 再训学生 (两阶段，离线)
在线蒸馏: 教师和学生同时训练，互相学习

Deep Mutual Learning (2018):
  模型 A 和模型 B 同时训练
  A 的软标签监督 B，B 的软标签监督 A

  L_A = CE(y, A(x)) + KL(B(x) || A(x))
  L_B = CE(y, B(x)) + KL(A(x) || B(x))

  两个模型都在进步，形成"良性竞争"

优势:
  - 不需要预训练大教师模型
  - 两个模型都能受益
  - 可扩展到多个模型互蒸馏

适用: 资源有限、无法获得预训练大模型的场景
```

### 4.2 多教师蒸馏

```
多个教师模型联合指导一个学生:

  Teacher_1 → soft label P_1 ─┐
  Teacher_2 → soft label P_2 ──┼→ 聚合 → P_ensemble → 指导 Student
  Teacher_3 → soft label P_3 ─┘

聚合方式:
  - 简单平均: P = (P_1 + P_2 + P_3) / 3
  - 加权平均: 按各教师在验证集上的表现加权
  - 学习权重: 用可训练的 gate network 动态选择

优势: 多教师的知识互补（如一个擅长细粒度，一个擅长全局）
应用: 模型集成的知识压缩到单一模型
```

## 五、蒸馏在 CV 中的应用

### 5.1 DeiT: ViT 的蒸馏训练 ⭐

DeiT (Data-efficient Image Transformer) 证明了蒸馏对 ViT 训练至关重要。

```
背景:
  原始 ViT 需要 JFT-300M (3 亿张图) 才能训好
  DeiT 的目标: 只用 ImageNet-1K (128 万张图) 训好 ViT

核心创新: Distillation Token

标准 ViT:
  [CLS] [patch_1] [patch_2] ... [patch_N] → Transformer → [CLS] → 分类头

DeiT:
  [CLS] [DIST] [patch_1] [patch_2] ... [patch_N] → Transformer
                                                       │
                                         ┌──────────────┤
                                         ↓              ↓
                                       [CLS]         [DIST]
                                         ↓              ↓
                                      分类头          蒸馏头
                                         ↓              ↓
                                    CE(y, pred)   CE(y_teacher, pred_dist)

  [DIST] 是新增的可学习 token
  它通过 self-attention 与所有 patch token 交互
  专门负责接收教师知识

  教师: RegNet (CNN) — 不是 ViT！
  → CNN 教师和 ViT 学生的归纳偏置互补
  → CNN 擅长的局部特征 + ViT 擅长的全局关系 = 最佳组合

结果:
  DeiT-B (86M) 在 ImageNet 上达到 83.4% top-1
  无需 JFT 预训练，只用 ImageNet-1K + 蒸馏
  训练成本降低数十倍
```

**为什么 CNN 教 ViT 比 ViT 教 ViT 效果更好？**

```
CNN 的归纳偏置 (locality, translation equivariance):
  → CNN 天然擅长捕捉局部纹理和边缘
  → 这恰好是 ViT 在数据不足时最欠缺的

ViT 教 ViT:
  → 教师和学生有相同的"盲点"
  → 暗知识的互补性不强

CNN 教 ViT:
  → 把 CNN 的局部感知能力"传授"给 ViT
  → ViT 同时保留自身的全局建模能力
  → 互补 = 最优
```

### 5.2 Diffusion 模型的蒸馏加速

Diffusion 模型推理需要几十到上百步采样，蒸馏是核心加速手段。

```
Progressive Distillation (2022):
  教师: N 步 DDPM
  学生: N/2 步 → 再蒸馏 → N/4 步 → ... → 最终 1~4 步

  每轮蒸馏:
    教师走 2 步: x_t → x_{t-1} → x_{t-2}
    学生走 1 步: x_t → x̂_{t-2}
    Loss: ||x̂_{t-2} - x_{t-2}||²

    学生用一步模拟教师两步的效果
    → 步数每轮减半

Consistency Distillation (2023, Consistency Models):
  更激进: 直接蒸馏出 1-step 生成模型
  
  核心: ODE 轨迹上的任意点都应该映射到同一个终点
  教师: 预训练的 diffusion model 定义了 ODE 轨迹
  学生: 学习"从轨迹上任意点直接跳到终点"

  f_θ(x_t, t) ≈ f_θ(x_{t-1}, t-1)  (自一致性约束)
  
  结果: 1~2 步即可生成高质量图像
  代价: 质量略低于完整采样，但速度提升 50~100 倍
```

### 5.3 目标检测中的蒸馏

```
检测蒸馏比分类更复杂，因为输出包含多个组件:

分类分支蒸馏:
  对齐每个 proposal/anchor 的类别分布 → 标准 KL

回归分支蒸馏:
  对齐 bbox 回归的输出 → L1 或 Smooth L1
  但回归值是连续的，不能直接用 softmax → 用 L2 距离

特征蒸馏 (更有效):
  对齐 backbone/FPN 的特征图
  通常在前景区域加权 (不关心背景的特征对齐)

代表: FGD (Focal and Global Distillation)
  - 前景区域: 精细对齐 (重要物体)
  - 全局区域: 粗粒度对齐 (场景理解)
```

## 六、蒸馏在 LLM 中的应用

### 6.1 黑盒蒸馏 vs 白盒蒸馏

```
┌─────────────────────────────────────────────────────────┐
│  黑盒蒸馏 (Black-box)                                   │
│  只能访问教师的输出文本，看不到 logits 或权重             │
│                                                         │
│  教师 (如 GPT-4) → 生成高质量回答                       │
│  学生 → 把这些回答当训练数据做 SFT                      │
│                                                         │
│  本质: 用教师生成的数据做 SFT                           │
│  代表: Alpaca (用 GPT-3.5 生成指令数据)                 │
│        Vicuna (用 ShareGPT 对话数据)                    │
│                                                         │
│  优势: 不需要教师模型权重 (API 就行)                    │
│  局限: 只学到教师的"行为"，学不到"思考过程"             │
│        (类比: 抄答案 vs 理解解题思路)                   │
└─────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────┐
│  白盒蒸馏 (White-box)                                   │
│  可以访问教师的完整权重和 logits                        │
│                                                         │
│  教师的每个 token 的 logits 分布都可以作为软标签         │
│  学生逐 token 对齐教师的 logits → KL loss               │
│                                                         │
│  对于序列生成:                                          │
│    教师: P(token_t | token_1...token_{t-1})             │
│    学生: Q(token_t | token_1...token_{t-1})             │
│    L = Σ_t KL(P_t || Q_t)                              │
│                                                         │
│  优势: 保留了完整的暗知识 (每个位置的 token 分布)       │
│  局限: 需要教师模型完整权重，计算量大                   │
│        32K 词表 × 每个 token 都要算 KL → 开销显著       │
│  优化: 只在 top-k token 上计算 KL (如 k=100)           │
└─────────────────────────────────────────────────────────┘
```

### 6.2 Chain-of-Thought 蒸馏

让小模型学会大模型的"推理过程"，而不仅是最终答案：

```
传统蒸馏:
  教师输出: "答案是 42"
  学生学: "答案是 42" → 学到了结果，但不知道为什么

CoT 蒸馏:
  教师输出: "首先分析条件 A，然后由 A 推出 B，再结合 C 得到 D，所以答案是 42"
  学生学: 整个推理链 → 学到了"怎么思考"

实现方式:
  1. 用大模型生成带 CoT 的回答 (黑盒)
  2. 用这些 CoT 数据 SFT 小模型
  3. 小模型学会了"先想后答"的模式

代表: 
  - Orca (2023): 用 GPT-4 的详细解释训练 13B 模型
  - 效果: 13B 模型在推理任务上接近 GPT-3.5 水平
```

### 6.3 DeepSeek-R1 的蒸馏实践 ⭐

DeepSeek-R1 (2025) 是 LLM 蒸馏的里程碑案例：

```
教师: DeepSeek-R1 (671B MoE, 具备强推理能力)
学生: Qwen-2.5 系列、LLaMA-3 系列 (1.5B~70B)

蒸馏方式: 黑盒 (只用教师生成的数据)

关键步骤:
  1. 用 R1 生成 80 万条高质量推理数据
     - 每条包含完整的思考链 (<think>...</think>) + 最终答案
     - 覆盖数学、代码、科学推理等任务

  2. 用这些数据对学生模型做 SFT
     - 学生不仅学最终答案，还学整个推理过程
     - 本质上是 CoT 蒸馏的大规模实践

结果 (令人震惊):
  DeepSeek-R1-Distill-Qwen-32B:
    AIME 2024: 72.6% (vs GPT-4o 的 79.2%)
    MATH-500: 94.3% (超过 GPT-4o 的 74.6%)
    
  一个 32B 的蒸馏模型在数学推理上超过了 GPT-4o！

关键发现:
  - 蒸馏 > 直接 RL: 在小模型上，蒸馏大模型的推理数据
    比直接在小模型上做 RL 效果更好
  - 原因: 小模型的 RL 探索能力有限，发现不了好的推理路径
    但大模型已经找到了，蒸馏直接"告诉"小模型正确的推理方式
```

### 6.4 词表维度的挑战

LLM 蒸馏在词表维度有独特困难：

```
CV 分类: 1000 类 (ImageNet) → softmax 1000 维 → 完整 KL 计算没问题
LLM:     150K 词表 → softmax 150K 维 → 每个 token 位置都要算 150K 维的 KL

优化策略:
  1. Top-K KL: 只在教师 top-k (如 k=50~100) 个 token 上算 KL
     剩余 token 的概率质量很小，贡献有限

  2. 分层蒸馏: 不同层用不同 loss
     浅层用 feature loss (隐含对齐)
     深层用 logit loss (显式对齐)

  3. 选择性蒸馏: 只在"难 token"上蒸馏
     教师置信度低的位置 → 暗知识丰富 → 重点蒸馏
     教师置信度高的位置 → 信息少 → 可以只用 hard label
```

## 七、蒸馏在 VLM / 多模态中的应用

### 7.1 视觉编码器蒸馏

VLM 的视觉编码器常用蒸馏来增强能力：

```
CLIP ViT 的蒸馏训练:
  教师: CLIP ViT-G/14 (大模型)
  学生: CLIP ViT-L/14 (部署模型)
  
  蒸馏目标:
    - 对齐图像特征: ||F_student - F_teacher||²
    - 对齐图文相似度矩阵: KL(sim_T || sim_S)
    → 学生保留教师的跨模态对齐能力

InternViT 的渐进式蒸馏 (InternVL2):
  InternViT-6B → 蒸馏 → InternViT-300M
  特点: 
    - 不只对齐最终特征，还对齐中间层
    - 渐进式: 先蒸馏浅层，逐步蒸馏深层
    → 保留了大 ViT 的视觉理解能力，推理成本大幅降低
```

### 7.2 跨模态蒸馏

一个模态的知识迁移到另一个模态：

```
视觉→文本蒸馏:
  图文对 (image, text):
    CLIP image encoder → image feature
    CLIP text encoder → text feature
    对齐: 让 text feature 捕获 image feature 中的信息
    → 纯文本模型可以"想象"没见过的图像

文本→视觉蒸馏 (更常见):
  用语言模型的知识增强视觉模型
  如 DINO-v2 训练中融入了文本监督信号
  → 视觉特征自带语义理解能力

音频→文本蒸馏:
  Whisper 的训练本质上也是一种蒸馏
  用大量带字幕的音频做弱监督 → 音频模型学到了语言知识
```

### 7.3 VLM 端到端蒸馏

将大 VLM 蒸馏为小 VLM：

```
教师: InternVL2-76B (大 VLM)
学生: InternVL2-8B (小 VLM)

蒸馏点:
  1. 视觉 token: 对齐投影模块输出的视觉 token
  2. LLM logits: 对齐 LLM 的 token 输出分布
  3. 回答数据: 用大 VLM 生成高质量 VQA 数据，训练小 VLM

实践中:
  通常结合白盒 (logit 对齐) + 黑盒 (数据生成) 两种方式
  先用黑盒蒸馏做 SFT warm-up
  再用白盒蒸馏做精细对齐
```

## 八、蒸馏的理论理解

### 8.1 为什么蒸馏有效？（多角度解释）

```
角度 1: 标签平滑 (Label Smoothing)
  软标签本质上是一种自适应的标签平滑
  但比固定的 label smoothing 更好——每个样本的平滑方式不同
  由教师模型根据输入动态决定

角度 2: 正则化
  软标签提供了更丰富的监督信号
  → 梯度更稳定，不容易过拟合
  → 类似于一种隐式正则化

角度 3: 数据增强
  教师的暗知识等价于一种"虚拟数据增强"
  一个 hard label 变成了一个丰富的分布
  → 有效训练数据量被"放大"了

角度 4: 课程学习 (Curriculum Learning)
  教师的软标签告诉学生"这个样本在哪些类别上容易混淆"
  → 自动为学生提供难度信息
  → 类似于自适应的课程学习

角度 5: 暗知识传递
  最直接的解释：类间相似性是 hard label 不包含的信息
  "猫像虎而不像卡车"这个知识只存在于软标签中
```

### 8.2 蒸馏的局限

```
1. 容量差距问题 (Capacity Gap)
   教师太大、学生太小 → 蒸馏效果反而变差
   学生"学不动"教师的复杂知识
   → 解决: 渐进式蒸馏 (Teacher Assistant)
     大教师 → 中等助教 → 小学生

2. 任务迁移问题
   在任务 A 上蒸馏的知识，可能对任务 B 没用
   → 蒸馏的知识有任务特异性

3. 分布偏移
   如果教师在某些数据上有系统性偏差
   蒸馏会把偏差传递给学生 (甚至放大)
   → 需要数据质量控制

4. 成本
   白盒蒸馏需要教师模型的完整前向传播
   对于超大模型 (如 671B)，光教师的推理就很贵
```

## 九、实践指南

### 9.1 蒸馏方案选择

```
场景 → 推荐方案:

有教师模型权重 + 计算资源充足:
  → 白盒蒸馏 (logit + feature 对齐)
  → 效果最好，但成本最高

只有教师 API (如 GPT-4):
  → 黑盒蒸馏 (用教师生成训练数据)
  → 实操最常见，效果也不差

教师和学生架构差异大:
  → 关系蒸馏 > 特征蒸馏 (不需要维度匹配)
  → 或用投影层做特征对齐

学生容量很小:
  → 渐进式蒸馏 (Teacher → Assistant → Student)
  → 降低每一跳的容量差距

无教师模型:
  → 自蒸馏 (DINO 风格) 或在线互蒸馏
```

### 9.2 超参数调优建议

```
温度 T:
  - 从 T=4 开始尝试
  - 教师置信度高 (如 >95% accuracy) → T 可以大一些 (10~20)
  - 教师置信度一般 → T 不宜太大 (2~5)
  - 类别数多 → T 适当增大

平衡系数 α:
  - 教师非常强 → α=0.8~0.9 (主要靠蒸馏)
  - 教师一般 → α=0.3~0.5 (蒸馏+真实标签平衡)
  - 训练后期可以降低 α (逐渐转向真实标签)

学习率:
  - 蒸馏训练的学习率通常比标准训练小
  - 因为蒸馏梯度已经很"好"了，不需要大步探索
```

### 9.3 常见 Trick

```
1. 教师和学生用不同的数据增强
   教师用弱增强 → 提供更稳定的软标签
   学生用强增强 → 增强泛化能力

2. 多教师集成蒸馏
   多个教师的 soft label 取平均 → 比单一教师更鲁棒

3. 渐进解冻
   蒸馏初期冻结学生浅层 → 只训深层
   后期全部解冻 → 端到端微调
   → 类似 transfer learning 的 fine-tuning 策略

4. 损失加权退火
   初期: α 大 (主要靠蒸馏引导)
   后期: α 小 (主要靠真实标签微调)
   → 前期学知识，后期学精度
```

## 十、总结

### 方法全景图

| 维度 | 方法 | 核心思想 | 典型应用 |
|------|------|---------|---------|
| 输出蒸馏 | Hinton KD | 对齐 softmax 输出 | 通用分类 |
| 特征蒸馏 | FitNets / AT | 对齐中间层特征 | 检测、分割 |
| 关系蒸馏 | RKD | 对齐样本间关系 | 度量学习 |
| 自蒸馏 | DINO / BAN | 模型自己教自己 | 视觉预训练 |
| 在线蒸馏 | DML | 多模型互相学习 | 资源受限 |
| 黑盒蒸馏 | Alpaca / Orca | 用教师输出做 SFT | LLM 训练 |
| CoT 蒸馏 | DeepSeek-R1 | 学推理过程不只学答案 | 推理增强 |
| 扩散蒸馏 | Consistency | 减少采样步数 | 图像生成加速 |
| 跨模态蒸馏 | CLIP distill | 模态间知识迁移 | 多模态 |

### 核心 Takeaway

```
1. 蒸馏的本质: 用教师的暗知识 (soft distribution) 提供比 hard label 更丰富的监督信号

2. 适用场景极广: 从 CV 分类到 LLM 训练到 Diffusion 加速，蒸馏无处不在

3. 关键设计选择:
   - 蒸什么？ (logits / features / relations)
   - 教师哪来？ (预训练大模型 / 自蒸馏 / 互蒸馏)
   - 白盒还是黑盒？ (取决于能否访问教师权重)

4. 当前趋势:
   - LLM 领域: CoT 蒸馏成为主流，学推理过程比学答案更重要
   - CV 领域: 自蒸馏 (DINO 系列) 成为视觉基础模型的标准训练方式
   - 生成领域: 蒸馏是 Diffusion 加速的核心手段
```
