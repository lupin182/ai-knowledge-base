# Chain-of-Thought 与推理范式

## 一、什么是 CoT

Chain-of-Thought（思维链），就是让模型**在给出答案之前，先写出中间推理步骤**。

**普通回答**：
> 问：一个商店有 23 个苹果，卖了 15 个，又进了 8 个，现在有多少个？
> 答：16 个

**CoT 回答**：
> 问：同上
> 答：商店一开始有 23 个苹果。卖了 15 个，还剩 23 - 15 = 8 个。又进了 8 个，现在有 8 + 8 = 16 个。答案是 **16 个**。

看起来只是"多写了几步"，但这个简单的改变对模型能力有巨大影响——尤其是数学、逻辑推理、代码等需要多步思考的任务。

### 为什么 CoT 有效？

1. **分解复杂问题**：把一个难题拆成多个简单的子步骤，每步模型都能做对
2. **暴露中间状态**：模型可以在每步"看到"自己前面算的结果，避免心算出错
3. **增加计算量**：更多的输出 token = 更多的 Transformer 前向传播 = 更多的"思考时间"
4. **对齐训练分布**：如果训练数据中有推理过程，模型在生成推理过程时更"熟悉"

> 直觉类比：让一个人心算 347 × 28 很难，但给纸笔列竖式就容易多了。CoT 就是给模型的"草稿纸"。

## 二、CoT 的发展历程

### 2022: Few-shot CoT（开山之作）

**论文**: *Chain-of-Thought Prompting Elicits Reasoning in Large Language Models* (Wei et al., Google, 2022)

核心发现：在 few-shot prompt 中给出几个带推理步骤的示例，模型就能学会对新问题也写推理过程。

```
# Few-shot CoT Prompt 示例
Q: Roger有5个网球，他又买了2桶网球，每桶有3个，他现在有多少个？
A: Roger一开始有5个球。2桶×3个=6个新球。5+6=11。答案是11。

Q: 食堂有23个苹果，用了20个做午餐，又买了6个，现在有多少个？
A: [模型会自动生成推理步骤]
```

关键发现：
- CoT 只在**大模型**（>100B）上有效，小模型写了推理过程反而更差（"涌现能力"）
- 在 GSM8K（小学数学）上准确率从 17.9% 提升到 58.1%（PaLM 540B）

### 2022: Zero-shot CoT

**论文**: *Large Language Models are Zero-Shot Reasoners* (Kojima et al., 2022)

更简单的方法——不需要示例，只需在 prompt 后面加一句：

> **"Let's think step by step"（让我们一步步思考）**

这句"咒语"就能让模型自动生成推理链。虽然效果比 few-shot CoT 差一些，但胜在简单。

### 2023: CoT 的变体和改进

| 方法 | 核心思路 | 出处 |
|------|---------|------|
| **Self-Consistency** | 生成多条推理链，投票选最终答案（多数决） | Wang et al., Google |
| **Tree-of-Thought (ToT)** | 把推理过程组织成树结构，可以回溯和探索多个分支 | Yao et al., Princeton |
| **Graph-of-Thought** | 更复杂的图结构推理 | Besta et al. |
| **Program-of-Thought** | 把推理过程写成代码，用解释器执行 | Chen et al. |

**Self-Consistency** 特别值得一提：

```
同一个问题，采样 5 次：
  推理链 1 → 答案 A
  推理链 2 → 答案 B
  推理链 3 → 答案 A
  推理链 4 → 答案 A
  推理链 5 → 答案 C

多数投票 → 最终答案 A（3票）
```

这其实就是 **test-time compute scaling** 的雏形——用更多的推理计算换取更高的准确率。

### 2024-2025: 从 Prompt 技巧到内化能力

前面的 CoT 都是**推理时的 prompt 技巧**——模型本身没变，只是通过 prompt 引导它写推理过程。

真正的范式变革是：**把 CoT 能力训练进模型内部**。

## 三、推理模型：CoT 的终极形态

### OpenAI o1/o3 (2024-2025)

o1 是第一个大规模商用的推理模型。核心改变：

- **内化 CoT**：模型在回答前会自动进行长时间"思考"，生成（用户不可见的）推理过程
- **Test-time Compute Scaling**：推理时花更多计算 → 更好的结果。不同于传统的训练时 scaling（更大模型、更多数据），这是在推理阶段的 scaling
- **训练方式**：推测使用了 PPO + PRM（过程奖励模型），对推理的每一步打分，而不只是最终答案

### DeepSeek-R1 (2025)

开源推理模型，最震撼的发现：

**DeepSeek-R1-Zero**：纯 RL 训练（GRPO + 规则验证），**不给任何 CoT 数据**，模型自己学会了写推理链。

训练过程中观察到的涌现：
1. 一开始模型直接给答案（没有推理过程）
2. RL 训练几千步后，模型开始自发写出 `<think>...</think>` 标签
3. 推理链越来越长、越来越结构化
4. 最终在数学和代码任务上接近 o1 的水平

**为什么会涌现？** 因为奖励信号只看最终答案对不对。模型发现"先想一想再回答"→ 答案更容易对 → 奖励更高 → 强化这个行为。这就是 RL 的探索能力——模型自己发现了 CoT 这个"策略"。

**DeepSeek-R1**（完整版）的训练流程：

```
Stage 1: SFT 冷启动
  └── 用少量 CoT 数据做 SFT，给模型一个好的起点

Stage 2: RL 训练（GRPO）
  └── 数学/代码：规则验证奖励（答案对=+1，错=-1）
  └── 通用任务：RM 奖励

Stage 3: 拒绝采样 + SFT
  └── 用 Stage 2 的模型生成多个回答
  └── 选最好的做 SFT（蒸馏回来）

Stage 4: 再次 RL
  └── 最终对齐
```

### CoT 数据的来源分类

| 类型 | 怎么来的 | 代表 |
|------|---------|------|
| **人工标注** | 人写推理过程 | 早期 Google CoT 论文、GSM8K 数据集 |
| **模型蒸馏** | 用强模型生成推理过程，给弱模型训练 | DeepSeek-R1-Distill 系列（用 R1 生成 CoT，蒸馏到 1.5B~70B） |
| **RL 涌现** | 不给 CoT 数据，模型通过 RL 自己学会 | DeepSeek-R1-Zero |
| **Prompt 引导** | 推理时通过 prompt 诱导 | "Let's think step by step" |

## 四、Test-time Compute Scaling

这是 2024-2025 最重要的范式之一：**用更多的推理计算换取更好的结果**。

### 两种 Scaling 范式

| | Train-time Scaling（传统） | Test-time Scaling（新范式） |
|---|---|---|
| **怎么做** | 更大模型、更多数据、更多训练 | 更多推理时间、更长思考链 |
| **代表** | GPT-3 → GPT-4（参数增大） | o1 → o3（思考时间增大） |
| **边际收益** | 逐渐递减（Scaling Laws 的天花板） | 还在探索中，目前看递减较慢 |
| **核心公式** | Loss ~ N^{-α} | Accuracy ~ Compute^{β}（推理计算） |

### 实现方式

1. **Best-of-N**：生成 N 个回答，用 RM/验证器选最好的（Self-Consistency 的推广）
2. **长推理链**：让模型写更长的推理过程（o1/R1 的做法）
3. **搜索 + 验证**：Tree-of-Thought、MCTS 等搜索算法 + 验证器剪枝
4. **迭代修正**：模型自己检查、修改推理过程（self-correction）

## 五、CoT 全景图

```
CoT 的演进：

  Prompt 技巧（不改模型）              训练到模型里
  ─────────────────────              ────────────
  2022: Few-shot CoT                 2024: o1 (内化 CoT + PRM)
  2022: Zero-shot CoT                2025: DeepSeek-R1 (RL 涌现 CoT)
  2023: Self-Consistency             2025: o3, Claude 3.5
  2023: Tree-of-Thought              2026: 各家推理模型标配
         │                                    │
         └──── test-time compute ─────────────┘
               在推理时投入更多计算 = 更好的结果
```

**关键转折点**：CoT 从"prompt 里的技巧"变成了"训练进模型的能力"。这也是 o1 和 R1 为什么是范式变革——它们不是在用更大的模型，而是在用更深的思考。
