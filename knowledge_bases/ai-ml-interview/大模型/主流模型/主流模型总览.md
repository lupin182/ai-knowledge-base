## 二、GPT 系列 (OpenAI)

### 2.1 版本演进

```
GPT-1 (2018)  → 1.17亿参数，证明预训练+微调范式有效
    │
GPT-2 (2019)  → 15亿参数，zero-shot 能力涌现，"太危险不敢发布"
    │
GPT-3 (2020)  → 175B参数，in-context learning，few-shot prompting
    │            Scaling Laws 的巅峰验证
    │
GPT-3.5 (2022) → InstructGPT + RLHF，ChatGPT 发布，引爆 AI 浪潮
    │
GPT-4 (2023.3) → 多模态(文本+图像)，推理能力质变
    │              推测: 8×220B MoE (~1.8T 总参数)
    │
GPT-4 Turbo (2023.11) → 128K 上下文，更便宜
    │
GPT-4o (2024.5) → "omni" 原生多模态(文本+图像+音频+视频)
    │               速度快 2x，价格低 50%
    │
o1 (2024.9)   → "推理模型"，内置 Chain-of-Thought
    │            数学/代码能力大幅提升
    │
o3 (2025.1)   → o1 升级版，推理能力再次提升
                  在 ARC-AGI 上达到 87.5%
```

### 2.2 关键技术特点

- **架构**：Decoder-Only Transformer，具体细节未公开
- **GPT-4 的 MoE 推测**：根据泄露信息，GPT-4 可能是 8 个 ~220B 的专家，每次激活 2 个，总参数 ~1.8T，活跃参数 ~440B
- **o1/o3 系列**：引入 "thinking tokens"——模型在回答前先进行长链推理（类似 DeepSeek-R1），大幅提升数学和代码能力
- **RLHF 先驱**：InstructGPT (2022) 开创了 SFT → RM → PPO 的三步对齐流程

### 2.3 API 生态

```
模型层次:
  GPT-4o       → 综合旗舰，性价比最高
  GPT-4o-mini  → 轻量快速，替代 GPT-3.5
  o1/o3        → 复杂推理任务
  o3-mini      → 轻量推理

定价 (每 1M tokens, 2025):
  GPT-4o:      输入 $2.5,  输出 $10
  GPT-4o-mini: 输入 $0.15, 输出 $0.6
  o1:          输入 $15,   输出 $60
```

### 3.1 版本演进

```
Claude 1 (2023.3)   → 初版，强调安全性 (Constitutional AI)
    │
Claude 2 (2023.7)   → 100K 上下文，性能对标 GPT-3.5
    │
Claude 3 (2024.3)   → 三档: Haiku / Sonnet / Opus
    │                  Opus 在多项测试超越 GPT-4
    │
Claude 3.5 Sonnet (2024.6) → 性能超越 Claude 3 Opus
    │                         代码能力突出，SWE-bench 最佳
    │
Claude 4 (2025)     → Opus/Sonnet/Haiku 全面更新
                       扩展思考 (Extended Thinking)
```

### 3.2 关键技术特点

- **Constitutional AI (CAI)**：用一组"宪法规则"指导模型自我改进，减少对人工标注的依赖
- **超长上下文**：200K token 上下文窗口，长文本处理能力顶尖
- **代码能力**：Claude 3.5 Sonnet 在 SWE-bench 上一度排名第一
- **Extended Thinking**：Claude 4 引入的思考模式，类似 o1 的内置推理链
- **安全对齐**：Anthropic 以 AI 安全为核心使命，模型在安全性上投入最多

### 3.3 产品矩阵

```
Claude Opus    → 最强能力，复杂任务
Claude Sonnet  → 性能/成本最佳平衡，主力模型
Claude Haiku   → 最快速度，轻量任务

特色功能:
  - Artifacts: 在对话中生成可交互的代码/文档
  - Computer Use: 控制计算机桌面完成任务
  - Claude Code: 命令行编程助手
```

## 四、LLaMA 系列 (Meta)

### 4.1 版本演进

```
LLaMA-1 (2023.2)  → 7B/13B/33B/65B，开源权重
    │                65B 接近 GPT-3.5
    │                引爆开源 LLM 社区
    │
LLaMA-2 (2023.7)  → 7B/13B/70B，正式开源（含商用）
    │                训练数据 2T tokens (LLaMA-1 的 40%)
    │                引入 GQA (70B)
    │
Code Llama (2023.8) → 基于 LLaMA-2，代码特化
    │
LLaMA-3 (2024.4)  → 8B/70B，大幅升级
    │                15T tokens 训练 (over-training)
    │                GQA 全系列，128K 词表
    │
LLaMA-3.1 (2024.7) → 8B/70B/405B
    │                  128K 上下文 (YaRN)
    │                  405B 是最大的开源模型
    │
LLaMA-3.3 (2024.12) → 70B，媲美 LLaMA-3.1-405B
                        蒸馏 + 大量 over-training
```

### 4.2 架构细节（面试常考）

| 参数 | LLaMA-1-7B | LLaMA-2-7B | LLaMA-3-8B | LLaMA-3.1-405B |
|---|---|---|---|---|
| 层数 | 32 | 32 | 32 | 126 |
| 隐藏维度 | 4096 | 4096 | 4096 | 16384 |
| 注意力头 | 32 | 32 | 32 | 128 |
| KV 头 | 32 (MHA) | 32 (MHA) | 8 (GQA) | 8 (GQA) |
| FFN 维度 | 11008 | 11008 | 14336 | 53248 |
| 词表大小 | 32000 | 32000 | 128256 | 128256 |
| 训练 tokens | 1T | 2T | 15T | 15T |
| 上下文 | 2K | 4K | 8K→128K | 128K |
| 位置编码 | RoPE | RoPE | RoPE | RoPE + YaRN |
| 归一化 | RMSNorm | RMSNorm | RMSNorm | RMSNorm |
| 激活函数 | SwiGLU | SwiGLU | SwiGLU | SwiGLU |

### 4.3 LLaMA 的意义

```
为什么 LLaMA 如此重要？

1. 开源催化: 第一个性能接近 GPT-3.5 的开源模型
   → 催生 Alpaca, Vicuna, WizardLM 等数百个微调模型

2. 架构标杆: RMSNorm + SwiGLU + RoPE + GQA 成为行业标准
   → Qwen, Mistral, DeepSeek 等都采用类似架构

3. 训练方法论: 证明了 over-training 小模型的可行性
   → LLaMA-3-8B (15T tokens) 超越 LLaMA-2-70B

4. 开源生态: HuggingFace 上最多的模型基座
```

### 5.1 版本演进

```
Qwen-1 (2023.8)    → 7B/14B，中文能力强
    │
Qwen-1.5 (2024.2)  → 0.5B~72B，全系列升级
    │
Qwen-2 (2024.6)    → 0.5B~72B，架构优化
    │
Qwen-2.5 (2024.9)  → 0.5B/1.5B/3B/7B/14B/32B/72B
    │                 全面超越 LLaMA-3.1 同规模
    │                 18T tokens 训练
    │
QwQ (2024.11)      → 推理模型 (类似 o1)
    │
Qwen-2.5-Max (2025) → 闭源旗舰，MoE 架构
```

### 5.2 关键技术特点

- **中英双语最强开源**：词表 152K，中文 token 效率高
- **全系列覆盖**：从 0.5B 到 72B，覆盖手机到服务器
- **数学/代码特化版**：Qwen-2.5-Math, Qwen-2.5-Coder
- **多模态**：Qwen-VL（视觉语言）、Qwen-Audio
- **Agent 能力**：内置 function calling，工具使用能力强

### 5.3 架构特点

```
相比 LLaMA 的差异:
  - 词表更大: 152064 (vs LLaMA 128256)
  - GQA 更激进: 7B 模型只用 4 个 KV 头 (vs LLaMA 8 个)
  - Attention Bias: 部分版本保留了 QKV bias
  - Embedding tying: 词表大 → 不做 weight tying，避免头重脚轻
```

## 六、DeepSeek 系列 (深度求索)

### 6.1 版本演进

```
DeepSeek-V1 (2024.1)  → 7B/67B，中国开源之光
    │
DeepSeek-V2 (2024.5)  → 236B MoE (21B active)
    │                    MLA (Multi-head Latent Attention)
    │                    推理成本仅为 GPT-4 的 1/50
    │
DeepSeek-V3 (2025.1)  → 671B MoE (37B active)
    │                    训练成本仅 $5.5M (H800×2048, 2个月)
    │                    性能接近 GPT-4o / Claude 3.5
    │
DeepSeek-R1 (2025.1)  → 推理模型
                          纯 RL 训练 (GRPO)，无需 SFT 冷启动
                          数学/代码超越 o1
                          开源了完整训练流程
```

### 6.2 关键创新（面试重点）

**MLA (Multi-head Latent Attention)**

```
标准 GQA: 缓存 K, V → KV Cache 大
MLA: 把 KV 压缩到低维潜在向量 c_kv → 缓存量降到 GQA 的 1/4~1/8

→ 详见 长上下文详解.md 的 MLA 章节
```

**DeepSeekMoE (细粒度专家)**

```
标准 MoE (如 Mixtral):  8 个专家，每次激活 2 个
DeepSeekMoE:           256 个细粒度专家 + 1 个共享专家
                        每次激活 8 个细粒度专家 + 共享专家

优势:
  - 细粒度专家 → 更精细的知识分工
  - 共享专家 → 保证通用能力不丢失
  - 辅助 loss → 负载均衡

→ 详见 MoE详解.md
```

**FP8 混合精度训练**

```
DeepSeek-V3 首次大规模使用 FP8 训练:
  前向传播: FP8 (E4M3) 做 GEMM
  反向传播: FP8 (E5M2) 做 GEMM
  梯度累积: FP32

→ 训练速度提升 ~1.5x，是 $5.5M 低成本的关键之一
```

**DeepSeek-R1 的 RL 训练**

```
传统推理模型 (o1): SFT 冷启动 → RL 微调 (过程不公开)
DeepSeek-R1-Zero: 纯 RL (GRPO)，不用 SFT 冷启动
  → 模型自发学会了 CoT 推理、自我验证、反思

  训练信号:
    数学: 答案是否正确 (rule-based reward)
    代码: 测试用例是否通过
    其他: GRPO 的组内相对排名

DeepSeek-R1: R1-Zero + SFT 冷启动 + 多阶段 RL
  → 数学/代码超越 o1
  → 完全开源 (模型 + 训练方法 + 蒸馏模型)
```

### 6.3 DeepSeek 的成本优势

```
DeepSeek-V3 训练成本: ~$5.5M
  对比: LLaMA-3-405B 估计 ~$100M+
  对比: GPT-4 估计 ~$100M+

原因:
  1. MoE 稀疏激活 → 同等能力下 FLOPs 少
  2. MLA → KV Cache 小 → batch size 可以更大
  3. FP8 训练 → 硬件利用率高
  4. H800 (阉割版 H100) → 单卡便宜

推理成本:
  DeepSeek API: ¥1/M input tokens, ¥2/M output tokens
  GPT-4o:       ¥17.5/M input, ¥70/M output
  → DeepSeek 推理成本仅为 GPT-4o 的 ~3%
```

## 九、模型选型指南

### 9.1 按场景选模型

| 场景 | 推荐模型 | 理由 |
|---|---|---|
| 通用对话 | GPT-4o / Claude Sonnet | 综合能力最强 |
| 代码开发 | Claude Sonnet / DeepSeek-V3 | SWE-bench 最佳 |
| 数学推理 | o3 / DeepSeek-R1 | 专门的推理模型 |

**面试快速回答模板**：

> **Q: 对比一下 LLaMA、Qwen、DeepSeek 的技术特点？**
> A: 三者都基于 Decoder-Only + RoPE + GQA + SwiGLU。LLaMA 是架构标杆，15T tokens over-training，生态最大。Qwen 词表最大 (152K)，中文效率最高，全系列覆盖。DeepSeek 用 MoE+MLA 实现极低推理成本，V3 总参 671B 但活跃只有 37B，还首次用 FP8 训练把成本压到 $5.5M。
>
> **Q: DeepSeek 为什么推理这么便宜？**
> A: 三个原因：(1) MoE 稀疏激活，每次只用 37B/671B 的参数；(2) MLA 压缩 KV Cache 到传统方法的 1/8；(3) FP8 训练降低了训练成本传导到定价。
>
> **Q: o1/o3 和普通模型有什么区别？**
> A: o1/o3 是"推理模型"，内置了 Chain-of-Thought。模型在回答前会先生成大量"思考 token"（用户不可见），通过延长推理时间来提升复杂任务的准确率。DeepSeek-R1 开源了类似的训练方法——纯 RL (GRPO) 就能让模型学会自主推理。
