# 视频关键帧检测

关键帧检测是视频处理的基础环节，决定了从视频中选取哪些帧送入下游模型（如 VLM）进行分析。本文从简单到复杂，逐层展开各类方法。

---

## 一、均匀采样（Baseline）

最朴素的方法——按固定间隔抽帧：

```
60秒视频，30fps，共 1800 帧
每隔 1 秒取 1 帧 → 60 帧
每隔 2 秒取 1 帧 → 30 帧
```

| 优点 | 缺点 |
|------|------|
| 简单、确定性强、工程好实现 | 不区分内容重要性 |
| 无需额外计算 | 静止画面和剧烈变化被同等对待 |
| 适合做 baseline | 短暂出现的关键内容可能被跳过 |

> **典型问题**：违规内容可能只闪了 0.3 秒（如快速闪过的违禁品画面），均匀采样直接跳过。

实际中均匀采样通常作为 baseline 或第一步粗筛，后面还需要更精细的方法。

---

## 二、基于视觉差异的方法——像素级/特征级变化检测

核心思想：**相邻帧差异大的地方，可能是关键帧。**

### 2.1 帧差法（Frame Differencing）

最直接，计算相邻两帧的像素差异：

```python
diff = abs(frame[t] - frame[t-1])
score = mean(diff)  # 或者用直方图差异、SSIM 等
if score > threshold:
    mark as keyframe
```

**问题**：
- 对光照变化敏感（灯光闪了一下就触发）
- 对摄像头运动敏感（镜头平移时每帧都在变，但其实内容没变）
- 阈值不好调，不同视频差异很大

### 2.2 直方图差异

不比较逐像素差异，而是比较两帧的颜色直方图：

```python
hist1 = calc_histogram(frame[t])
hist2 = calc_histogram(frame[t-1])
diff = chi_square_distance(hist1, hist2)
```

比像素差异更鲁棒——对小幅度的像素位移不敏感。但对颜色分布相似但内容完全不同的场景（比如两个不同的蓝天场景）会漏判。

### 2.3 基于深度特征的差异

用预训练 CNN（ResNet、CLIP 视觉编码器等）提取每帧的特征向量，计算余弦相似度：

```python
feat1 = encoder(frame[t])      # 2048 维向量
feat2 = encoder(frame[t-1])
similarity = cosine_sim(feat1, feat2)
if similarity < threshold:
    mark as keyframe
```

**目前最常用的方案之一**。深度特征关注语义层面的变化（场景内容变了），而不是像素层面的噪声，比前两种鲁棒得多。

## 三、场景切换检测（Shot Boundary Detection）

视频本质上由多个"镜头（shot）"组成，每个镜头内部内容连续，镜头之间有切换。关键帧的一个自然定义：**每个镜头取一帧代表**。

### 3.1 切换类型

| 类型 | 描述 | 检测难度 |
|------|------|---------|
| **硬切（hard cut）** | 直接跳到下一个场景，相邻帧差异极大 | 帧差法就能检测 |
| **渐变（gradual transition）** | 淡入淡出、溶解、擦除等，变化渐进 | 需要看更长时间窗口 |

### 3.2 经典算法

- **双阈值法**：设一个高阈值检测硬切，一个低阈值检测渐变的起止点
- **滑动窗口**：用一个时间窗口内的差异统计量（均值、方差）来判断是否发生了转场

### 3.3 深度学习方法：TransNet V2

**TransNet V2** 是目前主流的场景切换检测模型：
- 用 3D CNN 直接在视频帧序列上做二分类（是否是切换点）
- 准确率很高，速度也很快
- 很多工业 pipeline 都在用

检测到镜头边界后，每个镜头内取中间帧或第一帧作为关键帧。

## 四、基于聚类的方法——内容多样性最大化

换一个思路：不关注"哪里变化大"，而是关注"**选出来的帧能否最大程度代表整个视频的内容多样性**"。

```python
features = [encoder(frame[t]) for t in range(total_frames)]
kmeans = KMeans(n_clusters=K).fit(features)
keyframes = [nearest_to_center(cluster) for cluster in kmeans.clusters]
```

| 优点 | 缺点 |
|------|------|
| 选出的帧保证多样性 | 需要对所有帧提取特征，计算量大 |
| K 值直接控制关键帧数量 | K-Means 对 K 的选择敏感 |

**改进**：用**层次聚类**，不需要预设 K，通过树状图自动找到合理的切分层级。

## 五、面向任务的关键帧选择（T&S 场景）

上面的方法都是通用关键帧检测。在 Trust & Safety 场景中，目标不是"哪些帧代表了视频内容"，而是"**哪些帧可能包含违规内容**"。

### 5.1 先验知识驱动

基于违规内容的特点设计采样策略：

| 策略 | 适用场景 |
|------|---------|
| **肤色检测** | 涉黄审核，肤色占比高的帧优先采样 |
| **运动突变** | 闪帧逃避审核（突然切到违规画面再切回来） |
| **文字出现** | 诈骗引流，大量屏幕文字的帧优先 |
| **人脸/人体出现** | 未成年人保护，有人出现的帧优先 |

这些可以组合使用，不同审核任务用不同的采样策略。

### 5.2 轻量分类器预筛

用一个很小很快的分类器（MobileNet 级别）对每帧打"可疑度"分数：

```
所有帧 → 轻量分类器（~1ms/帧）→ 可疑度排序 → Top-K 帧 → VLM 精审
```

轻量分类器不需要很准确，只需要保证**召回率高**（宁可多选、不能漏选），准确率交给 VLM。

### 5.3 CLIP 驱动的语义采样

如果已知要审核什么类型的违规，可以用 CLIP 做 **query-aware 采样**：

```python
query = "violence, weapon, blood"  # 文本描述
query_feat = clip.encode_text(query)

for frame in all_frames:
    frame_feat = clip.encode_image(frame)
    score = cosine_sim(query_feat, frame_feat)
    if score > threshold:
        select this frame
```

用文本描述来"搜索"视频中与特定违规类型相关的帧，可以同时用多个 query 覆盖多种违规类型。

## 六、自适应采样——动态决定采样密度

根据内容动态调整采样密度：

```
静止画面（如纯展示 PPT）  → 极低密度，几秒一帧
正常内容区间              → 中等密度
可疑区间                  → 极高密度，甚至逐帧分析
```

**两阶段实现**：

```
第一遍：极低密度（每 2 秒 1 帧）扫一遍全视频
  → 标记可疑时间段

第二遍：对可疑时间段用高密度（每 0.1 秒 1 帧）重新采样
  → 精确定位违规时刻
```

第一遍成本很低（30 帧就够了），第二遍只对局部做精细分析。

## 七、缓慢变化检测——"青蛙煮水"问题

典型场景：相邻帧差异极小（每帧只变化一点），任何基于帧差的方法都检测不到，但 30 秒前和 30 秒后画面已经完全不同。

### 7.1 远距离帧对比

不只比较相邻帧，同时比较不同跨度的帧对：

```
相邻帧对比：   frame[t] vs frame[t-1]     → 检测突变
中距离对比：   frame[t] vs frame[t-30]    → 检测 1 秒内的变化
远距离对比：   frame[t] vs frame[t-150]   → 检测 5 秒内的变化
```

多个时间尺度同时监控，就能捕获不同速度的变化。

### 7.2 累积差异追踪

维护一个"累积变化量"：

```python
cumulative_diff = 0
anchor_frame = frame[0]

for t in range(total_frames):
    diff = feature_distance(frame[t], anchor_frame)
    cumulative_diff += frame_diff(frame[t], frame[t-1])
    
    if diff > threshold or cumulative_diff > threshold:
        mark as keyframe
        anchor_frame = frame[t]  # 重置锚点
        cumulative_diff = 0
```

无论变化是一瞬间发生的还是缓慢发生的，只要**总变化量够大**就会被检测到。

### 7.3 滑动窗口 + 状态追踪

更适合 T&S 场景——不检测"哪里变化大"，而是追踪"内容状态"：

```
维护状态向量：[肤色面积占比, 衣物覆盖率, 人体关键点暴露程度]

每帧更新状态 → 画出状态随时间的变化曲线

当状态曲线越过安全阈值时触发告警
```

本质上把关键帧检测转化为**时序信号的阈值监控问题**。不管变化多慢，只要指标越过阈值就会被检测到。

## 八、从视频压缩原理出发——I 帧、P 帧、B 帧

这是一个非常巧妙的思路：**视频编码器已经帮你做了一遍"哪里变化大"的分析**。

### 8.1 视频压缩基本原理

视频编码（H.264/H.265）把帧分成三种类型：

| 帧类型 | 全称 | 说明 |
|--------|------|------|
| **I 帧** | Intra Frame（关键帧） | 完整编码，不依赖其他帧，类似 JPEG |
| **P 帧** | Predicted Frame（前向预测帧） | 只存储和前一个参考帧的差异 |
| **B 帧** | Bi-directional Frame（双向预测帧） | 参考前后两帧预测，压缩率最高 |

```
典型帧序列：
I B B P B B P B B P B B I B B P ...

I 帧之间的间隔叫 GOP（Group of Pictures），通常 1-2 秒
```

### 8.2 利用压缩域信息做关键帧检测

#### 利用一：直接用 I 帧作为关键帧

视频编码器在场景切换点会自动插入 I 帧（因为 P 帧预测失效），所以 **I 帧位置天然就是场景切换点**。

```bash
# 用 ffmpeg 直接提取 I 帧，几乎零计算成本
ffmpeg -i video.mp4 -vf "select=eq(pict_type\,I)" -vsync vfr keyframes_%03d.jpg
```

无需解码和分析像素，直接从视频编码结构中获取场景切换信息。

#### 利用二：P 帧残差能量反映变化剧烈程度

P 帧存储的是和参考帧的差异，**残差大小直接反映画面变化剧烈程度**：

```
帧序号:  1    2    3    4    5    6    7    8
帧类型:  I    P    P    P    P    P    P    I
残差大小: -   120  115  130  890  920  850   -
                              ↑
                         突然变大，发生了重要变化
```

不需要解码图像、不需要提特征、不需要跑任何模型。

#### 利用三：运动向量（Motion Vector）

编码器为 P/B 帧的每个宏块（通常 16×16 像素）计算运动向量，表示"这个块从参考帧的哪个位置移过来"：

| 运动向量特征 | 含义 |
|-------------|------|
| 幅度大 | 运动剧烈，可能是关键帧 |
| 方向一致 | 摄像头平移（不是内容变化） |
| 方向混乱 | 场景内多个物体在动（可能是重要事件） |

> **T&S 应用**：检测打架场景——运动向量大且方向混乱，说明多人在剧烈运动。

### 8.3 压缩域解决缓慢变化

从压缩角度看缓慢变化场景：

- 每个 P 帧的残差都很小（变化缓慢）
- 但直接比较相隔两个 GOP 的 I 帧，差异就很明显

```
GOP 1 的 I 帧（第 0 秒） → 初始状态
GOP 5 的 I 帧（第 10 秒）→ 已经变化明显
```

I 帧本身就是完整图像，直接对比不同 GOP 的 I 帧，计算量极小，效果很好。

### 8.4 综合利用压缩域信息

```
视频压缩流
├── I 帧位置       → 场景切换点（零成本）
├── P 帧残差能量   → 变化剧烈程度时间曲线
├── 运动向量       → 运动区域和运动模式
└── 宏块类型分布   → intra 块多说明该区域新内容多

综合这些信号 → 生成每帧的"重要度分数" → 选 Top-K 作为关键帧
```

整个过程在**压缩域（compressed domain）**完成，不需要完全解码视频，速度比像素级方法快一个数量级。

## 九、工程落地的实际考量

### 9.1 采样数量的 budget 约束

VLM 的上下文窗口和推理成本限制了最多能输入多少帧。如果 VLM 上下文只能容纳 32 帧，那关键帧检测就变成了"从 1800 帧中选出最有价值的 32 帧"。

### 9.2 帧的表示方式

选出关键帧后送给 VLM 的方式：

| 方式 | 说明 |
|------|------|
| 直接送原图 | 信息最完整，但占 token 多 |
| 拼成网格图 | 16 帧拼成 4×4 网格，一张图含 16 帧信息 |
| 附带时间戳 | 让模型知道每帧对应视频的第几秒 |

> Qwen-VL 和 InternVL 都支持带时间戳的帧输入，模型回答时能说出"第 XX 秒"。

### 9.3 音视频对齐

选了关键帧后，需要把对应时间段的 ASR 文本一起送给模型。帧的时间戳和 ASR 的时间戳要对齐，否则模型看到的画面和听到的话对不上。

### 9.4 缓存和复用

对于热门视频（播放量暴涨的），可能需要反复审核（初审→复审→申诉复查）。关键帧提取的结果和特征向量应该缓存起来，避免重复计算。

## 方法对比总结

| 方法 | 速度 | 精度 | 适用场景 |
|------|------|------|---------|
| 均匀采样 | 最快 | 最低 | baseline / 粗筛 |
| 帧差法 | 快 | 中 | 硬切检测 |
| 深度特征差异 | 中 | 较高 | 通用关键帧 |
| TransNet 场景检测 | 中 | 高 | 镜头边界 |
| 聚类 | 慢 | 高 | 内容覆盖最大化 |
| CLIP 语义采样 | 中 | 高 | 面向特定违规类型 |
| 压缩域分析 | 最快 | 较高 | 大规模视频处理 |
| 自适应两阶段 | 中 | 最高 | 生产环境推荐 |

> **生产环境推荐组合**：TransNet 做镜头分割 + 每个镜头内用轻量分类器选可疑帧 + 可疑帧送 VLM。
>
> **完整三层架构**：压缩域快速筛选 → 轻量模型二次筛选 → VLM 精审。
