## 1. Multi-Head Attention

**思路**: 将 Q/K/V 通过线性变换后拆分成多头，每个头独立做 Scaled Dot-Product Attention，最后拼接再投影。

核心公式: $\text{Attention}(Q,K,V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$

```python
import torch
import torch.nn as nn
import torch.nn.functional as F
import math

class MultiHeadAttention(nn.Module):
    def __init__(self, d_model: int, num_heads: int):
        super().__init__()
        assert d_model % num_heads == 0
        self.d_model = d_model
        self.num_heads = num_heads
        self.d_k = d_model // num_heads

        self.W_q = nn.Linear(d_model, d_model)
        self.W_k = nn.Linear(d_model, d_model)
        self.W_v = nn.Linear(d_model, d_model)
        self.W_o = nn.Linear(d_model, d_model)

    def forward(self, x, mask=None):
        """
        x: (batch, seq_len, d_model)
        mask: (batch, 1, 1, seq_len) 或 (batch, 1, seq_len, seq_len)
        """
        B, L, _ = x.shape

        # 线性变换 + 拆分多头: (B, L, d_model) -> (B, num_heads, L, d_k)
        Q = self.W_q(x).view(B, L, self.num_heads, self.d_k).transpose(1, 2)
        K = self.W_k(x).view(B, L, self.num_heads, self.d_k).transpose(1, 2)
        V = self.W_v(x).view(B, L, self.num_heads, self.d_k).transpose(1, 2)

        # Scaled Dot-Product Attention
        scores = torch.matmul(Q, K.transpose(-2, -1)) / math.sqrt(self.d_k)
        if mask is not None:
            scores = scores.masked_fill(mask == 0, float('-inf'))
        attn = F.softmax(scores, dim=-1)

        # 加权求和 + 拼接
        out = torch.matmul(attn, V)                          # (B, num_heads, L, d_k)
        out = out.transpose(1, 2).contiguous().view(B, L, -1) # (B, L, d_model)
        return self.W_o(out)
```

**面试要点**:
- 为什么除以 $\sqrt{d_k}$? 防止点积值过大导致 softmax 梯度消失
- `contiguous()` 的作用? `transpose` 后内存不连续，`view` 前需要调用
- mask 通常有两种: padding mask 和 causal mask (下三角)

## 2. Grouped-Query Attention

**思路**: GQA 是 MHA 和 MQA 的折中。Q 仍然有 `num_heads` 个头，但 K/V 只有 `num_kv_heads` 个头，多个 Q 头共享同一组 K/V。减少 KV Cache 显存。

```python
class GroupedQueryAttention(nn.Module):
    def __init__(self, d_model: int, num_heads: int, num_kv_heads: int):
        super().__init__()
        assert num_heads % num_kv_heads == 0
        self.num_heads = num_heads
        self.num_kv_heads = num_kv_heads
        self.num_groups = num_heads // num_kv_heads  # 每组共享的 Q 头数
        self.d_k = d_model // num_heads

        self.W_q = nn.Linear(d_model, num_heads * self.d_k)
        self.W_k = nn.Linear(d_model, num_kv_heads * self.d_k)
        self.W_v = nn.Linear(d_model, num_kv_heads * self.d_k)
        self.W_o = nn.Linear(d_model, d_model)

    def forward(self, x, mask=None):
        B, L, _ = x.shape

        Q = self.W_q(x).view(B, L, self.num_heads, self.d_k).transpose(1, 2)
        K = self.W_k(x).view(B, L, self.num_kv_heads, self.d_k).transpose(1, 2)
        V = self.W_v(x).view(B, L, self.num_kv_heads, self.d_k).transpose(1, 2)

        # 关键: 将 K/V 扩展以匹配 Q 的头数
        # (B, num_kv_heads, L, d_k) -> (B, num_heads, L, d_k)
        K = K.repeat_interleave(self.num_groups, dim=1)
        V = V.repeat_interleave(self.num_groups, dim=1)

        scores = torch.matmul(Q, K.transpose(-2, -1)) / math.sqrt(self.d_k)
        if mask is not None:
            scores = scores.masked_fill(mask == 0, float('-inf'))
        attn = F.softmax(scores, dim=-1)

        out = torch.matmul(attn, V)
        out = out.transpose(1, 2).contiguous().view(B, L, -1)
        return self.W_o(out)
```

**面试要点**:
- `num_kv_heads=1` 就退化为 MQA; `num_kv_heads=num_heads` 就是标准 MHA
- LLaMA 2 70B、Gemma、Qwen2 等均采用 GQA

## 3. LayerNorm

**思路**: 对每个样本的特征维度做归一化 (均值为0，方差为1)，再做仿射变换。区别于 BatchNorm 是沿 batch 维度归一化。

公式: $y = \frac{x - \mu}{\sqrt{\sigma^2 + \epsilon}} \cdot \gamma + \beta$

```python
class LayerNorm(nn.Module):
    def __init__(self, d_model: int, eps: float = 1e-5):
        super().__init__()
        self.gamma = nn.Parameter(torch.ones(d_model))
        self.beta = nn.Parameter(torch.zeros(d_model))
        self.eps = eps

    def forward(self, x):
        # x: (batch, seq_len, d_model)
        mean = x.mean(dim=-1, keepdim=True)
        var = x.var(dim=-1, keepdim=True, unbiased=False)
        x_norm = (x - mean) / torch.sqrt(var + self.eps)
        return self.gamma * x_norm + self.beta
```

**面试要点**:
- `unbiased=False`: LN 用总体方差而非样本方差 (除以 N 而非 N-1)
- 为什么 LLM 用 LayerNorm 而不是 BatchNorm? 序列长度不一，batch 维度统计不稳定；LN 对每个 token 独立归一化
- Post-Norm vs Pre-Norm: 现代 LLM 基本都用 Pre-Norm (先归一化再做 Attention/FFN)，训练更稳定

## 4. RMSNorm

**思路**: LayerNorm 的简化版，去掉减均值的步骤，只用均方根 (Root Mean Square) 做归一化。计算更快，效果相当。LLaMA 系列标配。

公式: $y = \frac{x}{\text{RMS}(x)} \cdot \gamma, \quad \text{RMS}(x) = \sqrt{\frac{1}{d}\sum_{i=1}^{d}x_i^2}$

```python
class RMSNorm(nn.Module):
    def __init__(self, d_model: int, eps: float = 1e-6):
        super().__init__()
        self.weight = nn.Parameter(torch.ones(d_model))
        self.eps = eps

    def forward(self, x):
        # x: (batch, seq_len, d_model)
        rms = torch.sqrt(x.pow(2).mean(dim=-1, keepdim=True) + self.eps)
        return x / rms * self.weight
```

**面试要点**:
- 相比 LayerNorm 少了减均值和加 beta，参数量减半，速度提升约 10-15%
- 没有 beta 参数 (无平移)，只有 gamma (缩放)

## 5. RoPE

**思路**: 旋转位置编码。不是把位置信息加到 embedding 上，而是对 Q/K 做旋转操作，使得 $q_m^T k_n$ 只依赖于相对位置 $m-n$。

核心思想: 将 d 维向量两两配对，每对做一个 2D 旋转，旋转角度与位置成正比。

```python
class RotaryPositionEmbedding(nn.Module):
    def __init__(self, d_k: int, max_len: int = 4096, base: float = 10000.0):
        super().__init__()
        # 频率: theta_i = base^(-2i/d), i=0,1,...,d/2-1
        freqs = 1.0 / (base ** (torch.arange(0, d_k, 2).float() / d_k))
        # 位置 x 频率 -> 角度
        t = torch.arange(max_len).float()
        angles = torch.outer(t, freqs)  # (max_len, d_k/2)
        # 预计算 cos 和 sin
        self.register_buffer('cos_cached', angles.cos())  # (max_len, d_k/2)
        self.register_buffer('sin_cached', angles.sin())

    def forward(self, q, k):
        """q, k: (batch, num_heads, seq_len, d_k)"""
        L = q.shape[2]
        cos = self.cos_cached[:L]  # (L, d_k/2)
        sin = self.sin_cached[:L]

        q_rot = self._apply_rope(q, cos, sin)
        k_rot = self._apply_rope(k, cos, sin)
        return q_rot, k_rot

    def _apply_rope(self, x, cos, sin):
        # 拆成前半和后半
        x1, x2 = x[..., ::2], x[..., 1::2]    # 偶数位, 奇数位
        # 旋转: [x1, x2] -> [x1*cos - x2*sin, x1*sin + x2*cos]
        out1 = x1 * cos - x2 * sin
        out2 = x1 * sin + x2 * cos
        return torch.stack([out1, out2], dim=-1).flatten(-2)
```

**面试要点**:
- RoPE 的优势: 天然编码相对位置、外推性好、无需额外参数
- `base` 越大，高频分量越少，长文本外推能力越强 (NTK-aware Scaling 就是调 base)
- LLaMA、Qwen、Mistral 等均使用 RoPE

## 6. SwiGLU FFN

**思路**: 现代 LLM 的 FFN 不再用简单的 ReLU，而是用 SwiGLU 激活。包含门控机制，一路做激活一路做门控，然后逐元素相乘。

$\text{SwiGLU}(x) = (\text{Swish}(xW_1)) \odot (xW_3) \cdot W_2$

其中 $\text{Swish}(x) = x \cdot \sigma(x)$，即 `F.silu()`

```python
class SwiGLU_FFN(nn.Module):
    def __init__(self, d_model: int, d_ff: int = None):
        super().__init__()
        # 通常 d_ff = 8/3 * d_model (LLaMA 风格)
        if d_ff is None:
            d_ff = int(8 / 3 * d_model)
            d_ff = ((d_ff + 63) // 64) * 64  # 对齐到 64 的倍数

        self.w1 = nn.Linear(d_model, d_ff, bias=False)  # gate
        self.w3 = nn.Linear(d_model, d_ff, bias=False)  # up
        self.w2 = nn.Linear(d_ff, d_model, bias=False)  # down

    def forward(self, x):
        return self.w2(F.silu(self.w1(x)) * self.w3(x))
```

**面试要点**:
- 为什么 SwiGLU 比 ReLU 好? 门控机制让网络更有表达力，Swish 平滑可导
- 参数量: 标准 FFN 用 `4d * d * 2 = 8d²`，SwiGLU 用 `(8/3)d * d * 3 = 8d²`，参数量相当但效果更好
- `bias=False` 是 LLaMA 风格，减少参数

## 7. Softmax

**思路**: 将任意实数向量转为概率分布。实现时需要减去最大值防止数值溢出。

```python
def softmax(x, dim=-1):
    """数值稳定的 softmax"""
    x_max = x.max(dim=dim, keepdim=True).values
    x_exp = torch.exp(x - x_max)
    return x_exp / x_exp.sum(dim=dim, keepdim=True)
```

**面试要点**:
- 为什么要减最大值? `exp(x)` 在 x 很大时会溢出 (inf)，减去最大值后最大的 exp 值为 1
- 温度参数: `softmax(x/T)`，T 越大分布越平滑，T 越小越尖锐 (趋近 argmax)

## 8. 交叉熵损失

**思路**: LLM 的训练损失。先对 logits 做 softmax 得到概率，再取真实标签对应位置的负对数。

```python
def cross_entropy(logits, targets):
    """
    logits: (batch, vocab_size) 未归一化的分数
    targets: (batch,) 真实标签索引
    """
    # 数值稳定: log_softmax = x - max - log(sum(exp(x - max)))
    log_probs = logits - logits.max(dim=-1, keepdim=True).values
    log_probs = log_probs - torch.log(torch.exp(log_probs).sum(dim=-1, keepdim=True))

    # 取出真实标签对应的 log probability
    B = logits.shape[0]
    loss = -log_probs[torch.arange(B), targets]
    return loss.mean()
```

**面试要点**:
- PyTorch 的 `F.cross_entropy` 内部就是 log_softmax + nll_loss
- 不要先算 softmax 再取 log，直接算 log_softmax 数值更稳定

## 9. KL 散度损失

**思路**: 衡量两个概率分布 P 和 Q 的差异。在知识蒸馏中，P 是教师模型的软标签，Q 是学生模型的预测。

公式: $D_{KL}(P \| Q) = \sum_i P(i) \log \frac{P(i)}{Q(i)}$

```python
def kl_divergence(logits_student, logits_teacher, temperature=1.0):
    """
    知识蒸馏中的 KL Loss
    logits_student: (batch, vocab_size) 学生模型的 logits
    logits_teacher: (batch, vocab_size) 教师模型的 logits
    temperature: 温度参数，越大分布越平滑，蒸馏效果越好
    """
    # 用温度缩放 logits
    p = F.softmax(logits_teacher / temperature, dim=-1)       # 教师分布
    log_q = F.log_softmax(logits_student / temperature, dim=-1)  # 学生 log 分布

    # KL = sum(p * (log_p - log_q)) = sum(p * log_p) - sum(p * log_q)
    # 等价于: KL = sum(p * log(p/q))
    kl = F.kl_div(log_q, p, reduction='batchmean')

    # 温度补偿: 梯度需要乘以 T^2
    return kl * (temperature ** 2)


def kl_divergence_manual(logits_student, logits_teacher, temperature=1.0):
    """不用 F.kl_div 的纯手撕版本"""
    log_p = F.log_softmax(logits_teacher / temperature, dim=-1)
    log_q = F.log_softmax(logits_student / temperature, dim=-1)
    p = log_p.exp()

    kl = (p * (log_p - log_q)).sum(dim=-1).mean()
    return kl * (temperature ** 2)
```

**面试要点**:
- KL 散度不对称: $D_{KL}(P\|Q) \neq D_{KL}(Q\|P)$。蒸馏用 forward KL (教师 P → 学生 Q)
- 为什么乘 $T^2$? 温度缩放后梯度被缩小了 $T^2$ 倍，需要补偿回来
- `F.kl_div` 的坑: 第一个参数是 **log 概率** (学生)，第二个是 **概率** (教师)，顺序容易搞反
- 蒸馏总 Loss 通常是: $\mathcal{L} = \alpha \cdot \text{KL} + (1-\alpha) \cdot \text{CE}$
- Forward KL vs Reverse KL: forward KL 倾向 mean-seeking (覆盖所有模式)，reverse KL 倾向 mode-seeking (集中某个模式)

## 10. Transformer Decoder Block

**思路**: 把上面的组件组装成一个完整的 Decoder Block。现代 LLM 使用 Pre-Norm 结构。

```
x -> RMSNorm -> Attention -> + (残差) -> RMSNorm -> SwiGLU FFN -> + (残差)
```

```python
class TransformerDecoderBlock(nn.Module):
    def __init__(self, d_model: int, num_heads: int, num_kv_heads: int = None):
        super().__init__()
        if num_kv_heads is None:
            num_kv_heads = num_heads

        self.norm1 = RMSNorm(d_model)
        self.attn = GroupedQueryAttention(d_model, num_heads, num_kv_heads)
        self.norm2 = RMSNorm(d_model)
        self.ffn = SwiGLU_FFN(d_model)

    def forward(self, x, mask=None):
        # Pre-Norm + 残差连接
        x = x + self.attn(self.norm1(x), mask=mask)
        x = x + self.ffn(self.norm2(x))
        return x
```

## 11. KV Cache

**思路**: 自回归生成时，每步只新增一个 token，但 Attention 需要看到所有历史 token。KV Cache 缓存已计算的 K/V，避免重复计算。

```python
class CachedAttention(nn.Module):
    def __init__(self, d_model: int, num_heads: int):
        super().__init__()
        self.num_heads = num_heads
        self.d_k = d_model // num_heads
        self.W_q = nn.Linear(d_model, d_model)
        self.W_k = nn.Linear(d_model, d_model)
        self.W_v = nn.Linear(d_model, d_model)
        self.W_o = nn.Linear(d_model, d_model)

    def forward(self, x, kv_cache=None):
        """
        x: (B, 1, d_model) 生成阶段每次只输入新 token
        kv_cache: (past_k, past_v) 缓存的历史 K/V
        返回: output, new_kv_cache
        """
        B, L, _ = x.shape

        Q = self.W_q(x).view(B, L, self.num_heads, self.d_k).transpose(1, 2)
        K = self.W_k(x).view(B, L, self.num_heads, self.d_k).transpose(1, 2)
        V = self.W_v(x).view(B, L, self.num_heads, self.d_k).transpose(1, 2)

        # 拼接历史 KV
        if kv_cache is not None:
            past_k, past_v = kv_cache
            K = torch.cat([past_k, K], dim=2)  # 在 seq_len 维度拼接
            V = torch.cat([past_v, V], dim=2)

        new_kv_cache = (K, V)

        # 计算注意力 (Q 只有当前步, K/V 是全部历史)
        scores = torch.matmul(Q, K.transpose(-2, -1)) / math.sqrt(self.d_k)
        attn = F.softmax(scores, dim=-1)
        out = torch.matmul(attn, V)

        out = out.transpose(1, 2).contiguous().view(B, L, -1)
        return self.W_o(out), new_kv_cache
```

**面试要点**:
- Prefill 阶段: 输入完整 prompt，缓存所有 K/V
- Decode 阶段: 每步输入 1 个 token，只计算新 K/V 并追加到 cache
- KV Cache 显存 = `2 * num_layers * num_kv_heads * d_k * seq_len * batch * dtype_size`

## 12. Beam Search

**思路**: 每步保留概率最高的 `beam_width` 个候选序列，扩展后再选 top-k。比贪心搜索效果更好。

```python
def beam_search(model, input_ids, beam_width=4, max_len=50, eos_id=2):
    """
    model: 输入 token ids, 输出 logits
    input_ids: (1, prompt_len)
    """
    B = 1
    # 初始化: 每个 beam 都从相同的 input 开始
    beams = [(input_ids, 0.0)]  # (序列, 累积 log prob)
    finished = []

    for _ in range(max_len):
        all_candidates = []
        for seq, score in beams:
            if seq[0, -1].item() == eos_id:
                finished.append((seq, score))
                continue

            logits = model(seq)[:, -1, :]           # (1, vocab_size)
            log_probs = F.log_softmax(logits, dim=-1)
            topk_logp, topk_ids = log_probs.topk(beam_width)

            for i in range(beam_width):
                new_seq = torch.cat([seq, topk_ids[:, i:i+1]], dim=1)
                new_score = score + topk_logp[0, i].item()
                all_candidates.append((new_seq, new_score))

        # 选出得分最高的 beam_width 个
        all_candidates.sort(key=lambda x: x[1], reverse=True)
        beams = all_candidates[:beam_width]

        if len(beams) == 0:
            break

    # 从 finished 和 beams 中选最优
    all_seqs = finished + beams
    all_seqs.sort(key=lambda x: x[1], reverse=True)
    return all_seqs[0][0]
```

## 13. Top-k / Top-p 采样

**思路**:
- **Top-k**: 只从概率最大的 k 个 token 中采样
- **Top-p (Nucleus)**: 按概率从大到小累加，当累积概率超过 p 时截断

```python
def top_k_top_p_sampling(logits, top_k=50, top_p=0.9, temperature=1.0):
    """
    logits: (vocab_size,) 单步的 logits
    """
    logits = logits / temperature

    # Top-k 过滤
    if top_k > 0:
        topk_vals, _ = logits.topk(top_k)
        threshold_k = topk_vals[-1]
        logits = logits.masked_fill(logits < threshold_k, float('-inf'))

    # Top-p 过滤
    if top_p < 1.0:
        sorted_logits, sorted_indices = logits.sort(descending=True)
        cum_probs = F.softmax(sorted_logits, dim=-1).cumsum(dim=-1)

        # 找到累积概率超过 p 的位置, 将后续的置为 -inf
        remove_mask = cum_probs - F.softmax(sorted_logits, dim=-1) >= top_p
        sorted_logits[remove_mask] = float('-inf')

        # 还原顺序
        logits = sorted_logits.scatter(0, sorted_indices, sorted_logits)

    probs = F.softmax(logits, dim=-1)
    return torch.multinomial(probs, num_samples=1)
```

**面试要点**:
- Top-k 的问题: 对分布平坦/尖锐的情况不自适应
- Top-p 更灵活: 分布集中时只选几个, 分布平坦时选更多
- 实际推理通常 Top-k + Top-p 同时使用

## 14. LoRA

**思路**: 冻结原始权重 W，只训练低秩分解矩阵 A 和 B。推理时 $W' = W + \frac{\alpha}{r} BA$，可合并回原权重无额外开销。

```python
class LoRALinear(nn.Module):
    def __init__(self, original_linear: nn.Linear, r: int = 8, alpha: float = 16.0):
        super().__init__()
        self.original = original_linear
        self.original.weight.requires_grad_(False)  # 冻结原始权重
        if self.original.bias is not None:
            self.original.bias.requires_grad_(False)

        in_features = original_linear.in_features
        out_features = original_linear.out_features
        self.r = r
        self.scaling = alpha / r

        # A: 高斯初始化, B: 零初始化 -> 初始 delta=0
        self.lora_A = nn.Parameter(torch.randn(r, in_features) * 0.01)
        self.lora_B = nn.Parameter(torch.zeros(out_features, r))

    def forward(self, x):
        # 原始输出 + LoRA 增量
        base_out = self.original(x)
        lora_out = F.linear(F.linear(x, self.lora_A), self.lora_B) * self.scaling
        return base_out + lora_out

    def merge_weights(self):
        """推理时将 LoRA 合并回原始权重"""
        self.original.weight.data += (self.lora_B @ self.lora_A) * self.scaling
```

**面试要点**:
- 为什么 B 零初始化? 保证训练开始时 delta=0，不破坏预训练权重
- `r` 通常取 8~64，太小表达力不够，太大失去效率优势
- LoRA 通常只加在 Attention 的 Q/V 投影上 (实验表明效果最好)
- QLoRA: 在 LoRA 基础上将原始权重量化到 4bit，进一步省显存
