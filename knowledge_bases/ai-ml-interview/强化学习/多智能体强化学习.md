# 多智能体强化学习 (Multi-Agent RL, MARL)

多个智能体在共享环境中同时学习和交互。

## 1. 问题设定

### 马尔可夫博弈 (Markov Game / Stochastic Game)

MDP 的多智能体扩展，定义为 $(N, \mathcal{S}, \{\mathcal{A}_i\}, P, \{R_i\}, \gamma)$：

| 符号 | 含义 |
|------|------|
| $N$ | 智能体数量 |
| $\mathcal{S}$ | 全局状态空间 |
| $\mathcal{A}_i$ | 智能体 $i$ 的动作空间 |
| $P(s' \mid s, a_1, \ldots, a_N)$ | 联合状态转移 |
| $R_i(s, a_1, \ldots, a_N)$ | 智能体 $i$ 的奖励函数 |

### 任务类型

| 类型 | 奖励关系 | 例子 |
|------|---------|------|
| **完全合作** | $R_1 = R_2 = \cdots = R_N$ | 机器人协作搬运 |
| **完全竞争（零和）** | $R_1 = -R_2$ | 围棋、StarCraft 对战 |
| **混合（一般和）** | 各智能体奖励独立 | 自动驾驶交通流 |

## 2. 核心挑战

### 非平稳环境 (Non-Stationarity)

每个智能体的策略都在变化，导致环境从其他智能体的视角看是**非平稳的**。打破了 MDP 的基本假设。

### 信用分配 (Credit Assignment)

团队获得共同奖励时，如何判断每个智能体的贡献？

### 可扩展性

联合动作空间 $|\mathcal{A}|^N$ 随智能体数量指数增长。

### 部分可观测

每个智能体通常只能观测到局部信息。

## 3. CTDE 范式

**Centralized Training, Decentralized Execution**（集中训练，分散执行）。

- **训练时**：可以访问全局状态和所有智能体信息
- **执行时**：每个智能体只能用自己的局部观测做决策

```
训练阶段:   全局状态 s → 中央 Critic → 指导各智能体学习
执行阶段:   局部观测 oᵢ → 各自的策略 πᵢ → 各自的动作 aᵢ
```

## 4. 值分解方法

### VDN (Value Decomposition Network)

将联合 Q 函数分解为各智能体 Q 值之和：

$$Q_{\text{total}}(s, \mathbf{a}) = \sum_{i=1}^{N} Q_i(o_i, a_i)$$

简单但限制大——只能表示加性关系。

### QMIX

放宽到单调混合：

$$Q_{\text{total}} = f_{\text{mix}}(Q_1, Q_2, \ldots, Q_N; s)$$

约束：$\frac{\partial Q_{\text{total}}}{\partial Q_i} \geq 0$（单调性）

通过 **Hypernetwork** 生成混合网络的权重（权重非负），保证单调性。

```
Q₁(o₁, a₁) ─┐
Q₂(o₂, a₂) ─┤── Mixing Network ──→ Q_total
Q₃(o₃, a₃) ─┘   (权重由 s 通过
                   Hypernetwork 生成)
```

**单调性保证**：$\arg\max_\mathbf{a} Q_{\text{total}} = (\arg\max_{a_1} Q_1, \ldots, \arg\max_{a_N} Q_N)$，即每个智能体可以独立贪心。

## 5. 多智能体策略梯度

### MAPPO (Multi-Agent PPO)

Yu et al., 2022。将 PPO 直接应用到多智能体设定：

- 每个智能体有自己的 Actor $\pi_i(a_i \mid o_i)$
- 共享一个中央 Critic $V(s)$（或 $V(o_i)$）
- 用 PPO-Clip 更新每个 Actor

**实践发现**：简单的参数共享 + PPO 在很多 benchmark 上就能取得 SOTA，不需要复杂的 MARL 特殊算法。

### MADDPG

Lowe et al., 2017。多智能体版本的 DDPG：
- 每个智能体有自己的 Actor（分散执行）
- 每个 Critic 可以访问所有智能体的动作（集中训练）

## 6. 通信机制

让智能体学习如何交流：

| 方法 | 通信方式 | 特点 |
|------|---------|------|
| CommNet | 平均池化所有智能体的隐状态 | 简单，全连接通信 |
| IC3Net | 学习何时通信（门控） | 选择性通信 |
| TarMAC | 注意力机制决定听谁的 | 软寻址 |
| DIAL | 通过梯度反向传播优化消息 | 端到端可微 |

## 7. 经典应用场景

| 场景 | 类型 | 代表工作 |
|------|------|---------|
| StarCraft 微操 (SMAC) | 合作 | QMIX, MAPPO |
| Dota 2 (OpenAI Five) | 合作 | 大规模 PPO |
| 围棋 (AlphaGo/Zero) | 竞争 | Self-Play + MCTS |
| 自动驾驶 | 混合 | 交通流模拟 |
| 机器人集群 | 合作 | 编队控制 |

## 8. 面试高频考点

1. **CTDE 范式**：为什么需要集中训练分散执行
2. **非平稳环境**：MARL 与单智能体 RL 的核心区别
3. **QMIX 的单调性约束**：为什么需要，如何保证
4. **MAPPO 为什么有效**：简单方法 + 工程技巧
5. **值分解 vs 策略梯度**在 MARL 中的对比
