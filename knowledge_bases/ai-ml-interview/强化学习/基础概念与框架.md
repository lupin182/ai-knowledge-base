# 强化学习基础概念与框架

## 1. 什么是强化学习

强化学习 (RL) 是机器学习的第三大范式（监督学习、无监督学习、强化学习）。核心思想：

- **智能体 (Agent)** 在 **环境 (Environment)** 中采取 **动作 (Action)**
- 环境返回 **状态 (State)** 和 **奖励 (Reward)**
- 目标：学习一个 **策略 (Policy)**，最大化 **累积奖励**

```
        action aₜ
Agent ─────────────→ Environment
  ↑                      │
  │  state sₜ₊₁          │
  │  reward rₜ₊₁          │
  ←────────────────────────
```

### RL 与监督学习的关键区别

| 维度 | 监督学习 | 强化学习 |
|------|---------|---------|
| 反馈 | 每个样本有正确标签 | 只有标量奖励信号，可能延迟 |
| 数据 | 独立同分布 (i.i.d.) | 序列数据，当前决策影响未来 |
| 目标 | 最小化预测误差 | 最大化累积回报 |
| 探索 | 不需要 | 需要主动探索环境 |

## 2. RL 的数学建模：从问题到框架

RL 本身是一个**思想**——智能体通过试错与环境交互来学习。要把这个思想变成可以推导、可以编程的东西，需要一个数学模型。最常用的模型就是 MDP（马尔可夫决策过程），但它不是唯一的：

| 数学框架 | 适用场景 | 与 MDP 的关系 |
|----------|---------|---------------|
| **MDP** | 完全可观测、单智能体 | 基础框架，绝大多数教科书从这里开始 |
| **POMDP** | 部分可观测（机器人、对话） | MDP + 观测函数，状态看不全 |
| **Dec-POMDP / 马尔可夫博弈** | 多智能体 | 多个智能体各自有策略 |
| **序列决策** | Decision Transformer 等 | 绕开 Bellman 方程，直接做序列预测 |

后面几节先讲 MDP（因为 Bellman 方程、值函数、策略梯度都在这个框架下推导），后续专题再展开其他框架。

> **注意区分**：RL ≠ MDP。RL 是问题，MDP 是描述这个问题的一种（最主流的）数学语言。

## 3. 马尔可夫决策过程 (MDP)

MDP 定义为五元组 $(\mathcal{S}, \mathcal{A}, P, R, \gamma)$：

| 符号 | 含义 | 说明 |
|------|------|------|
| $\mathcal{S}$ | 状态空间 | 所有可能状态的集合 |
| $\mathcal{A}$ | 动作空间 | 所有可能动作的集合 |
| $P(s' \mid s, a)$ | 状态转移概率 | 在状态 $s$ 执行 $a$ 后转移到 $s'$ 的概率 |
| $R(s, a, s')$ | 奖励函数 | 转移时获得的即时奖励 |
| $\gamma \in [0, 1)$ | 折扣因子 | 未来奖励的衰减系数 |

**马尔可夫性质**：未来只取决于当前状态，与历史无关：

$$P(s_{t+1} \mid s_t, a_t, s_{t-1}, a_{t-1}, \ldots) = P(s_{t+1} \mid s_t, a_t)$$

### 马尔可夫假设为什么这么强？

这个假设在很多现实场景中**显然不成立**：

| 场景 | 为什么违反马尔可夫性 |
|------|---------------------|
| 对话系统 | 当前回复取决于整段对话历史，而不仅是上一句话 |
| 股票交易 | 价格走势依赖长期趋势、周期模式，单个时间步远远不够 |
| 自动驾驶 | 仅凭当前一帧图像无法判断其他车辆的速度和意图 |
| 游戏 AI | Atari 游戏中单帧画面无法区分球的运动方向 |

**核心矛盾**：真实世界的绝大多数问题都有历史依赖性，但 MDP 要求"一个状态就够了"。

### 实践中如何绕过这个限制

关键洞察：**马尔可夫性不是关于"观测"的性质，而是关于"状态"的性质**。如果我们把状态定义得足够丰富，让它包含所有必要的历史信息，那马尔可夫性就自然满足了。

**1. 状态堆叠 (Frame Stacking)**

最经典的 trick。DQN 的做法是把最近 4 帧画面拼在一起作为"状态"：

$$s_t = (o_t, o_{t-1}, o_{t-2}, o_{t-3})$$

这样球的速度、方向都可以从状态中推断出来，近似满足马尔可夫性。

**2. 循环网络 (RNN/LSTM)**

用隐状态 $h_t$ 压缩整段历史：

$$h_t = f(h_{t-1}, o_t)$$

理论上 $h_t$ 编码了从开始到现在的所有信息，所以 $(h_t)$ 构成一个（近似的）马尔可夫状态。代表工作：DRQN (Deep Recurrent Q-Network)。

**3. Transformer + 序列决策**

近年来最有影响力的方向。Decision Transformer 直接把 RL 问题建模为序列预测：

$$\text{输入} = (R_1, s_1, a_1, R_2, s_2, a_2, \ldots, R_t, s_t)$$

Transformer 的自注意力机制天然处理长程依赖，**完全不需要马尔可夫假设**。这也是为什么 LLM 对齐（RLHF/PPO）可以工作——语言模型本身就是在处理序列。

**4. Belief State (POMDP 的理论解)**

POMDP 理论中，可以定义一个 **信念状态 (belief state)** $b_t$，它是对真实状态的后验概率分布：

$$b_t(s) = P(s_t = s \mid o_1, a_1, \ldots, o_t)$$

可以证明 $b_t$ 满足马尔可夫性。但实际上 $b_t$ 的维度等于状态空间大小，计算代价极高，所以一般只在小规模问题中使用。

### POMDP

当智能体无法观测完整状态时，MDP 退化为 **部分可观测 MDP (POMDP)**。此时智能体接收的是 **观测 (Observation)** $o_t$ 而非真实状态 $s_t$。

POMDP 在 MDP 五元组基础上额外引入：
- $\Omega$：观测空间
- $O(o \mid s', a)$：观测函数，给定转移后的真实状态和动作，输出观测的概率

**POMDP 与马尔可夫性的关系**：POMDP 的真实状态仍然是马尔可夫的，只是智能体**看不到**真实状态。换句话说，马尔可夫性没有被违反，被违反的是**完全可观测性**。

> **总结**：马尔可夫假设确实很强，但它更像一个"接口约定"——只要你能把状态定义对（frame stacking、RNN hidden state、belief state），就能把非马尔可夫问题转化为马尔可夫问题。实践中几乎没有人直接在原始观测上假设马尔可夫性。

## 4. 策略 (Policy)

策略 $\pi$ 定义了智能体的行为方式：

- **确定性策略**：$a = \pi(s)$，给定状态直接输出动作
- **随机策略**：$a \sim \pi(a \mid s)$，给定状态输出动作的概率分布

大多数现代 RL 算法使用随机策略，因为：
1. 天然包含探索
2. 在某些环境中，随机策略严格优于任何确定性策略（如石头剪刀布）
3. 策略梯度方法需要可微分的策略

## 5. 值函数与 Bellman 方程

### 回报 (Return)

从时刻 $t$ 开始的折扣累积奖励：

$$G_t = r_{t+1} + \gamma r_{t+2} + \gamma^2 r_{t+3} + \cdots = \sum_{k=0}^{\infty} \gamma^k r_{t+k+1}$$

### 状态值函数 (State Value Function)

在策略 $\pi$ 下，从状态 $s$ 出发的期望回报：

$$V^\pi(s) = \mathbb{E}_\pi[G_t \mid s_t = s] = \mathbb{E}_\pi\left[\sum_{k=0}^{\infty} \gamma^k r_{t+k+1} \mid s_t = s\right]$$

### 动作值函数 (Action Value Function / Q函数)

在策略 $\pi$ 下，从状态 $s$ 执行动作 $a$ 后的期望回报：

$$Q^\pi(s, a) = \mathbb{E}_\pi[G_t \mid s_t = s, a_t = a]$$

### 两者的关系

$$V^\pi(s) = \sum_{a \in \mathcal{A}} \pi(a \mid s) \, Q^\pi(s, a)$$

$$Q^\pi(s, a) = R(s,a) + \gamma \sum_{s'} P(s' \mid s, a) \, V^\pi(s')$$

### Bellman 期望方程

值函数的递归分解：

$$V^\pi(s) = \sum_{a} \pi(a \mid s) \left[ R(s,a) + \gamma \sum_{s'} P(s' \mid s, a) \, V^\pi(s') \right]$$

$$Q^\pi(s, a) = R(s,a) + \gamma \sum_{s'} P(s' \mid s, a) \sum_{a'} \pi(a' \mid s') \, Q^\pi(s', a')$$

### Bellman 最优方程

最优策略 $\pi^*$ 对应的值函数满足：

$$V^*(s) = \max_{a} \left[ R(s,a) + \gamma \sum_{s'} P(s' \mid s, a) \, V^*(s') \right]$$

$$Q^*(s, a) = R(s,a) + \gamma \sum_{s'} P(s' \mid s, a) \, \max_{a'} Q^*(s', a')$$

**核心直觉**：最优值函数不再对动作求期望，而是取最大值——因为最优策略总是选最好的动作。

## 6. 探索与利用 (Exploration vs. Exploitation)

RL 的核心困境：

- **利用 (Exploitation)**：选择当前已知最优的动作，最大化短期收益
- **探索 (Exploration)**：尝试未知动作，可能发现更好的策略

### 经典策略

| 策略 | 公式/描述 | 特点 |
|------|----------|------|
| $\epsilon$-greedy | 以 $\epsilon$ 概率随机，$1-\epsilon$ 概率选最优 | 简单有效，最常用 |
| Boltzmann (Softmax) | $P(a) \propto \exp(Q(a)/\tau)$ | 温度 $\tau$ 控制探索程度 |
| UCB | $a = \arg\max_a \left[Q(a) + c\sqrt{\frac{\ln t}{N(a)}}\right]$ | 对不确定性高的动作更乐观 |
| Thompson Sampling | 从后验分布中采样 | 贝叶斯方法，理论最优 |
| 内在奖励 | 好奇心驱动 (ICM)、RND | 适合稀疏奖励环境 |

## 7. RL 算法分类总览

```
                        RL 算法
                     ┌─────┴─────┐
                Model-Free     Model-Based
              ┌────┴────┐         │
          Value-Based  Policy-Based  (学习环境模型)
              │           │         Dyna / MBPO
          Q-Learning   REINFORCE    World Models
          DQN          A2C/A3C      MuZero
          Double DQN   TRPO
          Dueling DQN  PPO
          Rainbow      SAC
                    │
              Actor-Critic
              (两者结合)
```

### 关键分类维度

| 维度 | 选项 | 代表算法 |
|------|------|---------|
| 是否学环境模型 | Model-Free / Model-Based | DQN vs MuZero |
| 策略类型 | On-Policy / Off-Policy | SARSA vs Q-Learning |
| 值函数 vs 策略 | Value-Based / Policy-Based / Actor-Critic | DQN / REINFORCE / PPO |
| 数据来源 | Online / Offline | PPO / CQL |

### On-Policy vs Off-Policy

- **On-Policy**：用当前策略采集数据并更新（SARSA、PPO）
- **Off-Policy**：可以用旧策略的数据更新（Q-Learning、DQN、SAC）

Off-Policy 的优势是**数据效率更高**，可以复用历史经验（Experience Replay）。

## 8. 面试高频考点

2. **$V$ 和 $Q$ 的关系**以及 Bellman 方程的推导
3. **On-Policy vs Off-Policy** 的区别和代表算法
4. **探索与利用**的权衡，$\epsilon$-greedy 为什么有效
5. **折扣因子 $\gamma$** 的作用：$\gamma=0$ 只看即时奖励；$\gamma \to 1$ 更远视
6. **为什么 LLM 对齐用 RL 而不是纯监督学习**：因为"好的回答"难以穷举标注，但人类可以比较好坏
