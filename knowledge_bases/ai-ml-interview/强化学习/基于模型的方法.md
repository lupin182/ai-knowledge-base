# 基于模型的方法 (Model-Based RL)

学习环境的动态模型 $\hat{P}(s' \mid s, a)$ 和 $\hat{R}(s, a)$，利用模型进行规划。

## 1. Model-Free vs Model-Based

| 维度 | Model-Free | Model-Based |
|------|-----------|-------------|
| 是否学环境模型 | 否 | 是 |
| 数据效率 | 低（需要大量真实交互） | 高（可在模型中"想象"） |
| 计算效率 | 高 | 低（需要训练模型+规划） |
| 渐近性能 | 高（不受模型误差影响） | 可能受限于模型精度 |
| 代表场景 | 游戏、模拟环境 | 机器人（真实交互昂贵） |

## 2. Dyna 架构

最经典的 Model-Based 框架（Sutton, 1991）。核心思想：**真实经验 + 模拟经验**。

```
           ┌─────── 真实环境交互 ───────┐
           │                           ↓
        Agent ←── 直接 RL 更新 ←── (s, a, r, s')
           │                           ↑
           └── 查询学习到的模型 ──→ 模拟经验
```

### Dyna-Q 算法

```python
for each step:
    # 1. 真实交互
    a = epsilon_greedy(Q, s)
    s_next, r = env.step(a)

    # 2. 直接 RL（Q-Learning 更新）
    Q[s, a] += alpha * (r + gamma * max(Q[s_next]) - Q[s, a])

    # 3. 更新模型
    model.update(s, a, r, s_next)

    # 4. 模拟规划（额外 n 次更新）
    for _ in range(n):
        s_sim, a_sim = random_previously_visited()
        r_sim, s_next_sim = model.predict(s_sim, a_sim)
        Q[s_sim, a_sim] += alpha * (r_sim + gamma * max(Q[s_next_sim]) - Q[s_sim, a_sim])
```

每次真实交互后做 $n$ 次模拟更新，**数据效率提升 $n$ 倍**。

## 3. MBPO (Model-Based Policy Optimization)

Janner et al., 2019。用学习到的模型生成短 rollout，训练 Model-Free 算法（如 SAC）。

### 核心创新

- **短 rollout**：只在模型中预测几步（如 1~5 步），避免长期误差累积
- **模型集成 (Ensemble)**：训练多个动态模型，用预测不一致性衡量不确定性
- **分支 rollout**：从真实状态出发做短模拟，而非从初始状态做长模拟

### 为什么短 rollout 有效

模型误差会随步数指数级累积：

$$\text{error}(H) \approx O(\epsilon_{\text{model}}^H)$$

短 rollout 限制了误差传播，同时仍然提供了有用的梯度信息。

## 4. World Models

Ha & Schmidhuber, 2018。在潜在空间中学习世界模型。

### 架构

```
真实观测 ──→ VAE Encoder ──→ 潜在状态 z
                              ↓
                         RNN (MDN-RNN)
                              ↓
                         预测下一个 z
                              ↓
                     Controller（线性策略）
```

1. **VAE**：将高维观测压缩为低维潜在表示
2. **MDN-RNN**：在潜在空间中预测动态
3. **Controller**：在"梦境"中训练策略

### 核心优势

- 完全在潜在空间中规划，不需要渲染真实像素
- 可以"做梦"——在模型中生成无限训练数据

## 5. MuZero

DeepMind, 2020。**不需要学习完整的环境动态**，只学习对规划有用的部分。

### 对比

| 算法 | 知道规则 | 学习模型 | 搜索 |
|------|---------|---------|------|
| AlphaGo | 是 | 否 | MCTS |
| AlphaZero | 是 | 否 | MCTS |
| **MuZero** | **否** | **是** | MCTS |

### 三个学习到的函数

1. **表示函数** $h$：$s_0 = h(o_1, \ldots, o_t)$ — 将观测映射为隐状态
2. **动态函数** $g$：$r_k, s_k = g(s_{k-1}, a_k)$ — 在隐空间中预测转移和奖励
3. **预测函数** $f$：$p_k, v_k = f(s_k)$ — 预测策略和值（用于 MCTS）

### 关键创新

MuZero 不试图重建观测（不学 $\hat{o}_{t+1}$），只学习**对决策有用的抽象动态**。这使它能：
- 适用于任何环境（棋类、Atari、连续控制）
- 不受观测复杂度影响

## 6. 近期进展

### IRIS / DIAMOND

将 Transformer 或扩散模型作为 World Model：
- 用自回归 Transformer 预测下一帧
- 在学习到的世界模型中训练 RL 智能体

### Genie / SORA

大规模视频生成模型本质上也是世界模型——它们学习了物理世界的动态规律。
