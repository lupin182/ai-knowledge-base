# 策略梯度方法 (Policy Gradient Methods)

直接参数化策略 $\pi_\theta(a \mid s)$，通过梯度上升最大化期望回报。

## 1. 为什么需要策略梯度

值函数方法的局限：
- **连续动作空间**：$\max_a Q(s,a)$ 难以计算
- **随机策略**：值函数方法天然倾向确定性策略
- **策略本身就是目标**：有时我们直接想要一个策略网络（如 LLM 对齐中的策略模型）

## 2. 策略梯度定理

### 目标函数

最大化策略的期望回报：

$$J(\theta) = \mathbb{E}_{\tau \sim \pi_\theta}\left[\sum_{t=0}^{T} \gamma^t r_t\right] = \mathbb{E}_{\tau \sim \pi_\theta}[R(\tau)]$$

### 策略梯度定理 (Policy Gradient Theorem)

$$\nabla_\theta J(\theta) = \mathbb{E}_{\pi_\theta}\left[\sum_{t=0}^{T} \nabla_\theta \log \pi_\theta(a_t \mid s_t) \cdot \Psi_t\right]$$

其中 $\Psi_t$ 可以是多种形式：

| $\Psi_t$ | 名称 | 方差 |
|-----------|------|------|
| $R(\tau)$ | 轨迹总回报 | 最高 |
| $G_t = \sum_{k=t}^{T} \gamma^{k-t} r_k$ | Reward-to-go | 较低 |
| $G_t - b(s_t)$ | 带 baseline | 更低 |
| $A^\pi(s_t, a_t) = Q^\pi - V^\pi$ | 优势函数 | 最低 |

### 直觉理解

$$\nabla_\theta J = \mathbb{E}\left[\nabla_\theta \log \pi_\theta(a \mid s) \cdot \Psi\right]$$

- $\nabla_\theta \log \pi_\theta(a \mid s)$：让动作 $a$ 在状态 $s$ 下概率增大的方向
- $\Psi$：这个动作有多好
- 合起来：**好的动作概率增大，差的动作概率减小**

### Log-Derivative Trick

推导的关键一步：

$$\nabla_\theta \pi_\theta = \pi_\theta \cdot \frac{\nabla_\theta \pi_\theta}{\pi_\theta} = \pi_\theta \cdot \nabla_\theta \log \pi_\theta$$

这把对概率的梯度转化为对 log 概率的期望，可以用采样估计。

## 3. REINFORCE

最简单的策略梯度算法（Williams, 1992）。

$$\theta \leftarrow \theta + \alpha \sum_{t=0}^{T} \nabla_\theta \log \pi_\theta(a_t \mid s_t) \cdot G_t$$

```python
for episode in range(num_episodes):
    trajectory = collect_trajectory(env, policy)

    for t in range(len(trajectory)):
        G_t = sum(gamma**k * trajectory[t+k].reward
               for k in range(len(trajectory) - t))
        loss -= log_prob[t] * G_t  # 梯度上升 = 最小化负值

    optimizer.zero_grad()
    loss.backward()
    optimizer.step()
```

### REINFORCE with Baseline

减去一个与动作无关的 baseline $b(s)$（通常是 $V(s)$ 的估计）：

$$\nabla_\theta J = \mathbb{E}\left[\nabla_\theta \log \pi_\theta(a_t \mid s_t) \cdot (G_t - b(s_t))\right]$$

- **不改变梯度期望**（因为 $\mathbb{E}[\nabla \log \pi \cdot b(s)] = 0$）
- **显著降低方差**

## 4. Actor-Critic

同时学习策略 (Actor) 和值函数 (Critic)：

- **Actor** $\pi_\theta(a \mid s)$：策略网络，决定行为
- **Critic** $V_\phi(s)$ 或 $Q_\phi(s,a)$：值网络，评估行为

### 优势函数 (Advantage Function)

$$A^\pi(s, a) = Q^\pi(s, a) - V^\pi(s)$$

含义：在状态 $s$ 下，动作 $a$ 比"平均水平"好多少。

实践中用 TD 误差近似优势：

$$\hat{A}_t = r_t + \gamma V_\phi(s_{t+1}) - V_\phi(s_t)$$

### A2C (Advantage Actor-Critic)

同步版本：多个 worker 同步采集数据，计算梯度后平均更新。

### A3C (Asynchronous Advantage Actor-Critic)

异步版本（Mnih et al., 2016）：多个 worker 异步更新共享参数。实践中 A2C 通常足够。

## 5. TRPO (Trust Region Policy Optimization)

策略梯度的问题：步长太大 → 策略崩溃（性能断崖式下降）。

### 核心思想

限制每次更新的策略变化幅度，使用 KL 散度约束：

$$\max_\theta \quad \mathbb{E}_{s,a \sim \pi_{\theta_{\text{old}}}}\left[\frac{\pi_\theta(a \mid s)}{\pi_{\theta_{\text{old}}}(a \mid s)} A^{\pi_{\theta_{\text{old}}}}(s, a)\right]$$

$$\text{s.t.} \quad \mathbb{E}_s\left[D_{\text{KL}}\left(\pi_{\theta_{\text{old}}}(\cdot \mid s) \| \pi_\theta(\cdot \mid s)\right)\right] \leq \delta$$

### 重要性采样 (Importance Sampling)

概率比 $r_t(\theta) = \frac{\pi_\theta(a_t \mid s_t)}{\pi_{\theta_{\text{old}}}(a_t \mid s_t)}$ 允许用旧策略数据评估新策略。

### 求解方法

使用共轭梯度法 + 线搜索，避免直接计算 Fisher 信息矩阵的逆。实现复杂，因此催生了 PPO。

## 6. PPO (Proximal Policy Optimization)

TRPO 的简化版（Schulman et al., 2017），用裁剪代替 KL 约束。**当前最流行的策略梯度算法**。

### PPO-Clip

$$L^{\text{CLIP}}(\theta) = \mathbb{E}_t\left[\min\left(r_t(\theta) \hat{A}_t, \; \text{clip}(r_t(\theta), 1-\epsilon, 1+\epsilon) \hat{A}_t\right)\right]$$

其中 $r_t(\theta) = \frac{\pi_\theta(a_t \mid s_t)}{\pi_{\theta_{\text{old}}}(a_t \mid s_t)}$，$\epsilon$ 通常取 0.1~0.2。

### 裁剪机制的直觉

| 情况 | $\hat{A}_t > 0$（好动作） | $\hat{A}_t < 0$（差动作） |
|------|--------------------------|--------------------------|
| $r_t > 1+\epsilon$ | 裁剪，不再增大 | 不裁剪，继续减小 |
| $r_t < 1-\epsilon$ | 不裁剪，继续增大 | 裁剪，不再减小 |

**效果**：鼓励概率往好的方向变，但不要变太多。

### GAE (Generalized Advantage Estimation)

PPO 中常用的优势估计方法：

$$\hat{A}_t^{\text{GAE}(\gamma, \lambda)} = \sum_{l=0}^{\infty} (\gamma\lambda)^l \delta_{t+l}$$

其中 $\delta_t = r_t + \gamma V(s_{t+1}) - V(s_t)$。$\lambda=0$ 退化为单步 TD，$\lambda=1$ 退化为 MC。

### PPO 完整训练流程

```python
for iteration in range(num_iterations):
    # 1. 用当前策略采集数据
    trajectories = collect_trajectories(env, policy, num_steps=2048)

    # 2. 计算 GAE 优势
    advantages = compute_gae(trajectories, value_network, gamma=0.99, lam=0.95)
    advantages = (advantages - advantages.mean()) / (advantages.std() + 1e-8)

    # 3. 多轮 mini-batch 更新
    for epoch in range(num_epochs=10):  # PPO 允许多轮复用
        for batch in mini_batches(trajectories, batch_size=64):
            ratio = policy.prob(batch.a, batch.s) / batch.old_prob
            surr1 = ratio * batch.advantage
            surr2 = clip(ratio, 1-eps, 1+eps) * batch.advantage
            policy_loss = -min(surr1, surr2).mean()
            value_loss = MSE(value_network(batch.s), batch.returns)

            loss = policy_loss + 0.5 * value_loss - 0.01 * entropy
            optimizer.step(loss)
```

## 7. SAC (Soft Actor-Critic)

最大熵强化学习框架（Haarnoja et al., 2018），在最大化回报的同时最大化策略熵：

$$J(\pi) = \sum_t \mathbb{E}\left[r_t + \alpha \mathcal{H}(\pi(\cdot \mid s_t))\right]$$

- **更强探索**：熵正则化鼓励多样化行为
- **Off-Policy**：使用 replay buffer，数据效率高
- **连续控制 SOTA**：MuJoCo 等连续动作任务的首选

## 8. 算法对比总结

| 算法 | 类型 | On/Off-Policy | 动作空间 | 核心特点 |
|------|------|-------------|---------|---------|
| REINFORCE | Policy Gradient | On | 离散/连续 | 最简单，方差大 |
| A2C/A3C | Actor-Critic | On | 离散/连续 | 并行采集降方差 |
| TRPO | Actor-Critic | On | 离散/连续 | KL 约束，理论保证 |
| **PPO** | Actor-Critic | On | 离散/连续 | 裁剪，简单高效，最常用 |
| SAC | Actor-Critic | Off | 连续 | 最大熵，连续控制首选 |
