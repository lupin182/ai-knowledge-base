# 离线强化学习 (Offline RL)

从固定的历史数据集中学习策略，**不与环境交互**。

## 1. 为什么需要 Offline RL

| 场景 | 为什么不能在线交互 |
|------|-------------------|
| 医疗决策 | 不能在真人身上做 exploration |
| 自动驾驶 | 试错代价太高 |
| 推荐系统 | 已有大量日志数据 |
| LLM 对齐 | 人类反馈数据有限且昂贵 |

Offline RL 的目标：从数据集 $\mathcal{D} = \{(s, a, r, s')\}$ 中学到尽可能好的策略，数据由**行为策略** $\pi_\beta$ 收集。

## 2. 核心挑战：分布偏移 (Distribution Shift)

### 外推误差 (Extrapolation Error)

对于数据集中**没有覆盖到的** $(s, a)$ 对，Q 函数的估计不可靠。标准 Q-Learning 的 $\max_a Q(s, a)$ 会选择这些被高估的 OOD (Out-of-Distribution) 动作。

```
数据覆盖区域:    ████████
策略可能选到的:         ██████  ← OOD 区域，Q 值不可信
```

### 解决思路

| 方法类型 | 思路 | 代表算法 |
|---------|------|---------|
| 策略约束 | 让学到的策略接近行为策略 | BCQ, BEAR, TD3+BC |
| 值函数惩罚 | 对 OOD 动作的 Q 值做悲观估计 | CQL, PBRL |
| 模型约束 | 用模型的不确定性限制 rollout | MOPO, MOReL |
| 序列建模 | 把 RL 当条件序列生成 | Decision Transformer |

## 3. CQL (Conservative Q-Learning)

Kumar et al., 2020。通过正则化让 Q 函数对 OOD 动作给出**保守（低）估计**。

### 目标函数

$$\min_Q \quad \alpha \cdot \underbrace{\left(\mathbb{E}_{a \sim \mu}[Q(s,a)] - \mathbb{E}_{a \sim \hat{\pi}_\beta}[Q(s,a)]\right)}_{\text{降低 OOD Q 值，抬高数据内 Q 值}} + \underbrace{\frac{1}{2}\mathbb{E}_{(s,a,r,s') \sim \mathcal{D}}\left[(Q - \hat{\mathcal{B}}^\pi \hat{Q})^2\right]}_{\text{标准 Bellman 误差}}$$

- 第一项：惩罚 OOD 动作的 Q 值（$\mu$ 是某个宽分布如 uniform）
- 第二项：保证数据内的 Q 值仍然准确

**效果**：学到的 Q 值是真实 Q 值的**下界**，保证策略不会选择数据没覆盖的危险动作。

## 4. IQL (Implicit Q-Learning)

Kostrikov et al., 2022。完全避免查询 OOD 动作，通过**期望回归**只用数据内的动作。

### 核心思想

用分位数回归（expectile regression）拟合 $V(s) \approx \max_a Q(s, a)$，但只使用数据中出现过的动作：

$$L_\tau(V) = \mathbb{E}_{(s,a) \sim \mathcal{D}}\left[|\tau - \mathbf{1}(Q(s,a) - V(s) < 0)| \cdot (Q(s,a) - V(s))^2\right]$$

当 $\tau \to 1$ 时，$V(s)$ 近似数据中最大的 $Q(s,a)$，无需真正执行 max。

## 5. Decision Transformer

Chen et al., 2021。将 RL 重新表述为**序列建模问题**。

### 核心思想

不学值函数，不做 Bellman 更新。直接用 Transformer 建模轨迹：

$$\tau = (\hat{R}_1, s_1, a_1, \hat{R}_2, s_2, a_2, \ldots)$$

其中 $\hat{R}_t$ 是 **Return-to-Go**（从 $t$ 到结束的期望回报）。

### 推理方式

给定目标回报 $\hat{R}_1 = R_{\text{target}}$，自回归生成动作序列：

$$a_t = \text{Transformer}(\hat{R}_1, s_1, a_1, \ldots, \hat{R}_t, s_t)$$

**设定高目标回报 → 模型输出高质量动作**。

### 架构

```
输入序列:   [R̂₁, s₁, a₁, R̂₂, s₂, a₂, ...]
              ↓   ↓   ↓   ↓   ↓   ↓
          ┌─── GPT-style Transformer ───┐
              ↓   ↓   ↓   ↓   ↓   ↓
预测:        [-,  -, â₁,  -,  -, â₂, ...]
```

### 与传统 RL 的对比

| 维度 | 传统 Offline RL | Decision Transformer |
|------|----------------|---------------------|
| 核心机制 | Bellman 更新 + 策略约束 | 自回归序列建模 |
| 需要 Q 函数 | 是 | 否 |
| 稳定性 | 可能有 bootstrapping 误差 | 标准监督学习，稳定 |
| 长期信用分配 | 通过 TD 传播 | Transformer 的注意力机制 |
| 泛化能力 | 受限于训练分布 | 可能泛化到新的 return 条件 |

## 6. Offline RL 与 LLM 的联系

| RL 概念 | LLM 对齐中的对应 |
|---------|-----------------|
| 离线数据集 | 人类反馈数据 / 偏好数据 |
| 行为策略 $\pi_\beta$ | SFT 后的基础模型 |
| 策略约束（不离开数据分布） | DPO/RLHF 中的 KL 惩罚 |
| Decision Transformer | 条件生成（给定高质量前缀） |
| CQL 的悲观主义 | 保守的对齐策略，避免 reward hacking |
