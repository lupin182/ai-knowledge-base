# 大模型技术演进与趋势

> 最后更新：2026-03

## 一、架构演进：从百花齐放到收敛

### 2017-2022：架构探索期

- **2017**: Transformer 提出 (Attention Is All You Need)
- **2018-2019**: BERT vs GPT 路线之争（encoder-only vs decoder-only）
- **2020**: Scaling Laws (Kaplan et al.) 确立"大力出奇迹"的理论基础
- **2020-2022**: 各种架构变体涌现——Sparse Transformer、Performer、Linear Attention、RWKV 等

### 2023-2024：Dense Transformer 架构基本定型

文本 LLM 的主流架构收敛为 **decoder-only Transformer**，以下组件成为标配：

| 组件 | 标准选择 | 说明 |
|------|----------|------|
| 归一化 | RMSNorm (Pre-Norm) | 比 LayerNorm 更高效，Pre-Norm 比 Post-Norm 训练更稳定 |
| 位置编码 | RoPE | 支持长度外推，几乎所有主流模型采用 |
| 注意力 | GQA / MQA | 在质量和推理效率间取得平衡 |
| 激活函数 | SwiGLU | 比 ReLU/GELU 效果更好，已成默认选择 |
| 词表 | BPE (大词表, 100K+) | Llama 3 用 128K 词表，提升多语言和代码能力 |

### MoE 成为扩大规模的主流手段

- **Mixtral 8x7B** (2023.12): 开源 MoE 的里程碑
- **DeepSeek-V2/V3** (2024): MoE + MLA，极致的效率优化
- **Grok** (xAI): 大规模 MoE

MoE 的核心优势：**激活参数远小于总参数**，训练和推理成本大幅降低。

### 挑战者未能颠覆 Transformer

| 架构 | 代表 | 现状 |
|------|------|------|
| 状态空间模型 (SSM) | Mamba, Mamba-2 | 在部分任务上有竞争力，但通用能力仍不及 Transformer |
| 线性注意力 | RWKV, RetNet | 推理效率优势明显，但长距离依赖能力弱 |
| 混合架构 | Jamba (SSM+Transformer) | 试图结合两者优点，尚未成为主流 |

**结论**：截至 2026 年初，Transformer 仍然是无可争议的主流架构。

## 二、2025-2026：竞争维度的转移

当架构不再是核心差异化因素后，竞争转向以下维度：

### 1. 数据工程 — "Data is the new moat"

- **合成数据**成为核心能力：用强模型生成训练数据（蒸馏），再训练弱模型
- **数据质量 >> 数据数量**：DeepSeek 的成功很大程度归功于精细的数据处理 pipeline
- **数据配比**：代码、数学、多语言等不同类型数据的比例对模型能力影响巨大
- Web 数据逐渐被"挖完"，高质量增量数据成为稀缺资源

### 2. 后训练与对齐 — 从 PPO 到新范式

| 方法 | 时间 | 关键创新 |
|------|------|----------|
| RLHF + PPO | 2022 (InstructGPT) | 经典三阶段范式，验证有效但工程复杂 |
| DPO | 2023 | 去掉 RM 和 RL，直接从偏好数据优化，大幅简化 |
| KTO | 2024 | 只需要"好/坏"标签，不需要成对偏好数据 |
| GRPO | 2025 (DeepSeek-R1) | 去掉 Critic，用组内相对奖励，适合推理任务 |
| RLVR | 2025 | 基于可验证奖励的 RL，用规则而非模型打分 |
| PRM (过程奖励模型) | 2024-2025 | 对推理过程的每一步打分，而非只看最终答案 |

**趋势**：后训练方法在变得更简单（工程上）但更精细（效果上）。

### 3. Test-time Compute Scaling — 推理时的新维度

这是 2024-2025 最重大的范式变化之一：

- **OpenAI o1/o3** (2024): 通过长思考链 (CoT) 在推理时投入更多计算
- **DeepSeek-R1** (2025.01): 开源复现，证明 GRPO + 长 CoT 可以涌现推理能力
- **Claude extended thinking** (2025): Anthropic 的类似方案

**核心思想**：模型能力不只取决于参数量（训练时计算），还取决于推理时投入的计算量。让模型"多想一会儿"可以显著提升复杂任务的表现。

这实际上开辟了一个新的 Scaling 维度：
```
传统 Scaling: 更多参数 × 更多训练数据 × 更多训练计算
新 Scaling:   + 更多推理计算 (test-time compute)
```

### 4. Agent — 从对话到执行

- **工具调用** (Tool Use / Function Calling) 成为标配能力
- **多步推理与规划**：复杂任务拆解为子任务
- **代码执行**：模型写代码并执行，通过结果反馈迭代
- **Computer Use**：直接操作 GUI（Claude Computer Use, OpenAI Operator）
- **多 Agent 协作**：多个专门化的 Agent 协同完成复杂工作流

Agent 是 LLM 从"知识工具"向"工作助手"转变的关键。

### 5. RAG 与长上下文

两条路线并行发展：

| 路线 | 进展 | 适用场景 |
|------|------|----------|
| **长上下文** | Gemini 1M+ tokens, Claude 200K, GPT-4 128K | 单文档/少量文档的深度理解 |
| **RAG** | 混合检索、reranking、GraphRAG、agentic RAG | 大规模知识库、实时信息 |

**趋势**：长上下文和 RAG 不是替代关系，而是互补——长上下文解决"能放多少"，RAG 解决"怎么找到相关的"。

### 6. 推理效率 — 让大模型可部署

- **量化**：从 FP16 → INT8 → INT4，甚至 2-bit（能力损失越来越小）
- **投机解码** (Speculative Decoding)：小模型草稿 + 大模型验证，加速 2-3x
- **知识蒸馏**：把大模型能力压缩到小模型（如 Llama 3.2 1B/3B）
- **推理引擎**：vLLM, TensorRT-LLM, SGLang 等持续优化吞吐和延迟

### 7. 多模态 — 远未收敛

与文本架构不同，多模态架构仍在快速演化：

| 方向 | 现状 |
|------|------|
| 视觉编码器 | SigLIP vs InternViT vs 原生像素输入，没有定论 |
| 视频理解 | 帧采样 vs 时序建模，效率和效果难兼顾 |
| 音频/语音 | 实时语音对话 (GPT-4o) 刚起步 |
| 原生多模态 vs 拼接式 | 争论中，原生多模态成本高但上限高 |

## 三、总结

> **文本 LLM 的单次前向传播架构基本收敛，竞争维度从"模型结构"转向"数据 × 后训练 × 推理策略 × 系统工程"。**

```
2017-2022: 架构探索期   → "什么结构最好？"
2023-2024: 架构收敛期   → "Transformer + MoE 就是答案"
2025-2026: 系统工程期   → "怎么训得更好、对齐得更好、用得更好？"
```

未来的核心竞争力不在于谁有更花哨的架构，而在于：
1. **数据飞轮**：高质量数据的获取和生产能力
2. **对齐技术**：让模型真正有用且安全
3. **系统级创新**：推理时计算、Agent 框架、部署效率
4. **垂直场景落地**：在具体行业和任务上做到最优
