# 06 · 高斯 + 物理仿真

3DGS 的高斯天然像粒子。把它和粒子物理（MPM、SPH、弹簧质点）耦合起来，就能"看一段视频，得到一个能仿真的 3D 场景"——这是 2024 年最有意思的 GS 应用方向之一，也是 real-to-sim-to-real 流水线的关键拼图。

> **核心论文**:
> - PhysGaussian (Xie et al., CVPR 2024)
> - Spring-Gaus (Zhong et al., CVPR 2024)
> - GaussianSplashing (Feng et al., CVPR 2024)
> - VR-GS (Jiang et al., SIGGRAPH 2024)
> - PhysDreamer (Zhang et al., ECCV 2024)

---

## 一、为什么 GS 适合物理

NeRF 时代做"重建 + 物理"困难——MLP 是黑箱，物理引擎需要明确的 mesh / particle / SDF。GS 改变了三件事：

1. **粒子化天然**：每个高斯有位置、形状、密度，本身就像一个 MPM 粒子，把它喂进 MPM 引擎几乎不需要转换
2. **可微 rasterizer**：物理仿真后的高斯能直接渲染回图像，可微链路完整——能优化材料参数让仿真匹配视频
3. **重建质量高**：粒子起始位置由 GS 重建给出，比手工建模 + 撒粒子准确得多

这一切让 **"看视频 → 重建 GS → 推断材料 → 仿真"** 成为可能。

---

## 二、PhysGaussian：GS 等于 MPM 粒子

### 2.1 核心想法

PhysGaussian 是这条路线的开山工作。一句话总结：**3D 高斯的中心 + 协方差就是 MPM 粒子的位置 + 形变梯度**。

### 2.2 形变 MPM 简介

**Material Point Method (MPM)**：粒子-网格混合方法。每个粒子有位置 $\mathbf{x}_p$、速度 $\mathbf{v}_p$、形变梯度 $\mathbf{F}_p \in \mathbb{R}^{3\times3}$、质量 $m_p$。每个时间步：

1. **P2G**：粒子状态投到背景网格
2. **网格更新**：网格上做动量守恒积分（牛顿第二定律 + 弹性力）
3. **G2P**：网格状态插回粒子，更新 $\mathbf{x}_p, \mathbf{v}_p, \mathbf{F}_p$

MPM 能处理弹性、塑性、断裂、流体等多种材料，而所有信息都存在粒子上——**与 GS 的高斯结构高度同构**。

### 2.3 GS ↔ MPM 映射

PhysGaussian 把每个高斯当一个 MPM 粒子：

| GS 参数 | MPM 解释 |
|---------|---------|
| 位置 $\boldsymbol{\mu}$ | 粒子位置 $\mathbf{x}_p$ |
| 协方差 $\boldsymbol{\Sigma}$ | 形变后的体积分布，与 $\mathbf{F}_p \mathbf{F}_p^T$ 关联 |
| α、SH 系数 | 仿真过程中保持不变（材料属性不影响外观） |

**关键约束**：协方差按 MPM 形变梯度变换：
$$\boldsymbol{\Sigma}(t+\Delta t) = \mathbf{F}_p \boldsymbol{\Sigma}(t) \mathbf{F}_p^T$$

这让"高斯随物理变形"自动有了几何意义——形变梯度大的地方高斯被拉长。

### 2.4 用法

PhysGaussian 给重建好的 GS 场景**手动指定材料属性**（杨氏模量、泊松比、密度），就能跑物理：

- 弹性材料（果冻、橡胶）
- 塑性材料（橡皮泥）
- 颗粒材料（沙子）
- 流体（用粘性 MPM）

仿真后用标准 GS rasterizer 渲染。视觉效果非常震撼——"重建好的乐高积木实际能从架子上掉下来"。

### 2.5 局限

- **材料属性需要手动指定**：用户得知道杨氏模量是多少
- **MPM 网格分辨率限制精度**：细节运动易糊
- **仿真不可微（默认实现）**：不能从视频反推材料参数

## 三、Spring-Gaus：弹簧质点法

### 3.1 思路

PhysGaussian 用 MPM，需要背景网格。Spring-Gaus 走更轻的路线：**每个高斯绑成弹簧质点系统**。

- 每个高斯是一个质点（有质量）
- 邻近高斯之间用**结构弹簧**连接（保形）
- 整个系统按 Verlet / Symplectic Euler 积分

### 3.2 邻接图构建

- 用 KNN（k=8-12）建立高斯之间的弹簧连接
- 弹簧自然长度 = 重建时的距离
- 弹簧刚度 = 学习参数

### 3.3 关键能力：可微 + 学习材料

Spring-Gaus 的弹簧仿真**整段可微**，意味着可以从视频反推材料：

$$\theta^* = \arg\min_\theta \sum_t \| \text{Render}(\text{Sim}(\theta, t)) - I_t \|_1$$

输入一段视频（例如布料飘动），训练 → 学到每条弹簧的刚度。这就是 [03 篇](03-动态与4DGS.md)说的"看视频反推物理"的早期版本。

### 3.4 与 PhysGaussian 对比

| 维度 | PhysGaussian (MPM) | Spring-Gaus (弹簧) |
|------|---------------------|---------------------|
| 适合材料 | 弹塑性 / 颗粒 / 流体 | 弹性 / 布料 |
| 表达力 | 强（连续介质） | 中等（离散质点） |
| 仿真速度 | 慢（背景网格） | 快（图遍历） |
| 可微性 | 部分可微 | 完全可微 |
| 材料参数推断 | 手动 | 学习 |

## 四、GaussianSplashing：流体 + GS

### 4.1 想法

让一杯水**真的流过**重建的场景。GaussianSplashing 把 GS 和 PIC/FLIP 流体仿真耦合：

- 重建的场景高斯保持不变（作为静态边界）
- 流体粒子（独立的高斯集合）按 PIC/FLIP 演化
- 流体粒子和场景高斯之间做碰撞检测
- 渲染时统一 rasterize

### 4.2 工程关键

- **碰撞检测**：基于场景高斯的 SDF 近似
- **流体高斯材质**：透明 + 高折射率（fake refraction），渲染出水的视觉效果
- **粒子生成与回收**：源点持续生成水粒子，逃出场景的回收

### 4.3 应用

- 影视特效（在重建的真实场景里加水/烟雾/火）
- AR 游戏（用户客厅里的虚拟泼水效果）

## 五、VR-GS：实时交互

PhysGaussian 仿真离实时还远（一秒仿真要几秒到几分钟）。**VR-GS** 把这个推到实时：

- **二阶有限元 (XPBD)**：替代 MPM，更适合实时交互
- **GPU 并行**：所有粒子并行积分
- **手部跟踪输入**：在 VR/AR 设备里直接戳一下重建的物体，它就形变 / 弹跳

VR-GS 在 SIGGRAPH 2024 现场 demo 时是 "重建一个柔软玩偶 → VR 里直接捏" 这种效果。

## 六、PhysDreamer / DreamPhysics：LLM 控制材料

材料参数（杨氏模量、密度）是物理量，但用户可能只说"这是塑料"或"这是金属"。**PhysDreamer / DreamPhysics** 用 LLM 把语义映射到数值：

```
用户：这个物体是橡皮糖
        │
        ▼
   LLM (GPT-4)
        │
        ▼
   { 杨氏模量: 5e4 Pa,
     密度: 1200 kg/m³,
     泊松比: 0.45,
     塑性阈值: 0.1 }
        │
        ▼
   PhysGaussian / Spring-Gaus 仿真
```

进一步：DreamPhysics 用 SDS 直接从视频蒸馏材料参数（结合 [05 篇](05-生成式高斯.md)的 SDS 思路）——给一段"杯子摔碎"的目标视频，反推这是什么材料。

## 七、Real-to-Sim-to-Real Pipeline

把这些工作串起来，得到完整的 real-to-sim-to-real 流水线：

```
1. 拍摄真实场景（多视角图像/视频）
        │
        ▼
2. 3DGS / 2DGS 重建 ([01][02])
        │
        ▼
3. 材料属性推断
   ├── 手动（PhysGaussian 默认）
   ├── 学习（Spring-Gaus from video）
   └── LLM 语义（PhysDreamer）
        │
        ▼
4. 物理仿真（MPM / 弹簧 / FLIP）
        │
        ▼
5. 仿真状态渲染回图像（GS rasterizer）
        │
        ▼
6. 训练 policy / 生成数据 ([08])
```

这条线是机器人 manipulation 数据生成的关键基础设施，[08 篇](08-高斯+具身.md)展开。

## 八、面试高频问题

### Q1: 为什么 GS 和 MPM 能直接耦合？

**答**：MPM 粒子需要：位置、形变梯度、质量、材料参数。GS 高斯天然提供：位置（$\boldsymbol{\mu}$）、形变（$\boldsymbol{\Sigma}$ 和 $\mathbf{F}\mathbf{F}^T$ 同构）、可推断质量（α + 体积）。两边的"粒子"几乎是同一个东西，把高斯位置喂进 MPM 引擎、再用形变梯度更新协方差，就完成了仿真-渲染闭环。SH 系数和 α 在仿真中不变（外观不受形变影响）。

### Q2: PhysGaussian 和 Spring-Gaus 的本质差别？

**答**：
- **PhysGaussian (MPM)**：连续介质方法，背景网格 + 粒子，能处理弹塑性 / 颗粒 / 流体，表达力强但仿真慢、默认不可微。
- **Spring-Gaus (弹簧)**：离散质点 + 显式弹簧，速度快、完全可微，但只擅长弹性 / 布料类材料，不能处理流体或大塑性变形。

### Q3: 怎么从视频反推材料参数？

**答**：用可微仿真 + 渲染损失。给定参考视频 $\{I_t\}$ 和初始重建 GS，把材料参数 $\theta$ 设为可学习变量，每个时间步跑可微仿真 + GS rasterize 得到渲染图，与 $I_t$ 算 L1/L2 损失，反传更新 $\theta$。Spring-Gaus 是这条路线的早期实现，DreamPhysics 用 SDS 做更鲁棒的版本。

### Q4: GaussianSplashing 怎么处理流体和场景的交互？

**答**：场景高斯作为**静态碰撞边界**，从场景高斯的 α 场近似 SDF，用于流体粒子的碰撞响应。流体本身是独立的高斯粒子集合，按 PIC/FLIP 演化。渲染时把场景 + 流体的所有高斯一起 rasterize，流体高斯设高折射率参数模拟水的视觉。这种"分两套高斯 + 共享渲染"的设计是常见模式。

### Q5: 这条线对 robotics 的最大意义是什么？

**答**：让"重建真实场景 → 在仿真里做 manipulation → 把 policy 部署回真机"的循环可行。传统 sim 用 URDF / 手工建模，与真实环境差距大；用 GS 重建 + PhysGaussian 仿真，环境视觉外观和物理行为都从真实数据来，sim-to-real gap 显著缩小。结合 [08 篇](08-高斯+具身.md)的 SplatSim / RoboGSim，这是 2024-2026 真正能上规模的机器人数据生成方法。
