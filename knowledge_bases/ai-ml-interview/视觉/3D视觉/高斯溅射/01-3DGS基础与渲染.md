# 01 · 3DGS 基础与渲染

3D Gaussian Splatting (3DGS) — 用显式 3D 高斯基元表示场景，通过可微光栅化实现实时渲染。这一篇是整个高斯溅射专题的**地基**：原版 3DGS 的表示、投影、光栅化、密度控制、训练流程。后续的 2DGS / 4DGS / 加速 / 生成 / 物理 / SLAM / 具身专题都假设你已经掌握本篇内容。

> **核心论文**: *3D Gaussian Splatting for Real-Time Radiance Field Rendering* (Kerbl et al., SIGGRAPH 2023)
> **本专题导航**: [README](README.md)

## 一、背景与动机

### 1.1 NeRF 的瓶颈

NeRF (Neural Radiance Fields) 在新视角合成上取得了惊人质量，但存在根本性的工程瓶颈：

- **渲染慢**：每个像素需要沿射线采样 ~64–192 个点，每个点都要做一次 MLP 前向传播。渲染一张 1080p 图像需要 ~200M 次 MLP 推理。
- **训练慢**：反向传播需要经过体渲染积分链 + 大量采样点的 MLP，训练一个场景通常需要 **数小时到数天**。
- **隐式表示不可编辑**：场景编码在 MLP 权重里，无法直接操作场景中的物体。

```
NeRF 渲染流程（每个像素）:

Camera ──ray──▶ [ sample 64+ points along ray ]
                     │
                     ▼
              [ MLP(x,y,z,θ,φ) → (c, σ) ] × 64+
                     │
                     ▼
              [ Volume Rendering Integral ]
                     │
                     ▼
                  Pixel Color
```

### 1.2 3DGS 的核心思路

3DGS 提出了一个完全不同的范式：

| | NeRF | 3DGS |
|---|---|---|
| 场景表示 | 隐式 (MLP 权重) | 显式 (一组 3D 高斯) |
| 渲染方式 | 射线步进 (Ray Marching) | 光栅化 (Splatting) |
| 可微性来源 | 沿射线积分可微 | 高斯投影 + α-blending 可微 |

**核心 Insight**：3D 高斯函数具备三个关键性质——
1. **可微**：参数（位置、形状、颜色）对渲染结果的梯度可解析计算
2. **高效投影**：3D 高斯投影到 2D 仍是高斯，无需数值积分
3. **稀疏性**：每个高斯只影响局部区域，天然适合 tile-based 并行

## 二、3D 高斯表示

### 2.1 单个高斯的参数

每个 3D 高斯 $G_i$ 由以下参数定义：

| 参数 | 符号 | 维度 | 说明 |
|------|------|------|------|
| 位置 (均值) | $\boldsymbol{\mu}$ | $\mathbb{R}^3$ | 高斯中心在世界坐标的位置 |
| 协方差矩阵 | $\boldsymbol{\Sigma}$ | $\mathbb{R}^{3\times3}$ | 控制高斯的形状和朝向 |
| 不透明度 | $\alpha$ | $[0, 1]$ | 该高斯的透明程度 |
| 球谐系数 | SH coeffs | $\mathbb{R}^{k}$ | 视角相关的颜色，$k = 3 \times (\ell_{\max}+1)^2$ |

3D 高斯的概率密度函数为：

$$G(\mathbf{x}) = \exp\left( -\frac{1}{2} (\mathbf{x} - \boldsymbol{\mu})^T \boldsymbol{\Sigma}^{-1} (\mathbf{x} - \boldsymbol{\mu}) \right)$$

### 2.2 协方差矩阵的参数化

直接优化 $\boldsymbol{\Sigma}$ 会遇到问题：协方差矩阵必须是**半正定**的，无约束优化无法保证这一点。

3DGS 的解决方案是将 $\boldsymbol{\Sigma}$ 分解为**缩放 (Scaling)** 和**旋转 (Rotation)**：

$$\boldsymbol{\Sigma} = R S S^T R^T$$

其中：
- $S = \text{diag}(s_x, s_y, s_z)$，三个缩放因子，存储为 $\mathbf{s} \in \mathbb{R}^3$
- $R$ 是 3×3 旋转矩阵，用**四元数** $\mathbf{q} = (q_w, q_x, q_y, q_z)$ 表示（4 个参数，归一化约束）

这样实际优化的参数是 $\mathbf{s}$ 和 $\mathbf{q}$，共 7 个标量，且**任意值都能保证 $\boldsymbol{\Sigma}$ 半正定**。

```
协方差参数化示意:

  四元数 q ──▶ 旋转矩阵 R (3×3)
                    │
  缩放向量 s ──▶ S = diag(s)
                    │
                    ▼
          Σ = R · S · S^T · R^T
          （自动保证半正定）
```

### 2.3 球谐函数 (Spherical Harmonics, SH)

为什么用球谐函数而不是直接存 RGB？

**因为真实场景中颜色是视角相关的**：高光、反射等现象导致同一个表面从不同角度看颜色不同。

球谐函数是球面上的正交基函数，类似于球面版的傅里叶级数：

$$c(\mathbf{d}) = \sum_{\ell=0}^{\ell_{\max}} \sum_{m=-\ell}^{\ell} c_{\ell}^{m} \cdot Y_{\ell}^{m}(\mathbf{d})$$

其中 $\mathbf{d}$ 是观察方向（单位向量），$Y_{\ell}^m$ 是 SH 基函数，$c_{\ell}^m$ 是待优化的系数。

| SH 阶数 $\ell_{\max}$ | 系数数 (每通道) | 总参数 (RGB) | 表达能力 |
|---|---|---|---|
| 0 | 1 | 3 | 仅漫反射 (Lambertian) |
| 1 | 4 | 12 | + 简单方向性 |
| 2 | 9 | 27 | + 高光 |
| 3 | 16 | 48 | 3DGS 默认，足够大多数场景 |

**SH 的优势**：
- 低阶 SH 自然表示低频光照变化，不会过拟合
- 从系数到颜色的计算只是加权求和，非常快
- 旋转等变性：旋转场景等价于旋转 SH 系数

### 2.4 每个高斯的参数量统计

| 参数 | 维度 | 说明 |
|------|------|------|
| $\boldsymbol{\mu}$ | 3 | 位置 |
| $\mathbf{s}$ | 3 | 缩放 |
| $\mathbf{q}$ | 4 | 旋转 (四元数) |
| $\alpha$ | 1 | 不透明度 |
| SH 系数 | 48 | $3 \times 16$，3 阶 SH |
| **合计** | **59** | 每个高斯 ~236 Bytes (float32) |

一个典型场景可能有 **100 万 – 500 万** 个高斯，存储量约为 **200 MB – 1 GB**（远大于 NeRF 的几 MB 网络权重，这是 3DGS 的主要缺点之一）。

## 三、3D 到 2D 投影

### 3.1 高斯的投影性质

3D 高斯经过仿射变换后仍然是高斯——这是 3DGS 高效渲染的数学基础。

给定相机的**观察变换矩阵** $W$（world → camera），3D 高斯在相机坐标系下的均值和协方差为：

$$\boldsymbol{\mu}' = W \boldsymbol{\mu}, \quad \boldsymbol{\Sigma}' = W \boldsymbol{\Sigma} W^T$$

### 3.2 透视投影的近似

透视投影是非线性的，不能直接用矩阵乘法。3DGS 使用**局部仿射近似**（EWA Splatting, Zwicker et al., 2001）：

在高斯中心附近对投影函数做一阶 Taylor 展开，得到雅可比矩阵 $J$：

$$\boldsymbol{\Sigma}_{2D} = J W \boldsymbol{\Sigma} W^T J^T$$

其中 $J \in \mathbb{R}^{2 \times 3}$ 是投影函数对相机坐标的雅可比矩阵：

$$J = \begin{bmatrix} \frac{f_x}{z} & 0 & -\frac{f_x \cdot x}{z^2} \\ 0 & \frac{f_y}{z} & -\frac{f_y \cdot y}{z^2} \end{bmatrix}$$

其中 $f_x, f_y$ 是焦距，$(x, y, z)$ 是高斯中心在相机坐标系下的坐标。

投影后得到 **2D 高斯**，用其 2×2 协方差 $\boldsymbol{\Sigma}_{2D}$ 和 2D 均值 $\boldsymbol{\mu}_{2D}$ 来计算每个像素处的贡献。

```
3D → 2D 投影流程:

3D 高斯 (μ, Σ)                  2D 高斯 (μ_2D, Σ_2D)
    │                                  │
    │  W: 世界→相机变换                │
    │  J: 投影雅可比                   │
    ▼                                  ▼
  Σ_2D = J · W · Σ · W^T · J^T     → 2D 椭圆
  μ_2D = project(W · μ)            → 2D 中心
```

## 四、可微光栅化渲染

### 4.1 α-blending 渲染方程

对于图像上的每个像素 $\mathbf{p}$，其颜色由覆盖该像素的所有高斯按深度排序后 α-blending 得到：

$$C(\mathbf{p}) = \sum_{i=1}^{N} c_i \cdot \alpha_i' \cdot T_i$$

其中：
- $c_i$ 是第 $i$ 个高斯的颜色（由 SH 系数根据观察方向计算）
- $\alpha_i' = \alpha_i \cdot G_{2D}(\mathbf{p})$ 是不透明度乘以 2D 高斯在像素处的值
- $T_i = \prod_{j=1}^{i-1}(1 - \alpha_j')$ 是累积透射率

**与 NeRF 体渲染的对应关系**：

NeRF 的体渲染公式为：

$$C(\mathbf{r}) = \int_{t_n}^{t_f} T(t) \cdot \sigma(\mathbf{r}(t)) \cdot c(\mathbf{r}(t), \mathbf{d}) \, dt, \quad T(t) = \exp\left(-\int_{t_n}^{t}\sigma(\mathbf{r}(s))\,ds\right)$$

离散化后等价于 α-compositing。3DGS 的 α-blending 本质上就是这个离散化版本，只是"采样点"不是沿射线均匀分布，而是**由高斯的位置决定**。

### 4.2 Tile-based 光栅化

这是 3DGS 实现实时渲染的关键工程设计。

```
Tile-based 光栅化流程:

┌─────────────────────────────────────────────────┐
│  Step 1: 视锥剔除 (Frustum Culling)              │
│  排除相机视锥外的高斯                              │
├─────────────────────────────────────────────────┤
│  Step 2: 将图像划分为 16×16 的 Tile               │
│                                                   │
│  ┌────┬────┬────┬────┐                           │
│  │ T0 │ T1 │ T2 │ T3 │ ...                      │
│  ├────┼────┼────┼────┤                           │
│  │ T4 │ T5 │ T6 │ T7 │ ...                      │
│  └────┴────┴────┴────┘                           │
│                                                   │
├─────────────────────────────────────────────────┤
│  Step 3: 每个高斯投影为 2D 椭圆                    │
│  计算该椭圆覆盖了哪些 Tile（一个高斯可覆盖多个 Tile）│
├─────────────────────────────────────────────────┤
│  Step 4: 对每个 Tile 内的高斯按深度排序             │
│  使用 GPU radix sort（基数排序）                    │
├─────────────────────────────────────────────────┤
│  Step 5: 每个 Tile 启动一个 GPU thread block       │
│  每个线程处理一个像素，按排序顺序做 α-blending       │
│  当 T_i < ε 时提前终止（该像素已"不透明"）          │
└─────────────────────────────────────────────────┘
```

**为什么用光栅化而不用光线追踪？**

| | 光线追踪 (Ray Tracing) | 光栅化 (Rasterization) |
|---|---|---|
| 思路 | 从像素发射射线，查找碰到的高斯 | 从高斯出发，投影到像素 |
| 排序 | 每条射线单独排序 | 全局排序一次，所有像素共享 |
| GPU 利用 | 射线访问不规则，缓存不友好 | Tile 内存连续，缓存友好 |
| 性能 | 慢 | **快 10-100 倍** |

### 4.3 前向与反向传播

**前向传播**：按上述流程计算每个像素的颜色。

**反向传播**（关键的可微部分）：

由于所有操作（投影、α-blending）都是可微的，可以直接反向传播到每个高斯的参数：

$$\frac{\partial \mathcal{L}}{\partial \boldsymbol{\mu}_i}, \quad \frac{\partial \mathcal{L}}{\partial \mathbf{s}_i}, \quad \frac{\partial \mathcal{L}}{\partial \mathbf{q}_i}, \quad \frac{\partial \mathcal{L}}{\partial \alpha_i}, \quad \frac{\partial \mathcal{L}}{\partial \text{SH}_i}$$

**实现细节**：反向传播时需要知道每个像素的 $T_i$ 值和排序顺序。3DGS 使用一个技巧——**从后往前遍历**，递推恢复中间状态，避免存储所有前向中间结果。

## 五、自适应密度控制 (Adaptive Density Control)

### 5.1 初始化

从 **COLMAP** 等 SfM (Structure from Motion) 工具得到的稀疏点云初始化高斯：

- 每个 SfM 点 → 一个高斯
- $\boldsymbol{\mu}$：直接用点的 3D 坐标
- $\mathbf{s}$：设为到最近邻点距离的某个比例
- $\mathbf{q}$：单位四元数（无旋转）
- $\alpha$：初始化为较低值（如 0.1）
- SH 系数：0 阶用点的 RGB 颜色初始化，高阶设为 0

典型初始点云：**数千到数万点**。训练结束后通常增长到 **数百万点**。

### 5.2 密度控制策略

每隔一定迭代（论文中为每 100 次迭代），执行以下操作：

```
自适应密度控制决策树:

对每个高斯 G_i:
    │
    ├── 位置梯度 ‖∂L/∂μ‖ > τ_grad ?   (梯度阈值)
    │   │
    │   ├── YES → 该高斯需要更多细节
    │   │   │
    │   │   ├── max(s) < τ_size ?   (高斯太小?)
    │   │   │   │
    │   │   │   ├── YES → 克隆 (Clone)
    │   │   │   │         在梯度方向复制一个相同的高斯
    │   │   │   │
    │   │   │   └── NO  → 分裂 (Split)
    │   │   │             拆成 2 个更小的高斯
    │   │   │             缩放因子除以 φ=1.6
    │   │   │
    │   │   └── (执行 Clone 或 Split)
    │   │
    │   └── NO  → 不操作
    │
    ├── α_i < ε_α ?   (不透明度太低?)
    │   │
    │   ├── YES → 剪枝 (Prune)，删除该高斯
    │   └── NO  → 保留
    │
    └── 高斯体积过大 (在屏幕空间投影面积过大)?
        │
        ├── YES → 剪枝
        └── NO  → 保留
```

### 5.3 克隆 vs 分裂的直觉

- **克隆 (Clone)**：高斯太小，无法覆盖实际需要的区域。解决方案是**增加一个**高斯来扩大覆盖。
  - 场景：一个小高斯试图表示一个纹理细节区域，梯度指示它应该移动但又不能离开当前位置。

- **分裂 (Split)**：高斯太大，无法表示局部细节。解决方案是**拆成两个更小的**高斯。
  - 场景：一个大高斯覆盖了一个边缘区域，边缘两侧颜色不同，单个大高斯无法同时表示。

### 5.4 周期性 α 重置

每隔 N 次迭代（论文中为 3000 次），将**所有高斯的不透明度重置为接近 0 的值**。

**目的**：
- 清除"浮动伪影"——那些误导性地变得不透明的高斯
- 给系统一个"重新竞争"的机会，真正有用的高斯会快速恢复不透明度，无用的会被后续剪枝清除

## 六、训练流程

### 6.1 总体流程

```
3DGS 训练流程:

多视角图像 + COLMAP 位姿/点云
          │
          ▼
  ┌─────────────────────────┐
  │  初始化高斯 (从 SfM 点云) │
  └────────────┬────────────┘
               │
               ▼
  ┌────────────────────────────────────────────┐
  │  For each iteration:                        │
  │                                             │
  │  1. 随机选一张训练图像                        │
  │  2. 可微光栅化 → 渲染图像                     │
  │  3. 计算损失 L (渲染 vs GT)                   │
  │  4. 反向传播 → 更新所有高斯参数                │
  │  5. 每 100 iter: 自适应密度控制               │
  │  6. 每 3000 iter: α 重置                     │
  │                                             │
  │  总共约 30,000 次迭代                         │
  └────────────────────────────────────────────┘
               │
               ▼
  最终高斯集合 → 可实时渲染
```

### 6.2 损失函数

$$\mathcal{L} = (1-\lambda)\,\mathcal{L}_1 + \lambda\,\mathcal{L}_{\text{D-SSIM}}$$

- $\mathcal{L}_1 = \| I_{\text{render}} - I_{\text{GT}} \|_1$：像素级 L1 损失，提供精确的颜色监督
- $\mathcal{L}_{\text{D-SSIM}} = \frac{1 - \text{SSIM}(I_{\text{render}}, I_{\text{GT}})}{2}$：结构相似性损失，关注局部结构和对比度
- $\lambda = 0.2$（论文默认值）

**为什么混合两个损失？**
- 纯 L1 会导致过于"模糊"——它不惩罚结构错误
- 纯 SSIM 对绝对颜色不敏感
- 混合使用兼顾像素精度和感知质量

### 6.3 优化细节

| 参数 | 优化器 | 初始学习率 | 调度 |
|------|--------|-----------|------|
| 位置 $\boldsymbol{\mu}$ | Adam | $1.6 \times 10^{-4}$ | Exponential decay |
| 缩放 $\mathbf{s}$ | Adam | $5 \times 10^{-3}$ | — |
| 旋转 $\mathbf{q}$ | Adam | $1 \times 10^{-3}$ | — |
| 不透明度 $\alpha$ | Adam | $5 \times 10^{-2}$ | — |
| SH 系数 | Adam | $2.5 \times 10^{-3}$ (DC) / $1.25 \times 10^{-4} / \ell$ (高阶) | — |

**关键设计**：
- 位置使用较小学习率 + decay，因为初始化已经给出了合理位置
- SH 高阶系数学习率更小，采用 coarse-to-fine 的思路
- 所有参数使用 Adam，但 $\beta_1, \beta_2$ 保持默认 (0.9, 0.999)

### 6.4 性能数据

| 指标 | 3DGS | Instant-NGP | Mip-NeRF 360 |
|------|------|------------|--------------|
| 训练时间 | ~25–40 min | ~5–10 min | ~48 hours |
| 渲染 FPS (1080p) | **100–200** | ~10 | ~0.06 |
| PSNR (Mip-NeRF 360 数据集) | **25.6** | 25.3 | 27.7 |
| 存储 | ~200–700 MB | ~50 MB | ~5 MB |

3DGS 在**渲染速度**上有压倒性优势，质量接近最好的 NeRF 方法，但存储开销显著更大。

## 七、与 NeRF 的全面对比

| 维度 | NeRF | 3DGS |
|------|------|------|
| 场景表示 | 隐式 (MLP 或哈希网格) | 显式 (3D 高斯集合) |
| 渲染方法 | 体渲染 (ray marching) | 光栅化 (splatting) |
| 渲染速度 | 慢 (0.05–10 FPS) | **实时 (>100 FPS)** |
| 训练速度 | 慢 (数小时) | 较快 (~30 min) |
| 存储开销 | 小 (数 MB) | **大 (数百 MB)** |
| 场景编辑 | 困难 (改 MLP 权重无物理意义) | **容易 (直接增删移动高斯)** |
| 渲染质量 | 高 | 高 (在大多数场景上持平或略优) |
| 处理反射/透明 | 较好 (MLP 有拟合能力) | 较难 (需要大量高斯叠加) |
| 对初始化的依赖 | 低 (MLP 可从随机开始) | **高 (需要 SfM 点云)** |
| 适合场景 | 高精度重建，小规模场景 | 实时应用，场景编辑，大规模场景 |

**总结**：3DGS 用存储换速度，用显式性换可编辑性。如果对实时性和交互有要求，3DGS 是更好的选择；如果追求极致压缩或处理复杂光学现象，NeRF 系列仍有优势。

## 八、关键数学推导

### 8.1 2D 投影协方差推导

设 3D 高斯在世界坐标系下的协方差为 $\boldsymbol{\Sigma}$，经过视图变换 $W$ 到相机坐标系：

$$\boldsymbol{\Sigma}_{\text{cam}} = W \boldsymbol{\Sigma} W^T$$

投影函数 $\pi: \mathbb{R}^3 \to \mathbb{R}^2$ 将相机坐标 $(x, y, z)$ 映射到像素坐标 $(u, v)$：

$$u = f_x \cdot \frac{x}{z} + c_x, \quad v = f_y \cdot \frac{y}{z} + c_y$$

在高斯中心 $\mathbf{t} = (t_x, t_y, t_z)$ 处做一阶近似：

$$J = \frac{\partial \pi}{\partial \mathbf{x}} \bigg|_{\mathbf{x}=\mathbf{t}} = \begin{bmatrix} \frac{f_x}{t_z} & 0 & -\frac{f_x \cdot t_x}{t_z^2} \\ 0 & \frac{f_y}{t_z} & -\frac{f_y \cdot t_y}{t_z^2} \end{bmatrix}$$

最终 2D 协方差：

$$\boldsymbol{\Sigma}_{2D} = J \boldsymbol{\Sigma}_{\text{cam}} J^T = J W \boldsymbol{\Sigma} W^T J^T$$

### 8.2 α-blending 梯度推导

对于渲染颜色 $C = \sum_{i} c_i \alpha_i' T_i$，其中 $T_i = \prod_{j<i}(1-\alpha_j')$：

$$\frac{\partial C}{\partial c_i} = \alpha_i' T_i$$

$$\frac{\partial C}{\partial \alpha_i'} = c_i T_i - \frac{1}{1-\alpha_i'}\sum_{j>i} c_j \alpha_j' T_j$$

后一项可以通过**从后向前遍历**高效计算——这正是 3DGS 反向传播采用的策略。

## 十、面试高频问题

### Q1: 3DGS 每个高斯有哪些参数？为什么这样设计？

**答**：位置 $\boldsymbol{\mu}$ (3)、缩放 $\mathbf{s}$ (3)、旋转四元数 $\mathbf{q}$ (4)、不透明度 $\alpha$ (1)、球谐系数 (48)，共 59 个参数。协方差用 $R, S$ 分解来保证半正定性；用球谐函数是为了表示视角相关颜色。

### Q2: Splatting 和体渲染有什么本质区别？

**答**：
- **体渲染**是 image-centric 的：从每个像素发射射线，沿射线积分。复杂度与采样点数成正比。
- **Splatting** 是 object-centric 的：从每个 3D 元素出发，投影到图像上。复杂度与高斯数成正比。
- 两者在数学上等价（都是 α-compositing），但工程实现差异巨大：splatting 天然适合 GPU 光栅化管线，因此快得多。

### Q3: 自适应密度控制的具体策略是什么？为什么需要？

**答**：初始化来自稀疏 SfM 点云，远不够表示场景细节。需要自动增减高斯：
- **克隆**：位置梯度大 + 高斯小 → 在梯度方向复制（欠覆盖区域）
- **分裂**：位置梯度大 + 高斯大 → 拆成两个小高斯（过覆盖区域）
- **剪枝**：不透明度过低或投影面积过大 → 删除
- **α 重置**：周期性把所有 α 设为低值，迫使无用高斯被剪枝

### Q4: 3DGS 为什么需要 SfM 点云初始化？能不能随机初始化？

**答**：理论上可以，但实践中 SfM 点云提供了关键的先验——让高斯从正确的位置开始优化。随机初始化需要更多迭代才能收敛，且容易陷入局部最优。后续工作（如 DUSt3R + 3DGS）在探索减少对 SfM 的依赖。

### Q5: 3DGS 的主要局限是什么？

**答**：
1. **存储大**：百万级高斯的参数，通常数百 MB
2. **依赖 SfM 初始化**：需要 COLMAP 预处理
3. **处理反射/透明困难**：高斯是不透明的体积元，模拟复杂光学需要大量堆叠
4. **训练视角覆盖要求高**：未被观测区域容易出现空洞
5. **高斯数量不可控**：自适应密度控制可能导致高斯爆炸

### Q6: 3DGS 适合什么应用场景？NeRF 适合什么？

**答**：
- **3DGS 适合**：实时渲染、VR/AR、交互式 3D 编辑、自动驾驶仿真、大规模场景
- **NeRF 适合**：高精度重建、小规模精细场景、存储受限场景、需要处理复杂光学效应
