# 08 · 高斯 + 具身

GS 进入具身智能（embodied AI / robotics）是 2024 下半年的事，但来势凶猛。原因很直接：**机器人需要的，正是 GS 能提供的**——重建真实环境作为仿真背景、可微渲染让 sim-to-real gap 缩小、3D 特征场为 manipulation 提供空间感。这一篇讲 GS 怎么落进 manipulation / VLA / real-to-sim 三个具身核心问题。

> **核心论文**:
> - SplatSim (Qureshi et al., 2024)
> - RoboGSim (Li et al., 2024)
> - Re³Sim (Han et al., 2024)
> - DISCOVERSE (Wang et al., 2024)
> - Feature 3DGS (Zhou et al., CVPR 2024)
> - LangSplat (Qin et al., CVPR 2024)

---

## 一、为什么具身需要 GS

机器人 manipulation 当前最大瓶颈不是算法，是**数据**。模仿学习需要演示，遥操作贵；强化学习需要环境，仿真环境与真实差距大。GS 在三个层面缓解：

1. **重建真实场景作仿真背景**：拍一段视频就能得到可仿真的真实场景，不必手工建模
2. **可微 sim-to-real 桥接**：[06 篇](06-高斯+物理仿真.md)的 PhysGaussian 让仿真后的视觉与真实一致，policy 训练后部署到真机的视觉差异最小化
3. **3D 特征场为 manipulation 提供空间表征**：相比 2D feature map，GS 上挂语义特征能直接做 6D pick / place

---

## 二、SplatSim：GS 作为机器人仿真背景

### 2.1 想法

机器人仿真器（PyBullet / IsaacSim / MuJoCo）通常用低保真背景（白板 + 简单纹理），policy 在这种环境训出来部署到真机，视觉上 sim-to-real gap 巨大。

SplatSim 的方法：**把 GS 重建的真实场景作为仿真背景**。

```
┌──────────────────────────────────────┐
│  传统仿真:                            │
│  ┌──────┐  + 真实物理                  │
│  │白色板 │  → policy 学到真实控制       │
│  └──────┘    但部署后视觉不同导致失败   │
└──────────────────────────────────────┘

┌──────────────────────────────────────┐
│  SplatSim:                           │
│  ┌──────────┐  + 真实物理              │
│  │GS 重建场景│  → policy 视觉与真实一致 │
│  └──────────┘    部署 transfer 大幅好转 │
└──────────────────────────────────────┘
```

### 2.2 实现

- 拍摄真实场景多视角图像 → 重建 3DGS
- 将 GS 加载进仿真器作为渲染背景
- 仿真器的物理（重力、碰撞）正常运行，与刚体物体（机械臂、目标物）交互
- 渲染时把仿真器输出的相机视角传给 GS rasterizer，得到逼真图像
- Policy 训练吃这个图像

### 2.3 收益

- Pick-and-place 任务从 sim 到 real 的成功率从 40% 提到 75%（典型实验）
- 不需要手工建模真实场景，扩展性强
- 缺点：背景仍是静态的，场景中可移动的物体（被操纵物）还是要手工建模

## 三、RoboGSim：完整 real-to-sim → policy

### 3.1 流程

RoboGSim 把这条路推到完整闭环：

```
Step 1: 拍摄真实场景（含机器人 workspace）
Step 2: 3DGS 重建 → 静态背景
Step 3: 用 SAM 分割可操作物体，单独重建为 GS + mesh
Step 4: 用 PhysGaussian 推断材料属性
Step 5: 在仿真里随机化物体初始位姿
Step 6: 跑 policy training (RL / IL)
Step 7: 部署到真机
```

### 3.2 关键 trick

- **物体高斯绑定 mesh**：物体既要参与物理（需 mesh），又要参与渲染（高斯），两者绑定
- **域随机化 + GS**：不只随机光照颜色，还随机 GS 的 SH 系数（生成"看起来略不同"的同一场景）
- **VLA fine-tune**：用生成数据 fine-tune Pi-0 / OpenVLA 等基础模型

### 3.3 表现

在物体 pick / drawer open / button press 等 8 个真实任务上，从仿真训出来的 policy 直接 zero-shot 到真机的成功率达到 60-80%（不同任务不同），是 2024 GS-based real-to-sim 的代表工作。

## 四、Re³Sim：真实-仿真-真实闭环

Re³Sim 强调**双向**：不只 real → sim → real，还要 sim → real → sim 反馈。

### 4.1 反馈机制

- 真机部署时记录失败 case（policy 在哪些状态下做错）
- 把这些状态反馈进仿真器，专门生成更多类似数据
- Policy 重新训练，针对性补强

类似 ML 里的 active learning，但反馈对象是仿真生成器。

### 4.2 工程意义

实际部署是**长尾问题**：仿真训得 80% 任务，剩下 20% 失败 case 千奇百怪。Re³Sim 的反馈循环是这条线上必然要走的工程化方向。

---

## 五、DISCOVERSE：具身基础设施

DISCOVERSE 不是单一工作，而是一个**具身仿真平台**，把上述能力做成可复用基础设施：

- **3DGS 重建工具链**：从手机视频到 GS 一键完成
- **PhysGaussian + MPM 物理后端**：开箱即用
- **机器人模型库**：常见机械臂 / 灵巧手 / 移动平台
- **Policy 训练 SDK**：兼容 ACT / Diffusion Policy / OpenVLA

类比：DISCOVERSE 之于 GS-robotics，等于 IsaacGym 之于 RL-robotics。

## 六、3D Feature Field：GS + 语义特征

### 6.1 思路

要让 robot 理解"打开抽屉"，不只要重建几何，还要**给每个高斯挂语义特征**。

### 6.2 Feature 3DGS / LangSplat

- **Feature 3DGS** (Zhou et al., CVPR 2024)：除了 SH 颜色，每个高斯额外存一个 N 维特征向量（如 DINO / CLIP 特征）
- **LangSplat** (Qin et al., CVPR 2024)：特征向量编码 CLIP 文本特征，可用自然语言查询场景中的位置

```
"找到红色的杯子"
       │
       ▼ CLIP 文本编码
   text feature
       │
       ▼ 与每个高斯的特征算相似度
   高斯热图
       │
       ▼ argmax 或采样
   3D 位置 → 抓取规划
```

### 6.3 应用：语言驱动的 manipulation

- 输入自然语言指令
- 在 GS 场景中定位目标
- 规划末端轨迹
- 执行

这条线和 VLA（Vision-Language-Action 模型）正在融合。OpenVLA / Pi-0 这类大模型直接吃图像；用 GS feature field 做 perception 前端能给 VLA 提供更准确的空间锚。

### 6.4 F3RM, OpenSeg-3DGS 等

类似工作：**F3RM**（NeRF 时代版本，迁到 GS）、**OpenSeg-3DGS**（开放词汇分割）、**SAM3D-GS**（点 prompt 分割 GS）。

## 七、VLA × GS 的可能性

VLA 模型（π₀ / GR00T / OpenVLA）当前主要吃 2D 图像。把 GS 接进 VLA 有几条路：

1. **GS 作 perception 前端**：分割 / 抓取候选 / 语义查询走 GS，再给 VLA 一个 cleaner 输入
2. **GS 作仿真数据生成器**：给 VLA 训练数据（参考 SplatSim / RoboGSim）
3. **GS feature 融合进 VLA**：把高斯的语义特征作为额外 token 输进 VLA transformer
4. **3D-aware VLA**：直接基于 GS 的 spatial encoding 做 VLA backbone（研究阶段）

这是 2026 年仍在开放的研究问题。当前各家做法不一致，但方向明确。

## 八、real-to-sim-to-real 全栈视图

把 GS 在具身里的所有用途串起来：

```
┌─────────── Real World ─────────────┐
│ 摄像头拍真实场景 + 机器人运动数据    │
└────────────┬─────────────────────┘
             │
             ▼ 3DGS / 2DGS 重建 ([01][02])
┌─────────── 静态 GS 地图 ───────────┐
│ 几何 + 外观，作仿真背景             │
└────────────┬─────────────────────┘
             │
             ▼ 加物理 ([06])
┌─────────── 物理化 GS ──────────────┐
│ PhysGaussian / Spring-Gaus         │
│ 物体能动、能形变                    │
└────────────┬─────────────────────┘
             │
             ▼ 加语义 (3D feature field)
┌─────────── 语义 GS ────────────────┐
│ Feature 3DGS / LangSplat           │
│ "找到红色的杯子" 可执行              │
└────────────┬─────────────────────┘
             │
             ▼ 训练 policy
┌─────────── Policy ─────────────────┐
│ ACT / Diffusion Policy / VLA       │
└────────────┬─────────────────────┘
             │
             ▼ 部署
┌─────────── Real Robot ─────────────┐
│ 真机执行                           │
└─────────────────────────────────── ┘
```

## 九、面试高频问题

### Q1: 为什么 GS 突然成了具身的热点？

**答**：具身瓶颈是数据，数据瓶颈是仿真与真实的 gap。GS 同时提供"重建真实场景"（背景视觉一致）+ "可微渲染"（policy 训练时梯度通畅）+ "可与物理耦合"（[06](06-高斯+物理仿真.md)的 PhysGaussian），三件事把 sim-to-real gap 显著缩小。NeRF 时代受限于渲染慢 + 黑箱，GS 改变了游戏规则。

### Q2: SplatSim 和 RoboGSim 的区别？

**答**：
- **SplatSim**：只用 GS 当仿真**背景**——静态环境，可操纵物体仍手工建模。改动最小，工程上最快上手。
- **RoboGSim**：完整 real-to-sim 流水线——背景和物体都是 GS，物体绑 mesh + 物理参数，policy 直接在 GS 仿真里训练。改动大但 sim-to-real gap 更小。

### Q3: 3D feature field 比 2D feature map 强在哪？

**答**：(1) **空间一致性**——同一个物体从不同角度看的 feature 一致，2D feature 有 occlusion / view-dependent 问题；(2) **直接给 6D 位姿**——查询 "杯子" 直接得到 3D 位置 + 法线，2D 还要做 inverse projection；(3) **多视角融合**——多帧 feature 自然聚合到 3D，2D 多帧融合需要 attention / pooling。代价是要 GS 重建 + 特征蒸馏。

### Q4: VLA 模型已经很强了，为什么还要 GS？

**答**：VLA 强在"语言到动作"端到端，但弱在 (1) 数据需求大——VLA 训练动辄 100k 真机轨迹，GS 仿真数据可以补充；(2) 3D 空间精度——VLA 吃 2D 图像，对厘米级精度的 manipulation 不够，GS 的 3D feature 给精确空间锚；(3) 长尾任务泛化——GS-based simulation 能 cheaply 生成新任务、新场景训练数据。两者是协作不是替代。

### Q5: 这条线最大的开放问题是什么？

**答**：(1) **可形变物体**——衣服、布、绳索的 GS 重建 + 物理仿真还不够稳（同一个高斯被拉成大形变后协方差精度崩溃，弹簧 / MPM 在拓扑变化下也不稳）；(2) **拓扑变化**——切菜、剪纸这种从一块变两块的场景，GS 表示天然困难（高斯怎么"分裂成两个连通域"）；(3) **长时序累积误差**——仿真跑很久后小误差累积，policy 在长 horizon 任务上表现仍差；(4) **数据 cost**：尽管比真机便宜，GS 重建本身也要时间和算力，规模化生产仍贵。
