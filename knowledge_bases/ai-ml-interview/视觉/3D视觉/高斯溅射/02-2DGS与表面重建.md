# 02 · 2DGS 与表面重建

3DGS 渲染漂亮，但**几何粗糙**——抽出来的 mesh 抖动、法线乱、重建出的表面像棉花团。这一篇讲为什么会这样，以及 2DGS / SuGaR / GOF 这一系列工作怎么把"渲染高质量"和"几何高质量"重新对齐。

> **核心论文**:
> - 2D Gaussian Splatting (Huang et al., SIGGRAPH 2024)
> - SuGaR (Guédon & Lepetit, CVPR 2024)
> - Gaussian Opacity Fields / GOF (Yu et al., SIGGRAPH 2024)
> - PGSR (Chen et al., 2024)

---

## 一、3DGS 为什么几何不行

回顾 [01](01-3DGS基础与渲染.md)：每个 3D 高斯是一个**有体积**的椭球，rasterizer 对所有椭球做 α-blending。这种表示对**渲染**没问题——观察方向上某个像素的颜色由若干高斯加权得来，结果非常逼真。

但对**几何**是有问题的：

1. **没有明确表面**：高斯是体积元，理想表面应该是高斯密度的某个等值面，但训练目标只是渲染颜色，没有任何力让表面"对齐"到几何。
2. **法线不可靠**：3D 椭球的最短轴方向不一定指向真实表面法线。训练好的高斯经常"歪着"贴在表面上。
3. **多视角不一致**：从不同角度看，同一个表面可能由不同高斯主导，等价于隐式承认"这个区域允许多个表面"。
4. **Mesh 提取困难**：标准做法是先渲深度图、再 TSDF/Poisson 融合，但深度图本身不一致，融合出来的 mesh 抖动严重。

定量：在 DTU / Tanks-and-Temples 上，3DGS 的 Chamfer distance 比 NeuS 这种 SDF 方法差 1.5-3 倍，这是 PSNR 看不出来的硬伤。

## 二、2D Gaussian Splatting（2DGS）

**核心想法**：把 3D 椭球**压扁成 2D 圆盘**，强制其对齐到表面。

### 2.1 从 3D 高斯到 2D 圆盘

2DGS 把每个 primitive 表示为 3D 空间中的一个 **2D 平面盘**：

- 中心 $\boldsymbol{\mu} \in \mathbb{R}^3$
- 两个正交切向量 $\mathbf{t}_u, \mathbf{t}_v \in \mathbb{R}^3$（控制盘的形状和朝向）
- 法线 $\mathbf{n} = \mathbf{t}_u \times \mathbf{t}_v$（自动得到）
- 缩放 $s_u, s_v$、不透明度 $\alpha$、SH 系数同 3DGS

每个 primitive 在 3D 空间中是一个椭圆形圆盘，没有"厚度"。

### 2.2 精确投影（不再用近似雅可比）

3DGS 用 EWA splatting 在屏幕空间近似投影；2DGS 不需要这一步——盘是 2D 的，可以**精确投影到屏幕**：

$$\mathbf{x}_{\text{world}}(u, v) = \boldsymbol{\mu} + s_u \cdot u \cdot \mathbf{t}_u + s_v \cdot v \cdot \mathbf{t}_v$$

光线 $\mathbf{r}(t)$ 与盘相交可以通过求解 ray-plane intersection 得到（解析解，无近似），交点处的高斯权重 $G(u, v) = \exp(-(u^2 + v^2)/2)$ 直接计算。

这一改动消除了 3DGS 在远场/近场的投影 artifact。

### 2.3 法线一致性正则

光做表示替换不够，2DGS 还加了两个关键损失：

- **法线一致性损失** $\mathcal{L}_n$：渲染出的深度图算 finite-difference 法线 $\mathbf{N}_d$，与 splatting 出的高斯法线 $\mathbf{N}$ 一致：
  $$\mathcal{L}_n = \sum_i w_i \cdot (1 - \mathbf{N}_i^T \mathbf{N}_{d,i})$$
- **深度畸变损失** $\mathcal{L}_d$：抑制射线方向上的深度展宽，迫使每条射线主要由一个 primitive 主导：
  $$\mathcal{L}_d = \sum_{i,j} w_i w_j |z_i - z_j|$$

整体损失：

$$\mathcal{L} = \mathcal{L}_{\text{rgb}} + \alpha \mathcal{L}_d + \beta \mathcal{L}_n$$

### 2.4 Mesh 提取

2DGS 的 mesh 提取是 **TSDF 融合 + Marching Cubes**：

1. 每个训练相机位姿渲染深度图
2. 用 Open3D 把深度图融合到 TSDF volume
3. Marching Cubes 抽取 mesh

由于深度更一致，融合出来的 mesh 比 3DGS 的提取版干净一个数量级。在 DTU 数据集上，2DGS 的 Chamfer distance 接近 NeuS（隐式 SDF）水平，但训练快了 100 倍以上。

## 三、SuGaR：3DGS 之上的 mesh 提取

如果不想换表示，能不能在 3DGS 上做后处理拿到 mesh？SuGaR 的答案：**加正则 + 重定位 + Poisson 重建**。

### 3.1 表面对齐正则

SuGaR 在 3DGS 训练后期加一项损失：

$$\mathcal{L}_{\text{align}} = \sum_i \min_j \| s_i^{\min} - 0 \|$$

意思是迫使每个高斯**最短缩放轴趋于 0**（变成扁盘），并让该方向与场景密度梯度对齐——本质上把 3DGS 朝 2DGS 的方向掰。

### 3.2 表面采样 + Poisson 重建

正则收敛后，SuGaR 在每个高斯上**采样表面点**（在椭球的"扁面"上撒点）+ 计算法线，然后送给 Poisson 重建得到 mesh。

### 3.3 Refinement

得到 mesh 后，把高斯**绑到 mesh 三角形上**（每个三角形一组高斯），再训练几千 iter 优化外观——这样最终产物是"mesh + 绑定的高斯"，既能用 mesh 做几何（碰撞、物理、AR），又能保留 3DGS 的渲染质量。

SuGaR 的工程价值：**已有 3DGS pipeline 想加 mesh，不用换表示，只加 fine-tune 阶段**。

## 四、Gaussian Opacity Fields（GOF）

GOF 走另一条路：**把 3D 高斯解释为不透明度场**，再从场上抽等值面。

### 4.1 从高斯到 opacity field

任意空间点 $\mathbf{x}$ 处的 opacity 定义为：

$$O(\mathbf{x}) = 1 - \prod_i (1 - \alpha_i \cdot G_i(\mathbf{x}))$$

这是从所有高斯"穿过"这个点时的累积不透明度。

### 4.2 表面 = 等值面

定义表面为 $O(\mathbf{x}) = 0.5$ 的等值面，用 **Tetrahedra Marching** 算法（比 Marching Cubes 更适合不规则采样）抽取 mesh。

### 4.3 关键 trick：射线-高斯精确积分

GOF 给出了射线穿过单个 3D 高斯的解析积分公式，这让 opacity field 的计算和 rendering 完全一致——避免了 2DGS/SuGaR 那种"渲染时一种表示、提 mesh 时另一种"的不一致性。

GOF 在 Tanks-and-Temples 上是 2024 年高斯方法的几何 SOTA。

## 五、其他重要工作

| 方法 | 核心思路 | 适用场景 |
|------|---------|---------|
| **PGSR** (2024) | 平面化 GS + 多视角光度一致性 + 单视角法线 | 室内大场景 |
| **TrimGS** (2024) | 训练时按透视面积剪枝大高斯 | 减少内存 + 改善边缘 |
| **RaDe-GS** (2024) | Rasterized depth gradient 取代 finite difference | 改善 2DGS 法线 |
| **Gaussian Surfels** (2024) | 类似 2DGS 但走 surfel 路线 | 实时表面渲染 |
| **2D Gaussian Editor** | 在 2DGS 上做几何编辑 | 编辑 + 重建一体 |

## 六、四类方法对比

| 维度 | 3DGS（基线） | 2DGS | SuGaR | GOF |
|------|-------------|------|-------|-----|
| 几何精度 | ★ | ★★★★ | ★★★ | ★★★★★ |
| 渲染质量 (PSNR) | ★★★★★ | ★★★★ | ★★★★★ | ★★★★ |
| Mesh 提取 | 困难 | TSDF 融合 | Poisson 重建 | Tetrahedra Marching |
| 训练速度 | 30 min | 35 min | 40 min（+ refine） | 50 min |
| 表示形式 | 椭球 | 圆盘 | 椭球 + mesh 绑定 | opacity field |
| 适合应用 | 实时渲染 | AR/VR / 重建 | mesh + 渲染 一体化 | 高精度重建 |

## 七、应用案例

- **AR/VR 内容生产**：2DGS / SuGaR 的 mesh 直接进 Unity / Unreal，做碰撞、阴影、物理。
- **建筑/文物扫描**：GOF 的高几何精度适合需要厘米级精度的场景。
- **机器人 real-to-sim**：[08 篇](08-高斯+具身.md)会讲到，2DGS 的 mesh 是机器人仿真的天然资产。
- **3D 编辑**：mesh + 绑定高斯的混合表示能同时做几何编辑（拉伸、变形）和外观编辑。

## 八、面试高频问题

### Q1: 为什么 3DGS 渲染好但几何差？

**答**：3DGS 的训练目标只有像素级渲染损失（L1 + D-SSIM），没有任何力强迫高斯对齐到真实表面。3D 椭球本身是体积元，没有定义"表面"，最短轴方向也不必然指向法线。结果就是 PSNR 高但 mesh 抖动、法线乱。

### Q2: 2DGS 相比 3DGS 改了什么？

**答**：(1) 表示从 3D 椭球换成 3D 空间中的 2D 圆盘；(2) 投影从 EWA 近似换成 ray-plane 解析交；(3) 加法线一致性损失 $\mathcal{L}_n$ 和深度畸变损失 $\mathcal{L}_d$；(4) Mesh 提取走 TSDF + Marching Cubes，因为深度图更一致。

### Q3: SuGaR 和 2DGS 各适合什么场景？

**答**：
- **SuGaR** 适合"已有 3DGS pipeline、想加 mesh 输出"——不换表示，加正则 + Poisson 后处理，最终是 mesh + 绑定高斯的混合产物。
- **2DGS** 适合"从头做几何"——表示更干净，深度更一致，mesh 质量更好。

### Q4: GOF 为什么把高斯当不透明度场？

**答**：opacity field 是"和 rendering 完全一致"的几何定义——表面就是渲染时光线被吸收的地方（$O=0.5$）。这避免了 2DGS/SuGaR 中"渲染表示和几何表示不一致"的问题。GOF 提出的解析射线积分让这一切可计算。

### Q5: 什么时候不需要做几何优化，直接用 3DGS 就够？

**答**：只要不抽 mesh、不做物理仿真、不做几何编辑——纯 NVS（新视角合成）应用，比如自由视点视频、VR 看片，3DGS 已经够好。一旦下游要 mesh / 法线 / 碰撞，必须走 02 篇这条线。
