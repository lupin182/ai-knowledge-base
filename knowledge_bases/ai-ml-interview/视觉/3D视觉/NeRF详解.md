# NeRF 详解

Neural Radiance Fields — 用神经网络隐式表示 3D 场景，从多视角图像合成新视角。

> **核心论文**：
> - [Mildenhall et al., 2020 - NeRF: Representing Scenes as Neural Radiance Fields for View Synthesis](https://arxiv.org/abs/2003.08934)
> - [Müller et al., 2022 - Instant Neural Graphics Primitives (Instant-NGP)](https://arxiv.org/abs/2201.05989)
> - [Barron et al., 2021 - Mip-NeRF: A Multiscale Representation for Anti-Aliasing Neural Radiance Fields](https://arxiv.org/abs/2103.13415)
> - [Barron et al., 2022 - Mip-NeRF 360](https://arxiv.org/abs/2111.12077)
> - [Chen et al., 2022 - TensoRF: Tensorial Radiance Fields](https://arxiv.org/abs/2203.09517)
> - [Kerbl et al., 2023 - 3D Gaussian Splatting](https://arxiv.org/abs/2308.14737)

---

## 一、背景与动机

### 1.1 3D 表示方式对比

计算机视觉和图形学中，3D 场景有两大类表示方式：

```
┌──────────────────────────────────────────────────────────────────┐
│                      3D 表示方式                                 │
├─────────────── 显式 (Explicit) ──────────────────────────────────┤
│  点云 (Point Cloud)    : 无序 3D 点集合，无拓扑                  │
│  网格 (Mesh)          : 顶点 + 面片，渲染友好但建模复杂          │
│  体素 (Voxel)         : 3D 栅格，规则但内存 O(N³) 爆炸           │
├─────────────── 隐式 (Implicit) ──────────────────────────────────┤
│  SDF                  : F(x,y,z) → signed distance               │
│  Occupancy Field      : F(x,y,z) → [0,1] 占据概率               │
│  NeRF (辐射场)        : F(x,y,z,θ,φ) → (r,g,b,σ)               │
└──────────────────────────────────────────────────────────────────┘
```

| 表示方式 | 存储效率 | 分辨率 | 可微渲染 | 拓扑灵活性 |
|:---------|:---------|:-------|:---------|:-----------|
| 点云 | 高 | 受点数限制 | 困难 | 高 |
| 网格 | 中 | 受面片数限制 | 需要特殊处理 | 低（拓扑固定） |
| 体素 | 低（$O(N^3)$） | 受分辨率限制 | 容易 | 高 |
| SDF | 高（仅存网络参数） | 连续，无限分辨率 | 需要球追踪 | 中 |
| **NeRF** | **高（仅存网络参数）** | **连续，无限分辨率** | **天然可微** | **高** |

### 1.2 新视角合成 (Novel View Synthesis)

**问题定义**：给定一组已知相机位姿的图像（训练视角），生成任意新视角下的图像。

```
输入：{(I₁, P₁), (I₂, P₂), ..., (Iₙ, Pₙ)}
      Iᵢ = 图像, Pᵢ = 相机位姿 (外参 R,t + 内参 K)

输出：给定新位姿 P_new，生成图像 I_new
```

这是三维重建、VR/AR、自动驾驶仿真等领域的核心问题。

### 1.3 传统方法的局限

传统管线：

```
多视角图像 → SfM (估计位姿) → MVS (稠密点云) → 网格重建 → 纹理映射 → 渲染
              COLMAP              COLMAP           Poisson     UV mapping
```

**问题**：
1. **管线冗长**：每一步都有误差累积，且各步骤不可联合优化
2. **离散化损失**：从连续场景到离散网格，丢失细节（如半透明物体、头发、烟雾）
3. **不可端到端训练**：无法通过梯度反传联合优化表示和渲染

NeRF 的突破在于：**用一个神经网络直接表示连续的 3D 场景，并通过可微体渲染端到端训练**。

## 二、NeRF 原始方法 (2020, Mildenhall et al.)

### 2.1 核心思想

将场景建模为一个连续的 5D 函数：

$$F_\Theta: (\mathbf{x}, \mathbf{d}) \rightarrow (\mathbf{c}, \sigma)$$

其中：
- $\mathbf{x} = (x, y, z)$：3D 空间位置
- $\mathbf{d} = (\theta, \phi)$：观察方向（用球坐标或单位方向向量表示）
- $\mathbf{c} = (r, g, b)$：该点在该方向上的辐射颜色
- $\sigma$：体积密度（volume density），表示该点的不透明程度

**关键设计**：
- **密度 $\sigma$ 只依赖位置 $\mathbf{x}$**：物体在哪里是客观的，和你从哪个方向看无关
- **颜色 $\mathbf{c}$ 依赖位置和方向**：同一个点从不同角度看，颜色可以不同（高光、反射）

```
     输入                  MLP                   输出
  ┌─────────┐         ┌──────────┐          ┌──────────┐
  │ (x,y,z) │────────→│          │─────────→│  σ (密度) │
  │         │         │  8层MLP  │          │          │
  │ (θ,φ)  │────┐    │  256维   │─────────→│ (r,g,b)  │
  └─────────┘    │    └──────────┘          └──────────┘
                 │         ↑
                 └─────────┘
              (方向仅在最后几层注入)
```

### 2.2 体渲染方程 (Volume Rendering)

NeRF 的渲染过程基于经典的**体渲染方程**。对于从相机发出的一条光线 $\mathbf{r}(t) = \mathbf{o} + t\mathbf{d}$（$\mathbf{o}$ 为相机原点，$\mathbf{d}$ 为方向），其颜色为：

$$C(\mathbf{r}) = \int_{t_n}^{t_f} T(t) \, \sigma(\mathbf{r}(t)) \, \mathbf{c}(\mathbf{r}(t), \mathbf{d}) \, dt$$

其中累积透射率为：

$$T(t) = \exp\left(-\int_{t_n}^{t} \sigma(\mathbf{r}(s)) \, ds\right)$$

**物理直觉**：
- $\sigma(\mathbf{r}(t))$：$t$ 处的密度，密度越大越不透明
- $T(t)$：光线从 $t_n$ 到 $t$ 的路径上"存活"（没被遮挡）的概率
- $T(t) \cdot \sigma(\mathbf{r}(t))$：光线恰好在 $t$ 处被终止的概率密度
- 整个积分：沿光线加权求和各处的颜色贡献

**类比**：想象一束光穿过一排有色玻璃。每块玻璃（$\sigma$）吸收一部分光，让剩余的光继续前进（$T$），最终你看到的颜色是所有玻璃颜色按透过率加权的混合。

### 2.3 离散化近似

实际计算中，连续积分通过**数值积分**近似。将光线 $[t_n, t_f]$ 分为 $N$ 个区间，在每个区间内均匀采样一个点 $t_i$：

$$t_i \sim \mathcal{U}\left[t_n + \frac{i-1}{N}(t_f - t_n), \; t_n + \frac{i}{N}(t_f - t_n)\right]$$

离散化后的渲染公式：

$$\hat{C}(\mathbf{r}) = \sum_{i=1}^{N} T_i \, \alpha_i \, \mathbf{c}_i$$

其中：

$$\alpha_i = 1 - \exp(-\sigma_i \delta_i)$$

$$T_i = \prod_{j=1}^{i-1} (1 - \alpha_j)$$

- $\delta_i = t_{i+1} - t_i$：相邻采样点间距
- $\alpha_i$：第 $i$ 个采样点的不透明度（alpha 值）
- $T_i$：到达第 $i$ 个点之前的累积透射率

**注意**：在区间内随机采样引入了**分层采样 (stratified sampling)** 的效果，相当于正则化，防止网络只记住固定采样位置。

### 2.4 位置编码 (Positional Encoding)

直接将 $(x, y, z, \theta, \phi)$ 输入 MLP 效果很差——场景的高频细节（纹理、边缘）全都糊掉了。

**原因**：MLP 存在 **spectral bias（频谱偏好）**，即深度网络倾向于先学低频函数，高频信号收敛极慢。[Rahaman et al., 2019] 从理论上证明了这一点。

**解决方案**：先将低维输入映射到高维 Fourier 特征空间：

$$\gamma(p) = \left(\sin(2^0 \pi p), \cos(2^0 \pi p), \sin(2^1 \pi p), \cos(2^1 \pi p), \ldots, \sin(2^{L-1} \pi p), \cos(2^{L-1} \pi p)\right)$$

对每个标量坐标分别应用，得到 $2L$ 维向量。

- **位置坐标**：$L = 10$，即 $(x,y,z)$ 从 3 维 → $3 \times 2 \times 10 = 60$ 维
- **方向坐标**：$L = 4$，即 $(\theta,\phi)$ 或方向向量从 3 维 → $3 \times 2 \times 4 = 24$ 维

**为什么有效？**

1. **频率覆盖**：$2^0$ 到 $2^{L-1}$ 的频率覆盖了从低频到高频的完整频谱
2. **理论支撑**：Tancik et al. (2020, Fourier Features) 证明，经过 Fourier 映射后，对应的 NTK (Neural Tangent Kernel) 的带宽显著增大，使得网络能够拟合高频信号
3. **与 Transformer 的联系**：形式与 Transformer 的位置编码相同，但动机不同——Transformer 是为了编码序列顺序，NeRF 是为了克服频谱偏好

### 2.5 层级采样 (Hierarchical Sampling)

朴素均匀采样的问题：大量采样点落在空白区域或被遮挡区域，浪费计算。

**解决方案**：使用 coarse（粗）+ fine（精）两个网络：

```
Step 1: Coarse 网络
  └─ 均匀采样 Nc = 64 个点
  └─ 得到每个点的 (σ, c)
  └─ 计算离散权重 wᵢ = Tᵢ αᵢ

Step 2: Fine 网络
  └─ 将 wᵢ 归一化为 PDF
  └─ 用逆 CDF 变换 (Inverse CDF Transform) 按该分布额外采样 Nf = 128 个点
  └─ 合并 Nc + Nf = 192 个点，重新排序后通过 fine 网络渲染

最终输出 = fine 网络的渲染结果
```

**效果**：fine 采样点集中在物体表面附近（密度高的区域），大幅提升渲染质量。

### 2.6 网络架构

```
位置编码后的坐标 (60D)
       │
       ▼
  ┌──────────┐
  │ FC 256   │ ← Layer 1
  │ ReLU     │
  ├──────────┤
  │ FC 256   │ ← Layer 2
  │ ReLU     │
  ├──────────┤
  │ FC 256   │ ← Layer 3
  │ ReLU     │
  ├──────────┤
  │ FC 256   │ ← Layer 4
  │ ReLU     │
  ├──────────┤
  │ FC 256   │ ← Layer 5  ◄── Skip Connection (拼接原始 60D 输入)
  │ ReLU     │
  ├──────────┤
  │ FC 256   │ ← Layer 6
  │ ReLU     │
  ├──────────┤
  │ FC 256   │ ← Layer 7
  │ ReLU     │
  ├──────────┤
  │ FC 256   │ ← Layer 8
  │ (无激活) │
  └────┬─────┘
       │
  ┌────┴─────┐
  │          │
  ▼          ▼
FC→σ      FC 256
(ReLU,    (拼接方向编码 24D)
 保证≥0)     │
             ▼
          FC 128
          ReLU
             │
             ▼
          FC→(r,g,b)
          Sigmoid
```

关键设计：
- **8 层 MLP，256 维隐藏层**
- **Layer 5 有 skip connection**：防止梯度消失，帮助保留位置信息
- **密度 $\sigma$ 在 Layer 8 之后直接输出**（ReLU 保证非负）
- **颜色 $\mathbf{c}$ 在注入方向信息后额外通过 1 层计算**：确保方向依赖仅影响颜色

### 2.7 训练流程

```
输入：多视角图像 {Iᵢ} + 对应相机位姿 {Pᵢ} (通常由 COLMAP 预计算)

每个训练步：
  1. 随机选一张图片，随机选一批像素 (batch of rays)
  2. 对每条光线 r = o + td：
     a. Coarse 采样 → MLP → 体渲染 → Ĉ_coarse(r)
     b. 基于 coarse 权重，fine 采样 → MLP → 体渲染 → Ĉ_fine(r)
  3. 计算损失：
     L = Σᵣ [ ||Ĉ_coarse(r) - C_gt(r)||² + ||Ĉ_fine(r) - C_gt(r)||² ]
  4. 反向传播更新两个网络的参数

训练配置（原论文）：
  - Batch size: 4096 rays
  - 迭代次数: ~200K steps
  - 优化器: Adam (lr = 5e-4, 指数衰减)
  - 训练时间: ~1-2 天 (单张 V100)
  - 每个场景独立训练一个网络
```

**注意**：NeRF 是 **per-scene optimization**，即每个场景需要从头训练一个新网络，没有跨场景泛化能力。

## 三、NeRF 的核心问题与改进

### 3.1 训练速度

原始 NeRF 训练一个场景需要 1-2 天，推理渲染一帧需要约 30 秒——完全无法实时应用。

#### Instant-NGP (2022, NVIDIA)

**核心贡献**：用**多分辨率哈希编码 (Multi-Resolution Hash Encoding)** 替代 MLP 中的位置编码和大量隐藏层。

```
传统 NeRF:
  坐标 → 位置编码(60D) → 8层MLP(256D) → (σ, c)
  瓶颈：每个采样点都需要 8 层前向传播

Instant-NGP:
  坐标 → 多分辨率哈希表查询 → 小型 MLP(2层,64D) → (σ, c)
  核心：用哈希表存储特征，MLP 只做轻量融合
```

**多分辨率哈希编码原理**：

1. 设置 $L$ 个分辨率等级（如 $L=16$），从粗到细
2. 每个等级维护一个固定大小的哈希表（如 $T = 2^{19}$ 个条目），每个条目存储 $F$ 维特征向量
3. 对于输入坐标 $\mathbf{x}$：
   - 在每个等级 $l$ 上，找到 $\mathbf{x}$ 所在的体素格子
   - 对格子 8 个顶点的坐标进行哈希：$h(\mathbf{x}) = \left(\bigoplus_{i=1}^{3} x_i \cdot \pi_i\right) \mod T$
   - 查表获取顶点特征，三线性插值得到该等级的特征
4. 拼接所有等级特征 → 输入小型 MLP

**为什么快？**
- 哈希表查询是 $O(1)$，替代了 MLP 的 $O(D \times W^2)$ 计算
- 哈希冲突通过梯度自动解决（重要区域的梯度大，自然占据哈希表）
- 训练从 **小时级 → 5 秒级**，推理达到实时

#### TensoRF (2022)

用**张量分解**表示辐射场：将 3D 体素特征分解为低秩的向量和矩阵分量。

$$\mathcal{G} \approx \sum_{r=1}^{R} \mathbf{v}_r^X \circ \mathbf{M}_r^{YZ} + \mathbf{v}_r^Y \circ \mathbf{M}_r^{XZ} + \mathbf{v}_r^Z \circ \mathbf{M}_r^{XY}$$

- $\mathbf{v}_r$ 是向量分量，$\mathbf{M}_r$ 是矩阵分量
- 存储从 $O(N^3)$ 降为 $O(N^2)$
- 训练约 30 分钟，渲染质量与 NeRF 持平

### 3.2 泛化能力

原始 NeRF 是 per-scene 的，每个场景需要独立训练。如何让模型泛化到新场景？

#### pixelNeRF (2021)

```
输入图像 → CNN 提取特征图 → 特征 + 坐标 → MLP → (σ, c)
```

- 将图像特征作为条件注入 NeRF，实现 **1-2 张图像即可重建**
- 关键：用图像特征提供场景先验，不再需要从零学习

#### IBRNet (2021)

- 多视角图像特征聚合 + Transformer 注意力
- 泛化到未见场景，无需 per-scene 微调

#### NeRF in the Wild (NeRF-W, 2021)

处理真实世界的 **非受控拍摄**（光照变化、行人遮挡等）：
- 为每张图片学习一个 **外观嵌入 (appearance embedding)**：处理光照变化
- 引入 **瞬态物体建模**：用额外的网络分支预测瞬态物体的密度和颜色
- 可以从互联网旅游照片重建地标建筑

### 3.3 动态场景

原始 NeRF 只能表示静态场景。

#### D-NeRF (2021)

```
静态 NeRF:  F(x, d) → (c, σ)
D-NeRF:     (x, t) → Δx = deformation_net(x, t)
            F(x + Δx, d) → (c, σ)
```

- 引入 **变形场 (deformation field)**：将每个时刻的场景变形回一个规范空间
- 适用于拓扑不变的运动（如人体姿态变化）

#### Nerfies (2021)

- 类似思路，但使用 SE(3) 变形场（刚体变换）
- 专注于人脸/自拍场景

#### HyperNeRF (2021)

- 引入高维 ambient space 处理**拓扑变化**（如张嘴、切割物体）
- 将 3D 坐标扩展到高维 $\mathbf{x} \to (\mathbf{x}, w(\mathbf{x}, t))$，其中 $w$ 是学习到的高维切片坐标

### 3.4 大规模场景

原始 NeRF 适用于小型物体/室内场景，无法处理城市级规模。

#### Block-NeRF (2022, Waymo)

```
城市场景 → 分成多个重叠区域 → 每个区域训练一个 NeRF → 渲染时按距离混合
```

- 可处理旧金山整条街道的规模
- 支持增量更新（只重训改变的区域）

#### Mega-NeRF (2022)

- 类似的分块策略 + 飞行器采集的大规模场景

#### Mip-NeRF (2021) & Mip-NeRF 360 (2022)

**Mip-NeRF** 解决 **锯齿 (aliasing)** 问题：

- 原始 NeRF 对每条光线采样离散点 → 不同距离的物体采样密度不同 → 近处过采样，远处欠采样
- Mip-NeRF 用**锥形区域 (cone)** 替代光线，对锥截面内的高斯分布积分，实现多尺度表示

$$\gamma_{\text{IPE}}(\mu, \sigma^2) = \exp(-2^{2l}\pi^2 \sigma^2/2) \cdot \begin{pmatrix} \sin(2^l\pi\mu) \\ \cos(2^l\pi\mu) \end{pmatrix}$$

**Mip-NeRF 360** 进一步扩展到 **360° 无界场景**：
- 引入非线性坐标收缩 (contraction)，将无限远映射到有限空间
- Proposal network 替代 coarse 网络，更高效的层级采样

## 四、NeRF 与 3DGS 的对比

3D Gaussian Splatting (3DGS, 2023) 是 NeRF 之后最重要的新视角合成方法。

| 维度 | NeRF | 3D Gaussian Splatting |
|:-----|:-----|:---------------------|
| **表示方式** | 隐式连续（MLP / 哈希表） | 显式离散（3D 高斯椭球集合） |
| **渲染方式** | 体渲染（ray marching） | 光栅化（splatting） |
| **训练速度** | 慢（原始版小时级，Instant-NGP 秒级） | 快（约 30 分钟） |
| **渲染速度** | 慢（需逐像素发射光线） | 快（实时 100+ FPS） |
| **内存占用** | 小（仅网络参数） | 大（百万个高斯，每个存储位置/协方差/颜色/透明度） |
| **编辑性** | 差（隐式表示难以直接操作） | 好（可直接移动/删除/添加高斯） |
| **质量** | 高（连续表示天然抗锯齿） | 高（但可能有 floater 伪影） |
| **拓扑灵活** | 高（连续场天然表示任意拓扑） | 中（需要足够多高斯覆盖） |
| **训练数据** | 多视角图像 + 位姿 | 多视角图像 + 位姿 + SfM 点云初始化 |

**总结**：
- NeRF 在**紧凑性**和**理论优雅性**上更优
- 3DGS 在**速度**和**可编辑性**上更优
- 实际应用中，3DGS 因为实时渲染能力逐渐成为主流
- 两者都在快速发展，且有融合趋势（如用高斯表示但用体渲染训练）

## 五、面试高频问题

### Q1: 体渲染方程的物理含义是什么？

**回答要点**：

体渲染方程描述了光线穿过半透明介质的累积过程：

$$C(\mathbf{r}) = \int_{t_n}^{t_f} T(t) \, \sigma(\mathbf{r}(t)) \, \mathbf{c}(\mathbf{r}(t), \mathbf{d}) \, dt$$

- $T(t) = \exp(-\int_{t_n}^{t} \sigma(s) ds)$ 是光线到达 $t$ 处时还"剩余"多少比例（透射率）
- $\sigma(t)$ 是 $t$ 处的密度，表示该点吸收/散射光线的能力
- $T(t) \cdot \sigma(t) \cdot dt$ 是光线在 $[t, t+dt]$ 区间被终止的概率
- 最终颜色 = 沿光线对每个位置的颜色按"被终止概率"加权求和

离散化后，$\alpha_i = 1 - e^{-\sigma_i \delta_i}$ 是 alpha compositing 中的透明度，$T_i = \prod_{j<i}(1-\alpha_j)$ 是前面所有层的累积透过率。这与传统图形学中的 **alpha blending** (从前到后) 完全一致。

### Q2: 为什么需要位置编码？

**回答要点**：

1. **根本原因**：MLP 存在 spectral bias，倾向于学习低频函数。直接输入 $(x,y,z)$ 这样的低维坐标，网络无法拟合高频的纹理和边缘细节。

2. **位置编码的作用**：$\gamma(p) = (\sin(2^l \pi p), \cos(2^l \pi p))_{l=0}^{L-1}$ 将输入映射到 $2L$ 维空间。不同频率的正弦函数提供了从低频到高频的完整频谱基底。

3. **理论解释**：Tancik et al. (2020) 从 NTK (Neural Tangent Kernel) 角度证明，Fourier 特征映射改变了核函数的带宽——原始输入对应窄带核（只能拟合低频），Fourier 映射后对应宽带核（可以拟合高频）。

4. **$L$ 的选择**：$L$ 过小 → 高频细节丢失；$L$ 过大 → 训练不稳定（过度关注高频噪声）。位置取 $L=10$，方向取 $L=4$（方向的变化通常更平滑）。

### Q3: NeRF 的主要瓶颈是什么？

1. **训练速度慢**：每个场景需要独立训练，原始版本需要 1-2 天
2. **渲染速度慢**：逐像素发射光线 + 每条光线数百次 MLP 前向传播 → 无法实时
3. **无泛化能力**：per-scene optimization，无法迁移到新场景
4. **依赖精确位姿**：需要 COLMAP 等预计算精确的相机内外参
5. **静态场景假设**：无法处理动态物体
6. **有界场景**：原始方法仅适用于小型有界场景

### Q4: Instant-NGP 如何加速？

**核心思路**：用**多分辨率哈希表查询**替代**大型 MLP 的前向传播**。

1. **编码加速**：传统位置编码后仍需 8 层 MLP 解码。Instant-NGP 用哈希表存储空间特征，查表是 $O(1)$，MLP 缩小到 2 层 64 维。
2. **多分辨率设计**：$L=16$ 个分辨率等级，粗等级捕获全局结构，细等级捕获细节。每个等级独立查表 + 三线性插值。
3. **哈希冲突处理**：不同位置可能哈希到同一个 entry（冲突）。但重要区域（物体表面）的梯度更大，训练时自然"挤走"不重要区域的信息。
4. **GPU 友好**：哈希表操作可以高度并行化，完美适配 GPU。

**结果**：训练从小时级降到 5 秒，渲染达到实时（~15 FPS）。

### Q5: NeRF 中的 coarse-to-fine 采样为什么有效？

均匀采样的问题：光线经过的大部分空间是空的（密度 $\sigma \approx 0$），有效采样点很少。

Coarse 网络先用少量均匀采样点"探路"，得到权重分布 $w_i = T_i \alpha_i$。这个分布告诉我们光线在哪些位置与物体相交（权重大的地方）。Fine 网络根据这个分布做重要性采样，将更多点集中在物体表面附近。

本质上是一种 **importance sampling**：用 coarse 网络的输出作为 proposal distribution，指导 fine 网络的采样。

### Q6: NeRF 为什么将密度和颜色的依赖分开？

- **密度 $\sigma$ 仅依赖位置 $\mathbf{x}$**：物体是否存在于某个位置，是与观察方向无关的物理事实
- **颜色 $\mathbf{c}$ 依赖位置和方向 $(\mathbf{x}, \mathbf{d})$**：同一点的颜色会随视角变化（镜面反射、高光等非朗伯效应）

如果密度也依赖方向，网络可能学到"从某个角度看过去物体消失了"这样的非物理解，导致多视角不一致。
