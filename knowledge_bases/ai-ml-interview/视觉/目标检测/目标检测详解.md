## 一、问题定义与评估指标

### 1.1 任务定义

目标检测 (Object Detection) 的任务是：给定一张图像，**同时定位和识别**图中所有感兴趣的目标。

$$\text{输入: } I \in \mathbb{R}^{H \times W \times 3} \quad \longrightarrow \quad \text{输出: } \{(c_i, b_i)\}_{i=1}^{N}$$

其中 $c_i$ 是类别标签，$b_i = (x, y, w, h)$ 是边界框 (bounding box)。

与分类不同，检测需要回答 **三个问题**：有没有目标？目标在哪里？目标是什么？

### 1.2 IoU (Intersection over Union)

IoU 衡量预测框与真实框的重叠程度：

$$\text{IoU} = \frac{|B_{\text{pred}} \cap B_{\text{gt}}|}{|B_{\text{pred}} \cup B_{\text{gt}}|}$$

```
  ┌──────────┐
  │  B_pred   │
  │     ┌─────┼────┐
  │     │ ∩∩∩ │    │
  └─────┼─────┘    │
        │   B_gt   │
        └──────────┘

IoU = 交集面积 / 并集面积
```

- IoU = 1 表示完美重合
- IoU > 0.5 通常判定为正确检测 (True Positive)

### 1.3 mAP 计算

**AP (Average Precision)**：单个类别的 Precision-Recall 曲线下面积。

1. 将所有预测按置信度降序排列
2. 逐个判断是 TP (IoU ≥ 阈值) 还是 FP
3. 计算每个位置的 Precision 和 Recall
4. 对 PR 曲线做插值后积分

$$\text{AP} = \int_0^1 p(r) \, dr$$

**mAP (mean AP)**：所有类别 AP 的平均值。

| 指标 | 说明 |
|------|------|
| AP@0.5 | IoU 阈值 = 0.5，PASCAL VOC 主指标 |
| AP@0.75 | 更严格的匹配 |
| AP@[0.5:0.95] | IoU 从 0.5 到 0.95 步长 0.05 的平均，**COCO 主指标** |
| AP_S / AP_M / AP_L | 按目标面积分：小 (<32²)、中 (32²~96²)、大 (>96²) |

### 1.4 NMS (Non-Maximum Suppression)

一个目标可能产生多个高置信度预测框，NMS 用来去除冗余：

```python
def nms(boxes, scores, iou_threshold=0.5):
    keep = []
    order = scores.argsort(descending=True)
    while order.numel() > 0:
        i = order[0]
        keep.append(i)
        if order.numel() == 1:
            break
        ious = compute_iou(boxes[i], boxes[order[1:]])
        mask = ious <= iou_threshold
        order = order[1:][mask]
    return keep
```

**NMS 的问题**：
- 阈值敏感：阈值高 → 冗余框多；阈值低 → 遮挡目标被误删
- 顺序贪心，非全局最优
- 后续的 Soft-NMS 改为衰减而非直接丢弃

## 二、两阶段检测器 (Two-Stage)

两阶段检测器的核心思想：**先提候选区域 (Region Proposal)，再精细分类和回归**。

### 2.1 R-CNN (2014)

R-CNN 是深度学习目标检测的开山之作，首次证明 CNN 特征在检测任务上的优越性。

**流程**：

```
输入图像
   │
   ▼
Selective Search ──→ ~2000 个候选框 (Region Proposals)
   │
   ▼ (对每个候选框)
Resize 到固定尺寸 (227×227) ──→ CNN 提取特征 ──→ SVM 分类
                                                    │
                                                    ▼
                                              BBox 回归微调
```

**Selective Search**：基于颜色、纹理等启发式合并超像素，生成约 2000 个候选区域。

**关键问题**：
- 每个候选框都要**单独过一遍 CNN**，一张图需要前向传播 ~2000 次，**极慢** (GPU 上约 13s/张)
- 多阶段训练：CNN 特征提取、SVM 分类、BBox 回归各自独立训练
- 特征存储到磁盘，占用大量空间

### 2.2 Fast R-CNN (2015)

**核心改进**：整张图像只过 CNN 一次，候选框在特征图上提取。

```
输入图像 ──→ CNN Backbone ──→ 整图特征图
                                  │
Selective Search ──→ 候选框 ──→ RoI Pooling ──→ FC ──→ 分类 + 回归
```

**RoI Pooling 详解**：

将任意大小的候选区域映射到固定大小 (如 7×7) 的特征图：

1. 将 RoI 对应的特征图区域划分为 $H \times W$ 的网格 (如 7×7)
2. 每个网格内做 max pooling
3. 输出固定大小的特征

```
原始 RoI 特征区域 (5×7)        RoI Pooling 输出 (2×2)
┌─┬──┬──┬──┬──┬──┬─┐         ┌───┬───┐
│3│ 1│ 4│ 1│ 5│ 9│2│         │ 4 │ 9 │  (max of each grid)
├─┼──┼──┼──┼──┼──┼─┤         ├───┼───┤
│6│ 5│ 3│ 5│ 8│ 9│7│         │ 6 │ 9 │
├─┼──┼──┼──┼──┼──┼─┤         └───┴───┘
│9│ 3│ 2│ 3│ 8│ 4│6│
├─┼──┼──┼──┼──┼──┼─┤
│2│ 6│ 4│ 3│ 3│ 8│3│
├─┼──┼──┼──┼──┼──┼─┤
│2│ 7│ 1│ 8│ 2│ 0│9│
└─┴──┴──┴──┴──┴──┴─┘
```

**多任务损失**：分类和回归联合训练

$$L = L_{\text{cls}}(p, u) + \lambda [u \geq 1] L_{\text{reg}}(t, v)$$

- $L_{\text{cls}}$：交叉熵损失
- $L_{\text{reg}}$：Smooth L1 损失 (对离群值更鲁棒)
- $[u \geq 1]$：只对非背景类回归

**速度提升**：训练快 9×，测试快 213× (对比 R-CNN)。但 Selective Search 仍然是瓶颈。

### 2.3 Faster R-CNN (2015)

**核心创新：Region Proposal Network (RPN)** — 用网络替代 Selective Search 生成候选框。

```
输入图像 ──→ CNN Backbone ──→ 共享特征图 ──┬──→ RPN ──→ 候选框
                                           │            │
                                           └──→ RoI Pooling ──→ 分类 + 回归
```

#### Anchor 机制详解

在特征图的**每个位置**放置 $k$ 个预定义的 anchor (锚框)：

```
原图某位置的 9 个 anchor（3 尺度 × 3 比例）:

  ┌─────────────────┐   比例 1:2
  │                 │
  └─────────────────┘
       ┌────────┐       比例 1:1
       │        │
       │        │
       └────────┘
     ┌───┐              比例 2:1
     │   │
     │   │
     │   │
     │   │
     └───┘

每种比例 × 3 种尺度 (128², 256², 512²) = 9 个 anchor
```

**RPN 结构**：

```
共享特征图 (H×W×C)
     │
     ▼
3×3 conv, 256 ──→ 中间特征
     │
     ├──→ 1×1 conv, 2k ──→ 前景/背景分类 (每 anchor 2 个值)
     │
     └──→ 1×1 conv, 4k ──→ 框回归 Δ(x,y,w,h) (每 anchor 4 个值)
```

**正负样本匹配**：
- **正样本**：与某个 GT 的 IoU ≥ 0.7，或者是某个 GT 的最大 IoU anchor
- **负样本**：与所有 GT 的 IoU < 0.3
- 其余 anchor 忽略不参与训练
- 正负比例 1:1，每个 mini-batch 256 个 anchor

**回归目标参数化**：

$$t_x = \frac{x - x_a}{w_a}, \quad t_y = \frac{y - y_a}{h_a}$$

$$t_w = \log\frac{w}{w_a}, \quad t_h = \log\frac{h}{h_a}$$

其中 $(x_a, y_a, w_a, h_a)$ 是 anchor 的坐标，$(x, y, w, h)$ 是目标框坐标。

**总损失函数**：

$$L = \frac{1}{N_{\text{cls}}} \sum_i L_{\text{cls}}(p_i, p_i^*) + \lambda \frac{1}{N_{\text{reg}}} \sum_i p_i^* L_{\text{reg}}(t_i, t_i^*)$$

其中 $p_i^*$ 是标签 (正样本为 1)，$L_{\text{reg}}$ 为 Smooth L1 Loss：

$$\text{Smooth}_{L_1}(x) = \begin{cases} 0.5x^2 & \text{if } |x| < 1 \\ |x| - 0.5 & \text{otherwise} \end{cases}$$

**训练方式演进**：
1. 交替训练 (原始方案)：RPN → Fast R-CNN → 微调 RPN → 微调 Fast R-CNN
2. 近似联合训练：合并 RPN 和 Fast R-CNN 损失，忽略 RoI 梯度
3. 端到端训练：使用 RoI Align (Mask R-CNN) 后完全可微

### 2.4 FPN (2017)

**Feature Pyramid Network** 解决了检测中的**多尺度问题**。

**动机**：CNN 高层特征语义强但分辨率低 (不利于小目标)，低层特征分辨率高但语义弱。

```
自底向上           自顶向下 + 横向连接 = FPN

C5 (1/32) ──────── P5 (1/32) ──→ 检测大目标
  │                  ↑
C4 (1/16) ──────── P4 (1/16) ──→ 检测中目标
  │                  ↑
C3 (1/8)  ──────── P3 (1/8)  ──→ 检测中小目标
  │                  ↑
C2 (1/4)  ──────── P2 (1/4)  ──→ 检测小目标

横向连接：1×1 conv 统一通道数
自顶向下：2× 上采样 + 逐元素相加
```

**对小目标的意义**：
- 小目标在高分辨率层 (P2) 上检测，特征更丰富
- 通过自顶向下路径，低层特征获得了高层语义信息
- FPN 成为后续几乎所有检测器的标配 neck

## 三、单阶段检测器 (One-Stage)

单阶段检测器**跳过候选框生成**，直接在特征图上回归类别和边界框，以速度换取精度。

### 3.1 YOLO 系列

#### YOLOv1 (2016)

**核心思想**：将检测转化为**回归问题**，一次前向传播同时预测所有目标。

```
输入图像 (448×448)
     │
     ▼
CNN Backbone (24 conv + 2 FC)
     │
     ▼
输出张量: S × S × (B × 5 + C)
         7 × 7 × (2 × 5 + 20) = 7 × 7 × 30

S = 7 (网格数)
B = 2 (每格预测 2 个框)
C = 20 (PASCAL VOC 类别数)
每个框预测: (x, y, w, h, confidence)
```

```
┌───┬───┬───┬───┬───┬───┬───┐
│   │   │   │   │   │   │   │
├───┼───┼───┼───┼───┼───┼───┤
│   │   │   │   │   │   │   │  每个格子预测：
├───┼───┼───┼───┼───┼───┼───┤  - B 个边界框
│   │   │ ★ │   │   │   │   │  - 每个框的置信度
├───┼───┼───┼───┼───┼───┼───┤  - C 个类别概率
│   │   │   │   │   │   │   │
├───┼───┼───┼───┼───┼───┼───┤  目标中心落在哪个格子，
│   │   │   │   │   │   │   │  该格子负责检测该目标
├───┼───┼───┼───┼───┼───┼───┤
│   │   │   │   │   │   │   │
├───┼───┼───┼───┼───┼───┼───┤
│   │   │   │   │   │   │   │
└───┴───┴───┴───┴───┴───┴───┘
```

**置信度定义**：

$$\text{Confidence} = P(\text{Object}) \times \text{IoU}_{\text{pred}}^{\text{gt}}$$

**优势**：45 FPS (Fast YOLO 155 FPS)，适合实时应用。

**问题**：每个格子只预测 2 个框，且只有 1 个类别 → 密集小目标检测差。

#### YOLOv2 / YOLOv3

| 改进 | YOLOv2 (2017) | YOLOv3 (2018) |
|------|---------------|---------------|
| Backbone | Darknet-19 | Darknet-53 (残差) |
| Anchor | 引入 anchor box (k-means 聚类) | 同 v2 |
| 多尺度 | passthrough layer | **3 个尺度的 FPN 式检测** |
| 分类 | softmax | sigmoid (支持多标签) |
| 其他 | Batch Norm、高分辨率预训练 | 更深网络 |

#### YOLOv4 / YOLOv5

引入大量工程优化，分为两类：

- **Bag of Freebies** (只增加训练代价，不增加推理代价)：
  - 数据增强：Mosaic (4 图拼接)、CutMix、MixUp
  - 标签平滑、Self-adversarial Training

- **Bag of Specials** (少量推理代价换取大精度提升)：
  - CSPNet backbone：减少计算量同时保持精度
  - SPP (Spatial Pyramid Pooling)
  - PANet (Path Aggregation Network)
  - Mish 激活函数

#### YOLOv8 ~ YOLOv11 及 YOLO-World

- **YOLOv8** (Ultralytics, 2023)：Anchor-Free 设计、C2f 模块、解耦 head
- **YOLOv9** (2024)：PGI (Programmable Gradient Information)、GELAN
- **YOLOv10** (2024)：NMS-Free 训练 (双标签分配策略)
- **YOLOv11** (2024)：C3k2 模块、SPPF 改进
- **YOLO-World** (2024)：开放词汇检测，文本引导的实时检测

**YOLO 核心设计哲学**：速度优先，适合边缘设备和实时部署。

### 3.2 SSD (2016)

**核心思想**：在 **多尺度特征图** 上直接预测，不同层负责不同大小的目标。

```
VGG-16 Backbone (截断)
     │
     ├──→ Conv4_3 (38×38) ──→ 检测小目标  (4 anchor/位置)
     │
     ├──→ Conv7    (19×19) ──→ 检测中目标  (6 anchor/位置)
     │
     ├──→ Conv8_2  (10×10) ──→                (6 anchor/位置)
     │
     ├──→ Conv9_2  (5×5)   ──→ 检测大目标  (6 anchor/位置)
     │
     ├──→ Conv10_2 (3×3)   ──→                (6 anchor/位置)
     │
     └──→ Conv11_2 (1×1)   ──→                (4 anchor/位置)

每个位置: cls (C+1 类) + reg (4 值) × anchor 数
总共 ~8732 个预测框
```

**与 YOLO 对比**：SSD 比 YOLOv1 精度高 (多尺度)，速度相当。

### 3.3 RetinaNet (2017)

**核心贡献**：提出 **Focal Loss**，揭示了单阶段检测器之前不如两阶段的根本原因——**正负样本极度不平衡**。

#### 为什么单阶段检测器之前比两阶段差？

- 两阶段检测器有 RPN 预筛选 → 第二阶段的正负比例约 1:3
- 单阶段检测器直接在所有位置预测 → 正负比例可达 **1:1000**
- 大量简单负样本 (背景) 的损失淹没了少量正样本的有效梯度

#### Focal Loss 推导

标准交叉熵：

$$\text{CE}(p_t) = -\log(p_t)$$

其中 $p_t = p$ (当 $y=1$) 或 $p_t = 1-p$ (当 $y=0$)。

问题：即使分类正确的简单样本 ($p_t$ 大)，仍有不可忽略的损失，累积起来淹没难样本。

**Focal Loss**：

$$\text{FL}(p_t) = -\alpha_t (1 - p_t)^\gamma \log(p_t)$$

- $(1 - p_t)^\gamma$ 是**调制因子**：当 $p_t \to 1$ (易分类)，因子 → 0，损失被大幅降低
- $\gamma$ 控制降低程度：$\gamma = 0$ 退化为 CE；**$\gamma = 2$ 效果最好**
- $\alpha_t$ 类别权重，一般正类 $\alpha = 0.25$

```
损失值
  │
  │  CE: ────────────────
  │      ╲
  │       ╲  γ=1
  │        ╲╲
  │         ╲╲  γ=2
  │          ╲╲╲
  │           ╲╲╲  γ=5
  │            ╲╲╲╲
  └──────────────────── p_t
  0                    1

易分类样本 (p_t → 1): Focal Loss → 0
难分类样本 (p_t → 0): Focal Loss ≈ CE
```

**直观理解**：Focal Loss 让模型**聚焦于难分类样本**，自动降低大量容易分类的背景样本的权重。

**RetinaNet 架构**：ResNet + FPN + 两个子网 (分类 + 回归)，配合 Focal Loss 首次让单阶段精度超越两阶段。

### 3.4 Anchor-Free 方法

Anchor-based 方法的问题：
- 需要手动设计 anchor 尺寸和比例
- 大量 anchor 导致正负样本不平衡
- anchor 与 GT 匹配规则复杂

#### FCOS (2019)

**Fully Convolutional One-Stage** — 逐像素预测，完全无 anchor。

```
FPN 特征图上的每个位置 (x, y)
     │
     ├──→ 分类: C 维 sigmoid
     │
     ├──→ 回归: (l, t, r, b) — 到 GT 框四边的距离
     │
     └──→ Centerness: 抑制远离目标中心的低质量预测

       l     t
    ◄────┐  ↑
         │  │
    (x,y)●──┘
         │
         ▼  ──────►
         b     r
```

**Centerness** 分支：

$$\text{centerness}^* = \sqrt{\frac{\min(l, r)}{\max(l, r)} \times \frac{\min(t, b)}{\max(t, b)}}$$

位于目标中心时 centerness = 1，边缘时 → 0。推理时将 centerness 乘以分类分数来降低边缘预测的权重。

**多尺度分配**：不同 FPN 层负责不同大小范围的目标 (按 $\max(l, t, r, b)$ 分配)。

#### CenterNet (2019)

**核心思想**：将目标检测建模为**关键点检测**问题 — 只预测目标的中心点。

```
输入图像
     │
     ▼
Backbone (Hourglass / DLA / ResNet)
     │
     ▼
输出 Heatmap (H/4 × W/4 × C)   ← 每个类别一个热力图
  + Offset (H/4 × W/4 × 2)      ← 中心点亚像素偏移
  + Size (H/4 × W/4 × 2)        ← 宽高回归
```

- 中心点用 Gaussian kernel 渲染到 heatmap
- 推理时取 heatmap 峰值 (local maxima) 作为检测结果
- **无需 NMS**：峰值提取天然去除冗余
- 同一框架可扩展到 3D 检测、人体姿态估计

## 四、基于 Transformer 的检测器

### 4.1 DETR (2020, Meta)

**Detection Transformer** — 目标检测领域的范式革命。

#### 核心创新

1. **去掉 NMS**：通过集合预测 + 二分图匹配，不再需要后处理
2. **去掉 Anchor**：用可学习的 Object Query 替代
3. **端到端**：真正的 end-to-end 检测

#### 架构

```
输入图像
     │
     ▼
CNN Backbone (ResNet-50)
     │
     ▼
特征图 (H/32 × W/32 × 2048)
     │
     ├──→ 1×1 conv 降维到 d=256
     │
     ├──→ + Positional Encoding (固定正弦位置编码)
     │
     ▼
Transformer Encoder (6 层)     ← 全局自注意力，建模像素间关系
     │
     ▼
Transformer Decoder (6 层)     ← 交叉注意力 + 自注意力
     ↑
Object Queries (N=100, 可学习)  ← 每个 query 负责预测一个目标
     │
     ▼
FFN (Feed-Forward Network)
     │
     ├──→ 分类: softmax (C+1 类, 含 "no object")
     └──→ 回归: sigmoid (归一化的 cx, cy, w, h)
```

#### Object Query 的直觉

- 100 个可学习向量，每个 query "学会" 关注图像的不同区域和不同类型的目标
- 可以理解为**可学习的"检测槽位"**
- query 之间的自注意力让它们互相协调，避免重复检测同一目标

#### 匈牙利匹配 (Hungarian Matching)

DETR 预测 $N$ 个结果 (固定为 100)，GT 有 $M$ 个目标 ($M \ll N$)，需要找最优一一对应：

$$\hat{\sigma} = \arg\min_{\sigma \in \mathfrak{S}_N} \sum_{i=1}^{N} \mathcal{L}_{\text{match}}(y_i, \hat{y}_{\sigma(i)})$$

**匹配代价**：

$$\mathcal{L}_{\text{match}} = -\mathbb{1}_{c_i \neq \varnothing} \hat{p}_{\sigma(i)}(c_i) + \mathbb{1}_{c_i \neq \varnothing} \left[ \lambda_{\text{L1}} \|b_i - \hat{b}_{\sigma(i)}\|_1 + \lambda_{\text{giou}} \mathcal{L}_{\text{GIoU}}(b_i, \hat{b}_{\sigma(i)}) \right]$$

使用 **匈牙利算法** (scipy.linear_sum_assignment) 在 $O(N^3)$ 时间内求解最优匹配。

```
预测 (100个)           GT (3个 + 97个∅)
   ŷ₁  ──────────────── y₁ (dog)
   ŷ₂  ──────────────── y₂ (cat)
   ŷ₃  ──────────────── y₃ (car)
   ŷ₄  ──────────────── ∅ (no object)
   ...                   ...
   ŷ₁₀₀ ─────────────── ∅ (no object)

匈牙利算法找到代价最小的一一匹配
```

#### 训练损失

匹配后的损失：

$$\mathcal{L} = \sum_{i=1}^{N} \left[ -\log \hat{p}_{\hat{\sigma}(i)}(c_i) + \mathbb{1}_{c_i \neq \varnothing} \mathcal{L}_{\text{box}}(b_i, \hat{b}_{\hat{\sigma}(i)}) \right]$$

$$\mathcal{L}_{\text{box}} = \lambda_{\text{L1}} \|b_i - \hat{b}\|_1 + \lambda_{\text{giou}} \mathcal{L}_{\text{GIoU}}(b_i, \hat{b})$$

**GIoU (Generalized IoU)**：

$$\text{GIoU} = \text{IoU} - \frac{|C \setminus (A \cup B)|}{|C|}$$

其中 $C$ 是包含 $A$ 和 $B$ 的最小外接矩形。GIoU ∈ [-1, 1]，比 IoU 对不重叠框也有梯度。

#### DETR 的问题

- **收敛慢**：需要 500 epoch (Faster R-CNN ~12 epoch)。Transformer 的全局注意力难以快速定位
- **小目标差**：只用最后一层特征图 (1/32 分辨率)，小目标信息丢失
- **计算量大**：全局自注意力 $O((HW)^2)$

### 4.2 Deformable DETR (2021)

针对 DETR 收敛慢和小目标差的问题，提出 **可变形注意力 (Deformable Attention)**。

#### 可变形注意力

传统注意力关注**所有位置**，可变形注意力只关注**参考点附近的少量采样点**：

$$\text{DeformAttn}(z_q, p_q, x) = \sum_{m=1}^{M} W_m \left[ \sum_{k=1}^{K} A_{mqk} \cdot x(p_q + \Delta p_{mqk}) \right]$$

- $p_q$：参考点位置
- $\Delta p_{mqk}$：可学习的偏移量 (每个 head $M$，$K$ 个采样点)
- $A_{mqk}$：注意力权重 (softmax 归一化)
- 默认 $K=4$ 个采样点

```
传统注意力:                    可变形注意力:
┌─────────────────┐          ┌─────────────────┐
│● ● ● ● ● ● ● ●│          │                 │
│● ● ● ● ● ● ● ●│          │     ●       ●   │
│● ● ● ● ● ● ● ●│          │         ★       │ ★ = 参考点
│● ● ● ● ● ● ● ●│          │       ●     ●   │ ● = 采样点
│● ● ● ● ● ● ● ●│          │                 │
└─────────────────┘          └─────────────────┘
关注所有 H×W 个位置             只关注 K=4 个点
O((HW)²)                     O(HW × K)
```

#### 多尺度特征

Deformable DETR 在**多尺度特征图**上采样 (类似 FPN)，采样点可跨尺度：

$$\text{MSDeformAttn}(z_q, \hat{p}_q, \{x^l\}) = \sum_{m=1}^{M} W_m \left[ \sum_{l=1}^{L} \sum_{k=1}^{K} A_{mlqk} \cdot x^l(\phi_l(\hat{p}_q) + \Delta p_{mlqk}) \right]$$

**效果**：
- 收敛速度提升 **10×** (50 epoch vs 500 epoch)
- 小目标精度大幅提升 (多尺度特征)
- 计算复杂度从 $O((HW)^2)$ 降至 $O(HW \cdot K)$

### 4.3 DINO (Detection) (2022)

> ⚠️ 这是 **DINO: DETR with Improved DeNoising Anchor Boxes** (检测)，不要和自监督学习的 DINO (Self-Distillation) 混淆。

DINO 是 DETR 系列的集大成者，在 COCO 上达到了最强性能 (63.3 mAP with Swin-L)。

#### 三大核心改进

**1. Contrastive Denoising (CDN) — 对比去噪训练**

在训练时给 GT 框加噪声，让模型学习去噪：

- **正样本**：在 GT 框附近加小噪声 (内圈)，目标是重建 GT
- **负样本**：加大噪声 (外圈)，目标是预测 "no object"
- 通过对比正负样本，模型学会更精确的定位

```
         ┌─────────────────┐
         │   外圈 (负样本)    │
         │  ┌───────────┐  │
         │  │ 内圈(正样本) │  │
         │  │  ┌─────┐  │  │
         │  │  │ GT  │  │  │
         │  │  └─────┘  │  │
         │  └───────────┘  │
         └─────────────────┘
```

**2. Mixed Query Selection**

DETR 的 object query 是纯可学习参数，与输入图像无关。DINO 改进：
- 从 encoder 输出中选择 top-K 特征作为**位置部分** (anchor)
- **内容部分**仍为可学习参数
- 结合了数据依赖的位置先验和可学习的内容信息

**3. Look Forward Twice**

在 decoder 的每一层都进行框预测，并将前一层的预测结果传递给下一层进行 refinement：

```
标准 Deformable DETR:          Look Forward Twice:
Layer 1 → box₁                 Layer 1 → box₁ ──→ 更新参考点
Layer 2 → box₂                 Layer 2 → box₂ ──→ 更新参考点  ← 梯度回传
Layer 3 → box₃                 Layer 3 → box₃ ──→ 更新参考点  ← 梯度更远
                                每层都能利用之前所有层的信息
```

- 关键在于让**梯度可以通过参考点更新反传到更早的层**
- 类似于 dense supervision，每层都有监督信号

### 4.4 Grounding DINO / YOLO-World

**开放词汇检测 (Open-Vocabulary Detection)**：不限于固定类别，用文本描述指定要检测的目标。

#### Grounding DINO (2024)

将 DINO 与 **语言-视觉融合** 结合：

```
图像 ──→ Image Backbone ──→ 图像特征 ──┐
                                       ├──→ 特征融合 ──→ 语言引导的检测
文本 ──→ Text Backbone  ──→ 文本特征 ──┘    (Cross-Modality)

示例:
  输入图像 + 文本 "a red car. a person walking."
  输出: 红色汽车的框 + 行走的人的框
```

**关键设计**：
- 在 encoder 中进行**双向跨模态融合** (图像↔文本)
- 文本 token 作为类别查询，与图像特征做匹配
- 支持任意文本描述的零样本检测

#### YOLO-World (2024)

将 YOLO 的实时性与开放词汇能力结合：

- **Re-Parameterizable Vision-Language PAN (RepVL-PAN)**：离线计算文本嵌入，在线只做视觉推理
- 推理时可以**预计算文本嵌入**，不增加实时性负担
- 在 V100 上达到 52.0 FPS (YOLO-World-L)

## 六、面试高频问题

### Q1: 两阶段和单阶段的本质区别？

**两阶段** 先生成与类别无关的候选框 (RPN)，再对候选框精细分类和回归。**单阶段** 直接在特征图的每个位置/anchor 上预测类别和框。

本质区别在于**是否有"候选区域"的显式筛选阶段**：
- 两阶段：RPN 预筛选 → 正负比例可控 (1:3) → 精度高
- 单阶段：无预筛选 → 正负比例极端 (1:1000) → 需要 Focal Loss 等特殊处理

### Q2: Anchor 机制如何工作？Anchor-Free 的优势？

**Anchor 工作流程**：
1. 预定义多组 (尺度, 宽高比) 的框铺设在特征图上
2. 将每个 anchor 与 GT 匹配 (基于 IoU)
3. 网络预测 anchor 的微调量 $(\Delta x, \Delta y, \Delta w, \Delta h)$ 和分类

**Anchor-Free 优势**：
- 无需手动设计 anchor (超参数少)
- 避免大量 anchor 带来的正负样本不平衡
- 更简洁，更容易迁移到新任务/数据集
- 代表：FCOS (逐像素), CenterNet (关键点)

### Q3: Focal Loss 的物理含义和公式推导？

$$\text{FL}(p_t) = -\alpha_t (1 - p_t)^\gamma \log(p_t)$$

**物理含义**：让模型**聚焦于难分类样本**。对已经分类正确的简单样本 ($p_t$ 大) 大幅降低其损失权重，让梯度来自于真正需要学习的难样本。

**为什么有效**：
- 单阶段检测器中，~99.9% 的位置是背景（简单负样本）
- 标准 CE 下，这些简单负样本的损失总和仍然远大于少量正样本
- Focal Loss 中，$p_t = 0.9$ 的样本损失降低 100×（$\gamma = 2$），有效解决了梯度被淹没的问题

### Q4: DETR 的匈牙利匹配怎么做的？

1. 模型固定输出 $N$ 个预测 (如 100)
2. GT 有 $M$ 个目标，补充 $N - M$ 个 "no object"
3. 计算 $N \times N$ 的代价矩阵 $C_{ij} = \mathcal{L}_{\text{match}}(\hat{y}_i, y_j)$
4. 用匈牙利算法在 $O(N^3)$ 内找到最小代价的**一一对应**
5. 基于匹配结果计算最终损失并反向传播

**为什么需要匈牙利匹配？** 因为预测是集合 (无序)，需要先建立预测与 GT 的对应关系才能计算损失。这避免了 NMS 的贪心后处理。

### Q5: NMS 的问题？DETR 如何避免 NMS？

**NMS 的问题**：
- 阈值敏感：调高 → 冗余多；调低 → 遮挡目标被误删
- 顺序贪心，非全局最优
- 作为后处理，不可微，无法端到端优化

**DETR 如何避免 NMS**：
- Object Query 之间的**自注意力**让每个 query "知道" 其他 query 在检测什么 → 自然避免重复
- **匈牙利匹配**保证每个 GT 只被分配给一个预测 → 训练时就鼓励不重复
- 最终实现了**真正的端到端检测**
