## 一、问题定义与分类

图像分割的目标是将图像中的每个像素分配到特定的语义或实例类别。根据分割粒度的不同，分为三大任务：

### 1.1 三种分割任务

```
输入图像（街景场景）
┌───────────────────────────────┐
│  🚗A   🚗B      🌳    🏠     │
│          🚶C                  │
│  ═══════道路═══════════       │
│         🌿草地                │
└───────────────────────────────┘

语义分割 (Semantic Segmentation)
┌───────────────────────────────┐
│  [车][车]    [树]  [建筑]     │  每个像素 → 类别标签
│        [人]                   │  不区分同类不同实例
│  ═══════[道路]═══════════     │  车A 和 车B 都标为"车"
│       [草地]                  │
└───────────────────────────────┘

实例分割 (Instance Segmentation)
┌───────────────────────────────┐
│  [车#1][车#2]  [树]  [建筑]   │  可数目标：逐实例区分
│        [人#1]                 │  车A=#1, 车B=#2
│  ═══════道路═══════════       │  不处理不可数类别(道路、天空)
│       草地                    │
└───────────────────────────────┘

全景分割 (Panoptic Segmentation)
┌───────────────────────────────┐
│  [车#1][车#2] [树#1] [建筑#1] │  语义 + 实例的统一
│        [人#1]                 │  Things: 逐实例编号
│  ═══════[道路]═══════════     │  Stuff: 仅类别标签
│       [草地]                  │  每个像素都有归属
└───────────────────────────────┘
```

### 1.2 评估指标

**mIoU (mean Intersection over Union)** — 语义分割的标准指标：

$$\text{IoU}_c = \frac{|P_c \cap G_c|}{|P_c \cup G_c|}$$
$$\text{mIoU} = \frac{1}{C} \sum_{c=1}^{C} \text{IoU}_c$$

其中 $P_c$ 为类别 $c$ 的预测区域，$G_c$ 为真值区域，$C$ 为类别总数。

**AP$^{\text{mask}}$ (Average Precision)** — 实例分割指标：

- 对每个预测实例计算与真值的 IoU
- 在不同 IoU 阈值（0.5:0.05:0.95）下计算 AP
- COCO 标准：$\text{AP} = \text{AP}_{50:95}$（多阈值平均）

**PQ (Panoptic Quality)** — 全景分割指标：

$$\text{PQ} = \underbrace{\frac{\sum_{(p,g) \in \text{TP}} \text{IoU}(p,g)}{|\text{TP}|}}_{\text{SQ (Segmentation Quality)}} \times \underbrace{\frac{|\text{TP}|}{|\text{TP}| + \frac{1}{2}|\text{FP}| + \frac{1}{2}|\text{FN}|}}_{\text{RQ (Recognition Quality)}}$$

- **SQ** 衡量匹配上的片段分割质量（IoU 均值）
- **RQ** 衡量识别质量（类似 F1 分数）
- 匹配规则：预测与真值 IoU > 0.5 视为 TP

## 二、语义分割

### 2.1 FCN — 全卷积网络 (2015)

**核心思想**：将分类网络（VGG/AlexNet）的全连接层替换为卷积层，实现任意尺寸输入的逐像素预测。

#### 架构

```
输入图像 (H×W×3)
    │
    ▼
┌──────────────────┐
│  VGG-16 backbone  │  去掉最后的全连接层
│  conv1 → pool1    │  → H/2
│  conv2 → pool2    │  → H/4
│  conv3 → pool3    │  → H/8
│  conv4 → pool4    │  → H/16
│  conv5 → pool5    │  → H/32
└──────────────────┘
    │
    ▼
┌──────────────────┐
│  fc6 → 1×1 conv   │  4096 通道
│  fc7 → 1×1 conv   │  4096 通道
│  score → 1×1 conv  │  C 通道（C = 类别数）
└──────────────────┘
    │
    ▼
  转置卷积上采样 → H×W×C  → argmax → 逐像素预测
```

#### Skip Connection

FCN 的关键创新是**跳跃连接**融合多尺度特征：

```
FCN-32s:  pool5 预测 ──────────────────→ ×32 上采样 → 粗糙
FCN-16s:  pool5 预测 ──→ ×2 上采样 ──┐
          pool4 预测 ────────────────┼→ 融合 → ×16 上采样 → 较细
FCN-8s:   pool5 → ×2 → + pool4 → ×2 ──┐
          pool3 预测 ───────────────────┼→ 融合 → ×8 上采样 → 最细
```

**转置卷积（Transposed Convolution）**：

- 也称反卷积（deconvolution），用于上采样
- 输出尺寸：$o = (i - 1) \times s - 2p + k$，其中 $i$ 为输入尺寸，$s$ 为步长，$p$ 为填充，$k$ 为核大小
- 实际上是卷积操作的转置，不是逆操作

**局限性**：

- 上采样倍率大（×8/×16/×32），细节丢失严重
- 没有全局上下文信息
- 简单的跳跃连接融合方式不够精细

### 2.2 U-Net (2015)

**核心思想**：对称的编码器-解码器结构 + 跳跃连接拼接（concatenation），专为小数据量医学图像设计。

#### 架构

```
输入 (572×572×1)
    │
    ▼
┌─────────┐                              ┌─────────┐
│ Conv 3×3 │ ──────── Crop & Copy ──────→ │ Conv 3×3 │  输出 (388×388×2)
│ 64 ch    │  (编码器特征拼接到解码器)      │ 64→2 ch │
└────┬─────┘                              └─────▲───┘
  Pool│2×2                                 UpConv│2×2
     ▼                                        │
┌─────────┐                              ┌─────────┐
│ Conv 3×3 │ ──────── Crop & Copy ──────→ │ Conv 3×3 │
│ 128 ch   │                              │ 128 ch   │
└────┬─────┘                              └─────▲───┘
  Pool│2×2                                 UpConv│2×2
     ▼                                        │
┌─────────┐                              ┌─────────┐
│ Conv 3×3 │ ──────── Crop & Copy ──────→ │ Conv 3×3 │
│ 256 ch   │                              │ 256 ch   │
└────┬─────┘                              └─────▲───┘
  Pool│2×2                                 UpConv│2×2
     ▼                                        │
┌─────────┐                              ┌─────────┐
│ Conv 3×3 │ ──────── Crop & Copy ──────→ │ Conv 3×3 │
│ 512 ch   │                              │ 512 ch   │
└────┬─────┘                              └─────▲───┘
  Pool│2×2                                 UpConv│2×2
     ▼                                        │
┌──────────────────────────────────────────────┐
│              Bottleneck: 1024 ch              │
└──────────────────────────────────────────────┘
```

#### 与 FCN / ResNet 跳跃连接的对比

| 特性 | U-Net Skip | ResNet Residual | FCN Skip |
|------|-----------|-----------------|----------|
| 操作 | **拼接** (concat) | **逐元素相加** (add) | **逐元素相加** |
| 通道数 | 倍增 | 不变 | 不变 |
| 连接层 | 编码器 → 解码器 | 相邻层之间 | 不同深度的分数图 |
| 目的 | 保留空间细节 | 梯度畅通 | 多尺度融合 |
| 信息量 | 完整保留编码器特征 | 学习残差 | 较少 |

**U-Net 的 concat 保留了编码器的完整空间信息**，这对于医学图像中精确的边界分割至关重要。ResNet 的 add 只学习残差，信息容量有限。

#### 技术细节

- **Overlap-tile 策略**：对大图像采用重叠裁剪推理，解决 GPU 内存限制
- **弹性变形数据增强**：通过随机弹性变形扩充训练数据，在小样本场景尤为有效
- **加权交叉熵**：对相邻细胞之间的边界像素给予更高权重

$$w(\mathbf{x}) = w_c(\mathbf{x}) + w_0 \cdot \exp\left(-\frac{(d_1(\mathbf{x}) + d_2(\mathbf{x}))^2}{2\sigma^2}\right)$$

其中 $d_1, d_2$ 是像素 $\mathbf{x}$ 到最近两个细胞边界的距离。

### 2.3 DeepLab 系列

DeepLab 系列是语义分割最重要的工作线之一，核心贡献是**空洞卷积**和 **ASPP**。

#### DeepLab v1 (2015)

- 使用 VGG 做分类 backbone
- 关键创新：**CRF（条件随机场）后处理**，利用像素间的空间关系细化分割边界
- 问题：CRF 是独立的后处理步骤，无法端到端训练

#### DeepLab v2 (2017) — 空洞卷积 + ASPP

**空洞卷积（Atrous / Dilated Convolution）**：在标准卷积核的元素之间插入间隔（holes），在不增加参数量的情况下扩大感受野。

对于输入信号 $x$ 和卷积核 $w$，空洞卷积定义为：

$$y[i] = \sum_{k} x[i + r \cdot k] \cdot w[k]$$

其中 $r$ 为膨胀率（dilation rate），$r=1$ 退化为标准卷积。

```
标准卷积 (r=1)          空洞卷积 (r=2)          空洞卷积 (r=4)
感受野 3×3               感受野 5×5               感受野 9×9
参数量 3×3=9             参数量 3×3=9             参数量 3×3=9

■ ■ ■                   ■ □ ■ □ ■               ■ □ □ □ ■ □ □ □ ■
■ ■ ■                   □ □ □ □ □               □ □ □ □ □ □ □ □ □
■ ■ ■                   ■ □ ■ □ ■               □ □ □ □ □ □ □ □ □
                        □ □ □ □ □               □ □ □ □ □ □ □ □ □
                        ■ □ ■ □ ■               ■ □ □ □ ■ □ □ □ ■
                                                □ □ □ □ □ □ □ □ □
■ = 参与计算的位置                                □ □ □ □ □ □ □ □ □
□ = 跳过的位置                                   □ □ □ □ □ □ □ □ □
                                                ■ □ □ □ ■ □ □ □ ■
```

**等效感受野大小**：$k_{\text{eff}} = k + (k-1)(r-1)$，3×3 核 + $r=2$ → 等效 5×5。

**ASPP (Atrous Spatial Pyramid Pooling)**：使用多个不同膨胀率的空洞卷积并行处理，捕获多尺度上下文信息。

```
            输入特征图
        ┌──────┼──────┐──────┐
        ▼      ▼      ▼      ▼
    ┌──────┐┌──────┐┌──────┐┌──────┐
    │r = 6 ││r = 12││r = 18││r = 24│  不同膨胀率的 3×3 空洞卷积
    │3×3   ││3×3   ││3×3   ││3×3   │
    └──┬───┘└──┬───┘└──┬───┘└──┬───┘
       └───────┼───────┼───────┘
               ▼
          Concatenate
               ▼
           1×1 Conv → 融合多尺度信息
```

#### DeepLab v3 (2017)

改进 ASPP：

- **新增全局平均池化分支**：弥补 ASPP 在极大膨胀率下退化的问题
- 使用 BatchNorm
- 移除 CRF 后处理（网络本身已足够强）

```
              输入特征图
    ┌──────┬──────┬──────┬──────────┐
    ▼      ▼      ▼      ▼          ▼
 ┌─────┐┌─────┐┌─────┐┌─────┐┌──────────┐
 │1×1  ││r=6  ││r=12 ││r=18 ││ GAP      │  全局平均池化
 │Conv ││3×3  ││3×3  ││3×3  ││→1×1 Conv │  → 双线性上采样
 └──┬──┘└──┬──┘└──┬──┘└──┬──┘└────┬─────┘
    └──────┴──────┴──────┴────────┘
                  ▼
            Concatenate
                  ▼
              1×1 Conv → 输出
```

#### DeepLab v3+ (2018)

在 v3 的基础上加入**编码器-解码器结构**，改善分割细节：

```
┌────────────────── 编码器 ──────────────────┐
│                                            │
│  Backbone (Modified Xception / ResNet)     │
│  ├── Low-level features (stride 4) ───────────────┐
│  └── High-level features (stride 16)       │      │
│           │                                │      │
│       Improved ASPP                        │      │
│           │                                │      │
└───────────┼────────────────────────────────┘      │
            ▼                                       │
┌────────────────── 解码器 ──────────────────┐      │
│                                            │      │
│   ×4 上采样  ──→  Concat  ←── 1×1 Conv ←──┘      │
│                     │                      │
│                  3×3 Conv                   │
│                     │                      │
│                  ×4 上采样 → 最终预测        │
│                                            │
└────────────────────────────────────────────┘
```

关键改进：

- 使用 **Depthwise Separable Convolution**（深度可分离卷积）替代标准卷积，减少计算量
- Low-level features 经过 1×1 Conv 降维后再与高层特征融合
- Backbone 可选 Modified Xception 或 ResNet

### 2.4 PSPNet — 金字塔场景解析网络 (2017)

**动机**：语义分割中的"上下文先验缺失"问题——局部特征无法判断全局语义（如把湖面上的船误识别为车）。

**Pyramid Pooling Module (PPM)**：

```
           输入特征图 (H×W×C)
    ┌──────────┬──────────┬──────────┐
    ▼          ▼          ▼          ▼
┌──────┐  ┌──────┐  ┌──────┐  ┌──────┐
│1×1池化│  │2×2池化│  │3×3池化│  │6×6池化│   不同尺度的自适应平均池化
│全局   │  │      │  │      │  │      │
└──┬───┘  └──┬───┘  └──┬───┘  └──┬───┘
   │  1×1Conv  │  1×1Conv │  1×1Conv │  1×1Conv → 降维到 C/4
   ▼          ▼          ▼          ▼
 上采样到    上采样到    上采样到    上采样到
 H×W        H×W        H×W        H×W
   └──────────┴──────────┴──────────┘
                  ▼
     Concat(原始特征, 4个尺度特征)
                  ▼
           最终卷积 → 预测
```

PPM 提供了从局部到全局的多尺度上下文信息，1×1 池化相当于全局语义，6×6 池化保留较多空间信息。

### 2.5 基于 Transformer 的语义分割

#### SETR (2021)

- **Segmentation Transformer**：直接用 ViT 做编码器，将图像切成 patch 序列
- 三种解码器方案：Naive（直接上采样）、PUP（渐进上采样）、MLA（多层聚合）
- 证明 Transformer 可以替代 CNN 做密集预测

#### SegFormer (2021)

SegFormer 是最具影响力的 Transformer 分割方法，设计极简且高效：

```
输入图像 (H×W×3)
    │
    ▼
┌──────────────────────────────────┐
│   Hierarchical Transformer       │
│   (Mix Transformer, MiT)        │
│                                  │
│   Stage 1: H/4 × W/4 × C₁      │  Overlap Patch Embedding
│   Stage 2: H/8 × W/8 × C₂      │  Efficient Self-Attention
│   Stage 3: H/16 × W/16 × C₃    │  Mix-FFN (3×3 DWConv)
│   Stage 4: H/32 × W/32 × C₄    │
└──────────────────────────────────┘
    │       │       │       │
    ▼       ▼       ▼       ▼
┌──────────────────────────────────┐
│   All-MLP Decoder                │
│                                  │
│   各 Stage → Linear → 统一通道   │
│   → 上采样到 H/4 × W/4          │
│   → Concat → Linear → 预测      │
└──────────────────────────────────┘
```

关键设计：

- **Efficient Self-Attention**：通过 Reduction Ratio $R$ 减少 KV 序列长度，将复杂度从 $O(N^2)$ 降低到 $O(N^2/R)$
- **Mix-FFN**：在 FFN 中加入 3×3 深度可分离卷积，引入局部信息，无需位置编码
- **All-MLP Decoder**：极简的只含 MLP 的解码器，证明强编码器 + 简单解码器即可

从 MiT-B0（3.8M 参数）到 MiT-B5（84.7M），提供不同规模的模型。

## 三、实例分割

### 3.1 Mask R-CNN (2017)

**核心思想**：在 Faster R-CNN 的基础上增加一个并行的 Mask 预测分支，实现实例分割。

#### 架构

```
输入图像
    │
    ▼
┌──────────────┐
│  Backbone     │  ResNet-50/101 + FPN
│  (特征提取)   │
└──────┬───────┘
       │
       ▼
┌──────────────┐        ┌──────────────┐
│     RPN       │ ──→   │  RoI Align    │  替代 RoI Pooling
│  (区域提议)   │        │  (7×7 或 14×14)│
└──────────────┘        └──────┬───────┘
                               │
                    ┌──────────┼──────────┐
                    ▼          ▼          ▼
              ┌──────────┐┌──────────┐┌──────────┐
              │ 分类分支  ││ 框回归    ││ Mask 分支 │  ← 新增
              │ FC → C+1 ││ FC → 4   ││ Conv → K  │
              └──────────┘└──────────┘└──────────┘
                                          │
                                     K 个 28×28
                                     二值掩码
```

#### RoI Align vs RoI Pooling

这是 Mask R-CNN 最关键的技术贡献。

**RoI Pooling 的问题 — 两次量化**：

```
原图上的 RoI: (x=10.6, y=20.3, w=100.7, h=80.2)

第一次量化: 坐标取整 → (10, 20, 100, 80)    ← 丢失亚像素位置
                    │
           映射到特征图 (stride=16)
                    │
第二次量化: (0.625, 1.25, 6.25, 5.0) → (0, 1, 6, 5)  ← 再次取整

→ 对分类影响不大，但对掩码预测造成严重的空间错位！
```

**RoI Align 的解决方案**：

```
1. 不做任何量化，保留浮点坐标
2. 将 RoI 均匀分成 k×k 个 bin
3. 每个 bin 内均匀采样 4 个点（2×2 网格）
4. 每个采样点用双线性插值从特征图获取值
5. 对 4 个采样点取 max 或 avg 作为该 bin 的值

┌─────────────────────┐
│  bin  │  bin  │  bin │
│  ·  · │  ·  · │  ·  ·│  · = 采样点
│  ·  · │  ·  · │  ·  ·│  通过双线性插值计算
├───────┼───────┼──────┤
│  ·  · │  ·  · │  ·  ·│
│  ·  · │  ·  · │  ·  ·│
└─────────────────────┘
```

#### Mask 分支的设计

关键设计决策：**解耦分类与分割**。

- 对每个 RoI 预测 $K$ 个类别的二值掩码（$K$ 个 $28 \times 28$ 的 mask），而非 $K+1$ 类的语义分割
- 推理时用分类分支的结果选择对应类别的掩码
- 避免了不同类别之间的竞争，显著提升分割质量

损失函数：

$$\mathcal{L} = \mathcal{L}_{\text{cls}} + \mathcal{L}_{\text{box}} + \mathcal{L}_{\text{mask}}$$

其中 $\mathcal{L}_{\text{mask}}$ 只对真值类别 $k$ 对应的掩码计算二值交叉熵：

$$\mathcal{L}_{\text{mask}} = -\frac{1}{m^2} \sum_{1 \le i,j \le m} \left[ y_{ij} \log \hat{y}_{ij}^{k} + (1 - y_{ij}) \log(1 - \hat{y}_{ij}^{k}) \right]$$

### 3.2 Cascade Mask R-CNN

**动机**：单阶段检测器在不同 IoU 阈值下表现差异大。低阈值（0.5）产生噪声框，高阈值（0.7）正样本太少。

**方法**：级联多个检测头，逐阶段提高 IoU 阈值：

```
RPN → RoI Align → Head₁ (IoU=0.5) → 精化框
                                   → RoI Align → Head₂ (IoU=0.6) → 精化框
                                                                  → RoI Align → Head₃ (IoU=0.7) → 最终预测
```

每阶段的检测头都重新学习分类和回归，使用更高的 IoU 阈值筛选正样本，逐步提升检测质量。Mask 分支可以加在最后一个 stage。

### 3.3 SOLO / SOLOv2 (2020)

**核心思想**：完全绕过检测框（box-free），按位置直接分割实例。

#### SOLO

将图像划分为 $S \times S$ 的网格，如果一个实例的中心落在某个网格内，该网格负责预测该实例的掩码。

```
输入图像
    │
    ▼
  Backbone + FPN
    │
    ├── Category 分支: S×S×C   每个网格位置预测 C 类的概率
    │
    └── Mask 分支: H×W×S²      共 S² 个通道，每个通道对应一个网格位置的掩码
```

推理时：对每个激活的网格位置 $(i, j)$，取 Category 分支的类别概率 + Mask 分支的第 $(i \times S + j)$ 个通道作为该实例的掩码。

#### SOLOv2

使用**动态卷积**替代直接输出 $S^2$ 个掩码通道，大幅减少计算量：

```
Mask Feature 分支: 生成统一的 mask feature (H×W×E)
Kernel 分支: 每个网格位置预测一个卷积核权重 (E×1×1)

实例掩码 = Mask_Feature ⊗ Kernel   (逐位置卷积)
```

### 3.4 Mask2Former — 统一分割架构 (2022)

**核心思想**：用可学习的查询向量（learnable queries）统一处理语义分割、实例分割、全景分割。

#### 架构

```
输入图像
    │
    ▼
┌──────────────────┐
│  Backbone + FPN   │  多尺度特征 {1/4, 1/8, 1/16, 1/32}
└──────┬───────────┘
       │
       ▼
┌──────────────────────────────────────────────────┐
│              Transformer Decoder                  │
│                                                  │
│   N 个可学习查询 Q ∈ ℝ^{N×d}                     │
│                                                  │
│   每层:                                           │
│   1. Masked Cross-Attention (Q 与特征图交互)       │
│   2. Self-Attention (查询之间交互)                  │
│   3. FFN                                          │
│                                                  │
│   输出: 更新后的 Q → 预测 N 个掩码 + N 个类别      │
└──────────────────────────────────────────────────┘
    │                │
    ▼                ▼
N 个类别预测      N 个掩码预测
(N × (C+1))      Q × 像素特征 → N 个 (H×W) 掩码
```

#### Masked Cross-Attention

这是 Mask2Former 最关键的创新。

标准交叉注意力让每个 query 关注整张特征图的所有位置，计算量大且容易被无关区域干扰。Masked Cross-Attention 让每个 query **只关注其预测掩码区域内的特征**：

$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d}} + \mathcal{M}\right) V$$

其中 $\mathcal{M}$ 是由上一层预测掩码生成的注意力掩码：

$$\mathcal{M}_{l,i,j} = \begin{cases} 0, & \text{if } \hat{M}_{l-1}(i,j) = 1 \\ -\infty, & \text{otherwise} \end{cases}$$

这样做的好处：

- **聚焦**：每个 query 只需关注它负责的区域
- **效率**：等价于局部注意力，减少无效计算
- **性能**：相比 MaskFormer 有 显著提升

#### 训练：匈牙利匹配

使用**二分匹配**（Hungarian Algorithm）将 $N$ 个预测与真值进行一对一匹配：

$$\hat{\sigma} = \arg\min_{\sigma \in \mathfrak{S}_N} \sum_{i=1}^{N} \mathcal{L}_{\text{match}}(y_i, \hat{y}_{\sigma(i)})$$

匹配代价包括分类损失 + Mask 损失（Dice + BCE）。

## 四、全景分割

### 4.1 定义与 PQ 指标

全景分割将图像中的每个像素分配到一个语义类别，并对"可数目标"（Things）进一步区分实例。

- **Things**（可数目标）：车、人、狗 — 需要实例级区分
- **Stuff**（不可数背景）：天空、道路、草地 — 只需语义标签

**PQ 分解**：

$$\text{PQ} = \text{SQ} \times \text{RQ}$$

- $\text{SQ} = \frac{\sum_{(p,g) \in \text{TP}} \text{IoU}(p,g)}{|\text{TP}|}$：匹配片段的平均 IoU
- $\text{RQ} = \frac{|\text{TP}|}{|\text{TP}| + \frac{1}{2}|\text{FP}| + \frac{1}{2}|\text{FN}|}$：类似 F1 分数
- 匹配规则：预测片段与真值片段 IoU > 0.5 视为 TP

PQ 可以分别对 Things 和 Stuff 类别计算，记为 $\text{PQ}^{\text{Th}}$ 和 $\text{PQ}^{\text{St}}$。

### 4.2 方法演进

#### Panoptic FPN (2019)

最早的全景分割方法之一，思路直接：在 Faster/Mask R-CNN 的 FPN 基础上增加语义分割头。

```
                    FPN 特征
                      │
         ┌────────────┼────────────┐
         ▼                         ▼
┌────────────────┐       ┌────────────────┐
│ 实例分割头      │       │ 语义分割头      │
│ (Mask R-CNN)   │       │ (轻量 FCN)     │
│ → Things 实例  │       │ → Stuff 类别   │
└────────┬───────┘       └────────┬───────┘
         └────────────┬───────────┘
                      ▼
              全景合并 (Panoptic Fusion)
              → 优先 Things 实例
              → 剩余像素用 Stuff 填充
```

问题：两个头独立训练，合并规则是启发式的（如重叠时实例优先）。

#### MaskFormer → Mask2Former

MaskFormer (2021) 首次提出：**将所有分割任务统一为"掩码分类"（mask classification）** 问题。

- 不再区分 Things 和 Stuff，统一用 $N$ 个查询预测 $N$ 个 (类别, 掩码) 对
- 语义分割：同类查询的掩码合并
- 实例分割：每个查询就是一个实例
- 全景分割：Things 查询给实例 ID，Stuff 查询合并

Mask2Former 在此基础上加入 Masked Attention 和多尺度特征交互，在三个任务上都达到 SOTA。

#### OneFormer (2023)

进一步统一：一个模型、一个权重、三个任务。

- 引入**任务条件查询**（task-conditioned queries）：训练时随机采样任务类型（语义/实例/全景），将任务 token 与查询拼接
- 推理时通过切换任务 token 执行不同任务
- 使用对比损失对齐查询与文本表示

## 五、SAM — Segment Anything Model (2023, Meta)

### 5.1 动机

NLP 领域已有 GPT 这样的 foundation model，可以零样本解决各种任务。SAM 旨在构建**分割领域的 foundation model**：

- 一个模型，通过 prompt 交互完成任意分割任务
- 零样本泛化到未见过的图像和类别
- 为此需要构建前所未有规模的分割数据集

### 5.2 架构

```
┌────────────────────────────────────────────────────────────┐
│                     SAM 架构                                │
│                                                            │
│  ┌─────────────────┐                                       │
│  │ Image Encoder    │  ViT-H/16 (632M 参数)                │
│  │ (仅运行一次)     │  MAE 预训练                           │
│  │                  │  输出: 16× 降采样特征图                │
│  └────────┬────────┘                                       │
│           │                                                │
│           │  ┌──────────────────┐                          │
│           │  │ Prompt Encoder    │                          │
│           │  │                  │                          │
│           │  │ 点/框 → 位置编码  │  稀疏 prompt            │
│           │  │        + 学习嵌入 │                          │
│           │  │ 掩码 → 卷积编码   │  密集 prompt            │
│           │  │ 文本 → CLIP 编码  │                          │
│           │  └────────┬─────────┘                          │
│           │           │                                    │
│           ▼           ▼                                    │
│  ┌─────────────────────────────────┐                      │
│  │     Mask Decoder (轻量)          │                      │
│  │                                 │                      │
│  │  2 层 Transformer Decoder       │                      │
│  │  ├── Token-to-Image Attention   │  prompt token → 图像 │
│  │  ├── Image-to-Token Attention   │  图像 → prompt token │
│  │  └── Self-Attention             │                      │
│  │                                 │                      │
│  │  输出: 3 个掩码 + 置信度分数     │  多义性处理            │
│  └─────────────────────────────────┘                      │
└────────────────────────────────────────────────────────────┘
```

### 5.3 Prompt 类型

| Prompt 类型 | 编码方式 | 示例 |
|-------------|---------|------|
| **点** (前景/背景) | 位置编码 + 学习的前景/背景嵌入 | 点击目标物体 |
| **边界框** | 左上角 + 右下角的位置编码 | 框选目标区域 |
| **粗掩码** | 卷积降采样后与图像特征逐元素相加 | 上一轮预测结果 |
| **文本** | CLIP 文本编码器 | "cat"、"building" |

### 5.4 多义性处理

一个 prompt 可能对应多个合理的分割结果（如点击衣服上的 logo，可以分割 logo、衣服或整个人）。SAM 同时预测 **3 个掩码**，并为每个掩码输出一个 IoU 置信度分数：

- 整体目标（whole）
- 部分目标（part）
- 子部分目标（subpart）

训练时只对 IoU 最高的掩码计算损失。

### 5.5 SA-1B 数据集

SAM 的成功离不开超大规模数据集 SA-1B：

- **11M 张图像**，来源于授权摄影师
- **1.1B 个掩码**，平均每张图像 ~100 个掩码
- 数据收集流程（3 个阶段）：
  1. **辅助标注**：SAM 初版辅助人工标注
  2. **半自动**：SAM 自动预测 + 人工补充遗漏
  3. **全自动**：32×32 网格点 prompt → 自动生成所有掩码

### 5.6 零样本泛化

SAM 的泛化能力来源于：

1. **大规模多样化数据**：11M 图像涵盖极广的场景和目标
2. **Prompt 驱动**：不预设类别，而是通过交互指定目标
3. **多义性输出**：覆盖不同粒度的合理分割
4. **强 backbone**：ViT-H + MAE 预训练提供了强大的视觉表示

零样本实验表明 SAM 在 23 个不同数据集上表现优秀，包括医学、遥感、农业等领域。

### 5.7 SAM 2 (2024)

SAM 2 将 Segment Anything 扩展到**视频分割**：

- **Streaming 架构**：逐帧处理，使用 Memory Attention 聚合历史信息
- **Memory Bank**：存储最近 $N$ 帧的特征和 prompted frame 的特征
- **SA-V 数据集**：51K 视频 + 643K 掩码序列（masklets）
- 比 SAM 在图像分割上也更好（得益于视频数据的多样性）

```
帧 t-2         帧 t-1          帧 t (当前)
  │               │               │
  ▼               ▼               ▼
Image Encoder  Image Encoder   Image Encoder
  │               │               │
  └──→ Memory ←───┘               │
       Bank                       │
         │                        │
         └─── Memory Attention ───┘
                    │
                    ▼
              Mask Decoder → 当前帧掩码
```

### 5.8 下游扩展

**Grounded-SAM**：将 Grounding DINO（开放词汇检测）与 SAM 结合：

```
文本提示: "red car, person"
    │
    ▼
Grounding DINO → 检测框 → SAM Prompt → 精确掩码
```

其他扩展：
- **HQ-SAM**：改善 SAM 在高质量边界上的表现
- **EfficientSAM**：蒸馏 SAM 到更小的模型
- **MedSAM**：针对医学图像微调

## 七、面试高频问题

### Q1: 语义分割 vs 实例分割 vs 全景分割的区别？

- **语义分割**：每个像素一个类别标签，不区分同类不同实例（两辆车都是"车"）
- **实例分割**：只处理可数目标（Things），区分同类不同实例，但不处理背景
- **全景分割**：统一二者，Things 有实例 ID，Stuff 有类别标签，每个像素都有归属

### Q2: U-Net 的 skip connection 和 ResNet 的有什么不同？

- **U-Net**：跨层跳跃（编码器 → 解码器），操作是 **concat**（通道拼接），保留编码器完整的空间信息
- **ResNet**：短距离跳跃（相邻层），操作是 **add**（逐元素相加），学习残差函数
- 本质区别：U-Net 的目的是**恢复空间细节**，ResNet 的目的是**缓解梯度消失**

### Q3: 空洞卷积如何扩大感受野？有什么问题？

空洞卷积在核元素之间插入间隔（dilation rate $r$），等效感受野 $k_{\text{eff}} = k + (k-1)(r-1)$。3×3 核 + $r=2$ → 等效 5×5，参数量不变。

**Gridding Effect（网格效应）**：

- 当连续使用相同膨胀率的空洞卷积时，只有间隔采样的像素参与计算
- 导致局部信息丢失，输出呈现棋盘格状伪影

解决方案：
- **混合膨胀率**（Hybrid Dilated Convolution, HDC）：使用不同膨胀率组合（如 1, 2, 5）
- 膨胀率之间不存在公因子（避免固定间隔采样）
- DeepLab v3 中结合全局平均池化分支弥补

### Q4: Mask R-CNN 的 RoI Align 和 RoI Pooling 的区别？

**RoI Pooling** 有两次量化：
1. 连续坐标 → 离散坐标（取整）
2. 特征图坐标 → bin 边界（取整）

这对分类无大碍，但对像素级掩码预测造成严重的空间错位。

**RoI Align**：
- 不做任何量化，保留浮点坐标
- 每个 bin 内均匀采样若干点（默认 4 个）
- 每个采样点通过**双线性插值**获取特征值
- 消除量化误差，mask AP 提升约 3 个点

### Q5: SAM 的泛化能力来源？Prompt 编码器怎么工作？

**泛化来源**：
1. SA-1B 超大规模数据集（11M 图像 + 1B 掩码）覆盖极广的视觉场景
2. Prompt-driven 设计不预设类别，而是通过交互指定目标
3. ViT-H + MAE 预训练提供强大的视觉表示基础
4. 多义性输出（3 个掩码）覆盖不同粒度

**Prompt 编码器**：
- **点和框**（稀疏 prompt）：用位置编码（正弦）表示坐标，加上学习的类型嵌入（前景/背景/左上角/右下角）
- **掩码**（密集 prompt）：用卷积网络降采样到与图像特征相同的分辨率，逐元素相加
- **文本**：通过 CLIP 文本编码器获取文本嵌入

### Q6: Mask2Former 如何统一三种分割任务？

核心思想：**将所有分割任务统一为掩码分类问题**。

- 使用 $N$ 个可学习查询，每个查询预测一个 (类别, 掩码) 对
- 语义分割：将同类查询的掩码合并
- 实例分割：每个查询即一个实例
- 全景分割：Things 查询分配实例 ID，Stuff 查询合并

关键技术 Masked Cross-Attention：让每个查询只关注其预测掩码区域内的特征，既提升效率又提升精度。
