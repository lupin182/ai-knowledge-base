知识库

每个知识库 = knowledge_bases/<slug>/ 下独立的一个文件夹，互不干扰。URL 形如 /kb/<slug>/。

＋ 新建一个空知识库
只能用字母 / 数字 / _ / -，决定 /kb/<这里>/ 的 URL。
📋 添加内容的格式 / 操作说明

两种加法：① 上面「新建空知识库」→ 在卡片里上传文件/文件夹；② 直接往 knowledge_bases/<slug>/ 放 markdown 再重建。

硬性格式：

  • slug 只能 A–Z a–z 0–9 _ -（1–64 字）；URL 大小写敏感。
  • 每个 KB 必须有 README.md —— 它是该 KB 首页，也是首页卡片标题/简介的来源。
  • 内容用 markdown；跨 KB 引用写绝对路径 /kb/<别的slug>/路径.md（写 .md，构建时自动转目录路由；大小写要和文件名一致）。
  • 图片直接相对引用 ![](./图.png) 即可（静态服务，不做优化）。

软约定：侧栏默认按文件夹结构自动生成；要精排就加 _sidebar.md（_ 前缀的文件不发布成页面）。加完内容要重建才在站点看到（AI/编辑器改完会自动触发，或重启）。

外部资源挂载

把仓库之外的文件夹（绝对路径，如 F:/onedrive/reports）挂进站点浏览。文件留在原地、不拷进仓库、不入 git，按 /<前缀>/... 访问。适合大体量 / 集群同步来的参考资料。改完点「保存并重建」生效。

说明 / 与「上传到知识库」的区别
  • 前缀：URL 段名，只用字母/数字/_/-；避开保留词 kb / api / server / docs / knowledge_bases。
  • 路径：绝对文件系统路径（Windows 用正斜杠如 F:/a/b 更稳）。路径不存在会被静默跳过。
  • vs 上传到知识库：上传会把文件拷进 knowledge_bases/（入库 + 随仓库同步）；外部挂载只引用原地文件，URL 走独立前缀，不入库。
  • 仅本机可改（安全：避免远程把任意目录挂出来）。保存后会自动重建一次把内容同步进站点。
